LDAP 接続の再バインド試行の待ち時間を変更する

「ディレクトリユーティリティ」を使用して、LDAP サーバが応答しない場合に再接続を試みるまでの待ち時間を指定できます。 この値を大きくすると、再接続を連続して試みないようにできます。

待機状態の LDAP クライアントが再バインドするまでの待ち時間を設定するには：

  1. 「システム環境設定」を開き、「アカウント」をクリックします。

  2. ロックアイコンがロックされている場合は、それをクリックしてロックを解除してから、管理者の名前とパスワードを入力します。

  3. 「ログインオプション」をクリックして、「編集」をクリックします。

  4. 「ディレクトリユーティリティを開く」をクリックします。

  5. ロックアイコンがロックされている場合は、それをクリックしてロックを解除してから、管理者の名前とパスワードを入力します。

  6. 「サービス」をクリックします。

  7. サービスのリストから「LDAPv3」を選択し、編集（/）ボタンをクリックします。

  8. サーバ構成のリストが隠れている場合は、「オプションを表示」をクリックします。

  9. リストでサーバ構成を選び、「編集」をクリックします。

  10. 「接続」をクリックしてから、「再バインドを試行するまでの時間」に値を入力します。

    デフォルトは、120 秒です。
