カスタム検索方式を定義する

「ディレクトリユーティリティ」を使用すると、Mac OS X コンピュータの認証検索方式とコンタクト検索方式でディレクトリドメインのカスタムリストが使用されるように構成できます。

カスタムリストは、コンピュータのローカルディレクトリドメインから開始し、Open Directory やその他の LDAP ディレクトリドメイン、Active Directory ドメイン、共有ディレクトリドメイン、BSD 構成ファイル、および NIS ドメインを含めることができます。

コンピュータのカスタム検索方式に指定されているディレクトリドメインを使用できない場合は、コンピュータが起動したときに遅延が発生します。

ディレクトリドメインのカスタムリストを検索方式に指定するには：

  1. 「システム環境設定」を開き、「アカウント」をクリックします。

  2. ロックアイコンがロックされている場合は、それをクリックしてロックを解除してから、管理者の名前とパスワードを入力します。

  3. 「ログインオプション」をクリックして、「接続」または「編集」をクリックします。

  4. 「ディレクトリユーティリティを開く」をクリックします。

  5. ロックアイコンがロックされている場合は、それをクリックしてロックを解除してから、管理者の名前とパスワードを入力します。

  6. 「検索方式」をクリックして、検索方式を選択します。

    • 「認証」：認証およびその他多くの管理データに使用する検索方式が表示されます。

    • 「コンタクト」：「アドレスブック」などのアプリケーションの連絡先情報に使用する検索方式が表示されます。

  7. 「検索」ポップアップメニューから「カスタムパス」を選びます。

  8. 「追加」をクリックしてディレクトリを選択し、再び「追加」をクリックして、必要なディレクトリドメインを追加します。

  9. リスト上のディレクトリドメインの順序を変更する必要があるときは、リスト内で上下にドラッグします。

  10. 検索方式に必要のないディレクトリドメインをリストから削除するときは、それらを選択して削除（－）ボタンをクリックします。

  11. 「OK」をクリックして削除することを確認してから、「適用」をクリックします。

使用可能なディレクトリにリストされていないディレクトリを追加するときは、コンピュータがそのディレクトリにアクセスするよう構成されていることを確認してください。詳しくは、以下を参照してください：

  • ディレクトリサービスの詳細設定を使用する

  • LDAP サービスの詳細設定を使用する

  • Active Directory サービスの詳細設定を使用する

  • NIS 設定を指定する

  • BSD 構成ファイルの設定を指定する
