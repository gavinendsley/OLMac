ホーム
Active Directory ユーザアカウントのホームフォルダを設定する

「ディレクトリユーティリティ」の Active Directory コネクタを使用するように構成されたコンピュータで、Active Directory ユーザアカウント用のネットワーク・ホーム・フォルダまたはローカル・ホーム・フォルダを有効または無効にできます。

ネットワーク・ホーム・フォルダを使用する場合、ユーザのログイン時に、ユーザの Windows ネットワーク・ホーム・フォルダが Mac OS X ホームフォルダとしてマウントされます。

Active Directory スキーマが Mac OS X の homeDirectory 属性を含むように拡張されている場合は、ネットワーク・ホーム・フォルダの場所を Active Directory の標準の homeDirectory 属性または Mac OS X の homeDirectory 属性のどちらから取得するかを決定します。

ローカル・ホーム・フォルダを使用する場合、ログインする各 Active Directory ユーザは、Mac OS X の起動ディスク上にホームフォルダを持ちます。また、ユーザのネットワーク・ホーム・フォルダが、共有ポイントのように、ネットワークボリュームとしてマウントされます。ユーザは、このネットワークボリュームとローカル・ホーム・フォルダ間でファイルをコピーできます。

Active Directory ユーザアカウントのホームフォルダを設定するには：

  1. 「システム環境設定」を開き、「アカウント」をクリックします。

  2. ロックアイコンがロックされている場合は、それをクリックしてロックを解除してから、管理者の名前とパスワードを入力します。

  3. 「ログインオプション」をクリックして、「接続」または「編集」をクリックします。

  4. 「ディレクトリユーティリティを開く」をクリックします。

  5. ロックアイコンがロックされている場合は、それをクリックしてロックを解除してから、管理者の名前とパスワードを入力します。

  6. 「サービス」をクリックします。

  7. サービスのリストから「Active Directory」を選択し、編集（/）ボタンをクリックします。

  8. 詳細オプションが表示されていない場合は、「詳細オプションを表示」をクリックします。

  9. 「ユーザ環境」をクリックします。

  10. Active Directory ユーザアカウントでコンピュータの「/ユーザ」フォルダ内にローカル・ホーム・フォルダが保持されるようにする場合は、「ローカル・ホームディレクトリを起動ディスクに設定」をクリックします。

    「ログイン時にモバイルアカウントを作成」が選択されている場合、このオプションは使用できません。

  11. ホームフォルダの場所として Active Directory の標準属性を使用するには、「Active Directory からの UNC パスを使用してネットワークホームを設定」を選択してから、ホームフォルダへのアクセスに使用するプロトコルを以下から選択します：

    • Windows 標準のプロトコル SMB を使用するときは、「使用するネットワークプロトコル」ポップアップメニューから「smb」を選択します。

    • Macintosh 標準のプロトコル AFP を使用するには、「使用するネットワークプロトコル」ポップアップメニューから「afp」を選択します。

  12. ホームフォルダの場所として Mac OS X の属性を使用するには、「Active Directory からの UNC パスを使用してネットワークホームを設定」の選択を解除します。

    Mac OS X 属性を使用するには、Active Directory のスキーマを拡張してそれを含める必要があります。

  13. 「OK」をクリックします。

Active Directory ドメインのユーザアカウントの名前を変更すると、そのユーザアカウントのホームフォルダ（およびサブフォルダ）が作成され、次回 Mac OS X コンピュータにログインするときにはそれらが使用されます。そのユーザは、「Finder」で古いホームフォルダに移動してその内容を確認できます。

ユーザが次回ログインする前に古いフォルダの名前を変更して、新しいホームフォルダが作成されないように設定することもできます。
