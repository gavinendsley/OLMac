ホーム
コンピュータを不当な DHCP サーバから保護する

アップルでは、セキュリティが重要な問題である環境では、自動認証検索方式を DHCP から提供される LDAP サーバや DHCP から提供される共有ディレクトリドメインと併用しないことをお勧めしています。

ネットワークにアクセスできる悪意のあるハッカーは、偽の DHCP サーバ、偽の LDAP ディレクトリまたは共有ディレクトリドメイン、およびお使いのコンピュータのルート・ユーザ・アカウントを使用して、コンピュータを制御することができます。

ハッカーがローカルネットワークにアクセスするには、ハッカーの偽の DHCP サーバがそのネットワークまたはサブネットの一部でなければなりません。つまり、お使いのコンピュータだけでローカルネットワークが構成されていて、Mac OS X Server NAT サービスまたは NAT ルーター経由でインターネットにアクセスしている場合には、このようなセキュリティ侵害はあり得ません。しかし、ローカルネットワークがワイヤレスで接続されている場合には、セキュリティが低下します。ハッカーにとっては、有線のローカルネットワークよりもワイヤレスのローカルネットワークに接続する方が簡単であるためです。

DHCP から提供される LDAP ディレクトリの使用を無効にし、ローカルディレクトリドメインのブロードキャストと DHCP バインディングを無効にして（または、ローカルディレクトリドメインを無効にして）、偽の DHCP サーバからの悪意のある攻撃からお使いの Mac を保護することができます。

コンピュータがネットワークに接続されているときに、LDAP サーバに接続しているモバイルコンピュータがあり、コンピュータの検索方式を（「ディレクトリユーティリティ」の「検索方式」の「認証」パネルで）自動からカスタムに変更する場合、コンピュータがネットワークに接続されていないときは起動が遅くなります。

このような遅延は、コンピュータがコンピュータのカスタム検索方式にリストされている特定のディレクトリドメインに接続できないために発生します。スリープ状態のときにネットワークから切断されたコンピュータのスリープを解除するときは、著しい遅延はありません。
