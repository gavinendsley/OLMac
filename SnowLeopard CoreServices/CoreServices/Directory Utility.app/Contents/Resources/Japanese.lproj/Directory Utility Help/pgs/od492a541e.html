パスワードを作成する

ユーザのアカウントに関連付けられたパスワードは、ユーザがログインまたはその他のサービスの認証を行うときにユーザが入力する必要があります。パスワードは大文字と小文字が区別され（SMB-LAN Manager のパスワードを除く）、入力時に画面上でマスクされます。

ユーザ用に選択するパスワードタイプに関係なく、Mac OS X Server のユーザアカウントのパスワードを作成するためのガイドラインを次に示します：

  • パスワードは文字、数字、および記号を組み合わせて、認証されていないユーザに簡単に推測されないようにしてください。 パスワードは単語で構成しないでください。適切なパスワードは、数字と記号（# や $ など）が含まれます。または、語句に含まれるすべての単語の最初の文字で構成されます。大文字と小文字の両方を使用してください。

  • スペースおよび Option キーの組み合わせは避けてください。

  • ユーザが使用するコンピュータで入力できない文字、または別のキーボードやプラットフォームで正しく入力するために特殊なキーストロークの組み合わせが必要になる可能性のある文字は避けてください。

  • 一部のネットワークプロトコルは、先行スペース、埋め込みスペース、または後続スペースを含むパスワードに対応していません。

  • 長さがゼロのパスワードはお勧めできません。Open Directory および一部のシステム（LDAP バインドなど）は、長さがゼロのパスワードに対応していません。

  • ユーザがアクセスする可能性のあるコンピュータおよびサービスとの互換性を最大にするために、パスワードには ASCII 文字だけを使用してください。
