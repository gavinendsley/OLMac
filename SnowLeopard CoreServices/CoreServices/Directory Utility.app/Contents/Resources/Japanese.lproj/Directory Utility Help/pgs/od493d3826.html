LDAP ディレクトリの接続設定を変更する

「ディレクトリユーティリティ」を使用すると、LDAPv3 または LDAPv2 ディレクトリへのアクセス方法を指定した構成で、その接続設定を変更できます。

LDAP ディレクトリへのアクセスの接続設定を変更するには：

  1. 「システム環境設定」を開き、「アカウント」をクリックします。

  2. ロックアイコンがロックされている場合は、それをクリックしてロックを解除してから、管理者の名前とパスワードを入力します。

  3. 「ログインオプション」をクリックして、「編集」をクリックします。

  4. 「ディレクトリユーティリティを開く」をクリックします。

  5. ロックアイコンがロックされている場合は、それをクリックしてロックを解除してから、管理者の名前とパスワードを入力します。

  6. 「サービス」をクリックします。

  7. サービスのリストから「LDAPv3」を選択し、編集（/）ボタンをクリックします。

  8. サーバ構成のリストが隠れている場合は、「オプションを表示」をクリックします。

  9. リストでサーバ構成を選び、「編集」をクリックします。

  10. 「接続」をクリックして、以下の設定を必要に応じて変更します：

    • 「構成名」：この構成は、LDAP ディレクトリ構成リスト上でこの名前で識別されます。（LDAP ディレクトリ構成リスト上での名前を変更することもできます。）

    • 「サーバ名または IP アドレス」：サーバの DNS 名または IP アドレスを指定します。（この設定は、LDAP ディレクトリ構成リスト上で変更することもできます。）

    • 「開く／閉じる操作のタイムアウトまでの秒数」：接続を試みる最大時間を指定します。この時間が経過すると、接続はキャンセルされます。

    • 「クエリー・タイム・アウトまでの秒数」：クエリーを継続する最大時間を指定します。この時間が経過すると、クエリーはキャンセルされます。

    • 「再バインドを試行するまでの時間」：LDAP サーバが応答しない場合、再接続を試みるまでに待つ秒数を指定します。再接続を何度も試みないようにするには、この値を大きくします。

    • 「接続が切断されるまでのアイドルタイム」：待機状態または応答のない状態で接続が開いたままになっていることを許可する分数を指定します。

    • 「SSL を使って暗号化」：SSL 接続を使用して、LDAP ディレクトリとの通信を暗号化するかどうかを決定します。（この設定は、LDAP ディレクトリ構成リスト上で変更することもできます。）「SSL」チェックボックスを選択する前に、SSL が必要かどうかを Open Directory 管理者に確認してください。

    • 「カスタムポートを使用」：LDAP 接続用の標準ポート（389：SSL を使用しない、636：SSL を使用する）以外のポート番号を指定します。

    • 「サーバの紹介を無視」：ほかの LDAP サーバまたは情報の複製を参照するため LDAP サーバの紹介を無視するか従うかどうかを決定します。 サーバの紹介はコンピュータが情報を見つけるのに役立ちますが、コンピュータがほかの LDAP サーバへの紹介を検証する必要がある場合、ログインが遅くなったり、ほかの処理が遅くなったりする可能性があります。

    • 「LDAPv2 を使用（読み出し専用）」：LDAP ディレクトリへの読み出し専用のアクセスに古い LDAPv2 プロトコルを使用するかどうかを決定します。
