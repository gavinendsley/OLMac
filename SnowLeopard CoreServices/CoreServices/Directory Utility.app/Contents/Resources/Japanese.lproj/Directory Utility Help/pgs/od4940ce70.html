Active Directory ドメインへの LDAP アクセスを設定する

「ディレクトリユーティリティ」を使用すると、Windows サーバ上の Active Directory ドメインにアクセスするための LDAPv3 構成を設定できます。 LDAPv3 構成は、Mac OS X のレコードタイプや属性を Active Directory のオブジェクトクラス、検索ベース、属性などにマッピングする際の細かい制御を提供します。

一意なユーザ ID（UID）など、Mac OS X の重要なレコードタイプや属性をマッピングする場合は、Active Directory スキーマを拡張する必要があります。

LDAPv3 構成には、「ディレクトリユーティリティ」にリスト表示される Active Directory コネクタの以下の機能は含まれていません。

  • 一意のユーザ ID とプライマリグループ ID を動的に生成する機能

  • ローカル Mac OS X ホームフォルダの作成

  • Windows ホームフォルダの自動マウント

  • 認証資格情報がキャッシュされたモバイル・ユーザ・アカウント

  • Active Directory フォレスト内のすべてのドメインを検出する機能

  • Active Directory の複製とフェイルオーバーのサポート

詳しくは、「Active Directory アクセスについて」を参照してください。

Active Directory サーバ構成を作成するには：

  1. 「システム環境設定」を開き、「アカウント」をクリックします。

  2. ロックアイコンがロックされている場合は、それをクリックしてロックを解除してから、管理者の名前とパスワードを入力します。

  3. 「ログインオプション」をクリックして、「接続」または「編集」をクリックします。

  4. 「ディレクトリユーティリティを開く」をクリックします。

  5. ロックアイコンがロックされている場合は、それをクリックしてロックを解除してから、管理者の名前とパスワードを入力します。

  6. 「サービス」をクリックします。

  7. サービスのリストから「LDAPv3」を選択し、編集（/）ボタンをクリックします。

  8. 「新規」をクリックして、Active Directory サーバの DNS 名または IP アドレスを入力します。

  9. ディレクトリにアクセスするために以下のオプションから適切なオプションを選択し、「続ける」をクリックして Active Directory サーバから情報を取得します。

    • Open Directory から Active Directory サーバに接続するときに SSL を使用する場合は、「SSL を使って暗号化」を選択します。「SSL」チェックボックスを選択する前に、SSL が必要かどうかを Open Directory 管理者に確認してください。

    • ユーザがログインやサービスの認証に使用するユーザアカウントがこのディレクトリに含まれている場合は、「認証に使用」を選択します。

    • 「アドレスブック」で使用するメールアドレスなどの情報がこのディレクトリに含まれている場合は、「コンタクトに使用」を選択します。

    「ディレクトリユーティリティ」が Active Directory サーバに接続できない場合は、メッセージが表示され、手動でアクセスを構成するか、設定プロセスをキャンセルする必要があります。詳しくは、「LDAP ディレクトリへのアクセスを手動で構成する」を参照してください。

    「認証に使用」オプションまたは「コンタクトに使用」オプションを選択した場合は、Active Directory ドメインへの LDAPv3 接続が、「ディレクトリユーティリティ」の「認証」または「コンタクト」パネルのカスタム検索方式に追加されます。

    設定した接続がコンピュータで使用されるように、「サービス」パネルで LDAPv3 が有効になっていることを確認します。詳しくは、「LDAP ディレクトリサービスを有効にする／無効にする」を参照してください。

  10. ダイアログが展開されてマッピングオプションが表示されたら、ポップアップメニューから「Active Directory」を選択し、検索ベースを入力して、「続ける」をクリックします。

    Mac OS X のレコードタイプや属性のいくつかは、LDAPv3 の構成の Active Directory マッピングテンプレートを使用して、Active Directory の標準スキーマに含まれていないオブジェクトクラスや属性にマップされます。テンプレートで定義されているマッピングを変更したり、Active Directory のスキーマを拡張することもできます。

    または、LDAPv3 ではなく Active Directory コネクタを介して Active Directory ドメインにアクセスできる場合もあります。詳しくは、「Active Directory ドメインへのアクセスを構成する」を参照してください。

  11. ダイアログが展開されて接続オプションが表示されたら、Active Directory ユーザアカウントの識別名とパスワードを入力します。

  12. 「OK」をクリックして、LDAP 接続の作成を終了します。

  13. 「OK」をクリックして、LDAPv3 オプションの構成を終了します。
