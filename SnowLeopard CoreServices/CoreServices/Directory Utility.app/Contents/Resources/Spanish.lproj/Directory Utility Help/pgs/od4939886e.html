Especificación de ajustes de los archivos de configuración BSD

Los ordenadores UNIX siempre han almacenado los datos administrativos en archivos de configuración, como "/etc/master.passwd", "/etc/group" y "/etc/hosts". Mac OS X se basa en una versión BSD de UNIX, pero suele obtener los datos administrativos a partir de sistemas de directorios.

Mac OS X Server admite un conjunto fijo de archivos de configuración BSD. No puede especificar qué archivos de configuración utilizar ni puede asignar su contenido a los tipos de registro y atributos de Mac OS X.

En Mac OS X 10.2 o posterior (incluido Mac OS X Server 10.2 o posterior), Open Directory puede recuperar datos administrativos de archivos de configuración BSD. Esta capacidad permite a las organizaciones que disponen de archivos de configuración BSD utilizar copias de los archivos existentes en ordenadores con Mac OS X. Los archivos de configuración BSD pueden utilizarse solos o con otros dominios de directorio.

Para utilizar archivos de configuración BSD:

  1. Compruebe que los archivos de configuración BSD contengan los datos requeridos por los servicios de directorio de Mac OS X.

    Para obtener más información, consulte Configuración de datos en los archivos de configuración BSD.

  2. Abra Preferencias del Sistema y haga clic en Cuentas.

  3. Si el icono del candado está cerrado, desbloquéelo haciendo clic en él e introduciendo el nombre y la contraseña de un administrador.

  4. Haga clic en "Opc. inicio sesión" y luego haga clic en Acceder o Editar.

  5. Haga clic en “Abrir Utilidad de Directorios”.

  6. Si el icono del candado está cerrado, desbloquéelo haciendo clic en él e introduciendo el nombre y la contraseña de un administrador.

  7. Haga clic en Servicios.

  8. En la lista de servicios, seleccione “BSD Flat File y NIS” y haga clic en el botón Editar (/).

  9. Seleccione “Usar registros de usuario y grupo en el nodo local BSD” y haga clic en Aceptar.

    El dominio de los archivos de configuración BSD se añade a la política de búsqueda de autenticación del ordenador con la forma "/BSD/local".
