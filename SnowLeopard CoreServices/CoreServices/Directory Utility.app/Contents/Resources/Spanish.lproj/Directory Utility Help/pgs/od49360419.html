Cómo configurar el acceso a un directorio LDAP manualmente

Puede crear manualmente una configuración que especifique cómo debe acceder Mac OS X a un directorio LDAPv3 o LDAPv2. Debe saber el nombre DNS o la dirección IP del servidor del directorio LDAP.

Si el directorio no está alojado en Mac OS X Server, deberá saber la base de búsqueda y la plantilla para asignar los datos de Mac OS X a los datos del directorio. Las plantillas de asignación compatibles son las siguientes:

  • Del Servidor, para un directorio que suministre sus propias asignaciones y base de búsqueda, como Mac OS X Server.

  • Servidor Open Directory, para un directorio que utilice el esquema de Mac OS X Server.

  • Active Directory, para un directorio alojado en un servidor Windows 2000, Windows 2003 o un servidor posterior.

  • RFC 2307, para la mayoría de los directorios alojados en servidores UNIX.

  • Personalizado, para directorios que no utilicen ninguna de las asignaciones anteriores.

El módulo LDAPv3 es totalmente compatible con la replicación y la conmutación por errores de Open Directory. Si el máster de Open Directory deja de estar disponible, el módulo recurre a una réplica próxima.

Importante:  si el nombre de su ordenador contiene un guión, es posible que no pueda acceder o enlazarse a un dominio de directorio como LDAP o Active Directory. Para establecer un enlace, utilice un nombre de ordenador que no contenga un guión.
Para configurar el acceso a un directorio LDAP manualmente:

  1. Abra Preferencias del Sistema y haga clic en Cuentas.

  2. Si el icono del candado está cerrado, desbloquéelo haciendo clic en él e introduciendo el nombre y la contraseña de un administrador.

  3. Haga clic en "Opc. inicio sesión" y luego haga clic en Acceder o Editar.

  4. Haga clic en “Abrir Utilidad de Directorios”.

  5. Si el icono del candado está cerrado, desbloquéelo haciendo clic en él e introduciendo el nombre y la contraseña de un administrador.

  6. Haga clic en Servicios.

  7. En la lista de servicios, seleccione LDAPv3 y haga clic en el botón Editar (/).

    Puede seleccionar LDAPv3 en la lista de servicios sin seleccionar la opción Activar correspondiente a LDAPv3.

  8. Haga clic en Nuevo y, a continuación, haga clic en Manual.

  9. Asigne un nombre a la configuración.

  10. Pulse Tabulador e introduzca el nombre DNS o la dirección IP del servidor que aloja el directorio LDAP al que desea acceder.

  11. Haga clic en el menú local situado junto al nombre DNS o la dirección IP y seleccione una plantilla o método de asignación:

    • Si selecciona "Del servidor", no se necesita sufijo de la base de búsqueda. En tal caso, Open Directory considera que el sufijo de la base de búsqueda es el del primer nivel del directorio LDAP.

    • Si selecciona una plantilla, introduzca el sufijo de la base de búsqueda para el directorio LDAP y haga clic en Aceptar. Debe introducir el sufijo de la base de búsqueda o el ordenador no podrá encontrar la información en el directorio LDAP.

      Normalmente, el sufijo de la base de búsqueda deriva del nombre del DNS del servidor. Por ejemplo, el sufijo de la base de búsqueda podría ser "dc=ods,dc=ejemplo, dc=com" para un servidor cuyo nombre DNS sea "ods.ejemplo.com".

    • Si selecciona Personalizado, tendrá que configurar las asignaciones entre los tipos de registros de Mac OS X y los atributos y las clases del directorio LDAP al que se esté conectando. Para obtener más información, consulte Cómo configurar búsquedas y mapas LDAP.

  12. Antes de seleccionar la opción “Encriptar con SSL”, consulte con el administrador de Open Directory para determinar si es necesario utilizar SSL.

  13. Para modificar los ajustes siguientes de esta configuración LDAP, haga clic en Editar para que se muestren las opciones de la configuración LDAP seleccionada, realice los cambios pertinentes y haga clic en Aceptar cuando termine de editar las opciones de configuración LDAP.

    • Haga clic en Conexión para ajustar las opciones de tiempo, especificar un puerto personalizado, ignorar las referencias del servidor o forzar el uso del protocolo LDAPv2 (solo lectura). Para obtener más información, consulte Cómo cambiar los ajustes de conexión de un directorio LDAP.

    • Haga clic en "Búsqueda y mapas" para configurar búsquedas y asignaciones de un servidor LDAP. Para obtener más información, consulte Cómo configurar un vínculo fiable con un directorio LDAP.

    • Haga clic en Seguridad para configurar una conexión autenticada (en lugar de un vínculo fiable) y otras opciones de las políticas de seguridad. Para obtener más información, consulte Cómo cambiar la política de seguridad de una conexión LDAP.

    • Haga clic en Enlazar para configurar vínculos fiables (si el directorio LDAP lo admite). Para obtener más información, consulte Cómo configurar un vínculo fiable con un directorio LDAP.

  14. Haga clic en Aceptar para finalizar manualmente la creación de la configuración para acceder a un directorio LDAP.

  15. Si desea que el ordenador acceda al directorio LDAP para el que ha creado una configuración, añada el directorio a una política de búsqueda personalizada en los paneles Autenticación y Contactos de "Política de búsqueda" en Utilidad de Directorios y compruebe que LDAPv3 esté activado en el panel Servicios.

Para obtener más información, consulte Cómo activar y desactivar servicios de directorio LDAP y Cómo definir políticas de búsqueda personalizadas.

Nota:  antes de poder utilizar Workgroup Manager para crear usuarios en un servidor LDAP que no sea de Apple y que utilice asignaciones RFC 2307 (UNIX), deberá editar la asignación del tipo de registro Usuarios. Para obtener más información, consulte Cómo editar mapas de RFC 2307 para activar la creación de usuarios.
Importante:  si cambia la dirección IP y el nombre de ordenador mediante changeip mientras está conectado a un servidor de directorios, deberá desconectarse y conectarse de nuevo a dicho servidor para actualizar el directorio con la dirección IP y el nombre de ordenador nuevos. De lo contrario, el directorio no se actualizará y continuará utilizando el nombre de ordenador y la dirección IP antiguos.
