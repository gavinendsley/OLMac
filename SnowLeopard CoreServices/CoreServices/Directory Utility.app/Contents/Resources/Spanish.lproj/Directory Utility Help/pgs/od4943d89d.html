Información acerca del acceso a Active Directory

Puede configurar Mac OS X para acceder a información básica de las cuentas de usuario en un dominio de Active Directory de un servidor Windows 2000 o posterior gracias a un conector de Active Directory para Utilidad de Directorios. Este conector de Active Directory se muestra en el panel Servicios de Utilidad de Directorios.

No es necesario efectuar cambios en el esquema del dominio de Active Directory para obtener información básica de la cuenta de usuario. Puede cambiar la lista de control de acceso (ACL) por omisión de atributos específicos para que las cuentas del ordenador puedan leer las propiedades de los usuarios.

El conector de Active Directory genera todos los atributos necesarios para la autenticación de Mac OS X a partir de los atributos estándar de las cuentas de usuario de Active Directory. Este conector también controla las políticas de autenticación de Active Directory, entre las que se incluyen cambios, caducidades, cambios forzados y opciones de seguridad de las contraseñas.

Mac OS X 10.6 admite opciones de firma de paquetes y encriptación de paquetes en todos los dominios Active Directory de Windows. Esta funcionalidad está activada por omisión como "permitir". Puede cambiar el ajuste por omisión a desactivado u obligatorio con la herramienta de línea de comandos dsconfigad. Las opciones de encriptación y de firma de paquetes garantizan la protección de todos los datos que entran y salen del dominio de Active Directory para las búsquedas de registros.

El conector de Active Directory genera dinámicamente un ID de usuario único y un ID de grupo primario basados en el ID único global (GUID) de la cuenta de usuario del dominio de Active Directory. Los ID de usuario y de grupo primario generados son los mismos para cada cuenta de usuario, incluso si la cuenta se utiliza para iniciar sesión en distintos ordenadores Mac OS X.

Por otro lado, se puede forzar que el conector de Active Directory asigne el ID de usuario a los atributos de Active Directory que se indiquen.

El conector de Active Directory genera un ID de grupo basado en el GUID de la cuenta de grupo de Active Directory. También puede forzar al módulo para que asigne el ID de grupo de las cuentas de grupo a los atributos de Active Directory que se indiquen.

Cuando un usuario inicia una sesión en un sistema Mac OS X con una cuenta de usuario de Active Directory, el conector de Active Directory puede montar la carpeta de inicio de red de Windows especificada en la cuenta de usuario de Active Directory como carpeta de inicio de Mac OS X del usuario. Puede especificar si desea utilizar la carpeta de inicio de red especificada en el atributo estándar “home Directory” de Active Directory o en el atributo “home Directory” de Mac OS X (en caso de que se haya ampliado el esquema de Active Directory para incluirlo).

También puede configurar el módulo para crear una carpeta de inicio local en el volumen de arranque del ordenador cliente con Mac OS X. En este caso, el módulo también monta la carpeta de inicio de red de Windows del usuario (que se especifica en la cuenta de usuario de Active Directory) como un volumen de red, es decir, como un punto de volumen compartido. Usando el Finder, el usuario puede a continuación copiar archivos entre el volumen de red de la carpeta de inicio de Windows y la carpeta de inicio local de Mac OS X.

El conector de Active Directory también permite crear cuentas móviles para los usuarios. Una cuenta móvil dispone de una carpeta de inicio local en el volumen de arranque del ordenador cliente con Mac OS X. (El usuario dispone también de una carpeta de inicio de red, tal y como se especifica en la cuenta de Active Directory del usuario).

Una cuenta móvil almacena en caché las credenciales de autenticación de Active Directory del usuario en el ordenador cliente con Mac OS X. Las credenciales almacenadas en caché permiten al usuario iniciar una sesión mediante el nombre y la contraseña de Active Directory cuando el ordenador cliente no está conectado al servidor Active Directory.

Una cuenta móvil dispone de una carpeta de inicio local en el volumen de arranque del ordenador cliente con Mac OS X. (El usuario dispone también de una carpeta de inicio de red, tal y como se especifica en la cuenta de Active Directory del usuario).

Si el esquema de Active Directory se ha ampliado para incluir tipos (clases de objeto) y atributos de registro de Mac OS X, el conector de Active Directory los detectará y accederá a ellos.

Por ejemplo, el esquema de Active Directory podría cambiarse utilizando las herramientas administrativas de Windows para incluir atributos de cliente gestionado de Mac OS X. Este cambio del esquema permite que el conector de Active Directory admita ajustes de cliente gestionado introducidos mediante la aplicación Workgroup Manager de Mac OS X Server.

Los clientes Mac OS X adoptan acceso de lectura total de los atributos que se añadan al directorio. Por consiguiente, puede ser necesario cambiar la ACL de dichos atributos para que los grupos del ordenador puedan leer estos atributos adicionales.

El conector de Active Directory detecta todos los dominios de un bosque de Active Directory. Puede configurar el módulo para permitir a los usuarios de cualquier dominio del dominio root (bosque) autenticarse en un ordenador Mac OS X. También puede permitir que solo se autentiquen dominios específicos en el cliente.

El conector de Active Directory es totalmente compatible con la replicación y la tolerancia a fallos de Active Directory. Detecta múltiples controladores de dominio y determina cuál es el más cercano. Si un controlador de dominio deja de estar disponible, el módulo recurre a otro controlador de dominio próximo.

El conector de Active Directory utiliza LDAP para acceder a las cuentas de usuario de Active Directory y Kerberos para autenticarlas. No utiliza las interfaces de servicio ADSI (Active Directory Services Interface) de Microsoft para obtener servicios de autenticación o de directorio.
