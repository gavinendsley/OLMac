Configuración del acceso LDAP a dominios de Active Directory

Mediante Utilidad de Directorios, puede realizar una configuración de LDAPv3 para acceder a un dominio de Active Directory en un servidor Windows. Una configuración LDAPv3 le proporciona control completo sobre la asignación de tipos y atributos de registro Mac OS X a clases de objetos, bases de búsqueda y atributos de Active Directory.

La asignación de algunos tipos y atributos de registro Mac OS X importantes, tal como el ID único de usuario (UID), requiere la ampliación del esquema de Active Directory.

Una configuración LDAPv3 no incluye las siguientes características del conector de Active Directory que figuran en Utilidad de Directorios.

  • Generación dinámica del ID único de usuario y del ID de grupo primario

  • Creación de una carpeta de inicio local de Mac OS X

  • Montaje automático de la carpeta de inicio de Windows

  • Cuentas de usuario móviles con credenciales de autenticación en memoria caché

  • Detección de todos los dominios en un bosque de Active Directory

  • Compatibilidad con la replicación y la tolerancia a fallos de Active Directory

Para obtener más información, consulte Información acerca del acceso a Active Directory.

Para crear una configuración de servidor Active Directory:

  1. Abra Preferencias del Sistema y haga clic en Cuentas.

  2. Si el icono del candado está cerrado, desbloquéelo haciendo clic en él e introduciendo el nombre y la contraseña de un administrador.

  3. Haga clic en "Opc. inicio sesión" y luego haga clic en Acceder o Editar.

  4. Haga clic en “Abrir Utilidad de Directorios”.

  5. Si el icono del candado está cerrado, desbloquéelo haciendo clic en él e introduciendo el nombre y la contraseña de un administrador.

  6. Haga clic en Servicios.

  7. En la lista de servicios, seleccione LDAPv3 y haga clic en el botón Editar (/).

  8. Haga clic en Nuevo e introduzca el nombre DNS o la dirección IP del servidor Active Directory.

  9. Seleccione una de las siguientes opciones de acceso al directorio y, a continuación, haga clic en Continuar para que Utilidad de Directorios obtenga información del servidor Active Directory.

    • Seleccione “Encriptar con SSL” si desea que Open Directory utilice conexiones SSL con el servidor Active Directory. Antes de seleccionar la opción SSL, pregunte al administrador de Open Directory si es necesario utilizar SSL.

    • Seleccione “Usar para autenticar” si este directorio contiene cuentas de usuario que alguien va a utilizar para iniciar sesión o autenticarse en los servicios.

    • Seleccione “Usar para los contactos” si este directorio contiene direcciones de correo y otra información que desee utilizar en la Agenda.

    Si Utilidad de Directorios no puede establecer contacto con el servidor Active Directory, se mostrará un mensaje y tendrá que configurar el acceso manualmente o cancelar el proceso de configuración. Para obtener más información, consulte Cómo configurar el acceso a un directorio LDAP manualmente.

    Si había seleccionado la opción "Usar para autenticar" o "Usar para los contactos", la conexión LDAPv3 al dominio de Active Directory se añadirá automáticamente a una política de búsqueda personalizada en los paneles Autenticación o Contactos de Utilidad de Directorios.

    Asegúrese de que la opción LDAPv3 esté activada en el panel Servicios para que el ordenador utilice la conexión que ha configurado. Para obtener más información, consulte Cómo activar y desactivar servicios de directorio LDAP.

  10. Cuando el cuadro de diálogo se expande para mostrar opciones de asignación, seleccione Active Directory en el menú local, introduzca la base de búsqueda y, a continuación, haga clic en Continuar.

    La plantilla de asignación de Active Directory para una configuración LDAPv3 asigna algunos tipos y atributos de registro Mac OS X a clases y atributos de objetos que no forman parte del esquema estándar de Active Directory. Puede cambiar las asignaciones definidas por la plantilla o ampliar el esquema de Active Directory.

    Por otro lado, también podría acceder a su dominio de Active Directory mediante el conector de Active Directory en lugar de utilizar LDAPv3. Para obtener más información, consulte Configuración del acceso a un dominio de Active Directory.

  11. Cuando el cuadro de diálogo se expanda para mostrar las opciones de conexión, escriba el nombre y la contraseña de una cuenta de usuario de Active Directory.

  12. Haga clic en Aceptar para terminar de crear la conexión LDAP.

  13. Haga clic en Aceptar para terminar de configurar las opciones LDAPv3.
