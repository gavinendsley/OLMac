Configuración del acceso a un dominio de Active Directory

Utilizando el conector de Active Directory que aparece en Utilidad de Directorios, puede configurar Mac OS X para acceder a información básica de las cuentas de usuario de un dominio de Active Directory de un servidor Windows.

El conector de Active Directory genera todos los atributos necesarios para la autenticación en Mac OS X. No son necesarios cambios en el esquema de Active Directory.

El conector de Active Directory detecta y accede a tipos y atributos de registro estándar de Mac OS X (como los atributos necesarios para la administración de clientes de Mac OS X), si el esquema de Active Directory ha sido ampliado para incluirlos.

ADVERTENCIA:  las opciones avanzadas del conector de Active Directory permiten asignar el ID de usuario único (UID), ID de grupo primario (GID) y el atributo GID de grupo de Mac OS X a los atributos apropiados que se hayan añadido al esquema de Active Directory. Si se cambia el ajuste de alguna de estas opciones de asignación posteriormente, los usuarios pueden perder el acceso a los archivos creados anteriormente.
Importante:  si el nombre de su ordenador contiene un guión, es posible que no pueda añadirse o enlazarse a un dominio de directorio como LDAP o Active Directory. Para establecer un enlace, utilice un nombre de ordenador que no contenga un guión.
Para configurar el acceso a un dominio de Active Directory:

  1. Abra Preferencias del Sistema y haga clic en Cuentas.

  2. Si el icono del candado está cerrado, desbloquéelo haciendo clic en él e introduciendo el nombre y la contraseña de un administrador.

  3. Haga clic en "Opc. inicio sesión" y luego haga clic en Acceder o Editar.

  4. Haga clic en “Abrir Utilidad de Directorios”.

  5. Si el icono del candado está cerrado, desbloquéelo haciendo clic en él e introduciendo el nombre y la contraseña de un administrador.

  6. Haga clic en Servicios.

  7. En la lista de servicios, seleccione "Active Directory" y haga clic en el botón Editar (/).

  8. Introduzca el nombre DNS del dominio de Active Directory al que desea enlazar el ordenador que está configurando.

    Puede consultar al administrador del dominio de Active Directory el nombre DNS que debe introducir.

  9. Si es necesario, modifique el ID del ordenador.

    El ID del ordenador es el nombre por el que se conoce al ordenador en el dominio de Active Directory. El ajuste predefinido corresponde al nombre del ordenador. Puede que tenga que cambiarlo para cumplir el esquema establecido de nomenclatura de ordenadores de su empresa en el dominio de Active Directory. Si no está seguro, consulte con el administrador del dominio de Active Directory.

  10. (Opcional) Ajuste las posiciones avanzadas.

    Si las opciones avanzadas están ocultas, haga clic en "Mostrar opciones avanzadas" y defina las opciones en los paneles "Opciones del usuario", "Asociaciones" y "Administración". También puede modificar más adelante los ajustes de las opciones avanzadas.

    Para obtener más información acerca de las opciones avanzadas, consulte:

    • Cómo configurar cuentas de usuario móviles en Active Directory

    • Cómo configurar carpetas de inicio para cuentas de usuario de Active Directory

    • Cómo definir una shell de UNIX para cuentas de usuario de Active Directory

    • Asignación del Identificador de usuario (UID) a un atributo de Active Directory

    • Asignación del ID de grupo primario a un atributo de Active Directory

    • Asignación del ID de grupo de cuentas de grupo a un atributo de Active Directory

    • Especificación del servidor Active Directory preferido

    • Cambio de los grupos de Active Directory que pueden administrar el ordenador

    • Control de autenticaciones desde todos los dominios del bosque (dominio root) de Active Directory

  11. Haga clic en Enlazar, use la información siguiente para autenticarse como usuario con privilegios para enlazar un ordenador con el dominio de Active Directory, seleccione las políticas de búsqueda a las que desee añadir Active Directory (véase abajo) y haga clic en Aceptar.

    • Nombre de usuario y contraseña: es posible que pueda autenticarse introduciendo el nombre y la contraseña de su cuenta de usuario de Active Directory, o puede que el administrador del dominio de Active Directory tenga que proporcionarle un nombre y una contraseña.

    • OU ordenador: introduzca la unidad organizativa (OU, "organizational unit") del ordenador que está configurando.

    • Usar para autenticar: utilice esta opción para determinar si Active Directory se añade a la política de búsqueda de autenticación del ordenador.

    • Usar para los contactos: utilice esta opción para determinar si Active Directory se añade a la política de búsqueda de contactos del ordenador.

    Cuando haga clic en Aceptar, Utilidad de Directorios configurará un vínculo fiable entre el ordenador que está configurando y el servidor Active Directory. Las políticas de búsqueda del ordenador se configuran de acuerdo con las opciones seleccionadas en el momento de la autenticación y Active Directory se activa en el panel Servicios de Utilidad de Directorios.

    Con los ajustes por omisión de las opciones avanzadas de Active Directory, el bosque de Active Directory se añade a la política de búsqueda de autenticación y a la política de búsqueda de contactos del ordenador en caso de que haya seleccionado "Usar para autenticar" o "Usar para los contactos".

    Sin embargo, si anula la selección de la opción "Permitir la autenticación desde cualquier dominio del bosque" en el panel de opciones avanzadas Administración antes de haber hecho clic en Enlazar, se añadirá el dominio de Active Directory más próximo en lugar del bosque.

    Podrá cambiar las políticas de búsqueda más adelante añadiendo o eliminado el bosque de Active Directory o dominios individuales. Para obtener más información, consulte Cómo definir políticas de búsqueda personalizadas.

  12. (Opcional) Conecte el servidor al reino Kerberos de Active Directory.

    • En el servidor o en el ordenador de un administrador que pueda conectarse al servidor, abra Server Admin y seleccione Open Directory para el servidor.

    • Haga clic en Settings y, a continuación, haga clic en General.

    • Haga clic en Join Kerberos y, a continuación, seleccione el reino Kerberos de Active Directory en el menú local e introduzca las credenciales de un administrador local de este servidor.
