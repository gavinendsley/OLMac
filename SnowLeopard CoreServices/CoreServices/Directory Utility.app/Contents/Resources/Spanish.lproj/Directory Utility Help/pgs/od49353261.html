Cómo cambiar la política de seguridad de una conexión LDAP

Mediante Utilidad de Directorios, puede configurar una política de seguridad para una conexión LDAPv3 más estricta que la política de seguridad del directorio LDAP. Por ejemplo, si la política de seguridad del directorio LDAP admite contraseñas de texto no cifrado, puede definir una conexión LDAPv3 para que no permita contraseñas de texto no cifrado.

La definición de una política de seguridad más estricta protege al ordenador de hackers malintencionados que intenten utilizar un servidor LDAP falso para obtener el control de su ordenador.

El ordenador debe comunicarse con el servidor LDAP para mostrar el estado de las opciones de seguridad. Por lo tanto, cuando cambie las opciones de seguridad de una conexión LDAPv3, la política de búsqueda de autenticación del ordenador deberá incluir la conexión LDAPv3.

Los ajustes que las opciones de seguridad de una conexión LDAPv3 pueden admitir están sujetos a los requisitos y funciones de seguridad del servidor LDAP. Por ejemplo, si el servidor LDAP no admite el sistema de autenticación Kerberos, varias opciones de seguridad de la conexión LDAPv3 estarán desactivadas.

Para cambiar las opciones de seguridad de una conexión LDAPv3:

  1. Abra Preferencias del Sistema y haga clic en Cuentas.

  2. Si el icono del candado está cerrado, desbloquéelo haciendo clic en él e introduciendo el nombre y la contraseña de un administrador.

  3. Haga clic en "Opc. inicio sesión" y luego haga clic en Editar.

  4. Haga clic en “Abrir Utilidad de Directorios”.

  5. Si el icono del candado está cerrado, desbloquéelo haciendo clic en él e introduciendo el nombre y la contraseña de un administrador.

  6. Haga clic en “Política de búsqueda”.

  7. Haga clic en Autenticación y compruebe que el directorio LDAPv3 que desee figure en la política de búsqueda.

    Para obtener más información acerca de cómo añadir un directorio LDAPv3 a la política de búsqueda de autenticación, consulte Cómo definir políticas de búsqueda personalizadas.

  8. Haga clic en Servicios.

  9. En la lista de servicios, seleccione LDAPv3 y haga clic en el botón Editar (/).

  10. Si la lista de configuraciones del servidor está oculta, haga clic en "Mostrar opciones".

  11. Seleccione la configuración del directorio que desee y, a continuación, haga clic en Editar.

  12. Haga clic en Seguridad y, a continuación, cambie cualquiera de los ajustes siguientes:

    Nota:  estos ajustes de seguridad y los del servidor LDAP correspondiente se determinan cuando se configura la conexión LDAP. Esos ajustes no se actualizan cuando se cambian los ajustes del servidor.

    Si alguna de las cuatro últimas opciones está seleccionada pero desactivada, significa que es necesaria para el directorio LDAP. Si alguna de estas opciones no está seleccionada y está desactivada, significa que el servidor LDAP no la admite.

    • Usar autenticación para conectar: determina si la conexión LDAPv3 se autentica a sí misma en el directorio LDAP suministrando el nombre y la contraseña especificados. Esta opción no se muestra si la conexión LDAPv3 utiliza un vínculo fiable con el directorio LDAP.

    • Enlazado con el directorio como: especifica las credenciales que la conexión LDAPv3 utiliza para el vínculo fiable con el directorio LDAP. Ni esta opción ni las credenciales pueden modificarse desde aquí. En lugar de ello, puede anular el enlace y, a continuación, volver a establecer un enlace con distintas credenciales.

      Para obtener más información, consulte Cómo detener un vínculo fiable con un directorio LDAP y Cómo configurar un vínculo fiable con un directorio LDAP.

      Esta opción no se mostrará, salvo que la conexión LDAPv3 utilice el vínculo fiable.

    • Desactivar contraseñas de texto no encriptado: determina si la contraseña se va a enviar como texto no cifrado en caso de que no pueda validarse mediante un método de autenticación que envíe una contraseña encriptada.

    • Firmar digitalmente todos los paquetes (requiere Kerberos): certifica que ningún otro ordenador ha interceptado ni modificado los datos de directorio del servidor LDAP mientras se enviaban a su ordenador.

    • Encriptar todos los paquetes (requiere SSL o Kerberos): requiere que el servidor LDAP encripte los datos de directorio mediante SSL o Kerberos antes de enviarlos a su ordenador. Antes de seleccionar la opción “Encriptar todos los paquetes (requiere SSL o Kerberos)”, pregunte a su administrador de Open Directory si necesita utilizar SSL.

    • Bloquear los ataques “Man-in-the-Middle” (requiere Kerberos): protege frente a servidores falsos que simulan ser el servidor LDAP. Se recomienda utilizarla junto con la opción “Firmar digitalmente todos los paquetes”.
