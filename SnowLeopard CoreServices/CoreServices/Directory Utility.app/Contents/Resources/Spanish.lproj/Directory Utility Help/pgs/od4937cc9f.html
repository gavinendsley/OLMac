Cómo configurar el acceso a un directorio LDAP

Con Utilidad de Directorios, puede especificar cómo debe acceder Mac OS X a un directorio LDAPv3 si sabe el nombre DNS o la dirección IP del servidor de directorios LDAP.

Si el directorio no está alojado en un servidor que suministre sus propias asignaciones, como Mac OS X Server, deberá saber la base de búsqueda y la plantilla para asignar los datos de Mac OS X a los datos del directorio.

Las plantillas de asignación compatibles son las siguientes:

  • Servidor Open Directory, para un directorio que utilice el esquema de Mac OS X Server.

  • Active Directory, para un directorio alojado en un servidor Windows 2000, Windows 2003 o un servidor posterior.

  • RFC 2307, para la mayoría de los directorios alojados en servidores UNIX.

El módulo LDAPv3 es totalmente compatible con la replicación y la conmutación por errores de Open Directory. Si el máster de Open Directory deja de estar disponible, el módulo recurre a una réplica próxima.

Para especificar asignaciones personalizadas para los datos del directorio, siga las instrucciones de Cómo configurar el acceso a un directorio LDAP manualmente en lugar de estas instrucciones.

Importante:  si el nombre de su ordenador contiene un guión, es posible que no pueda acceder o enlazarse a un dominio de directorio como LDAP o Active Directory. Para establecer un enlace, utilice un nombre de ordenador que no contenga un guión.
Para que Utilidad de Directorios le ayude a configurar el acceso a un directorio LDAP:

  1. Abra Preferencias del Sistema y haga clic en Cuentas.

  2. Si el icono del candado está cerrado, desbloquéelo haciendo clic en él e introduciendo el nombre y la contraseña de un administrador.

  3. Haga clic en "Opc. inicio sesión" y luego haga clic en Acceder o Editar.

  4. Haga clic en “Abrir Utilidad de Directorios”.

  5. Si el icono del candado está cerrado, desbloquéelo haciendo clic en él e introduciendo el nombre y la contraseña de un administrador.

  6. Haga clic en Servicios.

  7. En la lista de servicios, seleccione LDAPv3 y haga clic en el botón Editar (/).

    Puede seleccionar LDAPv3 en la lista de servicios sin seleccionar la opción Activar correspondiente a LDAPv3.

  8. Haga clic en Nuevo e introduzca el nombre DNS o la dirección IP del servidor LDAP.

  9. Seleccione las opciones para acceder al directorio:

    • Seleccione “Encriptar con SSL” si desea que Open Directory utilice SSL (Secure Sockets Layer) para realizar las conexiones con el directorio LDAP. Antes de seleccionar esta opción, pregunte a su administrador de Open Directory si es necesario utilizar SSL.

    • Seleccione “Usar para autenticar” si este directorio contiene cuentas de usuario que alguien va a utilizar para el inicio de sesión o la autenticación en los servicios.

    • Seleccione “Usar para los contactos” si este directorio contiene direcciones de correo y otra información que desee utilizar en la Agenda.

    Si Utilidad de Directorios no puede establecer contacto con el servidor LDAP, se mostrará un mensaje y tendrá que configurar el acceso manualmente o cancelar el proceso de configuración. Para obtener más información acerca de la configuración manual, consulte Cómo configurar el acceso a un directorio LDAP manualmente.

    Si el cuadro de diálogo se expande para mostrar las opciones de asignaciones, seleccione la plantilla de asignación que desee en el menú local, introduzca el sufijo de la base de búsqueda y, a continuación, haga clic en Continuar.

    Normalmente, el sufijo de la base de búsqueda deriva del nombre del DNS del servidor. Por ejemplo, el sufijo de la base de búsqueda podría ser "dc=ods,dc=ejemplo, dc=com" para un servidor cuyo nombre DNS sea "ods.ejemplo.com".

    Si ninguna de las plantillas de asignación disponibles es adecuada para la conexión que está configurando, haga clic en Manual. Para obtener más información, consulte Cómo configurar el acceso a un directorio LDAP manualmente.

  10. Para que Utilidad de Directorios obtenga información del servidor LDAP, haga clic en Continuar.

  11. Si el cuadro de diálogo se expande para mostrar las opciones de vínculo fiable, introduzca el nombre del ordenador y el nombre y la contraseña de un administrador del directorio. (El enlace puede ser opcional).

    El cuadro de diálogo le indica si el directorio LDAP requiere vínculos fiables o si es opcional. El vínculo fiable es mutuo: cada vez que el ordenador se conecta al directorio LDAP, ambos se autentican entre sí. Si se ha configurado un vínculo fiable o si el directorio LDAP no admite vínculos fiables, no se mostrará el botón Enlazar. Compruebe que ha introducido el nombre del ordenador correcto.

    Si se muestra un aviso que indica que existe un registro del ordenador, haga clic en Cancelar para volver atrás y cambiar el nombre del ordenador o haga clic en Borrar para reemplazar el registro del ordenador existente.

    El registro del ordenador existente puede haberse abandonado o puede pertenecer a otro ordenador.

    Si decide reemplazar un registro del ordenador existente, comuníqueselo al administrador del directorio LDAP por si al reemplazar el registro se desactiva otro ordenador. En ese caso, el administrador del directorio LDAP deberá dar otro nombre al ordenador desactivado y añadirlo de nuevo al grupo de ordenadores al que pertenecía.

  12. Si el cuadro de diálogo se expande para mostrar las opciones de conexión, seleccione “Usar autenticación para conectar” e introduzca el nombre y la contraseña de una cuenta de usuario del directorio.

    Aparecerán las opciones para una conexión autenticada si el servidor LDAP admite una conexión de este tipo pero no un vínculo fiable. Una conexión autenticada no es mutua: el servidor LDAP autentica al cliente, pero el cliente no autentica al servidor.

    La opción "Usar autenticación para conectar" ya está previamente seleccionada, pero no estará disponible si el servidor LDAP requiere que introduzca el nombre y la contraseña de una cuenta de usuario para una conexión autenticada.

    El nombre puede hacer referencia a cualquier cuenta de usuario que tenga permiso para ver los datos del directorio. Por ejemplo, una cuenta de usuario cuyo nombre corto sea "diraut" en un servidor LDAP y cuya dirección sea "ods.ejemplo.com" tendría el nombre "uid=diraut,cn=usuarios,dc=ods,dc=ejemplo,dc=com".

    Importante:  si el nombre o la contraseña son incorrectos, puede iniciar sesión en el ordenador utilizando las cuentas de usuario del directorio LDAP.

  13. Haga clic en Aceptar para terminar de crear la conexión LDAP.

  14. Haga clic en Aceptar para terminar de configurar las opciones LDAPv3.

Si había seleccionado la opción "Usar para autenticar" o "Usar para los contactos" en el paso 5, la configuración del directorio LDAP que acaba de crear se añadirá automáticamente a una política de búsqueda personalizada en los paneles Autenticación o Contactos de Utilidad de Directorios.

Asegúrese de que la opción LDAPv3 esté activada en el panel Servicios para que el ordenador utilice la configuración LDAP que acaba de crear. Para obtener más información, consulte Cómo activar y desactivar servicios de directorio LDAP.
