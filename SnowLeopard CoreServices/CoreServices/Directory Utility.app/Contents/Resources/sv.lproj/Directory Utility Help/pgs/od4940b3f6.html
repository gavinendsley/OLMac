Ställa in en UNIX-kommandotolk för användarkonton i Active Directory

På en dator som är konfigurerad till att använda Katalogverktygs Active Directory-katalogkan du ställa in vilken kommandoradstolk som användare med Active Directory-konton kommer att använda som förval när de arbetar i Mac OS X via programmet Terminal.

Den förvalda kommandotolken kommer också att användas för fjärranvändning via SSH (Secure Shell) och Telnet. Varje användare kan välja en annan kommandotolk än den förvalda genom att ändra en Terminal-inställning.

Så här ställer du in en UNIX-kommandotolk för användarkonton i Active Directory:

  1. Öppna Systeminställningar och klicka på Konton.

  2. Om symbolen med ett hänglås är låst så låser du upp den genom att klicka på den och anger ett administratörsnamn och -lösenord.

  3. Klicka på Inloggning och sedan på Anslut eller Redigera.

  4. Klicka på Katalogverktyg.

  5. Om symbolen med ett hänglås är låst så låser du upp den genom att klicka på den och anger ett administratörsnamn och -lösenord.

  6. Klicka på Tjänster.

  7. Markera Active Directory i listan över tjänster och klicka sedan på redigeraknappen (/).

  8. Om de avancerade alternativen är dolda klickar du på Visa avancerade alternativ.

  9. Klicka på Användarupplevelse.

  10. Välj förvald användarkommandotolk och ange sökvägen för den förvalda användarkommandotolken.

  11. Klicka på OK.
