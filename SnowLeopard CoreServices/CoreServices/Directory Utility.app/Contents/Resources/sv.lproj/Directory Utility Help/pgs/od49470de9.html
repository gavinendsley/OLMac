Definiera automatiska sökpolicyer

Med Katalogverktyg kan du ställa in att sökpolicyerna för autentisering och kontakter i Mac OS X ska definieras automatiskt.

En automatiskt definierad sökpolicy innefattar den lokala katalogdomänen. Den kan också innehålla en LDAP-katalog som anges av DHCP-tjänsten.

Detta är den förvalda inställningen för sökpolicierna för autentisering och kontakter.

Anm:   Vissa program, t.ex. Mac OS X Mail och Adressbok, kan kontakta LDAP-kataloger direkt, utan att använda Open Directory. Du ställer in att dessa program ska kontakta LDAP-kataloger direkt genom att öppna programmet och göra lämpliga inställningar.
Viktigt:   Om du konfigurerar Mac OS X till att använda en automatisk sökpolicy för autentisering och en LDAP-server eller delad katalogdomän angiven med DHCP, ökar risken för att en användare med icke-legitima avsikter tar kontroll över datorn. Risken ökar ytterligare om datorn är konfigurerad så att den kan ansluta till ett trådlöst nätverk. Mer information finns i hjälpavsnittet Skydda datorer från icke-legitima DHCP-servrar.
Så här anger du automatisk definiering av en sökpolicy:

  1. Öppna Systeminställningar och klicka på Konton.

  2. Om symbolen med ett hänglås är låst så låser du upp den genom att klicka på den och anger ett administratörsnamn och -lösenord.

  3. Klicka på Inloggning och sedan på Anslut eller Redigera.

  4. Klicka på Katalogverktyg.

  5. Om symbolen med ett hänglås är låst så låser du upp den genom att klicka på den och anger ett administratörsnamn och -lösenord.

  6. Klicka på Sökpolicy och välj en sökpolicy:

    • Autentisering: Visar sökpolicyn som används för autentisering och de flesta andra administrativa data.

    • Kontakter: Visar sökpolicyn som används för kontaktinformation i program som Adressbok.

  7. Välj Automatisk från popupmenyn Sök och klicka sedan på Utför.

  8. Kontrollera att inställningspanelen Nätverk i Systeminställningar är inställd på att använda DHCP eller DHCP med en manuellt angiven IP-adress.
