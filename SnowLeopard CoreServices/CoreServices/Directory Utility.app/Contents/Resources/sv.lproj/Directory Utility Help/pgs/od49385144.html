Ställa in hemmappar för användarkonton i Active Directory

På en dator som är konfigurerad till att använda Katalogverktygs Active Directory-kontakt kan du aktivera eller avaktivera användningen av nätverkshemmappar eller lokala hemmappar för Active Directory-användarkonton.

Med nätverkshemmappar kommer en användares Windows-nätverksmapp att visas som Mac OS X-hemmapp när användaren loggar in.

Du väljer om platsen för nätverkshemmappen hämtas från Active Directorys standard-homeDirectory-attribut eller från HomeDirectory-attributet i Mac OS X, om Active Directory-schemat har utökats till att inkludera det.

Med lokala hemmappar har varje Active Directory-användare som loggar in en hemmapp på startenheten för Mac OS X. Därtill visas användarens nätverkshemmapp som en nätverksvolym, som en delningspunkt. Användaren kan kopiera filer mellan nätverksvolymen och den lokala hemmappen.

Så här ställer du in hemmappar för Active Directory-användarkonton:

  1. Öppna Systeminställningar och klicka på Konton.

  2. Om symbolen med ett hänglås är låst så låser du upp den genom att klicka på den och anger ett administratörsnamn och -lösenord.

  3. Klicka på Inloggning och sedan på Anslut eller Redigera.

  4. Klicka på Katalogverktyg.

  5. Om symbolen med ett hänglås är låst så låser du upp den genom att klicka på den och anger ett administratörsnamn och -lösenord.

  6. Klicka på Tjänster.

  7. Markera Active Directory i listan över tjänster och klicka sedan på redigeraknappen (/).

  8. Om de avancerade alternativen är dolda klickar du på Visa avancerade alternativ.

  9. Klicka på Användarupplevelse.

  10. Klicka på ”Tvinga lokal hemmapp på startskiva” om du vill att Active Directory-användarkonton ska ha lokala hemmappar i mappen Användare på datorn.

    Alternativet finns inte tillgängligt om ”Skapa flyttbart konto vid inloggning” är valt.

  11. Vill du använda Active Directorys standardattribut för hemmappens plats, väljer du ”Använd UNC-sökväg från Active Directory för att få fram nätverkets hemplats” och sedan från följande protokoll för åtkomst till hemmappen:

    • Vill du använda Windows-standardprotokollet SMB väljer du smb från popupmenyn ”Nätverksprotokoll som ska användas”.

    • Vill du använda Mac-standardprotokollet AFP väljer du afp från popupmenyn ”Nätverksprotokoll som ska användas”.

  12. Vill du använda Mac OS X-attributet för hemmappens plats avmarkerar du ”Använd UNC-sökväg från Active Directory för att få fram nätverkets hemplats”.

    Innan du kan använda Mac OS X-attributet måste Active Directory-schemat vara utökat till att inkludera det.

  13. Klicka på OK.

Om du byter namn på ett användarkonto i Active Directory-domänen skapar servern en hemmapp (och undermappar) för användarkontot nästa gång det används för inloggning till en Mac OS X-dator. Användaren kan fortfarande navigera till den gamla hemmappen och visa innehållet i Finder.

Du kan förhindra ett en ny hemmapp skapas genom att byta namn på den gamla mappen innan användaren loggar in nästa gång.
