Ställa in åtkomst till en Active Directory-domän

Med Active Directory-kontakten i Katalogverktyg kan du konfigurera Mac OS X så att systemet kommer åt grundläggande information om användarkonton i en Active Directory-domän på en Windows-server.

Active Directory-kontakten genererar alla attribut som behövs för Mac OS X-autentisering. Inga ändringar krävs i Active Directory-schemat.

Active Directory-kontakten upptäcker och använder standardposttyper och -attribut för Mac OS X (t.ex. de attribut som behövs för Mac OS X-klienthantering) om Active Directory-schemat har utökats och inkluderar dessa attribut.

VARNING: Med de avancerade alternativen i Active Directory-kontakten kan du koppla unika användar-ID (UID), primära grupp-ID (GID) och grupp-GID-attribut i Mac OS X till lämpliga attribut som har lagts till i Active Directory-schemat. Om du senare ändrar dessa inställningar kan det innebära att användare inte längre kommer åt filer som skapats före ändringen.
Viktigt:   Om datorns namn innehåller ett bindestreck kanske du inte kan ansluta eller koppla till en katalogdomän, t.ex. av typen LDAP eller Active Directory. Använd ett datornamn som inte innehåller något bindestreck när du ansluter.
Så här ställer du in åtkomst till en Active Directory-domän:

  1. Öppna Systeminställningar och klicka på Konton.

  2. Om symbolen med ett hänglås är låst så låser du upp den genom att klicka på den och anger ett administratörsnamn och -lösenord.

  3. Klicka på Inloggning och sedan på Anslut eller Redigera.

  4. Klicka på Katalogverktyg.

  5. Om symbolen med ett hänglås är låst så låser du upp den genom att klicka på den och anger ett administratörsnamn och -lösenord.

  6. Klicka på Tjänster.

  7. Markera Active Directory i listan över tjänster och klicka sedan på redigeraknappen (/).

  8. Ange DNS-namnet för den Active Directory-domän till vilken du vill koppla datorn du konfigurerar.

    Be Active Directory-domänens administratör om DNS-namnet.

  9. Om det behövs redigerar du dator-ID:t.

    Dator-ID:t är det namn datorn har i Active Directory-domänen och det är förinställt till datorns namn. Du kan ändra namnet och anpassa det till ditt företags etablerade namngivningsmönster för datorer i Active Directory-domänen. Om du är osäker frågar du Active Directory-domänens administratör.

  10. (Valfritt) Ange avancerade alternativ.

    Om de avancerade alternativen är gömda klickar du på Visa avancerade alternativ och ställer in alternativ i panelerna Användarupplevelse, Kopplingar och Administrativt. Du kan också ändra de avancerade alternativen senare.

    Vill du veta mer om avancerade alternativ läser du:

    • Ställa in flyttbara användarkonton i Active Directory Setting

    • Ställa in hemmappar för användarkonton i Active Directory

    • Ställa in en UNIX-kommandotolk för användarkonton i Active Directory

    • Koppla UID till ett Active Directory-attribut

    • Koppla primärt grupp-ID till ett Active Directory-attribut

    • Koppla grupp-ID i gruppkonton till ett Active Directory-attribut

    • Ange en förvald Active Directory-server

    • Ändra vilka Active Directory-grupper som kan administrera datorn

    • Kontrollera autentisering från alla domäner i Active Directory-trädsamlingen

  11. Klicka på Koppla, använd följande till att autentisera som en användare som har behörighet att koppla en dator till Active Directory-domänen, välj de sökpolicyer du vill att Active Directory ska läggas till i (se nedan) och klicka på OK:

    • Användarnamn och lösenord: Du kan eventuellt autentisera dig genom att ange namn och lösenord för ditt Active Directory-användarkonto, eller så kanske du måste få namn och lösenord från Active Directory-domänens administratör.

    • Dator-OU: Ange organisationsenhet (OU) för datorn du konfigurerar.

    • Använd för autentisering: Ange om Active Directory automatiskt ska läggas till i datorns sökpolicy för autentisering.

    • Använd för kontakter: Ange om Active Directory automatiskt ska läggas till i datorns sökpolicy för kontakter.

    När du klickar på OK ställer Katalogverktyg in en betrodd koppling mellan datorn du konfigurerar och Active Directory-servern. Datorns sökpolicyer ställs in i enlighet med de alternativ du valde när du autentiserade och Active Directory aktiveras i panelen Tjänster i Katalogverktyg.

    Med de förvalda inställningarna för avancerade alternativ i Active Directory läggs Active Directory-trädsamlingen till i datorns autentiseringssökpolicy och kontaktsökpolicy om du markerat ”Använd för autentisering” och ”Använd för kontakter”.

    Om du avmarkerar ”Tillåt autentisering från alla domäner i katalogträdet” under avancerade alternativ i panelen Administrativt innan du klickar på Koppla loss, läggs den närmaste Active Directory-domänen till istället för katalogträdet.

    Du kan ändra sökpolicyerna senare genom att lägga till eller ta bort Active Directory-trädsamlingen eller enskilda domäner. Mer information finns i avsnittet Definiera anpassade sökpolicyer.

  12. (Valfritt) Anslut servern till Active Directory Kerberos-området.

    • Öppna Server Admin på den server eller administratörsdator som kan ansluta till servern och välj Open Directory för servern.

    • Klicka på Inställningar och sedan på Allmänt.

    • Klicka på Anslut till Kerberos. Välj sedan Active Directory Kerberos-område från popupmenyn och ange ID-information för en lokal administratör på servern.
