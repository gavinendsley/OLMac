Om Active Directory-åtkomst

Du kan konfigurera Mac OS X så att systemet kommer åt grundläggande användarkontoinformation i Active Directory-domäner på servrar med Windows 2000 eller senare. Det är möjligt tack vare Active Directory-kontakten till Katalogverktyg. Du hittar Active Directory-kontakten i panelen Tjänster i Katalogverktyg.

Du behöver inte ändra Active Directory-domänens schema för att få grundläggande användarkontoinformation. Du kan behöva ändra standardbehörighetslistan för åtkomst för vissa attribut så att datorkontona ska kunna läsa användaregenskaperna.

Active Directory-kontakten genererar alla attribut som krävs för Mac OS X-autentisering från standardattribut i Active Directory-användarkonton. Kontakten hanterar också Active Directory-autentiseringspolicyer, inklusive lösenordsändringar, tidsbegränsningar, tvingade ändringar och säkerhetsalternativ.

Mac OS X 10.6 stöder både alternativ för paketkryptering och paketsignering för alla Active Directory-domäner i Windows. Den här funktionen är inställd som förval på ”tillåt”. Du kan ändra den förvalda inställningen till avaktiverad eller krävd genom att använda kommandoradsverktyget dsconfigad. Alternativen för paketkryptering och paketsignering säkerställer att alla data för postkontroll till och från Active Directory-domänen skyddas.

Active Directory-kontakten genererar dynamiskt ett unikt UID och ett primärt GID baserat på användarkontots GUID i Active Directory-domänen. Det genererade UID och primära GID är samma för varje användarkonto, även om kontot används vid inloggning på andra Mac OS X-datorer.

Ett annat alternativ är att tvinga Active Directory-kontakten att koppla UID till Active Directory-attribut som du anger.

Active Directory-kontakten genererar ett GID baserat på Active Directory-gruppkontots GUID. Du kan också tvinga insticksfilen att koppla GID för gruppkonton till Active Directory-attribut som du anger.

När någon loggar in till Mac OS X med ett Active Directory-användarkonto kan Active Directory-kontakten göra Windows-nätverkshemmappen som är angiven i Active Directory-användarkontot till användarens Mac OS X-hemmapp. Du kan ange om den nätverkshemmapp som är angiven av Active Directorys standard-homeDirectory-attribut eller den som är angiven av HomeDirectory-attributet i Mac OS X ska användas (om Active Directory-schemat har utökats till att inkludera det).

Alternativt så kan du konfigurera insticksfilen så att den skapar en lokal hemmapp på startvolymen på Mac OS X-klientdatorn. Om du gör det kommer insticksfilen visa användarens nätverkshemmapp (som angetts i Active Directory-användarkontot) som en nätverksvolym, som en delningspunkt. I Finder kan användaren sedan kopiera filer mellan Windows-hemmappen i nätverket och den lokala Mac OS X-hemmappen.

Active Directory-kontakten kan också skapa flyttbara konton för användare. Ett flyttbart konto har en lokal hemmapp på startenheten på Mac OS X-klientdatorn. (Användaren har också en nätverkshemmapp som är angiven i användarens Active Directory-konto.)

Ett flyttbart konto placerar användarens Active Directory-autentiseringsinformation i Mac OS X-klientdatorns cacheminne. Med ID-informationen i cacheminnet kan användaren logga in med Active Directory-namnet och -lösenordet när klientdatorn inte är ansluten till Active Directory-servern.

Ett flyttbart konto har en lokal hemmapp på startenheten på Mac OS X-klientdatorn. (Användaren har också en nätverkshemmapp som är angiven i användarens Active Directory-konto.)

Om Active Directory-schemat har utökats till att innehålla Mac OS X-posttyper (objektklasser) och -attribut upptäcker och använder Active Directory-kontaken dem.

Active Directory-schemat kan t.ex. ändras med hjälp av Windows-administratörsverktygen till att inkludera Mac OS X-attribut för hanterade klienter. En sådan schemaändring innebär att Active Directory-kontakten får stöd för inställningar för hanterade klienter som gjorts i Mac OS X Servers program Workgroup Manager.

Mac OS X-klienter förutsätter att de har fullständiga läsrättigheter för attribut som läggs till i katalogen. Det kan därför vara nödvändigt att ändra standardbehörighetslistorna för åtkomst för dessa attribut så att datorgrupper kan läsa dem.

Active Directory-kontakten upptäcker alla domäner i en Active Directory-trädsamling. Du kan konfigurera insticksfilen så att användare från alla domäner i trädsamlingen autentiseras på en Mac OS X-dator. Om du vill kan du istället ange att endast vissa domäner ska kunna autentiseras på klienten.

Active Directory-kontakten har fullständigt stöd för Active Directory-replikering och failover. Insticksfilen upptäcker om det finns flera domänövervakare och bestämmer vilken som är närmast. Om en domänövervakare blir otillgänglig byter programmet till en närliggande domänövervakare.

Active Directory-kontaken använder LDAP för åtkomst till Active Directory-användarkonton och Kerberos vid autentiseringen av dem. Active Directory-kontakten använder inte Microsofts ADSI (Active Directory Services Interface) för katalog- eller autentiseringstjänster.
