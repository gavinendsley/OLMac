Ställa in LDAP-åtkomst till Active Directory-domäner

Med Katalogverktyg kan du ställa in en LDAPv3-konfiguration så att den kan ansluta till en Active Directory-domän på en Windows-server. En LDAPv3-konfiguration ger dig full kontroll över kopplingen av posttyper och attribut i Mac OS X till objektklasser, sökdatabaser och attribut i Active Directory.

Vid koppling av vissa viktiga attribut och posttyper i Mac OS X, som unika UID (User ID), krävs att Active Directory-schemat utökas.

En LDAP v3-konfiguration saknar följande funktioner som Active Directory-kontakten i listan Katalogverktyg har:

  • Dynamisk generering av unika användar-ID och primära grupp-ID

  • Möjlighet att skapa en lokal Mac OS X-hemmapp

  • Möjlighet att lägga till Windows-hemmappen automatiskt

  • Flyttbara användarkonton med cachad autentiseringsinformation

  • Identifiering av alla domäner i en Active Directory-trädsamling

  • Stöd för Active Directory-replikering och failover

Mer information finns i avsnittet Om Active Directory-åtkomst.

Så här skapar du en Active Directory-serverkonfiguration:

  1. Öppna Systeminställningar och klicka på Konton.

  2. Om symbolen med ett hänglås är låst så låser du upp den genom att klicka på den och anger ett administratörsnamn och -lösenord.

  3. Klicka på Inloggning och sedan på Anslut eller Redigera.

  4. Klicka på Katalogverktyg.

  5. Om symbolen med ett hänglås är låst så låser du upp den genom att klicka på den och anger ett administratörsnamn och -lösenord.

  6. Klicka på Tjänster.

  7. Markera LDAPv3 i listan över tjänster och klicka sedan på redigeraknappen (/).

  8. Klicka på Ny och ange Active Directory-serverns DNS-namn eller IP-adress.

  9. Välj från följande alternativ för åtkomst till mappen och klicka på Fortsätt så hämtar Katalogverktyg informationen från Active Directory-servern

    • Markera ”Kryptera med SSL” om du vill att Open Directory ska använda SSL för anslutningar till Active Directory-servern. Fråga Open Directory-administratören om SSL behövs innan du slår på SSL.

    • Markera ”Använd för autentisering” om katalogen innehåller användarkonton som någon kommer att använda för inloggning eller autentisering av tjänster.

    • Markera ”Använd för kontakter” om katalogen innehåller e-postadresser och annan information du vill använda i Adressbok.

    Om Katalogverktyg inte kan kontakta Active Directory-servern visas ett meddelande och du måste konfigurera åtkomst manuellt eller avbryta inställningsproceduren. Mer information finns i avsnittet Ställa in åtkomst till en LDAP-katalog manuellt.

    Om du valde ”Använd för autentisering” eller ”Använd för kontakter” läggs LDAPv3-anslutningen till Active Directory-domänen till i en anpassad sökpolicy i panelen Autentisering eller Kontakter i Katalogverktyg.

    Kontrollera att LDAP v3 är aktiverat i panelen Tjänster så att datorn använder den anslutning du ställt in. Mer information finns i avsnittet Aktivera och avaktivera LDAP-katalogtjänster.

  10. När dialogrutan blir större och visar kopplingsalternativ väljer du Active Directory från popupmenyn, anger sökdatabas och klickar på Fortsätt.

    Active Directory-kopplingsmallen för en LDAPv3-konfiguration kopplar vissa Mac OS X-posttyper och -attribut till objektklasser och attribut som inte ingår i ett vanligt Active Directory-schema. Du kan ändra de kopplingar som definieras av mallen, eller utöka Active Directory-schemat.

    Det kan också vara möjligt att komma åt Active Directory-domänen via Active Directory-insticksfilen istället för LDAP v3. Mer information finns i avsnittet Ställa in åtkomst till en Active Directory-domän.

  11. När dialogrutan blir större och visar anslutningsalternativ, anger du det distingerade namnet och lösenordet för ett Active Directory-användarkonto.

  12. Slutför skapandet av LDAP-anslutningen genom att klicka på OK.

  13. Avsluta konfigureringen av LDAPv3-alternativ genom att klicka på OK.
