Ange inställningar för BSD-konfigurationsfiler

UNIX-datorer har länge sparat administrativa data i konfigurationsfiler som /etc/master.passwd, /etc/group och /etc/hosts. Mac OS X bygger på en BSD-version av UNIX, men får normalt administrativa data från katalogsystem.

Mac OS X Server hanterar en bestämd uppsättning BSD-konfigurationsfiler. Du kan inte ange vilka konfigurationsfiler som ska användas och inte heller kan du koppla deras innehåll till Mac OS X-posttyper eller -attribut.

I Mac OS X 10.2 eller senare (inklusive Mac OS X Server 10.2 eller senare) kan Open Directory hämta administrativa data från BSD-konfigurationsfiler. Detta gör det möjligt för organisationer som har BSD-konfigurationsfiler att använda kopior av de befintliga filerna på Mac OS X-datorer. BSD-konfigurationsfiler kan användas för sig själva eller tillsammans med andra katalogdomäner.

Så här använder du BSD-konfigurationsfiler:

  1. Kontrollera att BSD-konfigurationsfilen innehåller de data som krävs av Mac OS X-katalogtjänster.

    Mer information finns i avsnittet Göra inställningar för data i BSD-konfigurationsfiler.

  2. Öppna Systeminställningar och klicka på Konton.

  3. Om symbolen med ett hänglås är låst så låser du upp den genom att klicka på den och anger ett administratörsnamn och -lösenord.

  4. Klicka på Inloggning och sedan på Anslut eller Redigera.

  5. Klicka på Katalogverktyg.

  6. Om symbolen med ett hänglås är låst så låser du upp den genom att klicka på den och anger ett administratörsnamn och -lösenord.

  7. Klicka på Tjänster.

  8. Välj ”BSD Flat File och NIS” i listan över tjänster och klicka på redigeraknappen (/).

  9. Markera ”Använd poster för användare och grupper i BSD:s lokala nod” och klicka på OK.

    BSD-konfigurationsfilernas domän läggs till i datorns autentiseringssökpolicy som /BSD/local.
