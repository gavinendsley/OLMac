Stoppa betrodd koppling med en LDAP-katalog

Med Katalogverktyg kan du stoppa betrodd koppling mellan en dator och en LDAP-katalog som tillåter men inte kräver betrodd koppling.

Så här stoppar du betrodd koppling med en LDAP-katalog:

  1. Öppna Systeminställningar och klicka på Konton.

  2. Om symbolen med ett hänglås är låst så låser du upp den genom att klicka på den och anger ett administratörsnamn och -lösenord.

  3. Klicka på Inloggning och sedan på Redigera.

  4. Klicka på Katalogverktyg.

  5. Om symbolen med ett hänglås är låst så låser du upp den genom att klicka på den och anger ett administratörsnamn och -lösenord.

  6. Klicka på Tjänster.

  7. Markera LDAPv3 i listan över tjänster och klicka sedan på redigeraknappen (/).

  8. Om listan över serverkonfigurationer inte visas klickar du på Visa alternativ.

  9. Markera den serverkonfiguration du vill redigera och klicka på Redigera.

  10. Klicka på Koppla loss och ange följande ID-information. Klicka sedan på OK.

    Ange namn och lösenord för en LDAP-katalogadministratör (inte en lokal datoradministratör).

    Om betrodd koppling inte har ställts in på datorn visas inte knappen Koppla loss.

    Om ett varningsmeddelande om att datorn inte får kontakt med LDAP-servern visas kan du tvångsavsluta betrodd koppling genom att klicka på OK.

    Om du tvångsavslutar betrodd koppling finns datorposten för datorn kvar i LDAP-katalogen. Meddela LDAP-katalogens administratör så att administratören kan ta bort datorn från datorgruppen.

  11. Klicka OK när du har stoppat betrodd koppling.
