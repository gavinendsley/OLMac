Zuordnen von Config-Datensatzattributen für LDAP-Verzeichnisse

Wenn Sie Informationen für die Verwaltung von Mac OS X-Benutzern in einem LDAP-Verzeichnis speichern wollen, das nicht von Apple stammt, müssen Sie die folgenden Attribute des Datensatztyps "Config" zuordnen: "RealName" und "DataStamp".

Ohne die Zuordnung dieser Attribute wird die folgende Fehlermeldung beim Versuch angezeigt, mit dem Programm "Arbeitsgruppenmanager" einen Benutzerdatensatz zu ändern, der sich im LDAP-Verzeichnis befindet:

Das Attribut "dsRecTypeStandard:Config" hat keine Zuordnung.

Sie können diese Meldung ignorieren, wenn Sie keine Verwaltungsfunktionen für Mac OS X-Clients/Benutzer verwenden, die von den Attributen "RealName" und "DataStamp" des Datensatztyps "Config" für einen Cache abhängen.
