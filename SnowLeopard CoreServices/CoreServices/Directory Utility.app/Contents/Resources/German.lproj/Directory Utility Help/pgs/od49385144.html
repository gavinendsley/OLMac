Einrichten von Benutzerordnern für Active Directory-Benutzer-Accounts

Auf einem Computer, der so konfiguriert ist, dass er die ADC-Komponente (Active Directory Connector) des Programms "Verzeichnisdienste" verwendet, können Sie die Verwendung der Netzwerk-Benutzerordner oder der lokalen Benutzerordner für Active Directory-Benutzer-Accounts aktivieren oder deaktivieren.

Bei Netzwerk-Benutzerordnern wird der Benutzerordner im Windows-Netzwerk als Mac OS X-Benutzerordner aktiviert, wenn sich der Benutzer anmeldet.

Sie können festlegen, ob der Pfad des Netzwerk-Benutzerordners aus dem standardmäßigen Attribut "homeDirectory" von Active Directory abgerufen wird oder alternativ aus dem Mac OS X-Attribut "homeDirectory" (vorausgesetzt, das Active Directory-Schema wurde entsprechend erweitert).

Bei lokalen Benutzerordnern hat jeder Active Directory-Benutzer, der sich anmeldet, einen Benutzerordner auf dem Mac OS X-Startvolume. Zusätzlich wird der Benutzerordner im Netzwerk als Netzwerkvolume aktiviert. Der Benutzer kann Dateien zwischen diesem Netzwerkvolume und dem lokalen Benutzerordner kopieren.

Gehen Sie wie folgt vor, um Benutzerordner für Active Directory-Benutzer-Accounts einzurichten:

  1. Öffnen Sie die Systemeinstellungen und klicken Sie auf "Benutzer".

  2. Klicken Sie auf das Schlosssymbol, falls es geschlossen ist, und geben Sie den Namen und das Kennwort eines Administrators ein.

  3. Klicken Sie auf "Anmeldeoptionen" und dann auf "Beitreten" oder "Bearbeiten".

  4. Klicken Sie auf "Verzeichnisdienste öffnen".

  5. Klicken Sie auf das Schlosssymbol, falls es geschlossen ist, und geben Sie den Namen und das Kennwort eines Administrators ein.

  6. Klicken Sie auf "Dienste".

  7. Wählen Sie in der Liste der Dienste "Active Directory" aus und klicken Sie auf die Taste "Bearbeiten" (/).

  8. Wenn die erweiterten Optionen ausgeblendet sind, klicken Sie auf "Erweiterte Optionen einblenden".

  9. Klicken Sie auf "Benutzereinstellungen".

  10. Klicken Sie auf "Lokalen Benutzerordner unbedingt auf dem Startvolume anlegen", wenn Active Directory-Benutzer-Accounts über lokale Benutzerordner im Ordner "Benutzer" des Computers verfügen sollen.

    Diese Option ist nicht verfügbar, wenn "Mobilen Account bei der Anmeldung erstellen" ausgewählt ist.

  11. Wählen Sie "UNC-Pfad von Active Directory verwenden, um den Benutzerordner im Netzwerk abzuleiten" und wählen Sie eines der folgenden Protokolle für den Zugriff auf den Benutzerordner aus, um das Active Directory-Standardattribut für den Pfad des Benutzerordners zu verwenden:

    • Wählen Sie "smb" aus dem Einblendmenü "Zu verwendendes Netzwerkprotokoll" aus, um das standardmäßige Windows-Protokoll SMB zu verwenden.

    • Wählen Sie "AFP" aus dem Einblendmenü "Zu verwendendes Netzwerkprotokoll" aus, um das standardmäßige Mac OS-Protokoll AFP zu verwenden.

  12. Deaktivieren Sie "UNC-Pfad von Active Directory verwenden, um den Benutzerordner im Netzwerk abzuleiten", um das Mac OS X-Attribut für den Pfad des Benutzerordners zu verwenden.

    Damit das Mac OS X-Attribut verwendet werden kann, muss das Active Directory-Schema entsprechend erweitert werden.

  13. Klicken Sie auf "OK".

Wenn Sie den Namen eines Benutzer-Accounts in der Active Directory-Domain ändern, erstellt der Server einen Benutzerordner (mit Unterordnern) für den Benutzer-Account, sobald dieser Account das nächste Mal für die Anmeldung bei einem Mac OS X-Computer verwendet wird. Der Benutzer kann den alten Benutzerordner nach wie vor auswählen und den zugehörigen Inhalt im Finder anzeigen.

Sie können das Erstellen eines neuen Benutzerordners verhindern, indem Sie den alten Ordner umbenennen, bevor sich der Benutzer das nächste Mal anmeldet.
