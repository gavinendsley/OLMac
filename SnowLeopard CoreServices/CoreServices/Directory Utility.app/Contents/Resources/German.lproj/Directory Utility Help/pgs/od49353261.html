Ändern der Sicherheitsrichtlinie für eine LDAP-Verbindung

Mithilfe des Programms "Verzeichnisdienste" können Sie eine Sicherheitsrichtlinie für eine LDAPv3-Verbindung konfigurieren, die restriktiver ist als die Sicherheitsrichtlinie des LDAP-Verzeichnisses. Wenn beispielsweise die Sicherheitsrichtlinie des LDAP-Verzeichnisses Kennwörter in Klartext zulässt, können Sie eine LDAPv3-Verbindung so einrichten, dass Klartextkennwörter nicht akzeptiert werden.

Eine restriktivere Sicherheitsrichtlinie schützt Ihren Computer besser vor Hackern, die versuchen, mittels eines illegalen LDAP-Servers (Rogue-Servers) die Kontrolle über Ihren Computer zu erhalten.

Der Computer muss Daten mit dem LDAP-Server austauschen, um den Status der Sicherheitsoptionen korrekt anzeigen zu können. Wenn Sie die Sicherheitsoptionen für eine LDAPv3-Verbindung ändern, muss daher der Suchpfad des Computers für den Bereich "Identifizierung" die betreffende LDAPv3-Verbindung umfassen.

Welche Einstellungen für die Sicherheitsoptionen einer LDAPv3-Verbindung zulässig sind, hängt von den Sicherheitsfunktionen und Sicherheitsanforderungen des jeweiligen LDAP-Servers ab. So werden beispielsweise für einen LDAP-Server, der die Identifizierung über Kerberos nicht unterstützt, verschiedene Sicherheitsoptionen für die LDAPv3-Verbindung deaktiviert.

Gehen Sie wie folgt vor, um die Sicherheitsoptionen für eine LDAPv3-Verbindung zu ändern:

  1. Öffnen Sie die Systemeinstellungen und klicken Sie auf "Benutzer".

  2. Klicken Sie auf das Schlosssymbol, falls es geschlossen ist, und geben Sie den Namen und das Kennwort eines Administrators ein.

  3. Klicken Sie auf "Anmeldeoptionen" und dann auf "Bearbeiten".

  4. Klicken Sie auf "Verzeichnisdienste öffnen".

  5. Klicken Sie auf das Schlosssymbol, falls es geschlossen ist, und geben Sie den Namen und das Kennwort eines Administrators ein.

  6. Klicken Sie auf "Suchpfad".

  7. Klicken Sie auf "Identifizierung" und vergewissern Sie sich, dass das LDAPv3-Verzeichnis ein Bestandteil des Suchpfads ist.

    Weitere Informationen über das Hinzufügen des LDAPv3-Verzeichnis zum Suchpfad für den Bereich "Identifizierung" finden Sie unter Definieren eigener Suchpfade.

  8. Klicken Sie auf "Dienste".

  9. Wählen Sie in der Liste der Dienste "LDAPv3" aus und klicken Sie auf die Taste "Bearbeiten" (/).

  10. Wenn die Liste der Serverkonfigurationen ausgeblendet ist, klicken Sie auf "Optionen einblenden".

  11. Wählen Sie die Konfiguration für das gewünschte Verzeichnis aus und klicken Sie auf "Bearbeiten".

  12. Klicken Sie auf "Sicherheit" und nehmen Sie die gewünschten Änderungen an einer oder mehreren der folgenden Einstellungen vor:

    Hinweis:   Die hier und auf dem korrespondierenden LDAP-Server festgelegten Sicherheitseinstellungen werden beim Konfigurieren der LDAP-Verbindung angegeben. Die Einstellungen werden nicht aktualisiert, wenn Servereinstellungen geändert werden.

    Ist eine der vier letztgenannten Optionen ausgewählt, aber deaktiviert, so bedeutet dies, dass sie vom LDAP-Verzeichnis benötigt werden. Ist eine dieser Optionen nicht ausgewählt und deaktiviert, so bedeutet dies, dass sie vom betreffenden LDAP-Server nicht unterstützt wird.

    • Beim Verbindungsaufbau identifizieren: Diese Option bestimmt, ob sich die LDAPv3-Verbindung durch die Bereitstellung des Namens und des Kennworts eigenständig gegenüber dem LDAP-Verzeichnis identifiziert. Diese Option wird nicht angezeigt, wenn eine LDAPv3-Verbindung eine vertrauenswürdige Bindung zum LDAP-Verzeichnis nutzt.

    • Eingebunden in das Verzeichnis als: Diese Option legt die Anmeldedaten fest, die die LDAPv3-Verbindung für die vertrauenswürdige Bindung mit dem LDAP-Verzeichnis nutzt. Diese Option und die Anmeldedaten können an dieser Stelle nicht geändert werden. Sie können jedoch die Bindung trennen und unter Verwendung anderer Anmeldedaten neu herstellen.

      Weitere Informationen finden Sie unter Trennen einer vertrauenswürdigen Bindung zu einem LDAP-Verzeichnis und Einrichten einer vertrauenswürdigen Bindung zu einem LDAP-Verzeichnis.

      Diese Option wird nur angezeigt, wenn die LDAPv3-Verbindung eine vertrauenswürdige Bindung verwendet.

    • Kennwörter im Klartext deaktivieren: Diese Option bestimmt, ob das Kennwort als Klartext gesendet wird, wenn es nicht möglich ist, es mithilfe einer Identifizierungsmethode zu überprüfen, die Kennwörter in verschlüsselter Form sendet.

    • Alle Pakete digital signieren (Kerberos erforderlich): Diese Option gewährleistet, dass die vom LDAP-Server abgerufenen Verzeichnisdaten während der Übertragung auf Ihren Computer nicht von einem anderen Computer abgefangen und geändert werden.

    • Alle Pakete verschlüsseln (SSL oder Kerberos erforderlich): Diese Option legt fest, dass der LDAP-Server die Verzeichnisdaten mithilfe von SSL oder Kerberos verschlüsseln muss, bevor er sie an Ihren Computer sendet. Fragen Sie sich vor Auswahl der Option "Alle Pakete verschlüsseln (SSL oder Kerberos erforderlich)" Ihren Open Directory-Administrator, ob eine SSL-Verschlüsselung erforderlich ist.

    • Man-in-the-Middle-Angriffe blockieren (Kerberos erforderlich): Diese Option verhindert, dass sich ein illegaler (Rogue-) Server als LDAP-Server ausgeben kann. Diese Option ist am effektivsten, wenn sie mit der Option "Alle Pakete digital signieren" kombiniert wird.
