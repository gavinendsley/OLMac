Active Directory-Plug-In

Sie können Mac OS X dafür konfigurieren, auf grundlegende Benutzer-Account-Informationen in einer Active Directory-Domain eines Servers mit Windows 2000 oder neuer zuzugreifen. Dies wird durch die ADC-Komponente (Active Directory Connector) für das Programm "Verzeichnisdienste" ermöglicht. Die ADC-Komponente wird im Bereich "Dienste" des Programms "Verzeichnisdienste" aufgeführt.

Sie müssen keine Schemaänderungen an der Active Directory-Domain vornehmen, um grundlegende Benutzer-Account-Informationen zu erhalten. Unter Umständen empfiehlt es sich, die standardmäßige Zugriffsliste (ACL, Access Control List) bestimmter Attribute zu ändern, damit Computer-Accounts die Benutzereigenschaften lesen können.

Die ADC-Komponente generiert alle für die Mac OS X-Identifizierung benötigten Attribute aus Standardattributen in Active Directory-Benutzer-Accounts. Die Komponente unterstützt auch Active Directory-Richtlinien zur Identifizierung, einschließlich Kennwortänderungen, Kennwortablauf, Änderungsmaßnahmen und Sicherheitsoptionen.

Mac OS X 10.6 unterstützt Optionen für die Paketverschlüsselung und Optionen für die Paketsignierung für alle Windows Active Directory-Domains. Für diese Funktionalität ist standardmäßig "Erlauben" festgelegt. Sie können die Standardeinstellung mit dem Befehlszeilenprogramm dsconfigad in "Deaktiviert" oder "Erforderlich" ändern. Die Optionen für die Paketverschlüsselung und Paketsignierung stellen sicher, dass alle abgefragten Daten zur und von der Active Directory geschützt sind.

Die ADC-Komponente generiert dynamisch eine eindeutige Benutzer-ID und eine primäre Gruppen-ID, basierend auf der GUID (Globally Unique ID) des Benutzer-Accounts in der Active Directory-Domain. Die generierte Benutzer-ID und die Primärgruppen-ID sind bei jedem Benutzer-Account identisch, auch wenn der Account an verschiedenen Mac OS X-Computern für die Anmeldung verwendet wird.

Alternativ können Sie die ADC-Komponente anweisen, die Benutzer-ID von Ihnen angegebenen Active Directory-Attributen zuzuordnen.

Die Komponente generiert eine Gruppen-ID auf Basis der GUID des Active Directory-Gruppen-Accounts. Sie können das Active Directory-Plug-In außerdem anweisen, die Gruppen-ID für Gruppen-Accounts zu von Ihnen angegebenen Active Directory-Attributen zuzuordnen.

Wenn sich ein Benutzer bei Mac OS X mit einem Active Directory-Benutzer-Account anmeldet, kann die ADC-Komponente den Benutzerordner des Windows-Netzwerks aktivieren, der im Active Directory-Benutzer-Account als Mac OS X-Benutzerordner definiert ist. Sie können festlegen, ob der Netzwerk-Benutzerordner verwendet werden soll, der durch das standardmäßige Attribut "homeDirectory" von Active Directory oder durch das Mac OS X-Attribut "HomeDirectory" definiert wird (vorausgesetzt, das Active Directory-Schema wurde entsprechend erweitert).

Alternativ können Sie das Plug-In so konfigurieren, dass ein lokaler Benutzerordner auf dem Startvolume des Mac OS X-Client-Computers angelegt wird. In diesem Fall aktiviert das Plug-In außerdem den Benutzerordner im Windows-Netzwerk (der im Active Directory-Benutzer-Account definiert ist) als Netzwerkvolume. Im Finder kann der Benutzer anschließend Dateien zwischen dem Netzwerkvolume des Windows-Benutzerordners und dem lokalen Mac OS X-Benutzerordner kopieren.

Die ADC-Komponente kann außerdem mobile Accounts für Benutzer erstellen. Ein mobiler Account hat einen lokalen Benutzerordner auf dem Startvolume des Mac OS X-Client-Computers. (Der Benutzer besitzt außerdem einen Netzwerk-Benutzerordner, der in seinem Active Directory-Account definiert ist.)

Ein mobiler Account kann die Zertifikate der Identifizierung auf dem Mac OS X-Client-Computer im Cache zwischenspeichern. Die zwischengespeicherten Anmeldedaten erlauben dem Benutzer, sich mithilfe des Active Directory-Namens und -Kennworts anzumelden, wenn der Client-Computer vom Active Directory-Server getrennt ist.

Ein mobiler Account hat einen lokalen Benutzerordner auf dem Startvolume des Mac OS X-Client-Computers. (Der Benutzer besitzt außerdem einen Benutzerordner im Netzwerk, der in seinem Active Directory-Account definiert ist.)

Wenn das Active Directory-Schema um Mac OS X-Datensatztypen (Objektklassen) und Attribute erweitert wurde, werden diese von der ADC-Komponente erkannt und verwendet.

Zum Beispiel könnte das Active Directory-Schema mithilfe von Windows-Verwaltungswerkzeugen geändert werden, um von Mac OS X verwaltete Client-Attribute einzubeziehen. Durch diese Schemaänderung ist die ADC-Komponente in der Lage, Einstellungen für verwaltete Clients zu unterstützen, die mithilfe des Mac OS X Server-Programms "Arbeitsgruppenmanager" vorgenommen wurden.

Mac OS X-Clients setzen vollen Lesezugriff auf Attribute voraus, die dem Verzeichnis hinzugefügt werden. Deshalb kann es notwendig sein, die ACL der betreffenden Attribute zu ändern, damit Computergruppen diese hinzugefügten Attribute lesen können.

Die ADC-Komponente erkennt alle Domains in einer Active Directory-Gesamtstruktur. Sie können die Komponente so konfigurieren, dass sich Benutzer aus einer beliebigen Domain in der Gesamtstruktur bei einem Mac OS X-Computer anmelden können. Darüber hinaus besteht die Möglichkeit, die Identifizierung auf dem Client auf bestimmte Domains zu beschränken.

Die ADC-Komponente unterstützt in vollem Umfang Active Directory-Replikation und Ausfallumschaltung (Failover). Die Komponente erkennt mehrere Domain-Controller und ermittelt den nächstgelegenen. Wenn ein Domain-Controller nicht mehr verfügbar ist, wechselt die Komponente zu einem anderen, in der Nähe befindlichen Domain-Controller.

Die ADC-Komponente verwendet LDAP, um auf Active Directory-Benutzer-Accounts zuzugreifen, und Kerberos, um sie zu identifizieren. Die proprietäre Microsoft-Schnittstelle ADSI (Active Directory Services Interface) wird von der ADC-Komponente nicht verwendet, um Verzeichnis- oder Identifizierungsdienste in Anspruch zu nehmen.
