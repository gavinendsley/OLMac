Ändern der Verbindungseinstellungen für ein LDAP-Verzeichnis

Sie können mit dem Programm "Verzeichnisdienste" die Verbindungseinstellungen einer Konfiguration ändern, die angibt, auf welche Weise der Computer auf ein LDAPv3- oder ein LDAPv2-Verzeichnis zugreift.

Gehen Sie wie folgt vor, um die Verbindungseinstellungen für den Zugriff auf ein LDAP-Verzeichnis zu ändern:

  1. Öffnen Sie die Systemeinstellungen und klicken Sie auf "Benutzer".

  2. Klicken Sie auf das Schlosssymbol, falls es geschlossen ist, und geben Sie den Namen und das Kennwort eines Administrators ein.

  3. Klicken Sie auf "Anmeldeoptionen" und dann auf "Bearbeiten".

  4. Klicken Sie auf "Verzeichnisdienste öffnen".

  5. Klicken Sie auf das Schlosssymbol, falls es geschlossen ist, und geben Sie den Namen und das Kennwort eines Administrators ein.

  6. Klicken Sie auf "Dienste".

  7. Wählen Sie in der Liste der Dienste "LDAPv3" aus und klicken Sie auf die Taste "Bearbeiten" (/).

  8. Wenn die Liste der Serverkonfigurationen ausgeblendet ist, klicken Sie auf "Optionen einblenden".

  9. Wählen Sie aus der Liste eine Serverkonfiguration aus und klicken Sie auf "Bearbeiten".

  10. Klicken Sie auf "Verbindung" und nehmen Sie die gewünschten Änderungen an einer oder mehreren der folgenden Einstellungen vor:

    • Konfigurationsname: Der Name identifiziert eine Konfiguration innerhalb der Liste von LDAP-Verzeichniskonfigurationen. (Sie können den Namen auch direkt in der Liste der LDAP-Verzeichniskonfigurationen ändern.)

    • Servername oder IP-Adresse: Dies ist der DNS-Name bzw. die IP-Adresse des Servers. (Sie können diesen Wert auch direkt in der Liste der LDAP-Verzeichniskonfigurationen ändern.)

    • Zeitlimit für Öffnen/Schließen: Dieser Wert legt fest, wie lange ein Verbindungsversuch längstens dauern darf, bevor er abgebrochen wird.

    • Zeitlimit für Abfrage: Dieser Wert legt fest, wie lange eine Abfrage längstens dauern darf, bevor sie abgebrochen wird.

    • Erneuter Verbindungsversuch in: Diese Einstellung legt fest, nach wie vielen Sekunden ein weiterer Verbindungsversuch gestartet wird, wenn der LDAP-Server beim vorhergehenden Versuch nicht reagiert. Wählen Sie einen höheren Wert, um eine rasche Abfolge von Verbindungsversuchen zu vermeiden.

    • Zeitlimit für Verbindung: Diese Einstellung legt fest, wie lange (in Minuten) eine Verbindung, die sich im Leerlauf befindet oder nicht reagiert, geöffnet bleiben soll.

    • Verschlüsselung mit SSL: Diese Einstellung bestimmt, ob die Kommunikation mit dem LDAP-Verzeichnis mithilfe von SSL verschlüsselt wird. (Sie können diese Einstellung auch direkt in der Liste der LDAP-Verzeichniskonfigurationen ändern.) Fragen Sie vor Auswahl dieses Markierungsfelds Ihren Open Directory-Administrator, ob eine SSL-Verschlüsselung erforderlich ist.

    • Eigenen Port verwenden: Mit dieser Einstellung können Sie für LDAP-Verbindungen einen anderen Port anstelle des Standardports auswählen (389 ohne SSL, 636 mit SSL).

    • Verweise des Servers ignorieren: Diese Einstellung bestimmt, ob der Verweis eines LDAP-Servers, auf anderen LDAP-Servern oder Replikaten nach Informationen zu suchen, ignoriert oder befolgt werden soll. Serververweise können die Suche eines Computers nach Informationen vereinfachen. Sie können allerdings auch zu Verzögerungen bei der Anmeldung und zu weiteren Verzögerungen führen, wenn der Computer die Verweise auf die anderen LDAP-Server überprüfen muss.

    • LDAPv2 verwenden (Schreibgeschützt): Diese Einstellung bestimmt, ob für den schreibgeschützten Zugriff auf ein LDAP-Verzeichnis das ältere LDAPv2-Protokoll verwendet wird.

Verwandte Themen
Ändern des Zeitlimits für Öffnen & Schließen für eine LDAP-Verbindung
Ändern des Zeitlimits für Abfragen für eine LDAP-Verbindung
Ändern der Verzögerungszeit für erneute Verbindungsversuche für eine LDAP-Verbindung
Ändern des Zeitlimits für eine im Leerlauf befindliche LDAP-Verbindung
Ignorieren von LDAP-Serververweisen
