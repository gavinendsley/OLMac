Erstellen eines Kennworts

Das einem Benutzer-Account zugeordnete Kennwort muss vom Benutzer eingegeben werden, wenn er sich bei der Anmeldung oder für die Nutzung von Diensten identifiziert. Beim Kennwort muss die Groß-/Kleinschreibung beachtet werden (dies gilt nicht für SMB-LAN Manager-Kennwörter). Das Kennwort wird bei der Eingabe auf dem Bildschirm maskiert (nicht als Klartext) angezeigt.

Unabhängig vom gewählten Kennworttyp für einen Benutzer gelten bei der Erstellung von Kennwörtern für Mac OS X Server-Benutzer-Accounts folgende Richtlinien:

  • Ein Kennwort sollte aus einer Kombination von Buchstaben, Zahlen und Symbolen bestehen, die von unbefugten Benutzern nicht leicht erraten werden kann. Für Kennwörter sollten keine Wörter verwendet werden. Gute Kennwörter enthalten Ziffern und Symbole (z. B. # oder $) oder bestehen aus den jeweils ersten Buchstaben aller Wörter eines Satzes. Verwenden Sie Groß- und Kleinbuchstaben.

  • Vermeiden Sie Leerzeichen und Kombinationen mit der Wahltaste.

  • Vermeiden Sie Zeichen, die auf vom Benutzer verwendeten Computern nicht eingegeben werden können oder für deren korrekte Eingabe mit anderen Tastaturen und Plattformen eine spezielle Tastenkombination verwendet werden muss.

  • Einige Netzwerkprotokolle unterstützen keine Kennwörter mit vorangestellten, nachgestellten bzw. in das Wort integrierten Leerzeichen.

  • Ein Kennwort mit einer Länge von null Zeichen wird nicht empfohlen. Open Directory und einige Systeme (wie LDAP-Bind) unterstützen keine Kennwörter mit einer Länge von null Zeichen.

  • Verwenden Sie für Kennwörter nur ASCII-Zeichen, um eine maximale Kompatibilität mit Computern und Diensten zu erreichen, auf die Ihre Benutzer ggf. zugreifen.
