Konfigurieren von LDAP-Suchen und -Zuordnungen

Mithilfe des Programms "Verzeichnisdienste" können Sie die Zuordnungen sowie den Suchbeginn und den Suchbereich bearbeiten, die bestimmen, auf welche Weise Mac OS X in einem LDAP-Verzeichnis nach bestimmten Datenobjekten sucht. Sie können diese Einstellungen separat für jede im Programm "Verzeichnisdienste" aufgeführte LDAP-Verzeichniskonfiguration bearbeiten. Eine LDAP-Verzeichniskonfiguration gibt an, auf welche Weise Mac OS X auf Daten in einem LDAPv3- oder einem LDAPv2-Verzeichnis zugreift.

Sie können folgende Einstellungen bearbeiten:

  • Sie können die Zuordnung jedes Mac OS X-Datensatztyps zu LDAP-Objektklassen bearbeiten

  • Sie können die Zuordnung der Datentypen und Attribute von Mac OS X zu den LDAP-Attributen eines Datensatztyps bearbeiten.

  • Sie können den LDAP-Suchbeginn und den LDAP-Suchbereich bearbeiten; sie bestimmen, wo Mac OS X in einem LDAP-Verzeichnis nach einem bestimmten Mac OS X-Datensatztyp sucht.

Wenn Sie Mac OS X-Benutzerattribute zu einer nicht schreibgeschützten LDAP-Verzeichnis-Domain zuordnen, müssen Sie beachten, dass das zu "RealName" zugeordnete LDAP-Attribut nicht zugleich auch das erste Attribut in der Liste der LDAP-Attribute für "RecordName" sein darf.

Beispielsweise darf das Attribut "cn" nicht den Anfang der Liste der LDAP-Attribute für "RecordName" bilden, wenn "cn" zu "RealName" zugeordnet wird.

Wenn das zu "RealName" zugeordnete LDAP-Attribut zugleich das erste Attribut in der zu "RecordName" zugeordneten Liste von Attributen ist, treten Probleme beim Versuch auf, den vollen (langen) Namen oder den ersten Kurznamen im Arbeitsgruppenmanager zu bearbeiten.

Weitere Informationen zu Mac OS X-Datensatztypen und Attributen finden Sie im Handbuch "Open Directory - Administration" (verfügbar unter www.apple.com/de/server/resources/).

Gehen Sie wie folgt vor, um Suchbeginn und Suchbereiche sowie die Zuordnungen für einen LDAP-Server zu bearbeiten:

  1. Öffnen Sie die Systemeinstellungen und klicken Sie auf "Benutzer".

  2. Klicken Sie auf das Schlosssymbol, falls es geschlossen ist, und geben Sie den Namen und das Kennwort eines Administrators ein.

  3. Klicken Sie auf "Anmeldeoptionen" und dann auf "Bearbeiten".

  4. Klicken Sie auf "Verzeichnisdienste öffnen".

  5. Klicken Sie auf das Schlosssymbol, falls es geschlossen ist, und geben Sie den Namen und das Kennwort eines Administrators ein.

  6. Klicken Sie auf "Dienste".

  7. Wählen Sie in der Liste der Dienste "LDAPv3" aus und klicken Sie auf die Taste "Bearbeiten" (/).

  8. Wenn die Liste der Serverkonfigurationen ausgeblendet ist, klicken Sie auf "Optionen einblenden".

  9. Wählen Sie aus der Liste eine Serverkonfiguration aus und klicken Sie auf "Bearbeiten".

  10. Klicken Sie auf "Suche & Pfade".

  11. Wählen Sie die Zuordnungen, die Sie als Ausgangspunkt verwenden wollen. Wählen Sie "Eigene", wenn Sie die Zuordnungen ohne irgendwelche Vorgaben festlegen wollen.

    Wenn Sie eine der LDAP-Zuordnungsvorlagen auswählen, wird ein Suffix für den Suchbeginn angezeigt, das Sie ändern können. Sie können aber auch das Standardsuffix für den Suchbeginn durch Klicken auf "OK" übernehmen.

    Wählen Sie aus dem Einblendmenü "Zugriff auf diesen LDAPv3-Server über" eine Zuordnungsvorlage aus, wenn Sie deren Zuordnungen als Ausgangspunkt verwenden wollen.

  12. Fügen Sie bei Bedarf weitere Datensatztypen hinzu und/oder ändern Sie den Suchbeginn und den Suchbereich.

    • Klicken Sie zum Hinzufügen von Datensatztypen unter der Liste mit den Datensatztypen und Attributen auf "Hinzufügen". Klicken Sie auf "Datensatztypen", wählen Sie Datensatztypen in der Liste aus und klicken Sie auf "OK".

    • Markieren Sie einen Datensatztyp, für den Sie den Suchbeginn und den Suchbereich ändern wollen, in der Liste der Datensatztypen und Attribute und bearbeiten Sie anschließend den Inhalt des Felds "Suchbeginn". Wählen Sie "In allen Zweigen", wenn der Suchbereich die gesamte LDAP-Verzeichnishierarchie unterhalb des Suchbeginns umfassen soll. Wählen Sie "Nur in erster Ebene", wenn der Suchbereich neben dem Suchbeginn nur noch die Ebene innerhalb der LDAP-Verzeichnishierarchie umfassen soll, die dem Suchbeginn unmittelbar untergeordnet ist.

    • Markieren Sie einen Datensatztyp, den Sie entfernen wollen, in der Liste der Datensatztypen und Attribute und klicken Sie auf "Löschen".

    • Markieren Sie einen Datensatztyp, für den Sie eine Zuordnung hinzufügen wollen, in der Liste der Datensatztypen und Attribute und klicken Sie auf "Hinzufügen" (unterhalb von "Umleiten auf __ Objekte in der Liste"). Geben Sie danach den Namen einer Objektklasse aus dem LDAP-Verzeichnis ein.

      Drücken Sie, wenn Sie eine andere LDAP-Objektklasse hinzufügen wollen, zunächst den Zeilenschalter und geben Sie danach den Namen der gewünschten Objektklasse ein. In diesem Fall müssen Sie zusätzlich angeben, ob die angebotenen LDAP-Objektklassen verwendet werden sollen. Sie können diese Festlegung im Einblendmenü über der Liste treffen.

    • Wählen Sie in der Liste der Datensatztypen und Attribute einen Datensatztyp aus, dessen vorhandene Zuordnung Sie ändern wollen. Wählen Sie anschließend in der Liste "Umleiten auf __ Objekte in der Liste" durch Doppelklicken die LDAP-Objektklasse aus, die Sie ändern wollen, und nehmen Sie die gewünschten Änderungen vor. Mithilfe des Einblendmenüs über der Liste können Sie angeben, ob die aufgelisteten LDAP-Objektklassen verwendet werden sollen.

    • Wenn Sie eine Zuordnung für einen Datensatz entfernen wollen, markieren Sie den betreffenden Datensatz in der Liste der Datensatztypen und Attribute, wählen Sie danach aus der Liste "Umleiten auf __ Objekte in der Liste" die LDAP-Objektklasse aus, die Sie entfernen wollen, und klicken Sie auf "Löschen" (unterhalb von "Umleiten auf __ Objekte in Liste").

  13. Fügen Sie bei Bedarf weitere Attribute hinzu und/oder ändern Sie die Zuordnungen.

    • Markieren Sie einen Datensatztyp, dem Sie Attribute hinzufügen wollen, in der Liste der Datensatztypen und Attribute, und klicken Sie auf "Hinzufügen" (unterhalb der Liste der Datensatztypen und Attribute). Wählen Sie im nachfolgenden Fenster zunächst "Attributtypen" aus. Wählen Sie im Anschluss daran Attributtypen aus der Liste aus und klicken Sie auf "OK".

    • Markieren Sie ein Attribut, für das Sie eine Zuordnung hinzufügen wollen, in der Liste der Datensatztypen und Attribute und klicken Sie auf "Hinzufügen" (unterhalb von "Umleiten auf __ Objekte in der Liste"). Geben Sie danach den Namen eines Attributs aus dem LDAP-Verzeichnis ein. Möchten Sie ein weiteres LDAP-Attribut hinzufügen, drücken Sie den Zeilenschalter und geben Sie den Namen des Attributs ein.

    • Wählen Sie in der Liste der Datensatztypen und Attribute ein Attribut durch Doppelklicken aus, dessen vorhandene Zuordnung Sie ändern wollen. Wählen Sie anschließend in der Liste "Umleiten auf __ Objekte in der Liste" das Objekt durch Doppelklicken aus, das Sie ändern wollen, und nehmen Sie die gewünschten Änderungen für das betreffende Objekt vor.

    • Wenn Sie eine Zuordnung für ein Attribut entfernen wollen, markieren Sie das betreffende Attribut in der Liste der Datensatztypen und Attribute, wählen Sie danach aus der Liste "Umleiten auf __ Objekte in der Liste" das Objekt aus, das Sie entfernen wollen, und klicken Sie auf "Löschen" [unterhalb von "Umleiten auf __ Objekte in Liste").

    • Bewegen Sie die angezeigten Attribute in der Liste nach oben oder unten, wenn Sie deren Reihenfolge ändern wollen.

  14. Klicken Sie auf "Vorlage sichern", wenn Sie die von Ihnen definierten Zuordnungen als Vorlage sichern wollen.

    Vorlagen, die Sie am Standardspeicherort sichern, werden beim nächsten Starten des Programms "Verzeichnisdienste" in den Einblendmenüs angezeigt, in denen LDAP-Zuordnungsvorlagen angeboten werden. Der Standardspeicherort für gesicherte Vorlagen ist der Benutzerordner des aktuellen Benutzers mit folgendem Pfad:

    ~/Library/Application Support/Directory Access/LDAPv3/Templates

  15. Klicken Sie auf "Auf Server schreiben", wenn die Zuordnungen im LDAP-Verzeichnis gesichert werden sollen, sodass sie den zugehörigen Client-Computern automatisch zur Verfügung gestellt werden können. Geben Sie zum Speichern der Zuordnungen anschließend den gewünschten Suchbeginn sowie den Namen eines Administrators oder eines anderen Benutzers mit Schreibberechtigung für den Suchbeginn (z. B.: uid=diradmin,cn=users,dc=ods,dc=example,dc=com) und das zugehörige Kennwort ein.

    Wenn Sie Zuordnungen auf einen Open Directory-LDAP-Server schreiben, lautet der richtige Suchbeginn cn=config,suffix (dabei steht suffix für das Suchbeginn-Suffix des Servers, wie z. B. dc=ods,dc=example,dc=com).

Das LDAP-Verzeichnis stellt die in ihm gesicherten Zuordnungen solchen Mac OS X-Clients zur Verfügung, deren eigene Suchpfade eine Verbindung umfassen, die so konfiguriert ist, dass die Zuordnungen vom LDAP-Server abgerufen werden.

Das LDAP-Verzeichnis stellt die in ihm enthaltenen Zuordnungen außerdem auch allen Mac OS X-Clients zur Verfügung, die über einen automatischen Suchpfad verfügen. Weitere Informationen finden Sie unter Konfigurieren des Zugriffs auf ein LDAP-Verzeichnis und Verwenden erweiterter Einstellungen für Suchpfade.
