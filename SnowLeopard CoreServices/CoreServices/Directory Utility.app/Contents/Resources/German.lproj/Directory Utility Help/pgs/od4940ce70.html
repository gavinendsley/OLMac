Einrichten des LDAP-Zugriffs auf Active Directory-Domains

Mithilfe des Programms "Verzeichnisdienste" können Sie eine LDAPv3-Konfiguration einrichten, um auf eine Active Directory-Domain auf einem Windows-Server zuzugreifen. Eine LDAPv3-Konfiguration bietet Ihnen umfassende Kontrolle über die Zuordnung von Mac OS X-Datensatztypen und -Attributen zu Active Directory-Objektklassen, -Suchbereichen und -Attributen.

Die Zuordnung einiger wichtiger Mac OS X-Datensatztypen und -Attribute (wie UID) erfordert eine Erweiterung des Active Directory-Schemas.

Eine LDAPv3-Konfiguration schließt die folgenden Funktionen der im Programm "Verzeichnisdienste" aufgeführten ADC-Komponente (Active Directory Connector) nicht ein:

  • Dynamische Generierung von eindeutigen Benutzer-IDs und primären Gruppen-IDs

  • Erstellung von lokalen Mac OS X-Benutzerordnern

  • Automatische Aktivierung von Windows-Benutzerordnern

  • Mobile Benutzer-Accounts mit zwischengespeicherten Benutzerdaten für die Identifizierung

  • Suche nach allen Domains in einer Active Directory-Gesamtstruktur

  • Unterstützung für Active Directory-Replikation und -Failover

Weitere Informationen finden Sie unter Active Directory-Plug-In.

Gehen Sie wie folgt vor, um eine Active Directory-Serverkonfiguration zu erstellen:

  1. Öffnen Sie die Systemeinstellungen und klicken Sie auf "Benutzer".

  2. Klicken Sie auf das Schlosssymbol, falls es geschlossen ist, und geben Sie den Namen und das Kennwort eines Administrators ein.

  3. Klicken Sie auf "Anmeldeoptionen" und dann auf "Beitreten" oder "Bearbeiten".

  4. Klicken Sie auf "Verzeichnisdienste öffnen".

  5. Klicken Sie auf das Schlosssymbol, falls es geschlossen ist, und geben Sie den Namen und das Kennwort eines Administrators ein.

  6. Klicken Sie auf "Dienste".

  7. Wählen Sie in der Liste der Dienste "LDAPv3" aus und klicken Sie auf die Taste "Bearbeiten" (/).

  8. Klicken Sie auf "Neu" und geben Sie den DNS-Namen oder die IP-Adresse des Active Directory-Servers ein.

  9. Wählen Sie eine der folgenden Optionen für den Zugriff auf das Verzeichnis aus. Klicken Sie anschließend auf "Fortfahren", damit das Programm "Verzeichnisdienste" Informationen vom Active Directory-Server abruft.

    • Aktivieren Sie die Option "Verschlüsselung mit SSL", wenn Open Directory für Verbindungen zum Active Directory-Server eine SSL-Verschlüsselung verwenden soll. Fragen Sie vor Auswahl dieses Markierungsfelds Ihren Open Directory-Administrator, ob eine SSL-Verschlüsselung erforderlich ist.

    • Wählen Sie "Für Identifizierung verwenden", wenn dieses Verzeichnis Benutzer-Accounts enthält, die ein Benutzer für die Anmeldung oder zur Identifizierung für Dienste verwendet.

    • Wählen Sie "Für Kontakte verwenden", wenn das betreffende Verzeichnis E-Mail-Adressen und andere Informationen enthält, die im Programm "Adressbuch" verwendet werden sollen.

    Wenn das Programm "Verzeichnisdienste" die Verbindung zum Active Directory-Server nicht herstellen kann, wird eine entsprechende Meldung angezeigt. In diesem Fall müssen Sie den Zugriff manuell konfigurieren oder den Konfigurationsprozess abbrechen. Weitere Informationen finden Sie unter Manuelles Konfigurieren des Zugriffs auf ein LDAP-Verzeichnis.

    Wenn Sie die Option "Für Identifizierung verwenden" oder "Für Kontakte verwenden" ausgewählt haben, wird die LDAPv3-Verbindung zur Active Directory-Domain dem eigenen Suchpfad im Bereich "Identifizierung" bzw. "Kontakte" des Programms "Verzeichnisdienste" hinzugefügt.

    Vergewissern Sie sich, dass "LDAPv3" im Bereich "Dienste" aktiviert wurde, da der Computer nur dann die erstellte Verbindung verwenden kann. Weitere Informationen finden Sie unter Aktivieren oder Deaktivieren von LDAP-Verzeichnisdiensten.

  10. Wenn im Dialogfenster Zuordnungsoptionen angezeigt werden, wählen Sie "Active Directory" aus dem Einblendmenü aus, geben den Suchbereich ein und klicken dann auf "Fortfahren".

    Die Active Directory-Zuordnungsvorlage für eine LDAPv3-Konfiguration bildet einige Mac OS X-Datensatztypen und Attribute auf Objektklassen und Attribute ab, die nicht Teil eines standardmäßigen Active Directory-Schemas sind. Sie können die durch die Vorlage definierten Zuordnungen ändern oder das Active Directory-Schema erweitern.

    Alternativ können Sie unter Umständen über die ADC-Komponente statt über LDAPv3 auf Ihre Active Directory-Domain zugreifen. Weitere Informationen finden Sie unter Konfigurieren des Zugriffs auf eine Active Directory-Domain.

  11. Wenn sich das Dialogfenster erweitert und Verbindungsoptionen anzeigt, geben Sie den Namen und das Kennwort eines Active Directory-Benutzer-Accounts ein.

  12. Klicken Sie auf "OK", um das Erstellen der LDAP-Verbindung abzuschließen.

  13. Klicken Sie auf "OK", um das Konfigurieren der LDAPv3-Optionen abzuschließen.
