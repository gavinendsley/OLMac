Verwenden erweiterter Einstellungen für Suchpfade

Das Programm "Verzeichnisdienste" definiert folgende Suchpfade:

  • Identifizierung: Mac OS X verwendet den Suchpfad für die Identifizierung, um nach Informationen zur Identifizierung von Benutzern und anderen administrativen Daten in Verzeichnis-Domains zu suchen und diese abzurufen.

  • Kontakte: Mac OS X verwendet den Suchpfad für Kontakte, um nach Namen, Adressen und anderen Kontaktinformationen in Verzeichnis-Domains zu suchen und diese abzurufen. Das Programm "Mac OS X-Adressbuch" verwendet diese Kontaktinformationen. Andere Programme können so programmiert werden, dass sie diese Informationen ebenfalls verwenden.

    Jeder Suchpfad besteht aus einer Liste von Verzeichnis-Domains. Die Reihenfolge, in der die Verzeichnis-Domains in der Liste angeordnet sind, definiert den Suchpfad. Am Anfang der Liste beginnend, durchsucht Mac OS X jede Verzeichnis-Domain in der Liste, bis die benötigten Informationen gefunden werden oder das Ende der Liste erreicht ist, ohne dass die Informationen gefunden wurden.

    Für die Suchpfade für die Identifizierung und für Kontakte kann eine der folgenden Einstellungen gewählt werden:

  • Automatisch: Beginnt mit der lokalen Verzeichnis-Domain und kann ein LDAP-Verzeichnis berücksichtigen, das von DHCP und Verzeichnis-Domains bereitgestellt wird, mit denen der Computer verbunden ist. Dies ist die Standardeinstellung für Mac OS X 10.2 (oder neuer) und bietet die größte Flexibilität für Mobilcomputer.

  • Lokales Verzeichnis: Berücksichtigt nur die lokale Verzeichnis-Domain.

  • Eigener Pfad: Beginnt mit der lokalen Verzeichnis-Domain und enthält Ihre Auswahl an LDAP-Verzeichnissen, eine Active Directory-Domain, gemeinsame Verzeichnis-Domains, BSD-Konfigurationsdateien und eine NIS-Domain.

    Der Ordner "/BSD/local" ist immer Teil des Suchpfads und wird immer grau dargestellt.

Wichtig:  Wenn Sie Mac OS X so konfigurieren, dass für die Benutzeridentifizierung ein automatischer Suchpfad in Verbindung mit einem über DHCP bereitgestellten LDAP-Server verwendet wird, steigt das Risiko, dass ein unbefugter Benutzer die Kontrolle über Ihren Computer erlangen kann. Dieses Risiko ist noch höher, wenn Ihr Computer für den Zugang zu einem drahtlosen Netzwerk konfiguriert ist. Weitere Informationen finden Sie unter Schutz von Computern vor Angriffen durch fingierte DHCP-Server.
