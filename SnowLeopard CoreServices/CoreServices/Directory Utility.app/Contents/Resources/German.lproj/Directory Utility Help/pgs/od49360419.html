Manuelles Konfigurieren des Zugriffs auf ein LDAP-Verzeichnis

Sie können die Konfiguration, die den Zugriff von Mac OS X auf ein bestimmtes LDAPv3- oder LDAPv2-Verzeichnis bestimmt, manuell erstellen. Ihnen muss dazu der DNS-Name oder die IP-Adresse des LDAP-Verzeichnisservers bekannt sein.

Wird das Verzeichnis nicht von Mac OS X Server bereitgestellt, müssen Sie zusätzlich den Suchbeginn sowie die Vorlage für die Zuordnung der Mac OS X-Daten zu den Verzeichnisdaten kennen. Die folgenden Zuordnungsvorlagen werden unterstützt:

  • Vom Sever - für ein Verzeichnis, das seine eigenen Zuordnungen vornimmt und selbst den Suchbeginn bereitstellt (zum Beispiel Mac OS X Server)

  • Open Directory-Server - für ein Verzeichnis, das das Schema von Mac OS X Server verwendet

  • Active Directory - für ein von einem Windows 2000- oder Windows 2003-Server und neueren Servern bereitgestelltes Verzeichnis

  • RFC 2307 - für die meisten von UNIX-Servern bereitgestellten Verzeichnisse

  • Eigene - für Verzeichnisse, die keine der oben genannten Zuordnungen verwenden

Das Plug-In (Zusatzmodul) "LDAPv3" unterstützt die Open Directory-Replikation und Ausfallumschaltung (Failover) in vollem Umfang. Sollte der Open Directory-Master ausfallen, schaltet das Plug-In automatisch zu einem verfügbaren Replikat um.

Wichtig: Falls der Name eines Computers einen Bindestrich (-) enthält, besteht die Gefahr, dass der Versuch scheitert, einer Verzeichnis-Domain wie LDAP oder Active Directory beizutreten und die Verbindung herzustellen. Für den Beitritt müssen Sie gegebenenfalls den Bindestrich aus dem Namen des betreffenden Computers entfernen.
Gehen Sie wie folgt vor, um den Zugriff auf ein LDAP-Verzeichnis manuell zu konfigurieren:

  1. Öffnen Sie die Systemeinstellungen und klicken Sie auf "Benutzer".

  2. Klicken Sie auf das Schlosssymbol, falls es geschlossen ist, und geben Sie den Namen und das Kennwort eines Administrators ein.

  3. Klicken Sie auf "Anmeldeoptionen" und dann auf "Beitreten" oder "Bearbeiten".

  4. Klicken Sie auf "Verzeichnisdienste öffnen".

  5. Klicken Sie auf das Schlosssymbol, falls es geschlossen ist, und geben Sie den Namen und das Kennwort eines Administrators ein.

  6. Klicken Sie auf "Dienste".

  7. Wählen Sie in der Liste der Dienste "LDAPv3" aus und klicken Sie auf die Taste "Bearbeiten" (/).

    Sie können "LDAPv3" aus der Liste der Dienste auswählen, ohne das Feld "Aktiv" für LDAPv3 zu markieren.

  8. Klicken Sie auf "Neu" und anschließend auf "Manuell".

  9. Geben Sie einen Namen für die Konfiguration ein.

  10. Drücken Sie die Tabulatortaste und geben Sie den DNS-Namen oder die IP-Adresse des Servers ein, der das LDAP-Verzeichnis enthält, auf das Sie zugreifen wollen.

  11. Klicken Sie auf das Einblendmenü neben dem DNS-Namen bzw. der IP-Adresse und wählen Sie eine Zuordnungsvorlage oder Zuordnungsmethode aus.

    • Wenn Sie "Vom Server" auswählen, ist kein Suffix für den Suchbeginn erforderlich. In diesem Fall geht Open Directory davon aus, dass das Suchbeginnsuffix der ersten Ebene des LDAP-Verzeichnisses entspricht.

    • Wenn Sie sich für eine Vorlage entscheiden, müssen Sie das Suchbeginnsuffix für das LDAP-Verzeichnis eingeben und danach auf "OK" klicken. Ohne das Suchbeginnsuffix ist der Computer nicht in der Lage, Informationen im LDAP-Verzeichnis zu finden.

      Im Normalfall wird das Suffix für den Suchbeginn vom DNS-Namen des Servers abgeleitet. Für einen Server mit dem DNS-Namen "ods.example.com" könnte das Suffix beispielsweise "dc=ods,dc=example, dc=com" lauten.

    • Wenn Sie sich für "Eigene" entscheiden, müssen Sie die Zuordnungen zwischen den Datensatztypen und Attributen von Mac OS X auf der einen Seite und den Klassen und Attributen des LDAP-Verzeichnisses, zu dem Sie die Verbindung herstellen, auf der anderen Seite festlegen. Weitere Informationen finden Sie unter Konfigurieren von LDAP-Suchen und -Zuordnungen.

  12. Wenden Sie sich vor Auswahl der Option "Verschlüsselung mit SSL" an Ihren Open Directory-Administrator, um festzustellen, ob eine SSL-Verschlüsselung erforderlich ist.

  13. Klicken Sie zum Ändern der folgenden Einstellungen für diese LDAP-Konfiguration auf "Bearbeiten", um die Optionen für die ausgewählte LDAP-Konfiguration einzublenden, nehmen Sie die Änderungen vor und klicken Sie auf "OK", wenn alle Änderungen wunschgemäß festgelegt sind.

    • Klicken Sie auf "Verbindung", um die Optionen für die Zeitüberschreitung festzulegen, einen eigenen Port anzugeben, Serververweise zu ignorieren oder die Verwendung des LDAPv2-Protokolls (schreibgeschützt) zu erzwingen. Weitere Informationen finden Sie unter Ändern der Verbindungseinstellungen für ein LDAP-Verzeichnis.

    • Klicken Sie auf "Suche & Pfade", um die Such- und Zuordnungseinstellungen für einen LDAP-Server festzulegen. Weitere Informationen finden Sie unter Einrichten einer vertrauenswürdigen Bindung für ein LDAP-Verzeichnis.

    • Klicken Sie auf "Sicherheit", um (anstelle einer vertrauenswürdigen Bindung) eine identifizierte Verbindung und andere für die Sicherheitsrichtlinien relevante Optionen einzurichten. Weitere Informationen finden Sie unter Ändern der Sicherheitsrichtlinie für eine LDAP-Verbindung.

    • Klicken Sie auf "Einbinden", um eine vertrauenswürdige Bindung einzurichten (sofern eine solche Verbindung vom LDAP-Verzeichnis unterstützt wird). Weitere Informationen finden Sie unter Einrichten einer vertrauenswürdigen Bindung für ein LDAP-Verzeichnis.

  14. Klicken Sie auf "OK", um die manuelle Erstellung der Konfiguration für den Zugriff auf ein LDAP-Verzeichnis abzuschließen.

  15. Wenn der Computer auf ein LDAP-Verzeichnis zugreifen soll, für das Sie eine Konfiguration erstellt haben, müssen Sie das betreffende Verzeichnis dem Suchpfad im Bereich "Identifizierung" bzw. "Kontakte" des Programms "Verzeichnisdienste" hinzufügen. Außerdem müssen Sie sich vergewissern, dass "LDAPv3" im Bereich "Dienste" aktiviert wurde.

Weitere Informationen finden Sie unter Aktivieren oder Deaktivieren von LDAP-Verzeichnisdiensten und Definieren eigener Suchpfade.

Hinweis: Damit Sie mit dem Programm "Arbeitsgruppenmanager" Benutzer auf einem LDAP-Server erstellen können, der nicht von Apple stammt und der RFC 2307-Zuordnungen (UNIX) verwendet, müssen Sie die Zuordnung des Datensatztyps "Users" bearbeiten. Weitere Informationen finden Sie unter Bearbeiten der RFC 2307-Zuordnung zum Aktivieren der Erstellung von Benutzern.
Wichtig:  Wenn Sie die IP-Adresse und den Computernamen mithilfe des Befehls changeip ändern, während Sie bei einem Verzeichnisserver angemeldet sind, müssen Sie die Verbindung trennen und erneut herstellen, damit das Verzeichnis mit dem neuen Computernamen und der neuen IP-Adresse aktualisiert wird. Wenn Sie die Verbindung zum Verzeichnisserver nicht trennen und wiederherstellen, wird das Verzeichnis nicht aktualisiert und verwendet weiterhin den alten Computernamen und die alte IP-Adresse.
