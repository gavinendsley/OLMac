Löschen einer Konfiguration für den Zugriff auf ein LDAP-Verzeichnis

Sie können im Programm "Verzeichnisdienste" eine Konfiguration löschen, die angibt, auf welche Weise der Computer auf ein LDAPv3- oder ein LDAPv2-Verzeichnis zugreift.

Eine von DHCP bereitgestellte LDAP-Konfiguration kann nicht geändert werden. Aus diesem Grund wird eine Konfiguration dieses Typs in der Liste der LDAP-Konfigurationen grau dargestellt.

Gehen Sie wie folgt vor, um eine Konfiguration für den Zugriff auf ein LDAP-Verzeichnis zu löschen:

  1. Öffnen Sie die Systemeinstellungen und klicken Sie auf "Benutzer".

  2. Klicken Sie auf das Schlosssymbol, falls es geschlossen ist, und geben Sie den Namen und das Kennwort eines Administrators ein.

  3. Klicken Sie auf "Anmeldeoptionen" und dann auf "Bearbeiten".

  4. Klicken Sie auf "Verzeichnisdienste öffnen".

  5. Klicken Sie auf das Schlosssymbol, falls es geschlossen ist, und geben Sie den Namen und das Kennwort eines Administrators ein.

  6. Klicken Sie auf "Dienste".

  7. Wählen Sie in der Liste der Dienste "LDAPv3" aus und klicken Sie auf die Taste "Bearbeiten" (/).

  8. Wenn die Liste der Serverkonfigurationen ausgeblendet ist, klicken Sie auf "Optionen einblenden".

  9. Wählen Sie in der Liste eine Serverkonfiguration aus, klicken Sie auf "Löschen" und dann auf "OK".

  10. Folgendes steht zur Auswahl:

    • Wenn eine Warnmeldung mit dem Hinweis angezeigt wird, dass der Computer an das LDAP-Verzeichnis gebunden ist und Sie die vertrauenswürdige Bindung trennen möchten, klicken Sie auf "OK" und geben Sie dann den Namen und das Kennwort eines LDAP-Verzeichnisadministrators ein (nicht eines lokalen Computeradministrators).

    • Wenn Sie in einer Warnmeldung darüber informiert werden, dass der Computer keine Verbindung zum LDAP-Server herstellen kann, können Sie durch Klicken auf "OK" das Beenden der vertrauenswürdigen Bindung erzwingen.

      Bei einer erzwungenen Beendigung der vertrauenswürdigen Bindung verbleibt der Computereintrag dieses Computers im LDAP-Verzeichnis. Sie sollten in diesem Fall den Administrator des LDAP-Verzeichnisses bitten, den Computer aus der Computergruppe zu löschen.

Die gelöschte Konfiguration wird aus den eigenen Suchpfaden in den Bereichen "Identifizierung" und "Kontakte" entfernt.
