Konfigurieren des Zugriffs auf ein LDAP-Verzeichnis

Mit dem Programm "Verzeichnisdienste" können Sie festlegen, wie Mac OS X auf ein LDAPv3-Verzeichnis zugreift. Ihnen muss dafür der DNS-Name oder die IP-Adresse des LDAP-Verzeichnisservers bekannt sein.

Im Falle eines Verzeichnisses, das nicht von einem Server bereitgestellt wird, der eigene Zuordnungen vergibt (wie etwa Mac OS X Server), müssen Sie zusätzlich den Suchbeginn und die Vorlage für die Zuordnung der Mac OS X-Daten zu den Verzeichnisdaten kennen.

Die folgenden Zuordnungsvorlagen werden unterstützt:

  • Open Directory-Server - für ein Verzeichnis, das das Schema von Mac OS X Server verwendet

  • Active Directory - für ein von einem Windows 2000- oder Windows 2003-Server und neueren Servern bereitgestelltes Verzeichnis

  • RFC 2307 - für die meisten von UNIX-Servern bereitgestellten Verzeichnisse

Das Plug-In (Zusatzmodul) "LDAPv3" unterstützt die Open Directory-Replikation und Ausfallumschaltung (Failover) in vollem Umfang. Sollte der Open Directory-Master ausfallen, schaltet das Plug-In automatisch zu einem verfügbaren Replikat um.

Sie können eigene Zuordnungen für die Verzeichnisdaten vornehmen. Befolgen Sie in diesem Falle anstelle der hier beschriebenen Schritte die Anleitung unter Manuelles Konfigurieren des Zugriffs auf ein LDAP-Verzeichnis.

Wichtig: Falls der Name eines Computers einen Bindestrich (-) enthält, besteht die Gefahr, dass der Versuch scheitert, einer Verzeichnis-Domain wie LDAP oder Active Directory beizutreten und die Verbindung herzustellen. Für den Beitritt müssen Sie gegebenenfalls den Bindestrich aus dem Namen des betreffenden Computers entfernen.
Gehen Sie wie folgt vor, um mit der Unterstützung durch das Programm "Verzeichnisdienste" den Zugriff auf ein LDAP-Verzeichnis zu konfigurieren:

  1. Öffnen Sie die Systemeinstellungen und klicken Sie auf "Benutzer".

  2. Klicken Sie auf das Schlosssymbol, falls es geschlossen ist, und geben Sie den Namen und das Kennwort eines Administrators ein.

  3. Klicken Sie auf "Anmeldeoptionen" und dann auf "Beitreten" oder "Bearbeiten".

  4. Klicken Sie auf "Verzeichnisdienste öffnen".

  5. Klicken Sie auf das Schlosssymbol, falls es geschlossen ist, und geben Sie den Namen und das Kennwort eines Administrators ein.

  6. Klicken Sie auf "Dienste".

  7. Wählen Sie in der Liste der Dienste "LDAPv3" aus und klicken Sie auf die Taste "Bearbeiten" (/).

    Sie können "LDAPv3" aus der Liste der Dienste auswählen, ohne das Feld "Aktiv" für LDAPv3 zu markieren.

  8. Klicken Sie auf "Neu" und geben Sie den DNS-Namen oder die IP-Adresse des LDAP-Servers ein.

  9. Wählen Sie die gewünschten Optionen für den Zugriff auf das Verzeichnis:

    • Wählen Sie "Verschlüsselung mit SSL", wenn Open Directory SSL (Secure Sockets Layer) für Verbindungen mit dem LDAP-Verzeichnis verwenden soll. Fragen Sie vor Auswahl dieser Option Ihren Open Directory-Administrator, ob eine SSL-Verschlüsselung erforderlich ist.

    • Wählen Sie "Für Identifizierung verwenden", wenn das betreffende Verzeichnis Benutzer-Accounts enthält, die für die Anmeldung oder für die Identifizierung beim Zugriff auf Dienste verwendet werden.

    • Wählen Sie "Für Kontakte verwenden", wenn das betreffende Verzeichnis E-Mail-Adressen und andere Informationen enthält, die im Programm "Adressbuch" verwendet werden sollen.

    Wenn das Programm "Verzeichnisdienste" die Verbindung zum LDAP-Server nicht herstellen kann, wird eine entsprechende Meldung angezeigt. In diesem Fall müssen Sie den Zugriff manuell konfigurieren oder den Konfigurationsprozess abbrechen. Weitere Informationen über die Schritte für die manuelle Konfiguration finden Sie unter Manuelles Konfigurieren des Zugriffs auf ein LDAP-Verzeichnis.

    Wenn das Dialogfenster erweitert wird und die Zuordnungsoptionen angezeigt werden, können Sie die Zuordnungsvorlage aus dem Einblendmenü auswählen. Geben Sie danach das Suffix für den Suchbeginn ein und klicken Sie auf "Fortfahren".

    Im Normalfall wird das Suffix für den Suchbeginn vom DNS-Namen des Servers abgeleitet. Für einen Server mit dem DNS-Namen "ods.example.com" könnte das Suffix beispielsweise "dc=ods,dc=example, dc=com" lauten.

    Klicken Sie auf "Manuell", wenn keine der verfügbaren Zuordnungsvorlagen für die Verbindung geeignet ist, die Sie einrichten wollen. Weitere Informationen finden Sie unter Manuelles Konfigurieren des Zugriffs auf ein LDAP-Verzeichnis.

  10. Klicken Sie auf "Fortfahren", damit das Programm "Verzeichnisdienste" die Informationen vom LDAP-Server abrufen kann.

  11. Wenn das Dialogfenster erweitert wird und die Optionen für eine vertrauenswürdige Bindung angezeigt werden, können Sie den Namen des Computers sowie den Benutzernamen und das Kennwort eines Verzeichnisadministrators eingeben. (Die Bindung ist nicht in allen Fällen erforderlich.)

    Sie werden im Dialogfenster informiert, ob für das ausgewählte LDAP-Verzeichnis eine vertrauenswürdige Bindung erforderlich ist oder nicht. Eine vertrauenswürdige Bindung ist wechselseitig: Sobald der Computer die Verbindung zum LDAP-Verzeichnis herstellt, identifizieren sich die Systeme gegenseitig. Wenn bereits eine vertrauenswürdige Bindung eingerichtet wurde oder das LDAP-Verzeichnis eine solche Verbindung nicht unterstützt, wird die Taste "Einbinden" nicht angezeigt. Vergewissern Sie sich, dass der von Ihnen eingegebene Computername richtig ist.

    Wenn Sie informiert werden, dass bereits ein Eintrag für den eingegebenen Computernamen existiert, können Sie auf "Abbrechen" klicken, wenn Sie den eingegebenen Computernamen ändern wollen. Klicken Sie auf "Überschreiben", wenn der vorhandene Computereintrag überschrieben werden soll.

    Bei einem vorhandenen Computereintrag besteht die Möglichkeit, dass er nicht mehr verwendet wird; es kann aber auch sein, dass er Bezug auf einen anderen Computer nimmt.

    Sie sollten daher stets den Administrator des LDAP-Verzeichnisses entsprechend informieren, wenn Sie einen vorhandenen Computereintrag ersetzen, da durch das Ersetzen möglicherweise ein anderer Computer deaktiviert wird. In diesem Fall muss der für das LDAP-Verzeichnis zuständige Administrator dem betroffenen Computer einen anderen Namen zuweisen und diesen Computer wieder der Gruppe hinzufügen, der er ursprünglich angehörte.

  12. Wenn das Dialogfenster erweitert wird und die Verbindungsoptionen angezeigt werden, können Sie "Beim Verbindungsaufbau identifizieren" auswählen und den Namen und das Kennwort eines Benutzer-Accounts im betreffenden Verzeichnis eingeben.

    Die Optionen für eine identifizierte Verbindung werden angezeigt, wenn der LDAP-Server eine solche Verbindung, nicht aber eine vertrauenswürdige Bindung unterstützt. Eine identifizierte Verbindung ist nicht wechselseitig: der LDAP-Server identifiziert den Client, der Client umgekehrt aber nicht den Server.

    Die Option "Beim Verbindungsaufbau identifizieren" wird automatisch ausgewählt, jedoch grau angezeigt, wenn der LDAP-Server voraussetzt, dass für eine identifizierte Verbindung der Name und das Kennwort eines Benutzer-Accounts eingeben werden.

    Der Name kann Bezug auf jeden Benutzer-Account nehmen, der die Berechtigung zum Lesen/Anzeigen der Daten im Verzeichnis hat. Für den Benutzer-Account mit dem Kurznamen "dirauth" auf dem LDAP-Server "ods.example.com" lautet beispielsweise der Name "uid=dirauth,cn=users,dc=ods,dc=example,dc=com".

    Wichtig:  Falls der Name oder das Kennwort ungültig ist, können Sie einen der Benutzer-Accounts im LDAP-Verzeichnis verwenden, um sich am betreffenden Computer anzumelden.

  13. Klicken Sie auf "OK", um das Erstellen der LDAP-Verbindung abzuschließen.

  14. Klicken Sie auf "OK", um das Konfigurieren der LDAPv3-Optionen abzuschließen.

Wenn Sie in Schritt 5 die Option "Für Identifizierung verwenden" oder "Für Kontakte verwenden" ausgewählt haben, wird die neu erstellte LDAP-Verzeichniskonfiguration automatisch dem eigenen Suchpfad im Bereich "Identifizierung" bzw. "Kontakte" des Programms "Verzeichnisdienste" hinzugefügt.

Vergewissern Sie sich, dass "LDAPv3" im Bereich "Dienste" aktiviert wurde, da nur dann der Computer die neu erstellte LDAP-Konfiguration verwenden kann. Weitere Informationen finden Sie unter Aktivieren oder Deaktivieren von LDAP-Verzeichnisdiensten.
