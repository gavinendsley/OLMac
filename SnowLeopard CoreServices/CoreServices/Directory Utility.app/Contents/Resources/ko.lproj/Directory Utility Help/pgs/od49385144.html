홈
Active Directory 사용자 계정의 홈 폴더 설정하기

디렉토리 유틸리티의 Active Directory 커넥터를 사용하도록 구성된 컴퓨터에서는, Active Directory 사용자 계정에 대하여 네트워크 홈 폴더 또는 로컬 홈 폴더의 사용을 활성화 또는 비활성화할 수 있습니다.

네트워크 홈 폴더의 경우에는, 사용자가 로그인하면 사용자의 Windows 네트워크 홈 폴더가 Mac OS X 홈 폴더로 마운트됩니다.

Active Directory 체계가 확장되어 포함하고 있는 경우, 네트워크 홈 폴더 위치를 Active Directory의 표준 HomeDirectory 속성 또는 Mac OS X의 HomeDirectory 속성으로부터 가져올 것인지를 결정할 수 있습니다.

로컬 홈 폴더의 경우에는, 로그인하는 각 Active Directory 사용자가 Mac OS X 시동 디스크에 홈 폴더를 가지고 있습니다. 또한, 사용자의 네트워크 홈 폴더가 공유점처럼 네트워크 볼륨에 마운트됩니다. 사용자는 이 네트워크 볼륨과 로컬 홈 폴더 사이에 파일을 복사할 수 있습니다.

Active Directory 사용자 계정의 홈 폴더를 설정하려면,

  1. 시스템 환경설정을 열고 계정을 클릭하십시오.

  2. 자물쇠 아이콘이 잠겨 있으면, 해당 아이콘을 클릭하고 관리자의 이름과 암호를 입력하여 잠금 해제하십시오.

  3. 로그인 옵션을 클릭한 다음 연결 또는 편집을 클릭하십시오.

  4. 디렉토리 유틸리티 열기를 클릭하십시오.

  5. 자물쇠 아이콘이 잠겨 있으면, 해당 아이콘을 클릭하고 관리자의 이름과 암호를 입력하여 잠금 해제하십시오.

  6. 서비스를 클릭하십시오.

  7. 서비스 목록에서 Active Directory를 선택하고 편집(/) 단추를 클릭하십시오.

  8. 고급 옵션이 가려져 있으면, 고급 옵션 보기를 클릭하십시오.

  9. 사용자 환경을 클릭하십시오.

  10. Active Directory 사용자 계정이 컴퓨터의 /사용자 폴더에 있는 로컬 홈 폴더를 사용하게 하려면, “로컬 홈 폴더를 시동 디스크로 강제 설정”을 클릭하십시오.

    "로그인 시 모바일 계정 생성"이 선택되어 있다면 이 옵션은 사용할 수 없습니다.

  11. 홈 폴더 위치에 Active Directory의 표준 속성을 사용하려면, “Active Directory의 UNC 경로를 사용하여 네트워크 홈 위치 유도”을 선택한 다음 아래에서 홈 폴더 접근을 위한 프로토콜을 선택하십시오.

    • 표준 Windows 프로토콜 SMB을 사용하려면 “사용되는 네트워크 프로토콜” 팝업 메뉴에서 smb를 선택하십시오.

    • 표준 Macintosh 프로토콜 AFP를 사용하려면 “사용되는 네트워크 프로토콜” 팝업 메뉴에서 afp를 선택하십시오.

  12. 홈 폴더 위치에 대해 Mac OS X의 속성을 사용하려면, “Active Directory의 UNC 경로를 사용하여 네트워크 홈 위치 유도”를 선택 해제하십시오.

    Mac OS X의 속성을 사용하려면, Active Directory 체계가 그것을 포함하도록 확장되어야 합니다.

  13. 승인을 클릭하십시오.

Active Directory 도메인에서 사용자 계정의 이름을 변경하면 다음에 Mac OS X 컴퓨터에 로그인하는 데 사용될 때 서버가 해당 사용자 계정에 대한 홈 폴더(및 하위 폴더)를 생성합니다. 사용자는 여전히 기존 홈 폴더로 이동하여 Finder에서 내용을 볼 수 있습니다.

사용자가 다음에 로그인하기 전에 기존의 폴더 이름을 변경하여 새로운 홈 폴더의 생성을 방지할 수 있습니다.
