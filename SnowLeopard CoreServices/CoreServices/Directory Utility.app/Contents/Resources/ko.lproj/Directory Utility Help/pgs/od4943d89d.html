Active Directory 접근에 관하여

Mac OS X에서 Windows 2000 또는 Windows 이상 서버의 Active Directory 도메인에 있는 기본 사용자 계정 정보에 접근하도록 설정할 수 있습니다. 이것은 디렉토리 유틸리티에 대해 Active Directory 커넥터가 가능하기 때문입니다. Active Directory 커넥터는 디렉토리 유틸리티의 서비스 패널에 나열됩니다.

기본 사용자 계정 정보를 얻기 위해 Acrive Directory 도메인의 체계를 변경할 필요는 없습니다. 특정 속성의 기본 ACL(연결 조절 목록)을 변경하여 컴퓨터 계정이 사용자 속성을 읽도록 할 수도 있습니다.

Active Directory 커넥터는 Active Directory 사용자 계정의 표준 속성에서 Mac OS X 인증에 필요한 모든 속성을 생성합니다. 커넥터는 암호 변경, 만료, 강제 변경 및 보안 옵션을 포함한 Active Directory 인증 방법을 지원하기도 합니다.

Mac OS X v10.6은 모든 Windows Active Directory 도메인용으로 패킷 암호화와 패킷 서명 옵션을 지원합니다. 이 기능은 기본적으로 “허용”으로 켜져 있습니다. 명령행 도구 dsconfigad를 사용하여 기본 설정을 비활성화 또는 요청대로 변경할 수 있습니다. 패킷 암호화 및 패킷 서명 옵션은 레코드 검색을 위해 Active Directory 도메인 내외부의 모든 데이터가 보호되도록 보장합니다.

Active Directory 커넥터는 Active Directory 도메인에서 사용자 계정의 GUID(Globally Unique ID)를 기반으로 한 특정한 사용자 ID와 기본 그룹 ID를 생성합니다. 생성된 사용자 ID와 기본 그룹 ID는 다른 Mac OS X 컴퓨터로 로그인하는데 사용된 계정이더라도 각 사용자 계정에 대해 항상 동일합니다.

또는, Active Directory 커넥터로 하여금, 지정하는 Active Directory 속성에 사용자 ID를 강제로 매핑하도록 할 수 있습니다.

Active Directory 커넥터는 Active Directory 그룹 계정의 GUID에 기반하여 그룹 ID를 생성합니다. 또한, 플러그인으로 하여금, 지정하는 Active Directory 속성에 그룹 계정에 대한 그룹 ID를 강제로 매핑하도록 할 수 있습니다.

어떤 사용자가 Active Directory 사용자 계정을 통해 Mac OS X에 로그인하면, Active Directory 커넥터는 Active Directory 사용자 계정에 해당 사용자의 Mac OS X 홈 폴더로 지정된 Windows 네트워크 홈 폴더를 마운트할 수 있습니다. Active Directory 체계가 확장되어 포함하고 있는 경우, Active Directory의 표준 HomeDirectory 속성 또는 Mac OS X의 HomeDirectory 속성에서 지정한 네트워크 홈을 사용할 것인지를 지정할 수 있습니다.

또는, Mac OS X 클라이언트 컴퓨터의 시동 볼륨에 로컬 홈 폴더를 생성하도록 플러그인을 구성할 수 있습니다. 이 경우에는, 플러그인도 사용자의 Windows 네트워크 홈 폴더(Active Directory 사용자 계정에 지정된)를 공유 지점과 같은 네트워크 볼륨으로 마운트합니다. 그런 다음 Finder를 사용하여, 사용자는 Windows 홈 폴더 네트워크 볼륨과 로컬 Mac OS X 홈 폴더 사이에서 파일을 복사할 수 있습니다.

Active Directory 커넥터는 또한 사용자에 대한 모바일 계정을 생성할 수도 있습니다. 모바일 계정은 Mac OS X 클라이언트 컴퓨터의 시동 볼륨에 로컬 홈 폴더를 가지고 있습니다. (사용자는 또한 사용자의 Active Directory 계정에 지정된 대로 네트워크 홈 폴더를 가지고 있습니다.)

모바일 계정은 Mac OS X 클라이언트 컴퓨터에 사용자의 Active Directory 인증 증명서를 캐시화합니다. 캐시화된 증명서는 클라이언트 컴퓨터가 Active Directory 서버에서 연결 해제되었을 때도 사용자가 Active Directory 이름 및 암호를 사용하여 로그인할 수 있도록 지원합니다.

모바일 계정은 Mac OS X 클라이언트 컴퓨터의 시동 볼륨에 로컬 홈 폴더를 가지고 있습니다. (사용자는 또한 사용자의 Active Directory 계정에 지정된 대로 네트워크 홈 폴더를 가지고 있습니다.)

Active Directory 체계가 Mac OS X 레코드 유형(객체 클래스) 및 속성을 포함하도록 확장되면, Active Directory 커넥터는 그러한 유형과 속성을 탐지하고 접근합니다.

예를 들어, Windows 관리 도구를 사용하여 Mac OS X 관리 클라이언트 속성을 포함시키도록 Active Directory 체계를 변경할 수 있습니다. 이 체계 변경을 통해 Active Directory 커넥터가 Mac OS X Server의 Workgroup Manager 응용 프로그램을 사용하여 설정된, 관리되는 클라이언트를 지원하도록 할 수 있습니다.

Mac OS X 클라이언트는 디렉토리에 추가된 속성에 대해 모든 읽기 접근 권한을 가집니다. 따라서, 이들 속성의 ACL을 변경하여 컴퓨터 목록에서 추가된 이들 속성을 읽을 수 있도록 해야할 것입니다.

Active Directory 커넥터는 많은 Active Directory에서 모든 도메인을 찾아냅니다. 어떤 도메인에 있는 사용자라도 Mac OS X 컴퓨터에서 인증할 수 있도록 플러그인을 설정할 수 있습니다. 또는 특정 도메인만 클라이언트상에 인증되도록 할 수 있습니다.

Active Directory 커넥터는 Active Directory 복제(replication) 및 장애 복구(failover)를 지원합니다. Active Directory 플러그인은 다중 도메인 컨트롤러를 찾고 가장 가까운 것을 결정합니다. 도메인 컨트롤러가 사용 불가능하게 되면, 플러그인은 다른 가까운 도메인 컨트롤러로 넘어갑니다.

Active Directory 커넥터는 LDAP을 사용하여 Active Directory 사용자 계정에 접근하고 Kerberos를 사용하여 사용자 계정을 인증합니다. Active Directory 커넥터는 디렉토리 또는 인증 서비스를 얻기 위해 Microsoft의 독자적인 ADSI(Active Directory Services Interface)를 사용하지 않습니다.
