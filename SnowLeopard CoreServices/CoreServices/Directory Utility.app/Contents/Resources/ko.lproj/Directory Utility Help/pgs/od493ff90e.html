LDAP 디렉토리에 접근하기 위한 구성 삭제하기

디렉토리 유틸리티를 사용하여 컴퓨터에서 LDAPv3나 LDAPv2 디렉토리로의 접근 방법을 지정한 구성을 삭제할 수 있습니다.

LDAP 구성이 DHCP에 의해 제공된다면 변경할 수 없으므로, LDAP 구성 목록에서 이 구성 옵션이 흐리게 표시됩니다.

LDAP 디렉토리에 접근하기 위한 구성을 삭제하려면,

  1. 시스템 환경설정을 열고 계정을 클릭하십시오.

  2. 자물쇠 아이콘이 잠겨 있으면, 해당 아이콘을 클릭하고 관리자의 이름과 암호를 입력하여 잠금 해제하십시오.

  3. 로그인 옵션을 클릭한 다음 편집을 클릭하십시오.

  4. 디렉토리 유틸리티 열기를 클릭하십시오.

  5. 자물쇠 아이콘이 잠겨 있으면, 해당 아이콘을 클릭하고 관리자의 이름과 암호를 입력하여 잠금 해제하십시오.

  6. 서비스를 클릭하십시오.

  7. 서비스 목록에서 LDAPv3를 선택하고 편집(/) 단추를 클릭하십시오.

  8. 서버 구성 목록이 가려져 있다면, 옵션 보기를 클릭하십시오.

  9. 목록에서 원하는 서버 구성을 선택하고 삭제를 클릭한 다음, 승인을 클릭하십시오.

  10. 다음에서 선택하십시오.

    • 컴퓨터가 LDAP 디렉토리에 바인딩되었다는 경고 메시지가 나타나고 신뢰 바인딩을 중단하고자 한다면, 승인을 클릭한 다음 LDAP 디렉토리 관리자(로컬 컴퓨터 관리자가 아닌)의 이름 및 암호를 입력하십시오.

    • 컴퓨터에서 LDAP 서버에 접근할 수 없다는 경고 메시지가 나타난다면, 승인을 클릭하여 신뢰 바인딩을 강제로 중단하십시오.

      신뢰 바인딩을 강제로 중단하면, 이 컴퓨터의 LDAP 디렉토리에 컴퓨터 레코드가 여전히 남게 됩니다. LDAP 디렉토리 관리자에게 통보하여 관리자가 컴퓨터 그룹에서 컴퓨터를 제거하도록 해야 합니다.

삭제된 구성은 인증 및 연락처용 사용자 설정 검색 정책에서 제거됩니다.
