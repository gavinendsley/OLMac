LDAP 디렉토리에 대한 신뢰 바인딩 설정하기

디렉토리 유틸리티를 사용하여 컴퓨터와 신뢰 바인딩을 지원하는 LDAP 디렉토리 사이에 신뢰 바인딩을 설정할 수 있습니다. 바인딩은 신뢰 바인딩을 설정하면 디렉토리에 생성되는 인증된 컴퓨터 레코드로 상호간에 인증됩니다.

컴퓨터는 DHCP 공급 LDAP 디렉토리와 신뢰 LDAP 바인딩 모두를 사용하도록 구성될 수 없습니다. 신뢰 LDAP 바인딩은 고유하게 정적이지만 DHCP 공급 LDAP는 동적입니다.

LDAP 디렉토리에 신뢰 바인딩을 설정하려면,

  1. 시스템 환경설정을 열고 계정을 클릭하십시오.

  2. 자물쇠 아이콘이 잠겨 있으면, 해당 아이콘을 클릭하고 관리자의 이름과 암호를 입력하여 잠금 해제하십시오.

  3. 로그인 옵션을 클릭한 다음 연결 또는 편집을 클릭하십시오.

  4. 디렉토리 유틸리티 열기를 클릭하십시오.

  5. 자물쇠 아이콘이 잠겨 있으면, 해당 아이콘을 클릭하고 관리자의 이름과 암호를 입력하여 잠금 해제하십시오.

  6. 서비스를 클릭하십시오.

  7. 서비스 목록에서 LDAPv3를 선택하고 편집(/) 단추를 클릭하십시오.

  8. 서버 구성 목록이 가려져 있다면, 옵션 보기를 클릭하십시오.

  9. 원하는 서버 구성 하나를 선택한 다음, 편집을 클릭하십시오.

  10. 바인드를 클릭한 다음 인증을 입력하고 승인을 클릭하십시오.

    컴퓨터의 이름 및 LDAP 디렉토리 도메인 관리자의 이름 및 암호를 입력하십시오. 컴퓨터 이름은 신뢰 바인딩 또는 기타 네트워크 서비스용으로 다른 컴퓨터에 의해 사용될 수 없습니다.

    바인드 단추가 나타나지 않는다면, LDAP 디렉토리가 신뢰 바인딩을 지원하지 않는 것입니다.

  11. 올바른 컴퓨터 이름을 제공했는지 확인하십시오.

    컴퓨터 레코드가 존재한다는 경고 메시지가 나타나면 취소를 클릭하여 뒤로 돌아가서 컴퓨터 이름을 변경하거나, 덮어쓰기를 클릭하여 기존 컴퓨터 레코드를 대치하십시오.

    기존의 컴퓨터 레코드가 없어졌거나 다른 컴퓨터에 있는 것 같습니다. 기존의 컴퓨터 레코드를 대치하고자 한다면, 레코드가 다른 컴퓨터를 비활성화할 경우를 대비해서 LDAP 디렉토리 관리자에게 통보해야 합니다.

    이런 경우에는 해당 컴퓨터에 대한 다른 이름을 사용하여 LDAP 디렉토리 관리자가 비활성된 컴퓨터에 다른 이름을 부여하고 속해있던 컴퓨터 그룹에 해당 이름을 추가해야 합니다.

  12. 신뢰 바인딩의 설정을 완료하려면 승인을 클릭하십시오.
