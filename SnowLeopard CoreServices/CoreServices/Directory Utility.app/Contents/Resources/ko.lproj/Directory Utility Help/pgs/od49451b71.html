홈
악의적인 DHCP 서버로부터 컴퓨터 보호하기

Apple은 보안이 중요시 되는 환경에서는 자동 인증 검색 정책을 DHCP 공급 LDAP 서버 또는 DHCP 공급 공유 디렉토리 도메인과 함께 사용하지 말 것을 권장합니다.

여러분의 네트워크에 접근할 수 있는 악의적인 해커가 허위 DHCP 서버 및 허위 LDAP 디렉토리(또는 공유 디렉토리 도메인)를 사용하여 루트 사용자 계정을 사용하여 여러분의 컴퓨터를 제어할 수 있습니다.

여러분의 네트워크에 접근하려는 해커의 경우에는 허위 DHCP 서버가 로컬 네트워크 또는 서브넷의 일부여야 합니다. 따라서 컴퓨터가 로컬 네트워크상에 있는 컴퓨터이고 Mac OS X Server NAT 서비스 또는 NAT 라우터를 통해 인터넷에 연결한다면 이 유형의 보안 위반은 불가능합니다. 하지만, 무선 로컬 네트워크의 경우, 해커가 유선 보다는 무선 로컬 네트워크에 훨씬 더 쉽게 접근할 수 있기 때문에, 보안이 감소합니다.

DHCP 공급 LDAP 디렉토리의 사용을 비활성화하고 로컬 디렉토리 도메인에 대한 DHCP 바인딩 및 전송을 비활성화하여(또는 로컬 디렉토리 도메인을 비활성화하여), Mac을 허위 DHCP 서버의 악의적인 침범으로부터 보호할 수 있습니다.

만약에 컴퓨터가 네트워크에 연결되어 있고 컴퓨터의 검색 정책을 자동에서 사용자 설정으로 변경하였을 때(디렉토리 유틸리티 내 검색 정책의 인증 패널에서) LDAP 서버에 연결된 휴대용 컴퓨터를 사용한다면, 컴퓨터가 네트워크에 연결되어 있지 않으면 시동 지연이 발생합니다.

컴퓨터가 컴퓨터의 사용자 설정 검색 정책에 나열된 특정 디렉토리 도메인에 연결할 수 없다면 지연이 발생합니다. 잠자는 중에 네트워크에서 연결 해제된 컴퓨터를 깨우면 지연을 확실히 느낄 수 있습니다.
