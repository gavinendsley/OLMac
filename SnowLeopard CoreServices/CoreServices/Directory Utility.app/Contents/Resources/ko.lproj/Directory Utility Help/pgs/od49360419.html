LDAP 디렉토리에 대한 접근 수동으로 구성하기

Mac OS X에서 LDAPv3 또는 LDAPv2 디렉토리에 접근하는 방법을 지정하는 구성을 수동으로 생성할 수 있습니다. LDAP 디렉토리 서버의 IP 주소 또는 DNS 이름을 알아야 합니다.

디렉토리가 Mac OS X Server에서 호스트되지 않는다면, 디렉토리의 데이터에 Mac OS X 데이터를 매핑하기 위한 검색 기반 및 템플릿을 알아야 합니다. 지원되는 매핑 템플릿은 다음과 같습니다.

  • Mac OS X Server와 같이 자체 매핑 및 검색 기반을 제공하는 디렉토리용 서버에서

  • Mac OS X Server 체계를 사용하는 디렉토리용 Open Directory 서버

  • Windows 2000, Windows 2003 이상 서버에서 호스트하는 디렉토리용 Active Directory

  • UNIX 서버에서 호스트하는 대부분의 디렉토리용 RFC 2307

  • 위 매핑들을 사용하지 않는 디렉토리용 사용자화

LDAPv3 플러그인은 Open Directory 복제 및 장애 복구를 완벽하게 지원합니다. Open Directory 마스터가 사용 불가능하게 되면, 플러그인은 다른 가까운 복제 항목으로 넘어갑니다.

중요:  컴퓨터 이름에 하이픈이 포함되어 있다면 LDAP 또는 Active Directory와 같은 디렉토리 도메인에 접속하거나 바인딩할 수 없습니다. 바인딩을 구축하려면 하이픈이 포함되어 있지 않은 컴퓨터 이름을 사용하십시오.
LDAP 디렉토리에 대한 접근을 수동으로 구성하려면,

  1. 시스템 환경설정을 열고 계정을 클릭하십시오.

  2. 자물쇠 아이콘이 잠겨 있으면, 해당 아이콘을 클릭하고 관리자의 이름과 암호를 입력하여 잠금 해제하십시오.

  3. 로그인 옵션을 클릭한 다음 연결 또는 편집을 클릭하십시오.

  4. 디렉토리 유틸리티 열기를 클릭하십시오.

  5. 자물쇠 아이콘이 잠겨 있으면, 해당 아이콘을 클릭하고 관리자의 이름과 암호를 입력하여 잠금 해제하십시오.

  6. 서비스를 클릭하십시오.

  7. 서비스 목록에서 LDAPv3를 선택하고 편집(/) 단추를 클릭하십시오.

    LDAPv3에 대한 활성화 체크상자를 선택하지 않고도 서비스 목록에서 LDAPv3을 선택할 수 있습니다.

  8. 신규를 클릭한 다음, 수동을 클릭하십시오.

  9. 구성에 대한 이름을 입력하십시오.

  10. Tab 키를 누르고 DNS 이름을 입력하거나, 접근하려는 LDAP 디렉토리를 호스팅하는 서버의 IP 주소를 입력하십시오.

  11. DNS 이름 또는 IP 주소 옆에 있는 팝업 메뉴를 클릭하고 매핑 템플릿 또는 방식을 선택하십시오.

    • 서버에서를 선택하면, 검색 기반 확장자가 필요 없습니다. 이 경우, Open Directory는 검색 기반 확장자가 LDAP 디렉토리의 첫번째 수준이라고 가정합니다.

    • 템플릿을 선택하면, LDAP 디렉토리에 대한 검색 기반 확장자를 입력하고 승인을 클릭하십시오. 검색 기반 확장자를 입력하지 않으면 컴퓨터가 LDAP 디렉토리에 있는 정보를 찾을 수 없습니다.

      일반적으로, 검색 기반 확장자는 서버의 DNS 이름에서 파생됩니다. 예를 들어, DNS 이름이 ods.example.com인 서버의 경우, 검색 기반 확장자는 "dc=ods,dc=example, dc=com"입니다.

    • 사용자화를 선택하면, Mac OS X 레코드 유형 및 속성과 연결된 LDAP 디렉토리의 클래스 및 속정 사이에 매핑을 설정해야 합니다. 추가 정보를 보려면 LDAP 검색 및 매핑 구성하기를 참조하십시오.

  12. “SSL을 사용한 암호화” 체크상자를 선택하기 전에 담당 Open Directory 관리자에게 문의하여 SSL이 필요한지 확인하십시오.

  13. 이 LDAP 구성에 대한 다음 설정을 변경하려면, 편집을 클릭하여 선택한 LDAP 구성에 대한 옵션을 표시하고 변경을 가한 다음, LDAP 구성 옵션의 편집을 완료하면 승인을 클릭하십시오.

    • 연결을 클릭하여 시간 초과 옵션을 설정하고 사용자 설정 포트를 지정하며, 서버 리퍼럴을 무시하거나 LDAPv2(읽기 전용) 프로토콜을 강제로 사용할 수 있습니다. 추가 정보를 보려면, LDAP 디렉토리의 연결 설정 변경하기를 참조하십시오.

    • 검색 및 매핑을 클릭하여 LDAP 서버에 대한 검색과 매핑을 설정하십시오. 추가 정보를 보려면 LDAP 디렉토리에 대한 신뢰 바인딩 설정하기를 참조하십시오.

    • 보안을 클릭하여 인증 연결(신뢰 바인딩 대신) 및 기타 보안 정책 옵션을 설정하십시오. 추가 정보를 보려면 LDAP 연결의 보안 정책 변경하기를 참조하십시오.

    • 바인드를 클릭하여 신뢰 바인딩을 설정하십시오(LDAP 디렉토리가 지원하는 경우). 추가 정보를 보려면 LDAP 디렉토리에 대한 신뢰 바인딩 설정하기를 참조하십시오.

  14. 승인을 클릭하면 LDAP 디렉토리에 대한 접근 구성의 수동 생성이 완료됩니다.

  15. 컴퓨터에서 구성을 생성한 LDAP 디렉토리에 접근하게 하려면 디스크 유틸리티 내의 검색 정책의 인증 패널 및 연락처 패널에 있는 사용자 설정 검색 정책에 디렉토리를 추가한 다음, 서비스 패널에 LDAPv3이 활성화되어 있는지 확인하십시오.

추가 정보를 보려면, LDAP 디렉토리 서비스 활성 또는 비활성화시키기 및 사용자 설정 검색 정책 지정하기를 참조하십시오.

참고:  RFC 2307(UNIX) 매핑을 사용하는 비 Apple LDAP 서버에서 Workgroup Manager를 사용하여 사용자를 생성하기 전에, Users 레코드 유형의 매핑을 편집해야 합니다. 추가 정보를 보려면, 사용자 생성을 활성화시키기 위해 RFC 2307 매핑 편집하기를 참조하십시오.
중요:  디렉토리 서버에 연결되어 있는 동안 changeip를 사용하여 IP 주소 및 컴퓨터 이름을 변경하면, 디렉토리 서버에 연결 해제 및 다시 연결하여 새로운 컴퓨터 이름 및 IP 주소로 디렉토리를 업데이트해야 합니다. 디렉토리 서버에 연결 해제하고 다시 연결하지 않으면 디렉토리가 업데이트되지 않으며 기존 컴퓨터 이름 및 IP 주소를 계속 사용하게 됩니다.
