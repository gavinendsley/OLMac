고급 검색 정책 설정 사용하기

디렉토리 유틸리티는 다음 검색 정책을 지정합니다.

  • 인증:Mac OS X은 인증 검색 정책을 사용하여 디렉토리 도메인으로부터 사용자 인증 정보 및 기타 관리 데이터를 찾고 검색합니다.

  • 연락처: Mac OS X은 연락처 검색 정책을 사용하여 디렉토리 도메인으로부터 이름, 주소 및 기타 연락 정보를 찾고 검색합니다. Mac OS X 주소록은 이 연락처 정보를 사용합니다. 다른 응용 프로그램도 이것을 사용하도록 프로그래밍할 수 있습니다.

    각 검색 정책은 디렉토리 도메인 목록으로 구성되어 있습니다. 목록에 있는 디렉토리 도메인의 순서는 검색 정책을 지정합니다. Mac OS X은 필요한 정보를 찾거나, 정보를 찾지 못한 채 목록의 마지막에 도달할 때까지, 목록의 처음부터 시작하여 디렉토리 도메인의 각 목록을 검색합니다.

    인증 및 연락처 검색 정책은 다음 설정 중 하나를 가질 수 있습니다.

  • 자동: 로컬 디렉토리 도메인에서 시작하고, DHCP에 의해 제공되는 LDAP 디렉토리와 컴퓨터에 연결된 디렉토리 도메인을 포함할 수 있습니다. 이것은 Mac OS X v10.2 이상에서는 기본 설정이며, 휴대용 컴퓨터에 보다 높은 유연성을 제공합니다.

  • 로컬 디렉토리: 로컬 디렉토리 도메인만을 포함합니다.

  • 사용자 설정 경로: 설정은 로컬 디렉토리 도메인에서 시작하고, 사용자가 선택한 LDAP 디렉토리, Active Directory 도메인, 공유 디렉토리 도메인, NetInfo 도메인, BSD 구성 파일, NIS 도메인을 포함합니다.

    /BSD/local 폴더에는 항상 검색 경로가 포함되어 있고 항상 흐리게 표시됩니다.

중요: 자동 인증 검색 정책 및 DHCP 공급 LDAP 서버를 사용하도록 Mac OS X을 구성했다면, 악의를 가진 사용자가 여러분의 컴퓨터에 침입할 수 있는 위험이 높아집니다. 또한, 컴퓨터가 무선 네트워크에 연결되도록 구성되어 있다면, 이 위험은 더욱 높아집니다. 추가 정보를 보려면 악의적인 DHCP 서버로부터 컴퓨터 보호하기 부분을 참조하십시오.
