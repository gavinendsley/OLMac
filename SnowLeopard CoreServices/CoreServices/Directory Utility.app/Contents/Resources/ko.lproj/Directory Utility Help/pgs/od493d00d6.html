주 그룹 ID를 Active Directory 속성으로 매핑하기

디렉토리 유틸리티의 Active Directory 커넥터를 사용하도록 구성된 컴퓨터에서, Mac OS X의 주 그룹 ID(GID) 속성에 매핑시키려는 Active Directory 속성을 지정할 수 있습니다.

일반적으로 Active Directory 체계는 주 GID에 매핑될 수 있는 속성을 포함하도록 확장되어야 합니다.

  • Active Directory 관리자가 Microsoft의 UNIX용 서비스를 설치하여 Active Directory 체계를 확장한다면, GID를 msSFU-30-Gid-Number 속성에 매핑할 수 있습니다.

  • Active Directory 관리자가 수동으로 Active Directory 체계를 확장하여 RFC 2307 속성을 포함시켰다면, GID를 gidNumber에 매핑할 수 있습니다.

  • Active Directory 관리자가 수동으로 Active Directory 체계를 확장하여 Mac OS X PrimaryGroupID 속성을 포함시켰다면, 주 GID를 이 속성에 매핑할 수 있습니다.

주 GID 매핑이 비활성되어 있으면, Active Directory 커넥터는 Active Directory의 표준 GUID 속성을 기반으로 주 GID를 생성합니다.

경고: 나중에 주요 GID의 매핑을 변경하였다면, 사용자들이 이전에 생성한 파일에 대한 접근 권한을 유실할 수도 있습니다.
주요 GID를 확장 Active Directory 체계에 매핑하려면,

  1. 시스템 환경설정을 열고 계정을 클릭하십시오.

  2. 자물쇠 아이콘이 잠겨 있으면, 해당 아이콘을 클릭하고 관리자의 이름과 암호를 입력하여 잠금 해제하십시오.

  3. 로그인 옵션을 클릭한 다음 편집을 클릭하십시오.

  4. 디렉토리 유틸리티 열기를 클릭하십시오.

  5. 자물쇠 아이콘이 잠겨 있으면, 해당 아이콘을 클릭하고 관리자의 이름과 암호를 입력하여 잠금 해제하십시오.

  6. 서비스를 클릭하십시오.

  7. 서비스 목록에서 Active Directory를 선택하고 편집(/) 단추를 클릭하십시오.

  8. 고급 옵션이 가려져 있으면, 고급 옵션 보기를 클릭하십시오.

  9. 매핑을 클릭하십시오.

  10. “사용자 GID를 다음 속성에 매핑”을 선택한 다음, 사용자 계정의 주 그룹 ID에 매핑시킬 Active Directory 속성의 이름을 입력하십시오.

  11. 승인을 클릭하십시오.
