LDAP 검색 및 매핑 구성하기

디렉토리 유틸리티를 사용하여 Mac OS X이 LDAP 디렉토리에서 특정 데이터를 검색하는 방법을 설정하는 매핑, 검색 기반, 검색 범위를 편집할 수 있습니다. 디렉토리 유틸리티의 목록에서 각 LDAP 디렉토리 구성에 대한 설정값을 별도로 편집할 수 있습니다. 각 LDAP 디렉토리 구성은 Mac OS X이 LDAPv3나 LDAPv2 디렉토리의 데이터에 접근하는 방법을 지정합니다.

다음을 편집할 수 있습니다.

  • 각 Mac OS X 레코드 유형을 LDAP 객체 클래스에 매핑

  • Mac OS X 데이터 유형 또는 속성을 각 레코드 유형에 대한 LDAP 속성에 매핑

  • Mac OS X이 LDAP 디렉토리의 Mac OS X 레코드 유형을 검색할 수 있도록 해주는 LDAP 검색 기반과 검색 범위

Mac OS X 사용자 속성을 읽기/쓰기 LDAP 디렉토리 도메인(읽기 전용이 아닌 LDAP 도메인)에 매핑할 때, RealName에 매핑되는 LDAP 속성은 RecordName에 매핑된 LDAP 속성의 목록에 있는 첫번째 속성과 동일해서는 안 됩니다.

예를 들어, cn 속성이 RealName으로 매핑되어 있다면, cn 속성은 RecordName으로 매핑되는 첫번째 속성이 아니어야 합니다.

RealName에 매핑된 LDAP 속성이 RecordName에 매핑된 첫번째 속성과 동일하면 작업 그룹 관리자에서 전체(긴) 이름이나 첫번째 짧은 이름을 편집하려고 할 경우 문제가 발생합니다.

Mac OS X 레코드 유형 및 속성에 대한 추가 정보를 보려면, Open Directory Administration(Open Directory 관리)(www.apple.com/kr/server/resources/에 있음)를 참조하십시오.

LDAP 서버에 대한 검색 기반 및 매핑을 편집하려면,

  1. 시스템 환경설정을 열고 계정을 클릭하십시오.

  2. 자물쇠 아이콘이 잠겨 있으면, 해당 아이콘을 클릭하고 관리자의 이름과 암호를 입력하여 잠금 해제하십시오.

  3. 로그인 옵션을 클릭한 다음 편집을 클릭하십시오.

  4. 디렉토리 유틸리티 열기를 클릭하십시오.

  5. 자물쇠 아이콘이 잠겨 있으면, 해당 아이콘을 클릭하고 관리자의 이름과 암호를 입력하여 잠금 해제하십시오.

  6. 서비스를 클릭하십시오.

  7. 서비스 목록에서 LDAPv3를 선택하고 편집(/) 단추를 클릭하십시오.

  8. 서버 구성 목록이 가려져 있다면, 옵션 보기를 클릭하십시오.

  9. 목록에서 서버 구성 하나를 선택한 다음, 편집을 클릭하십시오.

  10. 검색 및 매핑을 클릭하십시오.

  11. 시작점으로 사용하려는 매핑을 선택하고 아니면, 사용자화를 선택하여 미리 설정되지 않은 매핑으로 시작하십시오.

    LDAP 매핑 템플릿 중 하나를 선택하면 변경할 수 있는 검색 기반 확장자가 나타나거나 승인을 클릭하여 기본 검색 기반을 수용할 수 있습니다.

    “다음으로 이 LDAPv3 서버에 연결” 팝업 메뉴를 클릭하고 해당 매핑을 시작점으로 사용하기 위한 매핑 템플릿을 선택하십시오.

  12. 레코드 유형을 추가하고 필요에 따라 기본 검색을 변경하십시오.

    • 레코드 유형을 추가하려면 추가(레코드 유형 및 속성 목록 아래)를 클릭하고 나타나는 시트에서 레코드 유형을 선택한 다음, 목록에서 레코드 유형을 선택하고 승인을 클릭하십시오.

    • 검색 기반 및 레코드 유형의 검색 범위를 변경하려면 레코드 유형 및 속성 목록에서 해당 항목을 선택한 다음 “검색 기반” 필드를 편집하십시오. “모든 서브 트리”를 선택하여 검색 기반으로부터 LDAP 디렉토리의 계층을 포함하도록 검색 범위를 설정하거나 “1단계만”을 선택하여 LDAP 디렉토리의 계층에서 검색 기반 및 1단계 아래만 포함하도록 검색 범위를 설정하십시오.

    • 레코드 유형을 제거하려면, 레코드 유형과 속성 목록에서 해당 항목을 선택하고 제거를 클릭하십시오.

    • 레코드 유형에 대해 매핑을 추가하려면 레코드 유형 및 속성 목록에서 레코드 유형을 선택한 후 추가(”목록에 있는 __ 항목으로 매핑” 아래)를 클릭한 다음, LDAP 디렉토리로부터 객체 클래스의 이름을 입력하십시오.

      다른 LDAP 객체 클래스를 추가하려면 Return 키를 누르고 해당 객체 클래스의 이름을 입력한 다음, 목록 위에 있는 팝업 메뉴를 사용하여 나열된 LDAP 객체 클래스를 사용할 것인 지를 지정하십시오.

    • 레코드 유형에 대해 매핑을 변경하려면 레코드 유형 및 속성 목록에서 레코드 유형을 선택하고 “목록에 있는 __ 항목으로 매핑”에서 변경하려는 LDAP 객체 클래스를 이중 클릭한 다음 편집하십시오. 목록 위에 있는 팝업 메뉴를 사용하여, LDAP 객체 클래스를 사용할 지를 지정하십시오.

    • 레코드 유형에 대한 매핑을 제거하려면 레코드 유형 및 속성 목록에서 레코드 유형을 선택하고 “목록에 있는 __ 항목으로 매핑”에서 제거하려는 LDAP 객체 클래스를 선택한 다음, 삭제(“목록에서 __ 항목으로 매핑” 아래에)를 클릭하십시오.

  13. 속성을 추가하고 필요하다면 해당 매핑을 변경하십시오.

    • 레코드 유형에 속성을 추가하려면 레코드 유형 및 속성 목록에서 레코드 유형을 선택하고 추가(레코드 유형 및 속성 목록 아래)를 클릭한 다음, 나타나는 시트에서 속성 유형을 선택하고 속성 유형을 선택한 다음 승인을 클릭하십시오.

    • 속성에 대해 매핑을 추가하려면 레코드 유형 및 속성 목록에서 해당 속성을 선택한 후 추가(“목록에 있는 __ 항목으로 매핑” 아래)를 클릭한 다음 LDAP 디렉토리로부터 객체 클래스의 이름을 입력하십시오. 다른 LDAP 객체 클래스를 추가하려면 Return 키를 누른 다음 해당 객체 클래스의 이름을 입력하십시오.

    • 속성에 대한 매핑을 변경하려면 레코드 유형 및 속성 목록에서 해당 속성을 선택하고 “목록에 있는 __ 항목으로 매핑”에서 변경하려는 항목을 이중 클릭한 다음 항목 이름을 편집하십시오.

    • 속성에 대한 매핑을 제거하려면 레코드 유형 및 속성 목록에서 속성을 선택하고 “목록에 있는 __ 항목으로 매핑”에서 제거하려는 항목을 선택한 다음, 삭제(“목록에서 __ 항목으로 매핑” 아래에)를 클릭하십시오.

    • 오른쪽 목록에 나타난 속성들의 순서를 변경하려면, 목록에서 속성을 위 또는 아래로 드래그하십시오.

  14. 매핑을 템플릿으로 저장하려면 템플릿 저장을 클릭하십시오.

    기본 위치에 저장된 템플릿이 여러분이 다음에 디렉토리 유틸리티를 열 때 LDAP 매핑 템플릿의 팝업 메뉴에 나열됩니다. 저장된 템플릿에 대한 기본 위치는 사용자 홈 폴더의 다음 경로입니다.

    ~/라이브러리/Application Support/Directory Access/LDAPv3/Templates

  15. 매핑을 LDAP 디렉토리에 저장하여 클라이언트에게 자동으로 공급되게 하려면, 서버에 쓰기를 클릭하고 매핑을 저장할 검색 기반, 관리자나 해당 검색 기반에 대해 쓰기 권한이 있는 다른 사용자의 고유한 이름(예: uid=diradmin,cn=users,dc=ods,dc=example,dc=com), 그리고 암호를 입력하십시오.

    Open Directory LDAP 서버에 매핑을 작성한다면, 올바른 검색 기준은 cn=config, 확장자”(확장자는 서버의 검색 기반 확장자임. 예: dc=ods,dc=example,dc=com)입니다.

LDAP 디렉토리는 LDAP 서버로부터 매핑을 가져오도록 구성되어 있는 연결이 포함된 사용자 설정 검색 정책을 가진 Mac OS X 클라이언트에게 자신의 매핑을 제공합니다.

또한, LDAP 디렉토리는 자신의 매핑을 자동 검색 정책을 가지고 있는 모든 Mac OS X 클라이언트에게 제공합니다. 추가 정보를 보려면, LDAP 디렉토리로의 접근 구성하기 및 고급 검색 정책 설정 사용하기를 참조하십시오.
