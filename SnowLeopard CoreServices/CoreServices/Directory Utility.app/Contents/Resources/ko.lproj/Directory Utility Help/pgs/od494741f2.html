홈
LDAP 디렉토리에 접근하기 위한 구성 변경하기

디렉토리 유틸리티를 사용하여 LDAP 디렉토리 구성의 설정을 변경할 수 있습니다. 구성 설정은 Open Directory가 LDAPv3나 LDAPv2 디렉토리에 접근하는 방법을 지정합니다.

LDAP 구성이 DHCP에 의해 제공된다면 변경할 수 없으므로, LDAP 구성 목록에서 이 유형의 구성이 흐리게 표시됩니다.

LDAP 디렉토리에 접근하기 위한 구성을 편집하려면,

  1. 시스템 환경설정을 열고 계정을 클릭하십시오.

  2. 자물쇠 아이콘이 잠겨 있으면, 해당 아이콘을 클릭하고 관리자의 이름과 암호를 입력하여 잠금 해제하십시오.

  3. 로그인 옵션을 클릭한 다음 편집을 클릭하십시오.

  4. 디렉토리 유틸리티 열기를 클릭하십시오.

  5. 자물쇠 아이콘이 잠겨 있으면, 해당 아이콘을 클릭하고 관리자의 이름과 암호를 입력하여 잠금 해제하십시오.

  6. 서비스를 클릭하십시오.

  7. 서비스 목록에서 LDAPv3를 선택하고 편집(/) 단추를 클릭하십시오.

  8. 서버 구성 목록이 가려져 있다면, 옵션 보기를 클릭하십시오.

  9. 다음 설정을 필요한 대로 변경하십시오.

    • 활성화: 체크상자를 클릭하면 LDAP 디렉토리 서버에 대한 접근이 활성 또는 비활성화됩니다.

    • 구성 이름: 구성 이름을 이중 클릭하면 편집할 수 있습니다.

    • 서버 이름 또는 IP 주소: 서버 이름 또는 IP 주소를 이중 클릭하면 변경할 수 있습니다.

    • LDAP 매핑: 팝업 메뉴에서 템플릿을 선택한 다음 LDAP 디렉토리에 대한 검색 기반 확장자를 입력하고 승인을 클릭하십시오.

      템플릿을 선택했다면, 검색 기반 확장자를 입력해야 합니다. 그렇지 않으면, 컴퓨터는 LDAP 디렉토리에 있는 정보를 찾을 수 없습니다. 일반적으로, 검색 기반 확장자는 서버의 DNS 이름에서 파생됩니다. 예를 들어, DNS 이름이 ods.example.com인 서버의 경우, 검색 기반 확장자는 “dc=ods,dc=example,dc=com”입니다.

      템플릿이 아닌 서버로부터를 선택했다면 검색 기반 확장자가 필요하지 않습니다. 이 경우, Open Directory는 검색 기반 확장자가 LDAP 디렉토리의 첫번째 수준이라고 가정합니다.

      사용자화를 선택하면, Mac OS X 레코드 유형 및 속성과 연결된 LDAP 디렉토리의 클래스 및 속정 사이에 매핑을 설정해야 합니다. 추가 정보를 보려면 LDAP 검색 및 매핑 구성하기를 참조하십시오.

    • SSL: 체크상자를 클릭하여 SSL 프로토콜을 사용하여 암호화된 통신을 활성화 또는 비활성화합니다. SSL 체크상자를 선택하기 전에 Open Directory 관리자에게 문의하여 SSL이 필요한 지 확인하십시오.

  10. 이 LDAP 구성에 대한 다음 기본 설정을 변경하려면, 편집을 클릭하여 선택한 LDAP 구성에 대한 옵션을 표시하고 변경을 가한 다음, LDAP 구성 옵션의 편집을 완료하면 승인을 클릭하십시오.

    • 연결을 클릭하여 시간 초과 옵션을 설정하고 사용자 설정 포트를 지정하며, 서버 리퍼럴을 무시하거나 LDAPv2(읽기 전용) 프로토콜을 강제로 사용할 수 있습니다. 추가 정보를 보려면, LDAP 디렉토리의 연결 설정 변경하기를 참조하십시오.

    • 검색 및 매핑을 클릭하여 LDAP 서버에 대한 검색과 매핑을 설정하십시오. 추가 정보를 보려면 LDAP 디렉토리에 대한 신뢰 바인딩 설정하기를 참조하십시오.

    • 보안을 클릭하여 인증 연결(신뢰 바인딩 대신) 및 기타 보안 정책 옵션을 설정하십시오. 추가 정보를 보려면 LDAP 연결의 보안 정책 변경하기를 참조하십시오.

    • 바인드를 클릭하여 신뢰 바인딩을 설정하거나 바인드 해제를 클릭하여 신뢰 바인딩을 중단하십시오. (LDAP 디렉토리에서 신뢰 바인딩을 허용하지 않는다면 이 단추가 나타나지 않습니다.) 추가 정보를 보려면 LDAP 디렉토리에 대한 신뢰 바인딩 설정하기를 참조하십시오.

  11. LDAP 디렉토리에 대한 접근 구성의 변경을 완료하려면 승인을 클릭하십시오.
