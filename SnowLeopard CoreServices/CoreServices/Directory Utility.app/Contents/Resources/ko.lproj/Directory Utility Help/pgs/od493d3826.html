LDAP 디렉토리의 연결 설정 변경하기

디렉토리 유틸리티를 사용하여 컴퓨터에서 특정 LDAPv3나 LDAPv2 디렉토리로의 접근 방법을 지정한 구성의 연결 설정을 변경할 수 있습니다.

LDAP 디렉토리 접근에 대한 연결 설정을 변경하려면,

  1. 시스템 환경설정을 열고 계정을 클릭하십시오.

  2. 자물쇠 아이콘이 잠겨 있으면, 해당 아이콘을 클릭하고 관리자의 이름과 암호를 입력하여 잠금 해제하십시오.

  3. 로그인 옵션을 클릭한 다음 편집을 클릭하십시오.

  4. 디렉토리 유틸리티 열기를 클릭하십시오.

  5. 자물쇠 아이콘이 잠겨 있으면, 해당 아이콘을 클릭하고 관리자의 이름과 암호를 입력하여 잠금 해제하십시오.

  6. 서비스를 클릭하십시오.

  7. 서비스 목록에서 LDAPv3를 선택하고 편집(/) 단추를 클릭하십시오.

  8. 서버 구성 목록이 가려져 있다면, 옵션 보기를 클릭하십시오.

  9. 목록에서 서버 구성 하나를 선택한 다음, 편집을 클릭하십시오.

  10. 연결을 클릭하고 다음 설정 중에서 변경하십시오.

    • 구성 이름: LDAP 디렉토리 구성 목록에서 구성을 식별할 수 있습니다. (LDAP 디렉토리 구성 목록에서 이름을 변경할 수도 있습니다.)

    • 서버 이름 또는 IP 주소: 서버의 DNS 이름과 IP 주소를 지정합니다. (LDAP 디렉토리 구성 목록에서 이것을 변경할 수도 있습니다.)

    • 열기/닫기 시간 제한: 시도가 취소되기 전에 연결 시도가 지속될 수 있는 최대 시간을 지정합니다.

    • 질의 시간 제한: 질의가 취소되기 전에 질의가 지속될 수 있는 최대 시간을 지정합니다.

    • 다음 시간 후 바인드 재시도: LDAP 서버의 응답이 실패한 경우 재연결 시도 전에 대기할 시간을 초 단위로 지정합니다. 지속적인 재연결 시도를 방지하려면 이 값을 높이십시오.

    • 다음 시간 후 연결 대기 해제: 대기 또는 비응답 연결을 열린 채로 있게 할 시간을 분 단위로 지정합니다.

    • SSL을 사용하여 암호화: SSL 연결을 사용하여 LDAP 디렉토리로 통신을 암호화할 것인지를 결정합니다. (LDAP 디렉토리 구성 목록에서 이 설정을 변경할 수도 있습니다.) SSL 체크상자를 선택하기 전에 Open Directory 관리자에게 문의하여 SSL이 필요한 지 확인하십시오.

    • 사용자 설정 포트 사용: LDAP 연결을 위한 표준 포트 외의 다른 포트 번호를 지정합니다(389 - SSL을 사용하지 않을 경우, 636 - SSL을 사용할 경우).

    • 서버 리퍼럴 무시: 다른 LDAP 서버 또는 정보용 복제본을 참조하려는 LDAP 서버의 리퍼럴을 무시할지 아니면 따를지를 결정합니다. 서버 리퍼럴은 컴퓨터에서 정보를 쉽게 찾을 수 있도록 지원하지만, 컴퓨터가 다른 LDAP 서버에 대한 리퍼럴을 검사하는 경우에는 다른 지연을 일으킬 수도 있습니다.

    • LDAPv2 사용(읽기 전용): LDAP 디렉토리에 대한 읽기 전용 접근을 위해 기존의 LDAPv2 프로토콜을 사용할지 여부를 결정합니다.
