홈
Active Directory 도메인으로의 접근 구성하기

디렉토리 유틸리티에 나열된 Active Directory 커넥터를 사용하여, Mac OS X이 Windows 서버의 Active Directory 도메인에 있는 기본 사용자 계정 정보에 접근하도록 구성할 수 있습니다.

Active Directory 커넥터는 Mac OS X 인증에 필요한 모든 속성을 생성합니다. Active Directory 체계를 변경할 필요가 없습니다.

Active Directory 체계가 Mac OS X 레코드 유형 및 속성을 포함할 수 있도록 확장되면, Active Directory 커넥터는 Mac OS X 클라이언트 관리에 필요한 속성과 같은, 표준 Mac OS X 레코드 유형 및 속성을 찾고 접근합니다.

경고: Active Directory 커넥터의 고급 옵션을 사용하면 Mac OS X 고유 사용자 ID(UID), 주 그룹 ID(GID)및 그룹 GID 속성을 Active Directory 체계에 추가된 적절한 속성에 매핑할 수 있습니다. 나중에 이들 매핑 옵션을 변경하였다면, 사용자들이 이전에 생성한 파일에 대한 접근 권한을 유실할 수도 있습니다.
중요: 컴퓨터 이름에 하이픈이 포함되어 있다면 LDAP 또는 Active Directory와 같은 디렉토리 도메인에 접속하거나 바인딩할 수 없습니다. 바인딩을 구축하려면 하이픈이 포함되어 있지 않은 컴퓨터 이름을 사용하십시오.
Active Directory 도메인에 대한 접근을 구성하려면,

  1. 시스템 환경설정을 열고 계정을 클릭하십시오.

  2. 자물쇠 아이콘이 잠겨 있으면, 해당 아이콘을 클릭하고 관리자의 이름과 암호를 입력하여 잠금 해제하십시오.

  3. 로그인 옵션을 클릭한 다음 연결 또는 편집을 클릭하십시오.

  4. 디렉토리 유틸리티 열기를 클릭하십시오.

  5. 자물쇠 아이콘이 잠겨 있으면, 해당 아이콘을 클릭하고 관리자의 이름과 암호를 입력하여 잠금 해제하십시오.

  6. 서비스를 클릭하십시오.

  7. 서비스 목록에서 Active Directory를 선택하고 편집(/) 단추를 클릭하십시오.

  8. 구성하는 컴퓨터에 바인딩할 Active Directory 도메인의 DNS 이름을 입력하십시오.

    Active Directory 도메인의 관리자가 입력해야 할 DNS 이름을 알려줄 것입니다.

  9. 필요하다면, 컴퓨터 ID를 편집하십시오.

    컴퓨터 ID는 Active Directory 도메인에 알려질 컴퓨터의 이름이며, 컴퓨터의 이름으로 미리 설정되어 있습니다. 이 이름을 Activity Directory 도메인에서 자신이 속한 조직의 이름 지정 기준에 따라 변경할 수도 있습니다. 확실하지 않다면, Active Directory 도메인 관리자에게 문의하십시오.

  10. (옵션) 고급 옵션을 설정하십시오.

    고급 옵션이 숨겨져 있다면, 고급 옵션 보기를 클릭한 다음 사용자 환경, 매핑 및 관리자 패널에서 옵션을 설정하십시오. 또한, 나중에 고급 옵션 설정을 변경할 수도 있습니다.

    고급 옵션에 관한 추가 정보는 다음을 참조하십시오.

    • Active Directory에서 모바일 사용자 계정 설정하기

    • Active Directory 사용자 계정의 홈 폴더 설정하기

    • Active Directory 사용자 계정의 UNIX 셸 설정하기

    • UID를 Active Directory 속성으로 매핑하기

    • 주 그룹 ID를 Active Directory 속성으로 매핑하기

    • 그룹 계정의 그룹 ID를 Active Directory 속성으로 매핑하기

    • 권장 Active Directory 서버 지정하기

    • 컴퓨터를 관리할 수 있는 Active Directory 그룹 변경하기

    • Active Directory 포리스트의 모든 도메인으로부터 인증 제어하기

  11. 바인드를 클릭하고 다음을 사용하여 Active Directory 도메인에 컴퓨터를 바인딩할 수 있는 권한을 가진 사용자를 인증한 다음, Active Directory를 추가하려는 검색 정책을 선택한 후(아래 참조) 승인을 클릭하십시오.

    • 사용자 이름 및 암호: 자신의 Active Directory 사용자 계정의 이름 및 암호를 입력하여 인증할 수도 있고 Active Directory 도메인 관리자가 이름과 암호를 제공할 수도 있습니다.

    • 컴퓨터 OU: 구성하는 컴퓨터에 대한 조직 단위(OU)를 입력하십시오.

    • 인증 시 사용: Active Directory가 컴퓨터의 인증 검색 정책에 자동으로 추가되는지 여부를 결정합니다.

    • 연락처에 사용: Active Directory가 컴퓨터의 연락처 검색 정책에 자동으로 추가되는지 여부를 결정합니다.

    승인을 클릭하면, 디렉토리 유틸리티에서 구성하는 컴퓨터와 Active Directory 서버 사이에 신뢰 바인딩을 설정합니다. 컴퓨터의 검색 정책이 인증했을 때 선택한 옵션에 따라 설정되며, 디렉토리 유틸리티의 서비스 패널에서 Active Directory가 활성화됩니다.

    "인증에 사용" 또는 "연락처에 사용" 옵션을 선택하였다면, Active Directory 고급 옵션에 대한 기본 설정을 통해 컴퓨터의 인증 검색 정책 및/또는 연락처 검색 정책에 Active Directory 포리스트가 추가됩니다.

    하지만, 바인드를 클릭하기 전에 관리자 고급 옵션 패널에서 "포리스트의 모든 도메인으로부터 인증 허용"을 선택 해제했다면, 포리스트 대신에 가장 가까운 Active Directory 도메인이 추가됩니다.

    Active Directory 포리스트 또는 개별 도메인을 추가 또는 제거하여 검색 정책을 나중에 변경할 수 있습니다. 추가 정보는, 사용자 설정 검색 정책 지정하기를 참조하십시오.

  12. (옵션) 서버를 Active Directory Kerberos 영역에 연결하십시오.

    • 이 서버에 연결할 수 있는 서버 또는 관리자 컴퓨터에서 Server Admin을 열고 해당 서버에 대한 Open Directory를 선택하십시오.

    • 설정을 클릭한 다음, 일반을 클릭하십시오.

    • Kerberos 연결을 클릭하고 팝업 메뉴에서 Active Directory Kerberos 영역을 선택한 다음, 이 서버상의 로컬 관리자에 대한 증명서를 입력하십시오.
