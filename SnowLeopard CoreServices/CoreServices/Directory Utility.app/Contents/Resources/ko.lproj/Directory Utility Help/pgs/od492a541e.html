암호 작성하기

사용자가 로그인 또는 서비스를 인증할 때 사용자의 계정과 관련된 암호를 입력해야 합니다. 암호는 대/소문자를 구분하며(SMB-LAN Manager 암호 제외) 입력하면 화면에 가려져서 표시됩니다.

사용자가 선택한 암호 유형에 상관없이, Mac OS X Server 사용자 계정에 대한 암호를 작성하는 지침입니다.

  • 암호는 인증되지 않은 사용자가 쉽게 추측할 수 없도록 문자, 숫자 및 기호 조합으로 구성되어야 합니다. 단어로 구성되어서는 안됩니다. 좋은 암호는 숫자 및 기호(# 또는 $)가 포함되어 있거나 문구에 있는 모든 단어의 첫 문자로 구성되어 있습니다. 대문자 및 소문자 모두를 사용하십시오.

  • 빈칸 및 Option-키 조합은 피하십시오.

  • 사용자가 사용할 컴퓨터에 입력할 수 없거나 특별한 키입력 조합을 알아야 다른 키보드 및 플랫폼에 올바르게 입력할 수 있는 문자는 피하십시오.

  • 일부 네트워크 프로토콜은 맨 앞에 넣은 빈칸, 중간에 끼워 넣는 빈칸 또는 마지막에 넣는 빈칸이 포함된 암호를 지원하지 않습니다.

  • 길이가 0인 암호는 권장되지 않습니다. Open Directory 및 일부 시스템(LDAP 바인드)는 길이가 0인 암호를 지원하지 않습니다.

  • 사용자가 연결할 수 있는 다른 컴퓨터 및 서비스와 호환성을 최대한 늘리기 위해서는 암호에 ASCII 문자만 사용하십시오.
