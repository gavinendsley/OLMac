LDAP 연결 인증하기

디렉토리 유틸리티를 사용하여, LDAP 디렉토리에 대한 인증 연결을 설정할 수 있습니다. 이 인증은 한 방향입니다. 컴퓨터에서는 LDAP 디렉토리에 대해 자체적으로 자격을 인증하지만, LDAP 디렉토리는 컴퓨터에 대해 자격을 인증하지 않습니다. 상호적인 인증에 관해 보려면, LDAP 디렉토리에 대한 신뢰 바인딩 설정하기를 참조하십시오.

참고: 컴퓨터와 LDAP 디렉토리 사이에 이미 신뢰 바인딩이 설정되어 있다면, 인증 연결이 중복되므로 다시 설정할 수 없습니다.
인증된 LDAPv3 연결을 설정하려면,

  1. 시스템 환경설정을 열고 계정을 클릭하십시오.

  2. 자물쇠 아이콘이 잠겨 있으면, 해당 아이콘을 클릭하고 관리자의 이름과 암호를 입력하여 잠금 해제하십시오.

  3. 로그인 옵션을 클릭한 다음 연결 또는 편집을 클릭하십시오.

  4. 디렉토리 유틸리티 열기를 클릭하십시오.

  5. 자물쇠 아이콘이 잠겨 있으면, 해당 아이콘을 클릭하고 관리자의 이름과 암호를 입력하여 잠금 해제하십시오.

  6. 서비스를 클릭하십시오.

  7. 서비스 목록에서 LDAPv3를 선택하고 편집(/) 단추를 클릭하십시오.

  8. 서버 구성 목록이 가려져 있다면, 옵션 보기를 클릭하십시오.

  9. 목록에서 서버 구성 하나를 선택한 다음, 편집을 클릭하십시오.

  10. 보안을 클릭하십시오.

  11. “연결 시 인증 사용”을 선택한 다음, 사용자의 식별 이름과 암호를 입력하십시오.

    식별되는 이름은 디렉토리에서 데이터를 볼 수 있는 권한을 가진 모든 사용자 계정을 지정할 수 있습니다. 예를 들어, ods.example.com 주소의 LDAP 서버에 ID가 “authenticator”인 사용자 계정은 uid=authenticator,cn=users,dc=ods,dc=example,dc=com인 이름으로 식별됩니다.

    중요: 식별된 이름이나 암호가 올바르지 않으면 LDAP 서버로부터 사용자 계정을 사용하여 컴퓨터에 로그인할 수 있습니다.
