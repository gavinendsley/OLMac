LDAP 디렉토리로의 접근 구성하기

LDAP 디렉토리 서버의 DNS 이름 또는 IP 주소를 안다면, 디렉토리 유틸리티를 사용하여 Mac OS X이 LDAPv3에 접근하는 방식을 지정할 수 있습니다.

디렉토리가 Mac OS X Server처럼 자체 매핑을 제공하는 서버에 의해 호스트되고 있지 않다면, 해당 디렉토리 데이터에 대해 Mac OS X 데이터를 매핑하기 위한 검색 기반과 템플릿을 알아야 합니다.

지원되는 매핑 템플릿은 다음과 같습니다.

  • Mac OS X Server 체계를 사용하는 디렉토리용 Open Directory 서버

  • Windows 2000, Windows 2003 이상 서버에서 호스트하는 디렉토리용 Active Directory

  • UNIX 서버에서 호스트하는 대부분의 디렉토리용 RFC 2307

LDAPv3 플러그인은 Open Directory 복제 및 장애 복구를 완벽하게 지원합니다. Open Directory 마스터가 사용 불가능하게 되면, 플러그인은 다른 가까운 복제 항목으로 넘어갑니다.

디렉토리 데이터에 대해 사용자 설정 매핑을 지정해야 한다면, 여기에 나와있는 지침 대신에 LDAP 디렉토리에 대한 접근 수동으로 구성하기를 참조하십시오.

중요: 컴퓨터 이름에 하이픈이 포함되어 있다면 LDAP 또는 Active Directory와 같은 디렉토리 도메인에 접속하거나 바인딩할 수 없습니다. 바인딩을 구축하려면 하이픈이 포함되어 있지 않은 컴퓨터 이름을 사용하십시오.
디렉토리 유틸리티가 LDAP 디렉토리에 대한 접근을 구성하는 것을 지원하려면,

  1. 시스템 환경설정을 열고 계정을 클릭하십시오.

  2. 자물쇠 아이콘이 잠겨 있으면, 해당 아이콘을 클릭하고 관리자의 이름과 암호를 입력하여 잠금 해제하십시오.

  3. 로그인 옵션을 클릭한 다음 연결 또는 편집을 클릭하십시오.

  4. 디렉토리 유틸리티 열기를 클릭하십시오.

  5. 자물쇠 아이콘이 잠겨 있으면, 해당 아이콘을 클릭하고 관리자의 이름과 암호를 입력하여 잠금 해제하십시오.

  6. 서비스를 클릭하십시오.

  7. 서비스 목록에서 LDAPv3를 선택하고 편집(/) 단추를 클릭하십시오.

    LDAPv3에 대한 활성화 체크상자를 선택하지 않고도 서비스 목록에서 LDAPv3을 선택할 수 있습니다.

  8. 신규를 클릭하고 LDAP 서버의 DNS 이름 또는 IP 주소를 입력하십시오.

  9. 디렉토리 접근 옵션을 선택하십시오.

    • Open Directory가 LDAP 디렉토리 연결에 SSL(Secure Sockets Layer)을 사용하도록 하려면, “SSL을 사용한 암호화”를 선택하십시오. 이 항목을 선택하기 전에 Open Directory 관리자에게 문의하여 SSL이 필요한 지 확인하십시오.

    • 이 디렉토리에 누군가가 서비스 인증 또는 로그인에 사용할 사용자 계정을 가지고 있다면, “인증 시 사용”을 선택하십시오.

    • 이 디렉토리에 주소록에서 사용하고자 하는 메일 주소 및 기타 정보가 있다면, “연결에 사용”을 선택하십시오.

    디렉토리 유틸리티에서 LDAP 서버에 접속할 수 없다는 메시지가 나타나면 접근을 수동으로 구성하거나 설정 과정을 취소해야 합니다. 수동 구성 지침에 대한 추가 정보를 보려면 LDAP 디렉토리에 대한 접근 수동으로 구성하기를 참조하십시오.

    대화상자가 매핑 옵션을 표시하도록 확장되면, 팝업 메뉴에서 매핑 템플릿을 선택하고 검색 기반 확장자를 입력한 다음, 계속을 클릭하십시오.

    일반적으로, 검색 기반 확장자는 서버의 DNS 이름에서 파생됩니다. 예를 들어, DNS 이름이 ods.example.com인 서버의 경우, 검색 기반 확장자는 "dc=ods,dc=example, dc=com"입니다.

    사용 가능한 매핑 템플릿 중 설정하려는 연결에 적용된 것이 없다면 수동을 클릭하십시오. 추가 정보를 보려면 LDAP 디렉토리에 대한 접근 수동으로 구성하기를 참조하십시오.

  10. 디렉토리 유틸리티에서 LDAP 서버로부터 정보를 받게 하려면 계속을 클릭하십시오.

  11. 대화상자가 확장되면서 신뢰 바인딩에 대한 옵션이 나타나면, 컴퓨터의 이름 및 디렉토리 관리자의 암호를 입력하십시오. (바인딩은 옵션일 수 있음)

    LDAP 디렉토리에 신뢰 바인딩이 필수인지 아니면 옵션인지 표시하는 대화상자가 나타납니다. 신뢰 바인딩은 상관 관계가 있습니다. 컴퓨터가 LDAP 디렉토리에 연결될 때마다 서로를 인증합니다. 신뢰 바인딩이 설정되어 있거나 LDAP 디렉토리가 신뢰 바인딩을 지원하지 않는다면, 바인드 단추가 나타나지 않습니다. 올바른 컴퓨터 이름을 공급했는지 확인하십시오.

    컴퓨터 레코드가 존재한다는 경고 메시지가 나타나면 취소를 클릭하여 뒤로 돌아가서 컴퓨터 이름을 변경하거나, 덮어쓰기를 클릭하여 기존 컴퓨터 레코드를 대치하십시오.

    기존의 컴퓨터 레코드가 없어졌거나 다른 컴퓨터에 있는 것 같습니다.

    기존의 컴퓨터 레코드를 대치하고자 한다면, 레코드가 다른 컴퓨터를 비활성화할 경우를 대비해서 LDAP 디렉토리 관리자에게 통보해야 합니다. 이 경우, LDAP 디렉토리 관리자가 비활성화된 컴퓨터에 다른 이름을 부여하고 속해있던 컴퓨터 그룹에 다시 추가해야 합니다.

  12. 대화상자가 확장되면서 연결 옵션이 표시되면, “선택 시 인증 사용”을 선택하고 식별이 쉬운 이름과 디렉토리의 사용자 계정의 암호를 입력하십시오.

    LDAP 서버에서 인증된 연결을 지원하지만 신뢰 바인딩은 지원하지 않는다면, 인증된 연결에 대한 옵션이 나타납니다. 인증 연결은 상호적이지 않습니다. LDAP 서버는 클라이언트를 인증하지만 클라이언트는 서버를 인증하지 않습니다.

    LDAP 서버에서 사용자 계정의 이름과 인증 연결을 위한 암호를 입력할 것을 요청한다면, "선택시 인증 사용"이 선택되어 있지만 흐려져 있습니다.

    식별되는 이름은 디렉토리에서 데이터를 볼 수 있는 권한을 가진 모든 사용자 계정을 지정할 수 있습니다. 예를 들어, ods.example.com에 주소를 가지고 있으며 LDAP 서버 ID가 "dirauth"인 사용자 계정은 uid=dirauth,cn=users,dc=ods,dc=example,dc=com인 이름을 가지게 됩니다.

    중요: 식별된 이름이나 암호가 올바르지 않으면 LDAP 서버로부터 사용자 계정을 사용하여 컴퓨터에 로그인할 수 있습니다.

  13. 승인을 클릭하여 LDAP 연결 생성을 완료하십시오.

  14. 승인을 클릭하여 LDAPv3 옵션 구성을 완료하십시오.

5단계에서 "인증에 사용" 또는 "연락처에 사용" 옵션을 선택하였다면, 방금 생성한 LDAP 디렉토리 구성이 디렉토리 유틸리티의 인증 또는 연락처 패널에 있는 사용자 설정 검색 정책에 추가됩니다.

서비스 패널에 있는 LDAPv3이 활성화되어 있어야 컴퓨터에서 방금 생성한 LDAP 구성을 사용할 수 있습니다. 추가 정보를 보려면, LDAP 디렉토리 서비스 활성 또는 비활성화시키기를 참조하십시오.
