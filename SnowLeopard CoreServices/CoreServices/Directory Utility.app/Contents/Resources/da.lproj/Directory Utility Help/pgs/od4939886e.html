Angive indstillinger til BSD-konfigurationsarkiver

UNIX-computere lagrer normalt administrative data i konfigurationsarkiver som f.eks. /etc/master.passwd, /etc/group og /etc/hosts. Mac OS X er baseret på en BSD-version af UNIX, men får normalt administrative data fra bibliotekssystemer.

Mac OS X Server understøtter et fast sæt af BSD-konfigurationsarkiver. Du kan ikke vælge de konfigurationsarkiver, der skal bruges, og du kan heller ikke overføre deres indhold til Mac OS X-optegnelser og -egenskaber.

I Mac OS X v10.2 og nyere versioner (inklusive Mac OS X Server v10.2 og nyere versioner) kan Open Directory hente administrative data fra BSD-konfigurationsarkiver. På den måde kan firmaer, som har BSD-konfigurationsarkiver, bruge kopier af de eksisterende arkiver på Mac OS X-computere. BSD-konfigurationsarkiver kan bruges alene eller med andre biblioteksdomæner.

Du bruger BSD-konfigurationsarkiver på følgende måde:

  1. Sørg for, at BSD-konfigurationsarkiverne indeholder de data, som Mac OS X-bibliotekstjenesterne kræver.

    Der findes flere oplysninger i emnet Oprette data i BSD-konfigurationsarkiver.

  2. Åbn Systemindstillinger, og klik på Konti.

  3. Hvis hængelåsen er låst, skal du låse den op ved at klikke på den og skrive en administrators navn og adgangskode.

  4. Klik på Log ind nederst til venstre, og klik derefter på Opret forbindelse eller Rediger.

  5. Klik på Åbn Biblioteksværktøj.

  6. Hvis hængelåsen er låst, skal du låse den op ved at klikke på den og skrive en administrators navn og adgangskode.

  7. Klik på Tjeneste.

  8. Vælg “BSD Flat File og NIS” på listen over tjenester, og klik på knappen Rediger (/).

  9. Vælg “Brug bruger- og gruppeposter i lokalt BSD-knudepunkt”, og klik på OK.

    Domænet med BSD-konfigurationsarkiverne føjes til computerens søgestrategi til godkendelse som /BSD/local.
