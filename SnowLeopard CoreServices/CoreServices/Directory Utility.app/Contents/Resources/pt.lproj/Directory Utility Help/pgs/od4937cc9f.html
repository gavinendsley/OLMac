Como configurar o acesso a um diretório LDAP

Usando o Utilitário de Diretório, você pode especificar como o Mac OS X acessa um diretório LDAPv3 se você conhece o nome DNS ou o endereço IP do servidor de diretório LDAP.

Se o diretório não for hospedado por um servidor que fornece seus próprios mapeamentos, como o Mac OS X Server, você precisará saber a base de busca e o modelo para mapear os dados do Mac OS X aos dados do diretório.

Os modelos de mapeamento compatíveis são:

  • Servidor de Open Directory, para um diretório que use o esquema do Mac OS X Server

  • Active Directory, para um diretório hospedado por um servidor Windows 2000, Windows 2003 ou posterior

  • RFC 2307, para a maioria dos diretórios hospedados por servidores UNIX

O plug-in LDAPv3 é totalmente compatível com o redirecionamento de dados em caso de falha e a resposta do Open Directory. Se o Open Directory principal se tornar indisponível, o plug-in alternará para a réplica mais próxima.

Para especificar mapeamentos personalizados para os dados do diretório, siga as instruções contidas em Como configurar o acesso a um diretório LDAP manualmente em vez de usar as instruções aqui descritas.

Importante:  se o nome de seu computador contém um hífen, talvez não seja possível unir-se ou vincular-se a um Domínio de Diretório como LDAP ou Active Directory. Para estabelecer o vínculo, use um nome de computador que não contenha um hífen.
Para que o Utilitário de Diretório ajude você a configurar o acesso a um diretório LDAP:

  1. Abra as Preferências do Sistema e clique em Contas.

  2. Se o ícone de bloqueio estiver ativado, desative-o clicando sobre ele e digitando o nome e a senha de um administrador.

  3. Clique em Opções de Início de Sessão, depois clique em Conectar ou Editar.

  4. Clique em Abrir Utilitário de Diretório.

  5. Se o ícone de bloqueio estiver ativado, desative-o clicando sobre ele e digitando o nome e a senha de um administrador.

  6. Clique em Serviços.

  7. Na lista de serviços, selecione LDAPv3 e clique no botão Editar (/).

    Você pode selecionar LDAPv3 na lista de serviços sem selecionar a caixa Ativar do LDAPv3.

  8. Clique em Novo e digite o nome DNS ou o endereço IP do servidor LDAP.

  9. Selecione as opções para acessar o diretório:

    • Selecione “Encriptar usando SSL” se deseja que o utilitário Open Directory use SSL (Camada de Soquete Segura, na sigla em inglês) nas conexões com o diretório LDAP. Antes de selecionar isto, pergunte ao seu administrador do Open Directory para determinar se SSL é necessário.

    • Selecione “Usar para autenticação” se esse diretório contiver contas de usuário que serão usadas para iniciar sessão ou autenticar serviços.

    • Selecione “Usar para contatos” se este diretório contiver endereços de e-mail e outras informações que você deseja usar na Agenda.

    Se o Utilitário de Diretório não puder entrar em contato com o servidor LDAP, uma mensagem será exibida e você deverá configurar o acesso manualmente ou cancelar o processo de configuração. Para obter mais informações sobre as instruções de configuração manual, consulte Como configurar o acesso a um diretório LDAP manualmente.

    Se a caixa de diálogo expandir para mostrar opções de mapeamento, escolha o modelo de mapeamento no menu local, digite o sufixo da base de busca e clique em Continuar.

    Normalmente, o sufixo de base de busca é derivado do nome DNS do servidor. Por exemplo, o sufixo de base de busca poderia ser “dc=ods,dc=example,dc=com” para um servidor cujo nome DNS é ods.example.com.

    Se nenhum dos modelos de mapeamento disponíveis se aplicar à conexão que você está configurando, clique em Manual. Para obter mais informações, consulte Como configurar o acesso a um diretório LDAP manualmente .

  10. Para que o Utilitário de Diretório obtenha informações do servidor LDAP, clique em Continuar.

  11. Se a caixa de diálogo se expandir para mostrar opções para vinculação confiável, digite o nome do computador e o nome e senha de um administrador de diretório. (A vinculação pode ser opcional.)

    A caixa de diálogo confirma se o diretório LDAP requer uma vinculação confiável ou se é opcional. A vinculação confiável é mútua: sempre que o computador se conectar a um diretório LDAP, eles se autenticarão um ao outro. Se a vinculação confiável estiver configurada ou se o diretório LDAP não for compatível com a vinculação confiável, o botão Vincular não será exibido. Certifique-se de ter fornecido o nome correto para o computador.

    Se você vir um alerta dizendo que existe um registro do computador, clique em Cancelar para votar e alterar o nome do computador, ou clique em Sobrescrever para substituir o registro de computador existente.

    O registro do computador existente pode ter sido abandonado ou pode pertencer a outro computador.

    Se você substituir um registro de computador existente, avise ao administrador de diretório LDAP caso essa substituição desative um outro computador. Nesse caso, o administrador de diretório LDAP deve dar ao computador desativado um outro nome e adicioná-lo de volta ao grupo ao qual pertencia.

  12. Se a caixa de diálogo se expandir para mostrar opções de conexão, selecione “Usar autenticação ao selecionar” e digite um nome e uma senha exclusivos de uma conta de usuário no diretório.

    As opções para uma conexão autenticada aparecem se o servidor LDAP for compatível com uma conexão autenticada, mas não com a vinculação confiável. Uma conexão de autenticação não é mútua: o servidor LDAP autentica o cliente, mas o cliente não autentica o servidor.

    A opção "Usar autenticação ao selecionar" está pré-selecionada, mas acinzentada se os servidores LDAP necessitarem que você digite um nome e uma senha exclusivos de uma conta de usuário para uma conexão autenticada.

    O nome exclusivo pode especificar qualquer conta de usuário que possua a permissão de ver dados no diretório. Por exemplo, uma conta de usuário cujo nome abreviado é "dirauth" em um servidor LDAP e cujo endereço é ods.example.com teria o nome exclusivo uid=dirauth,cn=users,dc=ods,dc=example,dc=com.

    Importante:  se o nome ou senha exclusivos estiverem incorretos, você poderá fazer login no computador usando as contas de usuário do diretório LDAP.

  13. Clique em OK para encerrar a criação da conexão LDAP.

  14. Clique em OK para finalizar a configuração das opções LDAPv3.

Se você selecionou “Usar para autenticação” ou “Usar para contatos” no passo 5, a configuração do diretório LDAP que você criou será adicionada a uma política de busca personalizada no painel Autenticação ou Contatos do Utilitário de Diretório.

Certifique-se de que o LDAPv3 esteja ativado no painel Serviços para que o computador use a configuração LDAP que você criou. Para obter mais informações, consulte Como ativar ou desativar os serviços de diretório LDAP.
