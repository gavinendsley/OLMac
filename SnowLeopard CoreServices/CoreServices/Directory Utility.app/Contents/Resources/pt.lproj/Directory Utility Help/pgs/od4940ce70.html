Como configurar o acesso LDAP a domínios do Active Directory

Usando o Utilitário de Diretório, você pode ajustar uma configuração LDAPv3 para acessar um domínio do Active Directory em um servidor Windows. Uma configuração LDAPv3 fornece controle total sobre o mapeamento de tipos de registros e atributos do Mac OS X para classes de objetos, bases de busca e atributos do Active Directory.

O mapeamento de alguns atributos e tipos de registros importantes do Mac OS X, como o ID único de usuário (UID), requer extensão de esquema do Active Directory.

Uma configuração LDAPv3 não inclui os seguintes recursos do conector do Active Directory listados no Utilitário de Diretório:

  • Geração dinâmica de ID de usuário e ID de grupo principal únicas

  • Criação de uma pasta de início local do Mac OS X

  • Montagem automática da pasta de início do Windows

  • Contas de usuário móveis com credenciais de autenticação em cache

  • Descoberta de todos os domínios em uma floresta do Active Directory

  • Suporte para duplicação e recuperação após falha do Active Directory

Para obter mais informações, consulte Sobre o acesso ao Active Directory.

Para criar uma configuração de servidor do Active Directory:

  1. Abra as Preferências do Sistema e clique em Contas.

  2. Se o ícone de bloqueio estiver ativado, desative-o clicando sobre ele e digitando o nome e a senha de um administrador.

  3. Clique em Opções de Início de Sessão, depois clique em Conectar ou Editar.

  4. Clique em Abrir Utilitário de Diretório.

  5. Se o ícone de bloqueio estiver ativado, desative-o clicando sobre ele e digitando o nome e a senha de um administrador.

  6. Clique em Serviços.

  7. Na lista de serviços, selecione LDAPv3 e clique no botão Editar (/).

  8. Clique em Novo e digite o nome DNS ou o endereço IP do servidor do Active Directory.

  9. Selecione entre as seguintes opções de acesso ao diretório e, em seguida, clique em Continuar para que o Utilitário de Diretório obtenha informações do servidor do Active Directory.

    • Selecione “Encriptar usando SSL” se você desejar que o Open Directory use SSL para conexões com o servidor do Active Directory.” Antes de selecionar a opção SSL, pergunte ao seu administrador do Open Directory se é necessário SSL.

    • Selecione “Usar para autenticação” se este diretório contiver contas de usuário que alguém usará para iniciar a sessão ou autenticar-se nos serviços.

    • Selecione “Usar para contatos” se este diretório contiver endereços de e-mail e outras informações que você deseja usar na Agenda.

    Se o Utilitário de Diretório não pode entrar em contato com o servidor do Active Directory, uma mensagem será exibida e você deverá configurar o acesso manualmente ou cancelar o processo de configuração. Para obter mais informações, consulte Como configurar o acesso a um diretório LDAP manualmente .

    Se você selecionou “Usar para autenticação” ou “Usar para contato”, a conexão LDAPv3 para o domínio do Active Directory será adicionada a uma política de busca personalizada no painel Autenticação ou Contatos do Utilitário de Diretório.

    Certifique-se de que o LDAPv3 esteja ativado no painel Serviços, assim o computador usará a conexão que você configurou. Para obter mais informações, consulte Como ativar ou desativar os serviços de diretório LDAP.

  10. Se a caixa de diálogo se expandir para mostrar opções de mapeamento, escolha Active Directory no menu, digite a base de busca e clique em Continuar.

    O modelo de mapeamento do Active Directory para uma configuração LDAPv3 mapeia alguns tipos de registros e atributos do Mac OS X para classes de objetos e atributos que não fazem parte de um esquema padrão do Active Directory. Você pode alterar os mapeamentos definidos pelo modelo ou esquema ampliado do Active Directory.

    Opcionalmente, você poderá acessar o domínio do Active Directory através do conector do Active Directory, em vez de LDAPv3. Para obter mais informações, consulte Como configurar o acesso a um domínio do Active Directory.

  11. Quando a caixa de diálogo ampliar-se para exibir opções de conexão, digite um nome e uma senha únicos de uma conta de usuário Active Directory.

  12. Clique em OK para encerrar a criação da conexão LDAP.

  13. Clique em OK para finalizar a configuração das opções LDAPv3.
