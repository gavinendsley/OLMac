Como configurar buscas e mapeamentos LDAP

Usando o Utilitário de Diretório, você pode editar os mapeamentos, as bases de busca e os escopos de busca que especificam como o Mac OS X encontra itens de dados específicos em um diretório LDAP. Você pode editar esses ajustes separadamente para cada configuração de diretório LDAP listada no Utilitário de Diretório. Cada Configuração de Diretórios LDAP especifica como o Mac OS X acessa dados em um diretório LDAPv3 ou LDAPv2.

Você pode editar o seguinte:

  • O mapeamento de cada tipo de registro do Mac OS X para classes de objeto LDAP

  • O mapeamento dos tipos de dados ou atributos do Mac OS X, para atributos do LDAP para cada tipo de registro

  • A base e o escopo de busca do LDAP que determinam onde o Mac OS X deve procurar um tipo de registro do Mac OS X em um diretório LDAP

Ao mapear atributos de usuário do Mac OS X para um domínio de diretório LDAP de leitura/gravação (um domínio LDAP que não seja somente para leitura), o atributo LDAP mapeado para RealName não deve ser igual ao primeiro atributo de uma lista de atributos LDAP mapeada para RecordName.

Por exemplo, o atributo cn não deve ser o primeiro atributo mapeado para RecordName se cn é também mapeado para RealName.

Se o atributo do LDAP mapeado para o RealName for o mesmo que o primeiro atributo mapeado para o RecordName, ocorrerão problemas quando você tentar editar o nome completo (longo) ou o primeiro nome abreviado no Workgroup Manager.

Para obter mais informações sobre tipos de registro e atributos do Mac OS X, consulte Administração do Open Directory (disponível em www.apple.com/br/server/resources/).

Para editar as bases de busca e os mapeamentos de um servidor LDAP:

  1. Abra as Preferências do Sistema e clique em Contas.

  2. Se o ícone de bloqueio estiver ativado, desative-o clicando sobre ele e digitando o nome e a senha de um administrador.

  3. Clique em Opções de Início de Sessão, depois clique em Editar.

  4. Clique em Abrir Utilitário de Diretório.

  5. Se o ícone de bloqueio estiver ativado, desative-o clicando sobre ele e digitando o nome e a senha de um administrador.

  6. Clique em Serviços.

  7. Na lista de serviços, selecione LDAPv3 e clique no botão Editar (/).

  8. Se a lista de configurações de servidor estiver oculta, clique em Mostrar Opções.

  9. Selecione uma configuração de servidor na lista e clique em Editar.

  10. Clique em Busca e Mapeamentos.

  11. Selecione os mapeamentos que deseja usar como ponto de partida; caso contrário, escolha Personalizado para começar sem mapeamentos predefinidos.

    Se você escolher um dos modelos de mapeamento LDAP, um sufixo de base de busca que você pode alterar aparece, ou você pode aceitar o sufixo de base de busca padrão clicando em OK.

    Clique no menu local “Acessar este servidor LDAPv3 usando” e escolha um modelo de mapeamento para usar seus mapeamentos como ponto de partida.

  12. Adicione os tipos de registro e modifique as bases de busca conforme necessário:

    • Para adicionar tipos de registro, clique em Adicionar (abaixo da lista Tipos e Atributos de Registro); em seguida, na folha que aparece, selecione Tipos de Registro, selecione tipos de registro na lista e clique em OK.

    • Para alterar a base e o escopo de busca de um tipo de registro, selecione-o na lista Tipos e Atributos de Registro e, em seguida, edite o campo “Base de busca”. Selecione “todas as sub-árvores” para definir o escopo de busca para incluir a hierarquia do diretório LDAP da base de busca para baixo, ou selecione “apenas primeiro nível” para definir o escopo de busca para incluir apenas a base de busca e um nível abaixo dela na hierarquia do diretório LDAP.

    • Para remover um tipo de registro, na lista Tipos e Atributos de Registro, selecione o tipo e clique em Apagar.

    • Para adicionar um mapeamento para um tipo de registro, selecione o tipo de registro na lista Tipos e Atributos de Registro e, em seguida, clique em Adicionar (abaixo de “Mapear para __ itens da lista”) e digite o nome de uma classe de objeto no diretório LDAP.

      Para adicionar uma outra classe de objeto LDAP, pressione Retorno e digite o nome da classe de objeto; especifique se devem ser usadas as classes de objeto LDAP listadas usando o menu local acima da lista.

    • Para alterar um mapeamento para um tipo de registro, selecione o tipo de registro na lista Tipos e Atributos de Registro e, em seguida, clique duas vezes na classe de objeto LDAP que deseja alterar em “Mapear para __ itens da lista” e edite-o. Especifique se deseja utilizar as classes de objeto LDAP listadas usando o menu local situado acima da lista.

    • Para remover um mapeamento para um tipo de registro, na lista Tipos e Atributos de Registro, selecione o tipo de registro, selecione a classe de objeto LDAP que deseja remover de “Mapear para __ itens da lista” e, em seguida, clique em Apagar (abaixo de “Mapear para __ itens da lista”).

  13. Adicione os atributos e modifique seus mapeamentos conforme necessário:

    • Para adicionar atributos a um tipo de registro, na lista Tipos e Atributos de Registro, selecione o tipo de registro e clique em Adicionar (abaixo da lista Tipos e Atributos de Registro); em seguida, na folha que aparece, selecione Tipos de Atributo, selecione os tipos de atributos e clique em OK.

    • Para adicionar um mapeamento para um atributo, na lista Tipos e Atributos de Registro, selecione o atributo, clique em Adicionar (abaixo de “Mapear para __ itens da lista”) e digite o nome de um atributo do diretório LDAP. Para adicionar um outro atributo LDAP, pressione a tecla Retorno e digite o nome do atributo.

    • Para alterar o mapeamento para um atributo, na lista Tipos e Atributos de Registro, selecione o atributo, clique duas vezes no item que deseja alterar em “Mapear para __ itens da lista” e, em seguida, edite o nome do item.

    • Para remover um mapeamento para um atributo, na lista Tipos e Atributos de Registro, selecione o atributo, selecione o item que deseja remover de “Mapear para __ itens da lista” e, em seguida, clique em Apagar (abaixo de “Mapear para __ itens da lista”).

    • Para alterar a ordem dos atributos exibidos na lista da direita, arraste os atributos para cima ou para baixo na lista.

  14. Para salvar seus mapeamentos como modelo, clique em Salvar Modelo.

    Os modelos salvos em um local padrão serão listados nos menus locais de modelos de mapeamento de LDAP na próxima vez que abrir o Utilitário de Diretório. O local padrão para modelos salvos está em sua pasta de início neste caminho:

    ~/Library/Application Support/Directory Access/LDAPv3/Templates

  15. Para armazenar os mapeamentos no diretório LDAP para que possa fornecê-los automaticamente para seus clientes, clique em Gravar no Servidor e, em seguida, digite uma base de busca para armazenar os mapeamentos, um nome exclusivo de um administrador ou de outro usuário com permissão de gravação para a base de busca (por exemplo, uid=diradmin,cn=users,dc=ods,dc=example,dc=com), e uma senha.

    Se você está gravando mapeamentos em um servidor LDAP Open Directory, a base de busca correta é cn=config,sufixo (onde sufixo é o sufixo de base de busca do servidor, como dc=ods,dc=example,dc=com).

O diretório LDAP fornece os seus mapeamentos aos clientes Mac OS X cuja política de busca personalizada inclua uma conexão que esteja configurada para obter mapeamentos de um servidor LDAP.

O diretório LDAP também fornece seus mapeamentos a todos os clientes Mac OS X que possuam uma política de busca automática. Para obter mais informações, consulte Como configurar o acesso a um diretório LDAP e Como usar as configurações avançadas da Política de Busca.
