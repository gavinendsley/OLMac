Como alterar uma configuração para o acesso a um diretório LDAP

Você pode usar o Utilitário de Diretório para alterar os ajustes de uma configuração de diretório LDAP. Os ajustes de configuração especificam como o Open Directory acessa um diretório LDAPv3 ou LDAPv2.

Se a configuração LDAP foi fornecida pelo DHCP, ela não pode ser alterada, assim esse tipo de configuração fica acinzentada na lista de configurações LDAP.

Para editar uma configuração para acessar um diretório LDAP:

  1. Abra as Preferências do Sistema e clique em Contas.

  2. Se o ícone de bloqueio estiver ativado, desative-o clicando sobre ele e digitando o nome e a senha de um administrador.

  3. Clique em Opções de Início de Sessão, depois clique em Editar.

  4. Clique em Abrir Utilitário de Diretório.

  5. Se o ícone de bloqueio estiver ativado, desative-o clicando sobre ele e digitando o nome e a senha de um administrador.

  6. Clique em Serviços.

  7. Na lista de serviços, selecione LDAPv3 e clique no botão Editar (/).

  8. Se a lista de configurações de servidor estiver oculta, clique em Mostrar Opções.

  9. Faça as mudanças necessárias nos seguintes ajustes:

    • Ativar: clique nesta caixa para ativar ou desativar o acesso a um servidor de diretório LDAP.

    • Nome de Configuração: clique duas vezes em um nome de configuração para editá-lo.

    • Nome do Servidor ou Endereço IP: clique duas vezes no nome de um servidor ou no endereço IP para alterá-lo.

    • Mapeamento LDAP: no menu local, escolha um modelo, digite o sufixo da base de busca para o diretório LDAP e clique em OK.

      Se você escolheu um modelo, você deve digitar um sufixo de base de busca, caso contrário o computador não será capaz de encontrar informações no diretório LDAP. Normalmente, o sufixo de base de busca é derivado do nome DNS do servidor. Por exemplo, o sufixo de base de busca poderia ser “dc=ods,dc=example,dc=com” para um servidor cujo nome DNS é ods.example.com.

      Se você escolheu Do Servidor em vez de um modelo, não é necessário digitar um sufixo de base de busca. Neste caso, o Open Directory supõe que o sufixo da base de busca é o primeiro nível do diretório LDAP.

      Se você escolheu Personalizado, você precisa configurar os mapeamentos entre os tipos e atributos de registro do Mac OS X e as classes e atributos do diretório LDAP ao qual está se conectando. Para obter mais informações, consulte Como configurar buscas e mapeamentos LDAP.

    • SSL: clique na caixa para ativar ou desativar comunicações encriptadas usando o protocolo SSL. Antes de selecionar a opção SSL, pergunte ao seu administrador do Open Directory se é necessário SSL.

  10. Para alterar as seguintes configurações padrão para esta configuração LDAP, clique em Editar para exibir as opções da configuração LDAP selecionada, faça as alterações e clique em OK ao concluir a edição das opções de configuração LDAP:

    • Clique em Conexão para definir opções de tempo limite, especificar uma porta personalizada, ignorar orientações de servidores ou forçar o uso do protocolo LDAPv2 (somente leitura). Para obter mais informações, consulte Como alterar os ajustes de conexão de um diretório LDAP.

    • Clique em Busca e Mapeamentos para configurar buscas e mapeamentos para um servidor LDAP. Para obter mais informações, consulte Como configurar a vinculação confiável para um diretório LDAP.

    • Clique em Segurança para ajustar uma conexão de autenticação (em lugar de uma vinculação confiável) e outras opções da política de segurança. Para obter mais informações, consulte Como alterar a política de segurança de uma conexão LDAP.

    • Clique em Vincular para configurar a vinculação confiável ou em Desvincular para parar a vinculação confiável. (Talvez você não veja estes botões se o diretório LDAP não permitir a vinculação confiável.) Para obter mais informações, consulte Como configurar a vinculação confiável para um diretório LDAP.

  11. Para finalizar a configuração para acessar um diretório LDAP, clique em OK.
