Como usar as configurações avançadas da Política de Busca

O Utilitário de Diretório define as seguintes políticas de busca:

  • Autenticação: O Mac OS X usa a política de busca de autenticação para localizar e obter informações de autenticação de usuário e outros dados administrativos dos domínios de diretório.

  • Contatos: O Mac OS X usa a política de busca de contatos para localizar e obter o nome, o endereço e outras informações de contato dos domínios de diretório. A Agenda do Mac OS X usa essas informações de contato. Outros aplicativos também podem ser programados para usá-las.

    Cada política de busca consiste em uma lista de domínios de diretório. A ordem dos domínios de diretório da lista define a política de busca. Começando pela parte superior da lista, o Mac OS X busca cada domínio de diretório listado, até encontrar a informação necessária ou até chegar ao final da lista sem encontrar a informação.

    As políticas de busca de autenticação e de contatos podem ter um dos seguintes ajustes:

  • Automático: Começa com o domínio de diretório local e pode incluir um diretório LDAP fornecido por DHCP e domínios de diretório aos quais o computador está conectado. Esse é o ajuste padrão para Mac OS X v10.2 ou posterior, e oferece a maior flexibilidade para computadores móveis.

  • Diretório local: Inclui apenas o domínio de diretório local.

  • Caminho personalizado: Começa com o domínio de diretório local e inclui sua escolha de diretórios LDAP, um domínio do Active Directory, domínios de diretório compartilhados, arquivos de configuração BSD e um domínio NIS.

    A pasta /BSD/local está sempre incluída no caminho de busca e está sempre acinzentada.

Importante:  se você configurar o Mac OS X para usar uma política de busca de autenticação automática e um servidor LDAP fornecido por DHCP, você aumentará o risco de um usuário mal-intencionado obter o controle de seu computador. O risco é ainda maior se o seu computador estiver configurado para conectar-se a uma rede sem fio. Consulte Como proteger computadores de um servidor DHCP mal-intencionado para obter mais informações.
