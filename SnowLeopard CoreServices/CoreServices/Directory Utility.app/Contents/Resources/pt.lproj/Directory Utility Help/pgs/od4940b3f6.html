Como definir uma shell do UNIX para as contas de usuário do Active Directory

Em um computador configurado para usar o conector do Active Directory do Utilitário de Diretório, você pode definir a shell de linha de comando que os usuários com contas Active Directory usarão por padrão ao interagir com o Mac OS X no Terminal.

A shell padrão é também usada para interação remota via SSH (Secure Shell) ou Telnet. Cada usuário pode substituir a shell padrão alterando a preferência do Terminal.

Para definir uma shell do UNIX para as contas de usuário do Active Directory:

  1. Abra as Preferências do Sistema e clique em Contas.

  2. Se o ícone de bloqueio estiver ativado, desative-o clicando sobre ele e digitando o nome e a senha de um administrador.

  3. Clique em Opções de Início de Sessão, depois clique em Conectar ou Editar.

  4. Clique em Abrir Utilitário de Diretório.

  5. Se o ícone de bloqueio estiver ativado, desative-o clicando sobre ele e digitando o nome e a senha de um administrador.

  6. Clique em Serviços.

  7. Na lista de serviços, selecione Active Directory e clique no botão Editar (/).

  8. Se as opções avançadas estiverem ocultas, clique em "Mostrar Opções Avançadas".

  9. Clique em "Experiência do Usuário".

  10. Selecione shell do usuário, depois digite o caminho da shell do usuário.

  11. Clique em OK.
