Como alterar a política de segurança de uma conexão LDAP

Usando o Utilitário de Diretório, você pode configurar uma política de segurança mais estrita para uma conexão LDAPv3 que a política de segurança do diretório LDAP. Por exemplo, se a política de segurança do diretório LDAP permite senhas não criptografadas, você pode configurar uma conexão LDAPv3 para não permitir senhas não criptografadas.

A configuração de uma política de segurança mais estrita protege o seu computador contra um hacker mal-intencionado que tente usar um servidor LDAP invasor para obter o controle do seu computador.

O computador deve comunicar-se com o servidor LDAP para mostrar o estado das opções de segurança. Desta forma, quando você modifica opções de segurança para uma conexão LDAPv3, a política de busca de autenticação do computador deve incluir a conexão LDAP3.

Os ajustes admissíveis das opções de segurança da conexão LDAPv3 estão sujeitas aos recursos e requisitos de segurança do servidor LDAP. Por exemplo, se o servidor LDAP não for compatível com a autenticação Kerberos, várias opções de segurança da conexão LDAPv3 estarão desativadas.

Para alterar as opções de segurança de uma conexão LDAPv3:

  1. Abra as Preferências do Sistema e clique em Contas.

  2. Se o ícone de bloqueio estiver ativado, desative-o clicando sobre ele e digitando o nome e a senha de um administrador.

  3. Clique em Opções de Início de Sessão, depois clique em Editar.

  4. Clique em Abrir Utilitário de Diretório.

  5. Se o ícone de bloqueio estiver ativado, desative-o clicando sobre ele e digitando o nome e a senha de um administrador.

  6. Clique em Política de Busca.

  7. Clique em Autenticação e certifique-se de que o diretório LDAPv3 desejado esteja listado na política de busca.

    Para obter mais informações sobre como adicionar o diretório LDAPv3 à política de busca de autenticação, consulte Como definir políticas de busca personalizadas.

  8. Clique em Serviços.

  9. Na lista de serviços, selecione LDAPv3 e clique no botão Editar (/).

  10. Se a lista de configurações de servidor estiver oculta, clique em Mostrar Opções.

  11. Selecione a configuração para o diretório desejado e, em seguida, clique em Editar.

  12. Clique em Segurança e, em seguida, altere qualquer um dos ajustes seguintes.

    Nota:  as configurações de segurança daqui e do servidor LDAP correspondente são determinadas quando a conexão LDAP é configurada. As configurações não são atualizadas quando as configurações do servidor são alteradas.

    Se alguma das últimas quatro opções estiver selecionada, mas desativada, é porque o diretório LDAP a requer. Se alguma destas opções não estiver selecionada e estiver desativada, o servidor LDAP não é compatível com ela.

    • Usar autenticação ao conectar: determina se a conexão LDAPv3 autentica a si própria com o diretório LDAP fornecendo o nome e senha distintos especificados. Esta opção não é exibida se a conexão LDAPv3 usar uma vinculação confiável com o diretório LDAP.

    • Vinculado ao diretório como: especifica as credenciais que a conexão LDAPv3 utiliza para a vinculação confiável com o diretório LDAP. Esta opção e as credenciais não podem ser alteradas aqui. Ao contrário, você pode desvincular e depois vincular novamente com credenciais diferentes.

      Para obter mais informações, consulte Como parar a vinculação confiável com um Diretório LDAP e Como configurar a vinculação confiável para um diretório LDAP.

      Esta opção não é exibida ao menos que a conexão LDAPv3 use uma vinculação confiável.

    • Desativar limpar senhas de texto: determina se a senha deve ser enviada sem estar criptografada no caso da senha não poder ser validada usando um método de autenticação que envie uma senha criptografada.

    • Assinar digitalmente todos os pacotes (requer Kerberos): garante que os dados do diretório do servidor LDAP não sejam interceptados e modificados por outro computador quando em rota ao seu computador.

    • Encriptar todos os pacotes (requer SSL ou Kerberos): requer que o servidor LDAP encripte os dados do diretório usando SSL ou Kerberos antes de enviá-los ao seu computador. Antes de selecionar a opção “Encriptar todos os pacotes (requer SSL ou Kerberos)”, pergunte ao seu administrador do Open Directory se SSL é necessário.

    • Bloquear ataques “man-in-the-middle” (requer Kerberos) protege contra um servidor invasor passando-se por um servidor LDAP. É melhor se for usado com a opção “Assinar digitalmente todos os pacotes”.
