Como configurar o acesso a um diretório LDAP manualmente

Você pode criar manualmente uma configuração que especifique como o Mac OS X acessa um diretório LDAPv3 ou LDAPv2. Você deve saber o nome DNS ou o endereço IP do servidor de diretório LDAP.

Se o diretório não for hospedado pelo Mac OS X Server, você precisará conhecer a base de busca e o modelo para mapear os dados do Mac OS X aos dados do diretório. Os modelos de mapeamento compatíveis são:

  • Do Servidor, para um diretório que fornece seus próprios mapeamentos e base de busca, como o Mac OS X Server

  • Servidor Open Directory, para um diretório que usa o esquema do Mac OS X Server

  • Active Directory, para um diretório hospedado por um servidor do Windows 2000, Windows 2003 ou posterior

  • RFC 2307, para a maioria dos diretórios hospedados por servidores UNIX

  • Personalizado, para diretórios que não usam nenhum dos mapeamentos acima

O plug-in LDAPv3 é totalmente compatível com o redirecionamento de dados em caso de falha e a resposta do Open Directory. Se o Open Directory principal se tornar indisponível, o plug-in alternará para a réplica mais próxima.

Importante:  se o nome de seu computador contém um hífen, talvez não seja possível unir-se ou vincular-se a um Domínio de Diretório como LDAP ou Active Directory. Para estabelecer o vínculo, use um nome de computador que não contenha um hífen.
Para configurar o acesso a um diretório LDAP manualmente:

  1. Abra as Preferências do Sistema e clique em Contas.

  2. Se o ícone de bloqueio estiver ativado, desative-o clicando sobre ele e digitando o nome e a senha de um administrador.

  3. Clique em Opções de Início de Sessão, depois clique em Conectar ou Editar.

  4. Clique em Abrir Utilitário de Diretório.

  5. Se o ícone de bloqueio estiver ativado, desative-o clicando sobre ele e digitando o nome e a senha de um administrador.

  6. Clique em Serviços.

  7. Na lista de serviços, selecione LDAPv3 e clique no botão Editar (/).

    Você pode selecionar LDAPv3 na lista de serviços sem selecionar a caixa Ativar do LDAPv3.

  8. Clique em Novo e depois clique em Manual.

  9. Digite um nome para a configuração.

  10. Pressione Tabulação e digite o nome DNS ou o endereço IP do servidor que hospeda o diretório LDAP que você deseja acessar.

  11. Clique no menu próximo ao nome DNS ou ao endereço IP e escolha um modelo ou método de mapeamento:

    • Se você escolheu Do Servidor, não é necessário digitar um sufixo de base de busca. Neste caso, o Open Directory supõe que o sufixo da base de busca é o primeiro nível do diretório LDAP.

    • Se você escolheu um modelo, digite o sufixo da base de busca para o diretório LDAP e clique em OK. Você deve digitar um sufixo de base de busca ou o computador não será capaz de encontrar informações no diretório LDAP.

      Normalmente, o sufixo de base de busca é derivado do nome DNS do servidor. Por exemplo, o sufixo de base de busca poderia ser “dc=ods,dc=example,dc=com” para um servidor cujo nome DNS é ods.example.com.

    • Se você escolheu Personalizado, você precisa configurar os mapeamentos entre os tipos e atributos de registro do Mac OS X e as classes e atributos do diretório LDAP ao qual está se conectando. Para obter mais informações, consulte Como configurar buscas e mapeamentos LDAP.

  12. Antes de selecionar a caixa “Encriptar usando SSL”, verifique com seu administrador de Open Directory se o SSL é necessário.

  13. Para alterar os seguintes ajustes para essa configuração LDAP, clique em Editar para mostrar as opções para a configuração LDAP selecionada, faça as alterações, e clique em OK quando finalizar a edição.

    • Clique em Conexão para definir opções de tempo limite, especificar uma porta personalizada, ignorar orientações de servidores ou forçar o uso do protocolo LDAPv2 (somente leitura). Para obter mais informações, consulte Como alterar os ajustes de conexão de um diretório LDAP.

    • Clique em Busca e Mapeamentos para configurar buscas e mapeamentos para um servidor LDAP. Para obter mais informações, consulte Como configurar a vinculação confiável para um diretório LDAP.

    • Clique em Segurança para ajustar uma conexão de autenticação (em lugar de uma vinculação confiável) e outras opções da política de segurança. Para obter mais informações, consulte Como alterar a política de segurança de uma conexão LDAP.

    • Clique em Vincular para configurar vinculações confiáveis (se o diretório LDAP for compatível). Para obter mais informações, consulte Como configurar a vinculação confiável para um diretório LDAP.

  14. Clique em OK para finalizar a criação manual da configuração de acesso a um diretório LDAP.

  15. Se você desejar que o computador acesso o diretório LDAP para o qual você criou a configuração, adicione o diretório a uma política de busca personalizada no painel Autenticação da Política de Busca no Utilitário de Diretório, em seguida, certifique-se de que LDAPv3 esteja ativado no painel Serviços.

Para obter mais informações, consulte Como ativar ou desativar os serviços de diretório LDAP e Como definir políticas de busca personalizadas.

Nota:  antes de usar o Workgroup Manager para criar usuários em um servidor LDAP não Apple que usa mapeamentos RFC 2307 (UNIX), você deve editar o mapeamento do tipo de registro de Usuários. Para obter mais informações, consulte Como editar o mapeamento RFC 2307 para permitir a criação de usuários.
Importante:  se alterar seu endereço IP e o nome do computador usando changeip enquanto estiver conectado a um servidor de diretório, você deve desconectar e conectar novamente o servidor de diretório para atualizá-lo com o novo nome de computador e endereço IP. Se não desconectar e conectar novamente o servidor de diretório, o diretório não atualizará e continuará a usar o nome e endereço IP do antigo computador.
