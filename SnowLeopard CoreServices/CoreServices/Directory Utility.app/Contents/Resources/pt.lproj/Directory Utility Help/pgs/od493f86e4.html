Como configurar o acesso a um domínio do Active Directory

Usando o conector do Active Directory listado no Utilitário de Diretório, você pode configurar o Mac OS X para acessar as informações básicas da conta de usuário em um domínio do Active Directory de um servidor Windows.

O conector do Active Directory gera todos os atributos necessários para a autenticação do Mac OS X. Não é necessária nenhuma alteração ao esquema do Active Directory.

O conector do Active Directory detecta e acessa tipos de registro padrão e atributos do Mac OS X (como os atributos solicitados para o gerenciamento de clientes do Mac OS X ) se o esquema do Active Directory foi estendido para incluí-los.

AVISO:  com as opções avançadas do conector do Active Directory, você pode mapear os atributos do ID exclusivo de usuário (UID), do ID do grupo primário (GID) e GID do grupo do Mac OS X para os atributos corretos que foram adicionados ao esquema do Active Directory. Se você alterar o ajuste dessas opções de mapeamento mais tarde, os usuários podem perder o acesso a arquivos criados anteriormente.
Importante:  se o nome de seu computador contém um hífen, talvez não seja possível unir-se ou vincular-se a um Domínio de Diretório como LDAP ou Active Directory. Para estabelecer o vínculo, use um nome de computador que não contenha um hífen.
Para configurar o acesso a um domínio do Active Directory:

  1. Abra as Preferências do Sistema e clique em Contas.

  2. Se o ícone de bloqueio estiver ativado, desative-o clicando sobre ele e digitando o nome e a senha de um administrador.

  3. Clique em Opções de Início de Sessão, depois clique em Conectar ou Editar.

  4. Clique em Abrir Utilitário de Diretório.

  5. Se o ícone de bloqueio estiver ativado, desative-o clicando sobre ele e digitando o nome e a senha de um administrador.

  6. Clique em Serviços.

  7. Na lista de serviços, selecione Active Directory e clique no botão Editar (/).

  8. Digite o nome DNS do domínio do Active Directory ao qual deseja vincular o computador que você está configurando.

    O administrador do domínio do Active Directory pode dizer o nome do DNS a ser digitado.

  9. Se for necessário, edite o ID do Computador.

    A ID do Computador é o nome pelo qual o computador é conhecido no domínio do Active Directory e está predefinida com o nome do computador. Você pode alterar este ajuste para adaptar-se ao esquema de nomes de computador no domínio do Active Directory estabelecido na sua organização. Se não tiver certeza, pergunte ao administrador de domínio do Active Directory.

  10. (Opcional) Defina as opções avançadas.

    Se as opções avançadas estiverem ocultas, clique em "Mostrar Opções Avançadas" e defina as opções nos painéis Administrativo, Mapeamentos e Experiência do Usuário. Você também pode alterar os ajustes das opções avançadas posteriormente.

    Para obter mais informações sobre as opções avançadas, consulte:

    • Como ajustar contas de usuários móveis no Active Directory

    • Como ajustar as pastas Início das contas de usuários do Active Directory

    • Como definir uma shell do UNIX para as contas de usuário do Active Directory

    • Como mapear o UID para um atributo do Active Directory

    • Como mapear o ID do grupo primário para um atributo do Active Directory

    • Como mapear a ID do grupo em contas de grupo para um atributo do Active Directory

    • Como especificar um servidor preferido do Active Directory

    • Como alterar os grupos do Active Directory que podem administrar o computador

    • Como controlar a autenticação de todos os domínios na floresta do Active Directory

  11. Clique em Vincular, use as opções a seguir para autenticar como um usuário com direitos para vincular um computador ao domínio do Active Directory, selecione as políticas de busca às quais deseja que o Active Directory seja adicionado (veja abaixo) e clique em OK:

    • Nome de Usuário e Senha: talvez você possa autenticar inserindo o nome e a senha da sua conta de usuário do Active Directory ou talvez o administrador de domínio do Active Directory tenha de fornecer um nome e uma senha.

    • OU do Computador: insira a unidade organizacional (OU) do computador que está configurando.

    • Usar para autenticação: use para determinar se o Active Directory deverá ser adicionado à política de busca de autenticação do computador.

    • Usar para contatos: use para determinar se o Active Directory deverá ser adicionado à política de busca de contatos do computador.

    Quando você clica em OK, o Utilitário de Diretório define o vínculo de confiança entre o computador que está configurando e o servidor do Active Directory. As políticas de busca do computador estão configuradas de acordo com as opções selecionadas ao autenticar, e o Active Directory está ativado no painel Serviços do Utilitário de Diretório.

    Com os ajustes padrão para as opções avançadas do Active Directory, a floresta do Active Directory é adicionada à política de busca de autenticação do computador e à política de busca de contatos se você selecionou “Usar para autenticação” ou “Usar para contatos”.

    Entretanto, se você desmarcou a opção "Permitir autenticação de qualquer domínio da floresta" no painel de opções avançadas Administrativo antes de clicar em Vincular, será adicionado o domínio Active Directory mais próximo em lugar da floresta.

    Você pode alterar políticas de busca posteriormente adicionando ou removendo a floresta ou domínios individuais do Active Directory. Para obter mais informações, consulte Como definir políticas de busca personalizadas.

  12. (Opcional) Conecte o servidor ao reino Kerberos do Active Directory:

    • No computador servidor ou administrador que pode se conectar ao servidor, abra o Server Admin e selecione Open Directory para o servidor.

    • Clique em Ajustes e, em seguida, em Geral.

    • Clique em Conectar ao Kerberos e, em seguida, escolha o reino Kerberos do Active Directory no menu local e digite as credenciais para um administrador local nesse servidor.
