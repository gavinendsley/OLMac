Sobre o acesso ao Active Directory

Você pode configurar o Mac OS X para acessar as informações básicas de conta de usuário em um domínio do Active Directory de um servidor Windows 2000 ou mais recente. Isso é possível graças a um conector do Active Directory para o Utilitário de Diretório. Esse conector do Active Directory está listado no painel Serviços do Utilitário de Diretório.

Não é necessário fazer nenhuma modificação de esquema ao domínio do Active Directory para obter as informações básicas de conta de usuário. Você pode alterar a Lista de Controle de Acesso (ACL) padrão de atributos específicos para que as contas do computador possam ler as propriedades do usuário.

O conector do Active Directory gera todos os atributos necessários para a autenticação do Mac OS X a partir dos atributos padrão das contas de usuário do Active Directory. O conector também é compatível com as políticas de autenticação do Active Directory, incluindo alterações de senhas, validade, alterações forçadas e opções de segurança.

O Mac OS X v10.6 é compatível com a encriptação de pacotes e com as opções de sinalização de pacotes para todos os domínios do Active Directory do Windows. Essa função está ativada por padrão como “permitir”. Você pode alterar as configurações padrão para desativadas ou requeridas usando a ferramenta de linha de comando dsconfigad. As opções de encriptação do pacote e assinatura do pacote garantem que todos os dados de e para o Domínio de Active Directory para buscas de registro estejam protegidos.

O conector do Active Directory gera dinamicamente uma ID exclusiva de usuário e uma ID de grupo principal baseadas na ID Exclusiva Global (GUID) da conta do usuário no domínio do Active Directory. A ID de usuário gerada e a ID de grupo principal são as mesmas para cada conta de usuário, inclusive se a conta está sendo usada para iniciar a sessão em diferentes computadores Mac OS X.

Alternativamente, você pode forçar o conector do Active Directory para mapear a ID de usuário para os atributos do Active Directory que especificar.

O conector do Active Directory gera uma ID de grupo baseada na GUID da conta de grupo do Active Directory. Você também pode forçar o plug-in a mapear o ID de grupo de contas de grupo para os atributos do Active Directory que você especificar.

Quando alguém inicia a sessão no Mac OS X com uma conta de usuário do Active Directory, o conector do Active Directory pode montar a pasta de início da rede Windows especificada na conta de usuário do Active Directory como a pasta de início do Mac OS X do usuário. Você pode especificar se deseja usar o início da rede especificado pelo atributo Directory de início padrão do Active Directory ou pelo atributo Directory de início do Mac OS X (caso o esquema do Active Directory tenha sido expandido para incluí-lo).

Por outro lado, você pode configurar o plug-in para criar uma pasta de início local no volume de inicialização do computador cliente do Mac OS X. Nesse caso, o plug-in também monta a pasta de início de rede do Windows do usuário (especificado na conta de usuário do Active Directory) como o volume de rede, como um ponto de compartilhamento. Usando o Finder, o usuário pode copiar os arquivos entre o volume de rede da pasta de início do Windows e da pasta de início do Mac OS X.

O conector do Active Directory também pode criar contas móveis para usuários. Uma conta móvel possui uma pasta de início no volume de inicialização do computador cliente do Mac OS X. O usuário também possui uma pasta de início de rede conforme especificada na conta do Active Directory do usuário.

Uma conta móvel guarda as credenciais de autenticação do Active Directory do usuário no computador cliente do Mac OS X. As credenciais guardadas permitem ao usuário iniciar a sessão usando o nome e a senha do Active Directory quando o computador cliente estiver desconectado de um servidor Active Directory.

Uma conta móvel possui uma pasta de início no volume de inicialização do computador cliente do Mac OS X. (O usuário também possui uma pasta de início de rede conforme especificada na conta do Active Directory do usuário.)

Se o esquema do Active Directory tiver sido estendido para incluir tipos de registros (classes de objetos) e atributos do Mac OS X, o conector do Active Directory os detecta e acessa.

Por exemplo, o esquema do Active Directory pode ser alterado usando as ferramentas administrativas do Windows para incluir os atributos de cliente gerenciados pelo Mac OS X. Essa alteração de esquema ativa o conector do Active Directory para que seja compatível com as configurações de clientes gerenciados efetuadas no aplicativo Workgroup Manager do Mac OS X Server.

Os clientes do Mac OS X possuem acesso de leitura completo aos atributos que foram adicionados ao diretório. Conseqüentemente, talvez seja necessário alterar a ACL desses atributos para permitir que grupos de computadores possam ler esses atributos adicionados.

O conector do Active Directory descobre todos os domínios na floresta do Active Directory. Você pode configurar o plug-in para permitir que os usuários de qualquer domínio na floresta se autentiquem em um computador Mac OS X. Alternativamente, você pode permitir que somente domínios específicos sejam autenticados no cliente.

O conector do Active Directory é totalmente compatível com o redirecionamento de dados em caso de falha e resposta do Active Directory. Ele descobre vários controladores de domínio e determina o mais próximo. Se um controlador de domínio se torna indisponível, o plug-in alterna para o outro controlador de domínio mais próximo.

O conector do Active Directory usa LDAP para acessar as contas de usuários do Active Directory e Kerberos para autenticá-las. O conector do Active Directory não usa a interface de serviços do Active Directory (ADSI) de propriedade da Microsoft para obter os serviços de diretório e autenticação.
