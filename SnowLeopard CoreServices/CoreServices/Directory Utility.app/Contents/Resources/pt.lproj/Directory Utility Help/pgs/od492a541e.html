Como criar uma senha

A senha associada a uma conta de usuário deve ser digitada pelo usuário quando ele fizer a autenticação de início de sessão ou outro serviço. A senha faz distinção entre maiúsculas e minúsculas (exceto para senhas SMB-LAN Manager) e é mascarada na tela conforme é inserida.

Sem considerar o tipo de senha que você escolher para um usuário, aqui estão algumas diretrizes para criar uma senha para contas de usuário do Mac OS X Server:

  • Uma senha deve conter uma combinação de letras, números e símbolos que não sejam facilmente adivinhados por usuários não autorizados. As senhas não devem conter palavras. As boas senhas incluem dígitos e símbolos (como # ou $) ou possuem a primeira letra de todas as palavras de uma frase. Use letras maiúsculas e minúsculas.

  • Evite espaços e combinações de Opção-tecla.

  • Evite caracteres que não possam ser digitados em computadores que o usuário irá utilizar ou que necessitam de um conhecimento especial de combinação de teclas para que seja inserida corretamente em diferentes teclados e plataformas.

  • Alguns protocolos de rede não são compatíveis com senhas que contêm espaços à esquerda, espaços incorporados ou espaços nulos.

  • Uma senha de comprimento zero não é recomendável. O Open Directory e alguns sistemas (como a vinculação LDAP) não são compatíveis com uma senha de comprimento zero.

  • Para obter compatibilidade máxima com computadores e serviços acessados pelos usuários, use apenas caracteres ASCII para as senhas.
