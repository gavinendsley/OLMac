Como configurar o acesso a um directório LDAP

Utilizando o Utilitário de Directório, é possível especificar a forma como o Mac OS X acede a um directório LDAPv3, se souber o nome DNS ou o endereço IP do servidor de directórios LDAP.

Se o directório não estiver alojado num servidor que forneça os seus próprios mapeamentos (como o Mac OS X Server), terá de conhecer a base de pesquisa e o modelo para associar dados do Mac OS X aos dados do directório.

Os modelos de mapeamento suportados são:

  • “Servidor Open Directory”, para um directório a utilizar o esquema do Mac OS X Server

  • “Active Directory”, para um directório alojado num servidor a executar o Windows 2000, o Windows 2003 ou posterior

  • “RFC 2307”, para a maioria dos directórios alojados em servidores UNIX

O plug-in LDAPv3 suporta na íntegra a réplica e protecção contra falhas do Open Directory. Se o servidor principal do Open Directory se tornar indisponível, o plug-in recorre a uma réplica.

Para especificar mapeamentos personalizados para os dados do directório, siga as instruções especificadas em Como configurar manualmente o acesso a um directório LDAP, em vez das presentes instruções.

Importante:  se o nome do computador incluir um hífen, poderá não conseguir aceder ou ligar-se a um domínio de directórios como o LDAP ou Active Directory. Para estabelecer ligação, utilize um nome de computador sem hífen.
Para receber ajuda do Utilitário de Directório na configuração do acesso a um directório LDAP:

  1. Abra as Preferências do Sistema e clique em Contas.

  2. Se o ícone de cadeado estiver fechado, abra-o clicando no mesmo e introduzindo um nome e palavra-passe de administrador.

  3. Clique em Opções e depois clique em Aceder ou Editar.

  4. Clique em “Abrir o Utilitário de Directório”.

  5. Se o ícone de cadeado estiver fechado, abra-o clicando no mesmo e introduzindo um nome e palavra-passe de administrador.

  6. Clique em Serviços.

  7. Na lista de serviços, seleccione LDAPv3 e clique no botão de edição (/).

    Pode seleccionar LDAPv3 na lista de serviços sem seleccionar a opção Activar referente ao LDAPv3.

  8. Clique em Novo e digite o nome DNS ou o endereço IP do servidor LDAP.

  9. Seleccione as opções para aceder ao directório:

    • Seleccione “Encriptar usando SSL”, se pretender que o Open Directory utilize SSL (Secure Sockets Layer) para efectuar ligações ao directório LDAP. Antes de seleccionar esta opção, pergunte ao administrador do Open Directory se é necessário SSL.

    • Seleccione “Usar para autenticação”, se este directório incluir contas de utilizador que alguém irá utilizar para aceder a serviços ou proceder a autenticação.

    • Seleccione “Usar para contactos”, se este directório incluir endereços de correio e outras informações que pretenda utilizar na Agenda.

    Se o Utilitário de Directório não for capaz de contactar o servidor LDAP, será exibida uma mensagem e terá de configurar o acesso manualmente ou cancelar o processo de configuração. Para mais informações sobre instruções de configuração manual, consulte Como configurar manualmente o acesso a um directório LDAP.

    Se a caixa de diálogo se expandir para mostrar opções de mapeamento, seleccione o modelo de mapeamento no menu instantâneo, digite o sufixo da base de pesquisa e clique em Continuar.

    Normalmente, o sufixo da base de pesquisa deriva do nome DNS do servidor. Por exemplo, o sufixo da base de pesquisa pode ser “dc=ods,dc=exemplo,dc=com” para um servidor cujo nome DNS seja ods.exemplo.com.

    Se nenhum modelo de mapeamento disponível se aplicar à ligação que está a configurar, clique em Manual. Para mais informações, consulte Como configurar manualmente o acesso a um directório LDAP.

  10. Para que o Utilitário de Directório obtenha informações do servidor LDAP, clique em Continuar.

  11. Se a caixa de diálogo se expandir para apresentar opções relativas a ligação fidedigna, digite o nome do computador e o nome e palavra-passe de um administrador do directório (a ligação pode ser opcional).

    A caixa de diálogo informa-o se o directório LDAP requer ligação fidedigna ou se esta é opcional. A ligação fidedigna é mútua: sempre que o computador se ligar ao directório LDAP, autenticam-se um ao outro. Se a ligação fidedigna estiver configurada, ou se o directório LDAP não suportar a ligação fidedigna, o botão Ligar não aparece. Certifique-se de que indicou o nome de computador correcto.

    Se surgir um aviso a indicar “Registo já existente”, clique em Cancelar para voltar atrás e alterar o nome do computador, ou clique em Substituir para substituir o registo existente.

    O registo existente pode estar abandonado ou pertencer a outro computador.

    Se substituir um registo existente, avise o administrador do directório LDAP, para o caso de a substituição do registo desactivar outro computador. Neste caso, o administrador do directório LDAP terá de dar ao computador desactivado um nome diferente e adicioná-lo novamente ao grupo de computadores ao qual pertencia.

  12. Se a caixa de diálogo se expandir para apresentar opções de ligação, seleccione “Usar autenticação ao estabelecer ligação” e digite o nome distinto e a palavra-passe de uma conta de utilizador no directório.

    As opções para uma ligação autenticada aparecem se o servidor LDAP suportar uma ligação autenticada, mas não uma ligação fidedigna. Uma ligação autenticada não é mútua: o servidor LDAP autentica o cliente, mas este não autentica o servidor.

    “Usar autenticação ao estabelecer ligação” é seleccionado por predefinição, mas apresenta-se esbatido se o servidor LDAP exigir a introdução de um nome distinto e palavra-passe de uma conta de utilizador para uma ligação autenticada.

    O nome distinto pode especificar qualquer conta de utilizador com permissão para ver dados no directório. Por exemplo, uma conta de utilizador cujo nome abreviado seja dirauth num servidor LDAP e cujo endereço seja ods.exemplo.com, teria o nome distinto uid=dirauth,cn=utilizadores,dc=ods,dc=exemplo,dc=com.

    Importante:  se o nome distinto ou a palavra-passe estiverem incorrectos, pode aceder ao computador utilizando contas de utilizador do directório LDAP.

  13. Clique em OK para terminar a criação da ligação LDAP.

  14. Clique em OK para terminar a configuração das opções do LDAPv3.

Se tiver seleccionado “Usar para autenticação” ou “Usar para contactos” no passo 5, a configuração do directório LDAP que acabou de criar é adicionada a uma política de pesquisa personalizada no painel Autenticação ou Contactos do Utilitário de Directório.

Certifique-se de que LDAPv3 está activado no painel Serviços, para que o computador utilize a configuração LDAP que criou. Para mais informações, consulte Como activar ou desactivar os serviços de directório LDAP.
