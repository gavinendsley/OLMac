Como configurar pesquisas e mapeamentos LDAP

Utilizando o Utilitário de Directório, é possível editar os mapeamentos, bases de pesquisa e âmbitos de pesquisa que especificam a forma como o Mac OS X encontra itens de dados específicos num directório LDAP. Estas definições podem ser editadas separadamente para cada configuração de directório LDAP listada no Utilitário de Directório. Cada configuração de directório LDAP especifica a forma como o Mac OS X acede a dados num directório LDAPv3 ou LDAPv2.

É possível editar o seguinte:

  • O mapeamento de cada tipo de registo do Mac OS X para classes de objectos LDAP

  • O mapeamento dos tipos de dados do Mac OS X, ou atributos, para os atributos LDAP de cada tipo de registo

  • A base de pesquisa e o âmbito de pesquisa LDAP que determinam onde o Mac OS X deve procurar um tipo de registo do Mac OS X num directório LDAP

Ao associar atributos de utilizador do Mac OS X a um domínio de directórios LDAP de leitura/escrita (um domínio LDAP que não é de apenas leitura), o atributo LDAP associado a RealName não pode ser igual ao primeiro atributo de uma lista de atributos LDAP associados a RecordName.

Por exemplo, o atributo cn não pode ser o primeiro atributo associado a RecordName, se cn também estiver associado a RealName.

Se o atributo LDAP associado a RealName for igual ao primeiro atributo associado a RecordName, ocorrerão problemas quando tentar editar o nome completo (por extenso) ou o primeiro nome abreviado no Gestor do Grupo de Trabalho.

Para mais informações sobre os tipos de registos e atributos do Mac OS X, consulte Administração do Open Directory (disponível em www.apple.com/pt/server/resources/).

Para editar as bases de pesquisa e os mapeamentos de um servidor LDAP:

  1. Abra as Preferências do Sistema e clique em Contas.

  2. Se o ícone de cadeado estiver fechado, abra-o clicando no mesmo e introduzindo um nome e palavra-passe de administrador.

  3. Clique em Opções e depois clique em Editar.

  4. Clique em “Abrir o Utilitário de Directório”.

  5. Se o ícone de cadeado estiver fechado, abra-o clicando no mesmo e introduzindo um nome e palavra-passe de administrador.

  6. Clique em Serviços.

  7. Na lista de serviços, seleccione LDAPv3 e clique no botão de edição (/).

  8. Se a lista de configurações do servidor estiver oculta, clique em “Mostrar opções”.

  9. Na lista, seleccione uma configuração de servidor e clique em Edição.

  10. Clique em “Pesquisa & Mapeamentos”.

  11. Seleccione os mapeamentos que pretende utilizar como ponto de partida; em alternativa, seleccione Personalizado para começar sem mapeamentos predefinidos.

    Se seleccionar um dos modelos de mapeamento LDAP, surge um sufixo da base de pesquisa que pode alterar ou pode aceitar o sufixo predefinido clicando em OK.

    Clique no menu instantâneo “Aceder ao servidor LDAPv3 usando” e seleccione um modelo de mapeamento para utilizar os respectivos mapeamentos como ponto de partida.

  12. Adicione tipos de registos e altere as respectivas bases de pesquisa conforme o necessário:

    • Para adicionar tipos de registos, clique em Adicionar (por baixo da lista “Tipos e atributos dos registos”); em seguida, na folha que aparece, seleccione “Tipos de registos”, seleccione os tipos de registos da lista e clique em OK.

    • Para alterar a base de pesquisa e o âmbito de pesquisa de um tipo de registo, seleccione-o na lista “Tipos e atributos dos registos” e, em seguida, edite o campo Pesquisar. Seleccione “todos os sub-directórios” para definir um âmbito de pesquisa que inclua a hierarquia do directório LDAP a partir da base de pesquisa, ou seleccione “apenas o primeiro nível” para definir um âmbito de pesquisa que inclua apenas a base de pesquisa e um nível abaixo desta na hierarquia do directório LDAP.

    • Para remover um tipo de registo, na lista “Tipos e atributos dos registos” seleccione o tipo e clique em Apagar.

    • Para adicionar um mapeamento de um tipo de registo, seleccione o tipo de registo na lista “Tipos e atributos dos registos”, clique em Adicionar (por baixo de “Mapear em __ itens na lista”) e introduza o nome de uma classe de objectos do directório LDAP.

      Para adicionar outra classe de objectos LDAP, prima a tecla Return e introduza o nome da classe de objectos; especifique se pretende utilizar as classes de objectos LDAP da lista, utilizando o menu instantâneo localizado por cima da mesma.

    • Para alterar um mapeamento de um tipo de registo, seleccione o tipo de registo na lista “Tipos e atributos dos registos”, faça duplo clique na classe de objectos LDAP que pretende alterar em “Mapear em __ itens na lista” e edite-a. Especifique se pretende utilizar as classes de objectos LDAP da lista, utilizando o menu instantâneo localizado por cima da mesma.

    • Para remover um mapeamento de um tipo de registo, seleccione o tipo de registo na lista “Tipos e atributos dos registos”, seleccione a classe de objectos LDAP que pretende remover de “Mapear p/ __ itens na lista” e clique em Apagar (por baixo de “Mapear p/ __ itens na lista”).

  13. Adicione atributos e altere os respectivos mapeamentos conforme necessário:

    • Para adicionar atributos a um tipo de registo, na lista “Tipos e atributos dos registos” seleccione o tipo de registo e clique em Adicionar (por baixo da lista “Tipos e atributos dos registos”); em seguida, na folha que aparece, seleccione “Tipos de atributos”, seleccione os tipos de atributos e clique em OK.

    • Para adicionar um mapeamento de um atributo, na lista “Tipos e atributos dos registos”, seleccione o atributo, clique em Adicionar (por baixo de “Mapear em __ itens na lista”) e introduza o nome de um atributo do directório LDAP. Para adicionar outro atributo LDAP, prima Return e introduza o nome do atributo.

    • Para alterar um mapeamento de um atributo, seleccione o atributo na lista “Tipos e atributos dos registos”, faça duplo clique no item que pretende alterar em “Mapear p/ __ itens na lista” e edite o nome do item.

    • Para remover um mapeamento de um atributo, seleccione o atributo na lista “Tipos e atributos dos registos”, seleccione o item que pretende remover de “Mapear p/ __ itens na lista” e clique em Apagar (por baixo de “Mapear p/ __ itens na lista”).

    • Para alterar a ordem dos atributos que aparecem na lista à direita, arraste os atributos para uma posição mais acima ou mais abaixo na lista.

  14. Para guardar os seus mapeamentos como um modelo, clique em “Guardar modelo”.

    Os modelos guardados no local predefinido serão listados nos menus instantâneos dos modelos de mapeamento LDAP na próxima vez que abrir o Utilitário de Directório. O local predefinido para os modelos guardados é a sua pasta pessoal, no seguinte percurso:

    ~/Biblioteca/Application Support/Directory Access/LDAPv3/Templates

  15. Para guardar os mapeamentos no directório LDAP, para que este os possa fornecer automaticamente aos respectivos clientes, clique em “Escrever no servidor” e introduza uma base de pesquisa para guardar os mapeamentos, um nome distinto de um administrador ou de outro utilizador com permissão de escrita na base de pesquisa (por exemplo, uid=diradmin,cn=utilizadores,dc=ods,dc=exemplo,dc=com), e uma palavra-passe.

    Se estiver a escrever mapeamentos num servidor LDAP Open Directory, a base de pesquisa correcta é “cn=config,sufixo (em que sufixo significa o sufixo da base de pesquisa do servidor, como por exemplo dc=ods,dc=exemplo,dc=com).

O directório LDAP fornece os respectivos mapeamentos aos clientes Mac OS X cuja política de pesquisa personalizada inclua uma ligação configurada para obter mapeamentos do servidor LDAP.

O directório LDAP também fornece os respectivos mapeamentos a todos os clientes Mac OS X que possuam uma política de pesquisa automática. Para mais informações, consulte Como configurar o acesso a um directório LDAP e Como utilizar as definições avançadas das políticas de pesquisa.
