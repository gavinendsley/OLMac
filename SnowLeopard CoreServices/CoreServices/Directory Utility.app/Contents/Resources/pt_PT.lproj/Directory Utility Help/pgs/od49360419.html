Como configurar manualmente o acesso a um directório LDAP

É possível criar manualmente uma configuração que especifique a forma como o Mac OS X acede a um directório LDAPv3 ou LDAPv2. Tem de conhecer o nome DNS ou o endereço IP do servidor de directórios LDAP.

Se o directório não estiver alojado num Mac OS X Server, terá de conhecer a base de pesquisa e o modelo para associar dados do Mac OS X aos dados do directório. Os modelos de mapeamento suportados são:

  • “Do servidor”, para um directório que forneça os seus próprios mapeamentos e base de pesquisa, como o Mac OS X Server

  • “Servidor Open Directory”, para um directório a utilizar o esquema do Mac OS X Server

  • “Active Directory”, para um directório alojado num servidor a executar o Windows 2000, o Windows 2003 ou posterior

  • “RFC 2307”, para a maioria dos directórios alojados em servidores UNIX

  • Personalizado, para directórios que não utilizem nenhum dos mapeamentos acima indicados

O plug-in LDAPv3 suporta na íntegra a réplica e protecção contra falhas do Open Directory. Se o servidor principal do Open Directory se tornar indisponível, o plug-in recorre a uma réplica.

Importante:  se o nome do computador incluir um hífen, poderá não conseguir aceder ou ligar-se a um domínio de directórios como o LDAP ou Active Directory. Para estabelecer ligação, utilize um nome de computador sem hífen.
Para configurar manualmente o acesso a um directório LDAP:

  1. Abra as Preferências do Sistema e clique em Contas.

  2. Se o ícone de cadeado estiver fechado, abra-o clicando no mesmo e introduzindo um nome e palavra-passe de administrador.

  3. Clique em Opções e depois clique em Aceder ou Editar.

  4. Clique em “Abrir o Utilitário de Directório”.

  5. Se o ícone de cadeado estiver fechado, abra-o clicando no mesmo e introduzindo um nome e palavra-passe de administrador.

  6. Clique em Serviços.

  7. Na lista de serviços, seleccione LDAPv3 e clique no botão de edição (/).

    Pode seleccionar LDAPv3 na lista de serviços sem seleccionar a opção Activar referente ao LDAPv3.

  8. Clique em Novo e depois em Manual.

  9. Digite um nome para a configuração.

  10. Prima a tecla Tab e digite o nome DNS ou o endereço IP do servidor onde está alojado o directório LDAP ao qual pretende aceder.

  11. Junto ao nome DNS ou ao endereço IP, clique no menu instantâneo e seleccione um método ou modelo de mapeamento:

    • Se seleccionar “Do servidor”, não necessita do sufixo da base de pesquisa. Neste caso, o Open Directory assume que o sufixo da base de pesquisa é o primeiro nível do directório LDAP.

    • Se seleccionar um modelo, digite o sufixo da base de pesquisa do directório LDAP e clique em OK. É necessário introduzir um sufixo de base de pesquisa, caso contrário, o computador não consegue localizar informações no directório LDAP.

      Normalmente, o sufixo da base de pesquisa deriva do nome DNS do servidor. Por exemplo, o sufixo da base de pesquisa pode ser “dc=ods,dc=exemplo,dc=com” para um servidor cujo nome DNS seja ods.exemplo.com.

    • Se seleccionar Personalizado, terá de configurar mapeamentos entre os tipos de registos e atributos do Mac OS X e as classes e atributos do directório LDAP ao qual pretende ligar. Para mais informações, consulte Como configurar pesquisas e mapeamentos LDAP.

  12. Antes de seleccionar a opção “Encriptar usando SSL”, consulte o administrador do Open Directory para determinar se é necessário SSL.

  13. Para alterar as seguintes definições para esta configuração LDAP, clique em Editar para apresentar as opções relativas à configuração LDAP seleccionada, efectue as alterações e clique em OK quando terminar de editar as opções da configuração LDAP.

    • Clique em Ligação para definir as opções de tempo limite, especificar uma porta personalizada, ignorar as referências do servidor ou forçar a utilização do protocolo LDAPv2 (apenas leitura). Para mais informações, consulte Como alterar as definições de ligação de um directório LDAP.

    • Clique em “Pesquisa & Mapeamentos” para configurar pesquisas e mapeamentos para um servidor LDAP. Para mais informações, consulte Como configurar uma ligação fidedigna de um directório LDAP.

    • Clique em Segurança para configurar uma ligação autenticada (em vez de uma ligação fidedigna) e outras opções de política de segurança. Para mais informações, consulte Como alterar a política de segurança de uma ligação LDAP.

    • Clique em Ligar para configurar ligações fidedignas (se o directório LDAP as suportar). Para mais informações, consulte Como configurar uma ligação fidedigna de um directório LDAP.

  14. Clique em OK para terminar de criar manualmente a configuração para aceder a um directório LDAP.

  15. Se pretender que o computador aceda ao directório LDAP para o qual criou uma configuração, adicione o directório a uma política de pesquisa personalizada no painel Autenticação e no painel Contactos da “Política de pesquisa” no Utilitário de Directório; em seguida, certifique-se de que LDAPv3 está activado no painel Serviços.

Para mais informações, consulte Como activar ou desactivar os serviços de directório LDAP e Como definir políticas de pesquisa personalizadas.

Nota:  antes de ser possível utilizar o Gestor do Grupo de Trabalho para criar utilizadores num servidor LDAP que não seja Apple e que utilize mapeamentos RFC 2307 (UNIX), é necessário editar o mapeamento do tipo de registo Utilizadores. Para mais informações, consulte Como editar o mapeamento RFC 2307 a fim de permitir a criação de utilizadores.
Importante:  se alterar o endereço IP e o nome do computador utilizando o changeip enquanto está ligado a um servidor de directórios, é necessário desligar e voltar a ligar ao servidor de directórios para actualizar o directório com o nome e o endereço IP novos do computador. Se não desligar e voltar a ligar ao servidor de directórios, o directório não será actualizado e continuará a utilizar o nome e endereço IP antigos.
