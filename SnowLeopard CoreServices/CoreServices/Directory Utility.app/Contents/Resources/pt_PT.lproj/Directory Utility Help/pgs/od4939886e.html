Como especificar as definições de um ficheiro de configuração BSD

Historicamente, os computadores UNIX guardam os dados administrativos em ficheiros de configuração, tais como /etc/master.passwd, /etc/group e /etc/hosts. O Mac OS X baseia-se numa versão BSD do UNIX, mas normalmente obtém os dados administrativos a partir de sistemas de directórios.

O Mac OS X Server suporta um conjunto fixo de ficheiros de configuração BSD. Não é possível especificar quais os ficheiros de configuração a utilizar, nem associar o respectivo conteúdo a tipos de registos e atributos do Mac OS X.

Na versão 10.2, ou posterior, do Mac OS X (incluindo a versão 10.2, ou posterior, do Mac OS X Server), o Open Directory consegue obter os dados administrativos a partir de ficheiros de configuração BSD. Esta capacidade permite às organizações que possuem ficheiros de configuração BSD utilizar cópias dos ficheiros existentes em computadores Mac OS X. Os ficheiros de configuração BSD podem ser utilizados sozinhos ou com outros domínios de directórios.

Para utilizar ficheiros de configuração BSD:

  1. Certifique-se de que os ficheiros de configuração BSD contêm os dados necessários para os serviços de directório do Mac OS X.

    Para mais informações, consulte Como configurar dados em ficheiros de configuração BSD.

  2. Abra as Preferências do Sistema e clique em Contas.

  3. Se o ícone de cadeado estiver fechado, abra-o clicando no mesmo e introduzindo um nome e palavra-passe de administrador.

  4. Clique em Opções e depois clique em Aceder ou Editar.

  5. Clique em “Abrir o Utilitário de Directório”.

  6. Se o ícone de cadeado estiver fechado, abra-o clicando no mesmo e introduzindo um nome e palavra-passe de administrador.

  7. Clique em Serviços.

  8. Na lista de serviços, seleccione “BSD Flat File e NIS” e clique no botão de edição (/).

  9. Seleccione “Utilizar registos de utilizador e grupo no nó local do BSD” e clique em OK.

    O domínio dos ficheiros de configuração BSD é adicionado à política de pesquisa de autenticação do computador como /BSD/local.
