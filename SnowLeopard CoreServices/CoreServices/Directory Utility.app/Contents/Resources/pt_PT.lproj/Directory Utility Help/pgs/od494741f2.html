Como alterar uma configuração para aceder a um directório LDAP

É possível utilizar o Utilitário de Directório para alterar as definições de uma configuração de directório LDAP. As definições da configuração determinam a forma como o Open Directory acede a um directório LDAPv3 ou LDAPv2.

Se a configuração LDAP tiver sido obtida por DHCP, não é possível alterá-la, pelo que este tipo de configuração aparece esbatido na lista de configurações LDAP.

Para editar uma configuração para aceder a um directório LDAP:

  1. Abra as Preferências do Sistema e clique em Contas.

  2. Se o ícone de cadeado estiver fechado, abra-o clicando no mesmo e introduzindo um nome e palavra-passe de administrador.

  3. Clique em Opções e depois clique em Editar.

  4. Clique em “Abrir o Utilitário de Directório”.

  5. Se o ícone de cadeado estiver fechado, abra-o clicando no mesmo e introduzindo um nome e palavra-passe de administrador.

  6. Clique em Serviços.

  7. Na lista de serviços, seleccione LDAPv3 e clique no botão de edição (/).

  8. Se a lista de configurações do servidor estiver oculta, clique em “Mostrar opções”.

  9. Efectue as alterações necessárias às seguintes definições:

    • Activar: clique numa opção para activar ou desactivar o acesso a um servidor de directórios LDAP.

    • Configuração: faça duplo clique no nome de uma configuração para a editar.

    • Servidor ou endereço IP: faça duplo clique no nome de um servidor ou endereço IP para o alterar.

    • Mapeamentos LDAP: no menu instantâneo, seleccione um modelo, digite o sufixo da base de pesquisa do directório LDAP e clique em OK.

      Se seleccionar um modelo, é necessário introduzir o sufixo da base de pesquisa, caso contrário, o computador não consegue localizar informações no directório LDAP. Normalmente, o sufixo da base de pesquisa deriva do nome DNS do servidor. Por exemplo, para um servidor cujo nome DNS seja ods.exemplo.com o sufixo da base de pesquisa é “dc=ods,dc=exemplo,dc=com”.

      Se seleccionar “Do servidor” em vez de um modelo, não necessita do sufixo da base de pesquisa. Neste caso, o Open Directory assume que o sufixo da base de pesquisa é o primeiro nível do directório LDAP.

      Se seleccionar Personalizado, terá de configurar mapeamentos entre os tipos de registos e atributos do Mac OS X e as classes e atributos do directório LDAP ao qual pretende ligar. Para mais informações, consulte Como configurar pesquisas e mapeamentos LDAP.

    • SSL: clique na opção para activar ou desactivar as comunicações encriptadas por meio do protocolo SSL. Antes de seleccionar a opção SSL, pergunte ao administrador do Open Directory se é necessário SSL.

  10. Para alterar as seguintes predefinições para esta configuração LDAP, clique em Editar para apresentar as opções relativas à configuração LDAP seleccionada, efectue as alterações e clique em OK quando terminar de editar as opções da configuração LDAP.

    • Clique em Ligação para especificar as opções de tempo limite, especificar uma porta personalizada, ignorar as referências do servidor ou forçar a utilização do protocolo LDAPv2 (apenas leitura). Para mais informações, consulte Como alterar as definições de ligação de um directório LDAP.

    • Clique em “Pesquisa & Mapeamentos” para configurar pesquisas e mapeamentos para um servidor LDAP. Para mais informações, consulte Como configurar uma ligação fidedigna de um directório LDAP.

    • Clique em Segurança para configurar uma ligação autenticada (em vez de uma ligação fidedigna) e outras opções de política de segurança. Para mais informações, consulte Como alterar a política de segurança de uma ligação LDAP.

    • Clique em Ligar para configurar a ligação fidedigna ou clique em Desligar para parar a ligação fidedigna. (Poderá não ver estes botões se o directório LDAP não permitir ligações fidedignas.) Para mais informações, consulte Como configurar uma ligação fidedigna de um directório LDAP.

  11. Clique em OK para terminar de alterar a configuração para aceder a um directório LDAP.
