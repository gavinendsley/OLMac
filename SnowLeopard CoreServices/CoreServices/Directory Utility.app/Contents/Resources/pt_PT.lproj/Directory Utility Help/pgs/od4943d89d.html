Acerca do acesso ao Active Directory

Pode configurar o Mac OS X para aceder a informações básicas relativas a contas de utilizador num domínio do Active Directory de um servidor Windows 2000 ou posterior. Isto é possível graças a um conector do Active Directory para o Utilitário de Directório. Este conector do Active Directory está listado no painel Serviços do Utilitário de Directório.

Não precisa de efectuar alterações de esquema ao domínio do Active Directory para obter informações básicas relativas a contas de utilizador. Poderá alterar a lista de controlo de acesso (ACL) predefinida de atributos específicos, para que as contas do computador consigam ler as propriedades do utilizador.

O conector do Active Directory gera todos os atributos exigidos pelo Mac OS X para autenticação a partir dos atributos padrão nas contas de utilizador do Active Directory. O conector também suporta políticas de autenticação do Active Directory, incluindo alterações à palavra-passe, caducidade, alterações forçadas e opções de segurança.

O Mac OS X v10.6 suporta opções de encriptação de pacote e de assinatura de pacote para todos os domínios do Active Directory do Windows. Esta funcionalidade está ligada por predefinição como “permitir”. Pode alterar a predefinição para desactivado ou exigido utilizando a ferramenta de linha de comandos dsconfigad. As opções de encriptação e assinatura de pacotes garantem a protecção de todos os dados transferidos de e para o domínio do Active Directory para fins de procura de registos.

O conector do Active Directory gera de forma dinâmica um ID de utilizador exclusivo e um ID de grupo principal com base no GUID (Globally Unique ID) da conta de utilizador no domínio do Active Directory. O ID de utilizador e o ID de grupo principal gerados são iguais para cada conta de utilizador, mesmo que a conta seja utilizada para aceder a diferentes computadores Mac OS X.

Em alternativa, pode forçar o conector do Active Directory a associar o ID de utilizador aos atributos do Active Directory que especificar.

O conector do Active Directory gera um ID de grupo com base no GUID da conta de grupo do Active Directory. Também pode forçar o plug-in a associar o ID de grupo das contas de grupo aos atributos do Active Directory que especificar.

Quando alguém acede ao Mac OS X com uma conta de utilizador do Active Directory, o conector do Active Directory pode montar a pasta pessoal em rede do Windows especificada na conta de utilizador do Active Directory como a pasta pessoal do Mac OS X do utilizador. É possível especificar se pretende utilizar a pasta pessoal em rede especificada pelo atributo homeDirectory padrão do Active Directory ou pelo atributo homeDirectory do Mac OS X (se o esquema do Active Directory tiver sido ampliado para o incluir).

Em alternativa, pode configurar o plug-in para criar uma pasta pessoal local no volume de arranque do computador cliente do Mac OS X. Neste caso, o plug-in também monta a pasta pessoal em rede do Windows do utilizador (especificada na conta de utilizador do Active Directory) como volume de rede, como um ponto de partilha. Utilizando o Finder, o utilizador pode então copiar ficheiros entre o volume de rede da pasta pessoal do Windows e a pasta pessoal local do Mac OS X.

O conector do Active Directory também pode criar contas móveis para os utilizadores. Uma conta móvel possui uma pasta pessoal local no volume de arranque do computador cliente do Mac OS X. (O utilizador também possui uma pasta pessoal em rede, conforme especificado na conta do Active Directory do utilizador.)

Uma conta móvel coloca as credenciais de autenticação do Active Directory do utilizador em cache no computador cliente do Mac OS X. As credenciais em cache permitem ao utilizador aceder utilizando o nome e a palavra-passe do Active Directory quando o computador cliente está desligado do servidor do Active Directory.

Uma conta móvel possui uma pasta pessoal local no volume de arranque do computador cliente do Mac OS X. (O utilizador também possui uma pasta pessoal em rede, conforme especificado na conta do Active Directory do utilizador.)

Se o esquema do Active Directory tiver sido ampliado para incluir tipos de registos (classes de objectos) e atributos do Mac OS X, o conector do Active Directory detecta-os e acede aos mesmos.

Por exemplo, o esquema do Active Directory pode ser alterado, utilizando as ferramentas administrativas do Windows, para incluir atributos de clientes geridos pelo Mac OS X. Esta alteração ao esquema permite ao conector do Active Directory suportar definições de clientes geridos configuradas utilizando a aplicação Gestor do Grupo de Trabalho do Mac OS X Server.

Os clientes Mac OS X assumem acesso de leitura pleno aos atributos que são adicionados ao directório. Por conseguinte, poderá ser necessário alterar a ACL desses atributos para permitir que grupos de computadores leiam estes atributos adicionados.

O conector do Active Directory localiza todos os domínios numa floresta do Active Directory. É possível configurar o plug-in para permitir aos utilizadores de qualquer floresta autenticarem-se num computador com Mac OS X. Em alternativa, pode permitir que apenas domínios específicos sejam autenticados no cliente.

O conector do Active Directory suporta na íntegra a réplica e protecção contra falhas do Active Directory. Localiza vários controladores de domínio e determina qual o mais próximo. Se um controlador de domínio se tornar indisponível, o plug-in recorre a outro controlador de domínio nas proximidades.

O conector do Active Directory utiliza LDAP para aceder a contas de utilizador do Active Directory e Kerberos para as autenticar. O conector do Active Directory não utiliza a ADSI (Active Directory Services Interface) da Microsoft para obter serviços de directório ou de autenticação.
