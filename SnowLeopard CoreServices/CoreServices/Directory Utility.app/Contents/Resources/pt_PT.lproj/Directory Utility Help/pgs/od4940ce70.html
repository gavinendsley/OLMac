Como configurar o acesso por LDAP a domínios do Active Directory

Utilizando o Utilitário de Directório, é possível especificar uma configuração LDAPv3 para aceder a um domínio do Active Directory num servidor Windows. Uma configuração LDAPv3 proporciona-lhe controlo total sobre o mapeamento de tipos de registos e atributos do Mac OS X a classes de objectos, bases de pesquisa e atributos do Active Directory.

O mapeamento de alguns tipos de registos e atributos importantes do Mac OS X, tais como o ID de utilizador exclusivo (UID), requer a ampliação do esquema do Active Directory.

Uma configuração LDAPv3 não inclui as seguintes características do conector do Active Directory listado no Utilitário de Directório:

  • Geração dinâmica de ID de utilizador exclusivo e ID do grupo principal

  • Criação de uma pasta pessoal local do Mac OS X

  • Montagem automática da pasta pessoal do Windows

  • Contas de utilizador móveis com credenciais de autenticação em cache

  • Localização de todos os domínios numa floresta do Active Directory

  • Suporte de réplica e protecção contra falhas do Active Directory

Para mais informações, consulte Acerca do acesso ao Active Directory.

Para criar uma configuração de servidor do Active Directory:

  1. Abra as Preferências do Sistema e clique em Contas.

  2. Se o ícone de cadeado estiver fechado, abra-o clicando no mesmo e introduzindo um nome e palavra-passe de administrador.

  3. Clique em Opções e depois clique em Aceder ou Editar.

  4. Clique em “Abrir o Utilitário de Directório”.

  5. Se o ícone de cadeado estiver fechado, abra-o clicando no mesmo e introduzindo um nome e palavra-passe de administrador.

  6. Clique em Serviços.

  7. Na lista de serviços, seleccione LDAPv3 e clique no botão de edição (/).

  8. Clique em Novo e digite o nome DNS ou o endereço IP do servidor do Active Directory.

  9. Seleccione as seguintes opções para aceder ao directório e clique em Continuar, para o Utilitário de Directório obter informações a partir do servidor do Active Directory.

    • Seleccione “Encriptar usando SSL”, se pretender que o Open Directory utilize SSL para efectuar ligações ao servidor do Active Directory. Antes de seleccionar a opção SSL, pergunte ao administrador do Open Directory se é necessário SSL.

    • Seleccione “Usar para autenticação”, se este directório incluir contas de utilizador que alguém irá utilizar para aceder a serviços ou proceder a autenticação.

    • Seleccione “Usar para contactos”, se este directório incluir endereços de correio e outras informações que pretenda utilizar na Agenda.

    Se o Utilitário de Directório não for capaz de contactar o servidor do Active Directory, será exibida uma mensagem e terá de configurar o acesso manualmente ou cancelar o processo de configuração. Para mais informações, consulte Como configurar manualmente o acesso a um directório LDAP.

    Se tiver seleccionado “Usar para autenticação” ou “Usar para contactos”, a ligação LDAPv3 ao domínio do Active Directory é adicionada a uma política de pesquisa personalizada no painel Autenticação ou Contactos do Utilitário de Directório.

    Certifique-se de que LDAPv3 está activado no painel Serviços, para que o computador utilize a ligação que configurou. Para mais informações, consulte Como activar ou desactivar os serviços de directório LDAP.

  10. Quando a caixa de diálogo se expandir para mostrar opções de mapeamentos, seleccione “Active Directory” no menu instantâneo, digite a base de pesquisa e clique em Continuar.

    O modelo de mapeamento do Active Directory para uma configuração LDAPv3 associa alguns tipos de registos e atributos do Mac OS X a classes de objectos e atributos que não fazem parte de um esquema padrão do Active Directory. Pode alterar os mapeamentos especificados pelo modelo ou ampliar o esquema do Active Directory.

    Em alternativa, poderá ser possível aceder ao seu domínio do Active Directory através do conector do Active Directory em vez de utilizar o LDAPv3. Para mais informações, consulte Como configurar o acesso a um domínio do Active Directory.

  11. Quando a caixa de diálogo se expandir para mostrar as opções de ligação, digite o nome distinto e a palavra-passe de uma conta de utilizador do Active Directory.

  12. Clique em OK para terminar a criação da ligação LDAP.

  13. Clique em OK para terminar a configuração das opções do LDAPv3.
