Como configurar pastas pessoais para contas de utilizador do Active Directory

Num computador configurado para utilizar o conector do Active Directory do Utilitário de Directório, é possível activar ou desactivar pastas pessoais em rede ou pastas pessoais locais para contas de utilizador do Active Directory.

Relativamente às pastas pessoais em rede, a pasta pessoal em rede do Windows de um utilizador é montada como pasta pessoal do Mac OS X quando o utilizador acede.

É possível determinar se pretende obter a localização da pasta pessoal em rede a partir do atributo homeDirectory padrão do Active Directory ou do atributo homeDirectory do Mac OS X, se o esquema do Active Directory tiver sido ampliado para o incluir.

Relativamente às pastas pessoais locais, cada utilizador do Active Directory que inicia sessão possui uma pasta pessoal no disco de arranque do Mac OS X. Além disso, a pasta pessoal em rede do utilizador é montada como volume de rede, como um ponto de partilha. O utilizador pode copiar ficheiros entre este volume de rede e a pasta pessoal local.

Para configurar pastas pessoais para contas de utilizador do Active Directory:

  1. Abra as Preferências do Sistema e clique em Contas.

  2. Se o ícone de cadeado estiver fechado, abra-o clicando no mesmo e introduzindo um nome e palavra-passe de administrador.

  3. Clique em Opções e depois clique em Aceder ou Editar.

  4. Clique em “Abrir o Utilitário de Directório”.

  5. Se o ícone de cadeado estiver fechado, abra-o clicando no mesmo e introduzindo um nome e palavra-passe de administrador.

  6. Clique em Serviços.

  7. Na lista de serviços, seleccione “Active Directory” e clique no botão de edição (/).

  8. Se as opções avançadas estiverem ocultas, clique em “Mostrar opções avançadas”.

  9. Clique em Utilizador.

  10. Se pretender que as contas de utilizador do Active Directory possuam pastas pessoais locais na pasta Utilizadores do computador, clique em “Forçar directório pessoal local no disco de arranque”.

    Esta opção não está disponível se a opção “Criar conta móvel ao aceder” estiver seleccionada.

  11. Para utilizar o atributo padrão do Active Directory para a localização da pasta pessoal, seleccione “Usar percurso UNC do Active Directory para obter localização pessoal da rede” e seleccione entre os seguintes protocolos de acesso à pasta pessoal:

    • Para utilizar o protocolo SMB padrão do Windows, seleccione smb no menu instantâneo “Protocolo de rede a utilizar”.

    • Para utilizar o protocolo AFP padrão do Macintosh, seleccione afp no menu instantâneo “Protocolo de rede a utilizar”.

  12. Para utilizar o atributo do Mac OS X para a localização da pasta pessoal, anule a selecção de “Usar percurso UNC do Active Directory para obter localização pessoal da rede”.

    Para utilizar o atributo do Mac OS X, o esquema do Active Directory tem de ser ampliado para o incluir.

  13. Clique em OK.

Se alterar o nome de uma conta de utilizador no domínio do Active Directory, o servidor cria uma pasta pessoal (e subpastas) para a conta de utilizador na próxima vez que esta for utilizada para aceder a um computador Mac OS X. O utilizador continua a poder navegar na pasta pessoal antiga e ver os respectivos conteúdos no Finder.

Pode impedir a criação de uma nova pasta pessoal, mudando o nome da pasta antiga antes do acesso do utilizador seguinte.
