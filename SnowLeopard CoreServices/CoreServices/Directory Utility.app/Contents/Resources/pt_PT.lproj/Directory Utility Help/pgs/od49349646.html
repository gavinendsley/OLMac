Como utilizar as definições avançadas das políticas de pesquisa

O Utilitário de Directório define as seguintes políticas de pesquisa:

  • Autenticação: o Mac OS X utiliza a política de pesquisa de autenticação para localizar e obter informações de autenticação de utilizador e outros dados administrativos em domínios de directórios.

  • Contactos: o Mac OS X utiliza a política de pesquisa de contactos para localizar e obter o nome, endereço e outras informações de contacto em domínios de directórios. A Agenda do Mac OS X utiliza estas informações de contacto. Outras aplicações também podem ser programadas para as utilizarem.

    Cada política de pesquisa consiste numa lista de domínios de directórios. A ordem dos domínios de directórios na lista define a política de pesquisa. Começando pelo topo da lista, o Mac OS X pesquisa cada domínio de directórios incluído, até encontrar as informações de que necessita ou alcançar o fim da lista sem encontrar as informações.

    As políticas de pesquisa de autenticação e contactos podem ter uma das seguintes definições:

  • Automaticamente: começa pelo domínio de directórios local e pode incluir um directório LDAP obtido por DHCP e domínios de directórios aos quais o computador esteja ligado. Esta é a predefinição do Mac OS X v10.2 (e versões posteriores) e oferece o máximo de flexibilidade para computadores portáteis.

  • Directório local: inclui apenas o domínio de directórios local.

  • Percurso personalizado: começa pelo domínio de directórios local e inclui directórios LDAP à sua escolha, um domínio do Active Directory, domínios de directórios partilhados, ficheiros de configuração BSD e um domínio NIS.

    A pasta /BSD/local está sempre incluída no percurso de pesquisa e está sempre inactiva.

Importante:  se configurar o Mac OS X para utilizar uma política de pesquisa de autenticação automática e um servidor LDAP obtido por DHCP, aumenta o risco de um utilizador com intuitos maliciosos controlar o seu computador. O risco ainda é maior se o seu computador estiver configurado para se ligar a uma rede sem fios. Para mais informações, consulte Como proteger os computadores de um servidor DHCP malicioso.
