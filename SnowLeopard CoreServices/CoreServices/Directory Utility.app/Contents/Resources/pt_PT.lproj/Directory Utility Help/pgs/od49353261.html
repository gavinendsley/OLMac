Como alterar a política de segurança de uma ligação LDAP

Utilizando o Utilitário de Directório, é possível configurar uma política de segurança para uma ligação LDAPv3 mais rígida do que a política de segurança do directório LDAP. Por exemplo, se a política de segurança do directório LDAP permitir palavras-passe legíveis, pode definir uma ligação LDAPv3 para não permitir palavras-passe legíveis.

A definição de uma política de segurança mais rígida protege o seu computador contra piratas informáticos maliciosos que tentem utilizar um servidor LDAP fictício para assumir o controlo do seu computador.

O computador tem de comunicar com o servidor LDAP para mostrar o estado das opções de segurança. Por conseguinte, quando altera as opções de segurança de uma ligação LDAPv3, a política de pesquisa de autenticação do computador deverá incluir a ligação LDAPv3.

As definições admissíveis para as opções de segurança de uma ligação LDAPv3 estão sujeitas às capacidades e requisitos de segurança do servidor LDAP. Por exemplo, se o servidor LDAP não suportar autenticação Kerberos, várias opções de segurança da ligação LDAPv3 ficam desactivadas.

Para alterar as opções de segurança de uma ligação LDAPv3:

  1. Abra as Preferências do Sistema e clique em Contas.

  2. Se o ícone de cadeado estiver fechado, abra-o clicando no mesmo e introduzindo um nome e palavra-passe de administrador.

  3. Clique em Opções e depois clique em Editar.

  4. Clique em “Abrir o Utilitário de Directório”.

  5. Se o ícone de cadeado estiver fechado, abra-o clicando no mesmo e introduzindo um nome e palavra-passe de administrador.

  6. Clique em “Política de pesquisa”.

  7. Clique em Autenticação e certifique-se de que o directório LDAPv3 que pretende está listado na política de pesquisa.

    Para mais informações sobre como adicionar o directório LDAPv3 à política de pesquisa de autenticação, consulte Como definir políticas de pesquisa personalizadas.

  8. Clique em Serviços.

  9. Na lista de serviços, seleccione LDAPv3 e clique no botão de edição (/).

  10. Se a lista de configurações do servidor estiver oculta, clique em “Mostrar opções”.

  11. Seleccione a configuração para o directório que pretende e clique em Editar.

  12. Clique em Segurança e altere qualquer uma das seguintes definições.

    Nota:  as definições de segurança aqui e no servidor LDAP correspondente são determinadas quando a ligação LDAP é configurada. As definições não são actualizadas quando as definições do servidor forem alteradas.

    Se qualquer uma das quatro últimas opções estiver seleccionada, mas desactivada, significa que o directório LDAP necessita delas. Se qualquer uma destas opções não estiver seleccionada e estiver desactivada, significa que não é suportada pelo directório LDAP.

    • Usar autenticação ao estabelecer ligação: determina se a ligação LDAPv3 se autentica a si própria com o directório LDAP, fornecendo o nome distinto e a palavra-passe especificados. Esta opção não está visível se a ligação LDAPv3 utilizar ligação fidedigna com o directório LDAP.

    • Ligar ao directório como: especifica as credenciais que a ligação LDAPv3 utiliza para estabelecer uma ligação fidedigna com o directório LDAP. Esta opção e as credenciais não podem ser alteradas aqui. Para tal, deverá desligar e voltar a ligar com credenciais diferentes.

      Para mais informações, consulte Como parar uma ligação fidedigna a um directório LDAP e Como configurar uma ligação fidedigna de um directório LDAP.

      Esta opção não está visível, a não ser que a ligação LDAPv3 utilize ligação fidedigna.

    • Desactivar palavras-passe legíveis: determina se a palavra-passe deve ser enviada de forma legível, caso não possa ser validada através de um método de autenticação que envie uma palavra-passe encriptada.

    • Assinar digitalmente todos os pacotes (requer Kerberos): certifica que os dados do directório do servidor LDAP não foram interceptados e modificados por outro computador durante a transferência para o seu computador.

    • Encriptar todos os pacotes (requer SSL ou Kerberos): exige que o servidor LDAP encripte os dados do directório utilizando SSL ou Kerberos antes de os enviar para o seu computador. Antes de seleccionar a opção “Encriptar todos os pacotes (SSL ou Kerberos necessário)”, pergunte ao administrador do Open Directory se é necessário SSL.

    • Bloquear ataques “man-in-the-middle” (requer Kerberos): protege contra um servidor fictício que se faça passar pelo servidor LDAP. É melhor se for utilizado com a opção “Assinar digitalmente todos os pacotes”.
