Como configurar o acesso a um domínio do Active Directory

Utilizando o conector do Active Directory listado no Utilitário de Directório, é possível configurar o Mac OS X para aceder a informações básicas relativas a contas de utilizador num domínio do Active Directory num servidor Windows.

O conector do Active Directory gera todos os atributos exigidos pelo Mac OS X para autenticação. Não são necessárias alterações ao esquema do Active Directory.

O conector do Active Directory detecta e acede a tipos de registos e atributos padrão do Mac OS X (tais como os atributos necessários para a gestão de clientes do Mac OS X), se o esquema do Active Directory tiver sido ampliado para os incluir.

AVISO:  com as opções avançadas do conector do Active Directory, é possível associar o ID de utilizador exclusivo (UID), o ID do grupo principal (GID) e o GID do grupo do Mac OS X aos atributos correctos que foram adicionados ao esquema do Active Directory. Se alterar a definição destas opções de mapeamento mais tarde, os utilizadores poderão perder o acesso a ficheiros anteriormente criados.
Importante:  se o nome do computador incluir um hífen, poderá não conseguir aceder ou ligar-se a um domínio de directórios como o LDAP ou Active Directory. Para estabelecer ligação, utilize um nome de computador sem hífen.
Para configurar o acesso a um domínio do Active Directory:

  1. Abra as Preferências do Sistema e clique em Contas.

  2. Se o ícone de cadeado estiver fechado, abra-o clicando no mesmo e introduzindo um nome e palavra-passe de administrador.

  3. Clique em Opções e depois clique em Aceder ou Editar.

  4. Clique em “Abrir o Utilitário de Directório”.

  5. Se o ícone de cadeado estiver fechado, abra-o clicando no mesmo e introduzindo um nome e palavra-passe de administrador.

  6. Clique em Serviços.

  7. Na lista de serviços, seleccione “Active Directory” e clique no botão de edição (/).

  8. Digite o nome DNS do domínio do Active Directory que pretende ligar ao computador que está a configurar.

    O administrador do domínio do Active Directory pode informá-lo do nome DNS que deve introduzir.

  9. Se necessário, edite o ID do computador.

    O ID do computador é o nome pelo qual o computador é conhecido no domínio do Active Directory, sendo, por predefinição, o nome do computador. Poderá alterá-lo de acordo com o esquema estabelecido na sua organização para atribuir nomes aos computadores no domínio do Active Directory. Se não tiver a certeza, consulte o administrador do domínio do Active Directory.

  10. (Opcional) Defina as opções avançadas.

    Se as opções avançadas estiverem ocultas, clique em “Mostrar opções avançadas” e especifique as opções nos painéis Utilizador, Mapeamentos e Administração. Também pode alterar as opções avançadas mais tarde.

    Para mais informações sobre opções avançadas, consulte:

    • Como configurar contas de utilizador móveis no Active Directory

    • Como configurar pastas pessoais para contas de utilizador do Active Directory

    • Como definir uma shell UNIX para contas de utilizador do Active Directory

    • Como associar o UID a um atributo do Active Directory

    • Como associar o ID do grupo principal a um atributo do Active Directory

    • Como associar o ID do grupo em contas de grupo a um atributo do Active Directory

    • Como especificar um servidor do Active Directory preferido

    • Como alterar os grupos do Active Directory que podem administrar o computador

    • Como controlar a autenticação a partir de todos os domínios na floresta do Active Directory

  11. Clique em Ligar, utilize os dados seguintes para se autenticar como utilizador com direitos para ligar um computador ao domínio do Active Directory, seleccione as políticas de pesquisa às quais pretende adicionar o Active Directory (ver em baixo) e clique em OK:

    • Nome de utilizador e palavra-passe: poderá autenticar-se introduzindo o nome e a palavra-passe da sua conta de utilizador do Active Directory, ou o administrador do domínio do Active Directory poderá ter de lhe fornecer um nome e palavra-passe.

    • OU do computador: introduza a unidade organizacional (OU) do computador que está a configurar.

    • Usar para autenticação: utilize para determinar se o Active Directory é adicionado à política de pesquisa de autenticação do computador.

    • Usar para contactos: utilize para determinar se o Active Directory é adicionado à política de pesquisa de contactos do computador.

    Quando clica em OK, o Utilitário de Directório configura uma ligação fidedigna entre o computador que está a configurar e o servidor do Active Directory. As políticas de pesquisa do computador são especificadas de acordo com as opções que seleccionou quando se autenticou, e o Active Directory é activado no painel Serviços do Utilitário de Directório.

    Com as predefinições das opções avançadas do Active Directory, a floresta do Active Directory é adicionada à política de pesquisa de autenticação e à política de pesquisa de contactos do computador, se tiver seleccionado “Usar para autenticação” ou “Usar para contactos”.

    Contudo, se anular a selecção de “Permitir autenticação de cada domínio na raiz” nas opções avançadas do painel Administração, antes de clicar em Ligar, o domínio do Active Directory mais próximo é adicionado em vez da floresta.

    Pode alterar as políticas de pesquisa mais tarde, adicionando ou removendo a floresta do Active Directory ou domínios individuais. Para mais informações, consulte Como definir políticas de pesquisa personalizadas.

  12. (Opcional) Agrupe o servidor com o realm Kerberos do Active Directory.

    • No servidor ou num computador administrador que se possa ligar ao servidor, abra o Administrador do Servidor e seleccione “Abrir directório”.

    • Clique em Definições e depois em Geral.

    • Clique em “ Aceder a Kerberos”, seleccione o Active Directory Kerberos no menu instantâneo e introduza as credenciais de um administrador local deste servidor.
