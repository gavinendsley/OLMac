À propos de l’accès Active Directory

Vous pouvez configurer Mac OS X pour accéder à des informations élémentaires de comptes d’utilisateurs dans un domaine Active Directory sur un serveur Windows 2000 ou de version ultérieure. Le connecteur Active Directory vous offre cette possibilité pour Utilitaire d’annuaire. Il figure dans la sous-fenêtre Services d’Utilitaire d’annuaire.

Il est inutile de modifier le schéma du domaine Active Directory pour obtenir des informations élémentaires de comptes d’utilisateurs. Vous pouvez modifier la liste de contrôle d’accès (ACL) par défaut de certains attributs de sorte que les propriétés d’utilisateur puissent être lues par des comptes d’ordinateur.

Le connecteur Active Directory génère tous les attributs requis pour l’authentification Mac OS X à partir d’attributs standard de comptes utilisateur Active Directory. Le connecteur prend également en charge les règles d’authentification Active Directory, qui comprennent les modifications de mots de passe, les délais d’expiration, les modifications forcées et les options de sécurité.

Mac OS X 10.6 prend en charge les options de chiffrement et de signature de paquets pour l’ensemble des domaines Active Directory de Windows. Cette fonctionnalité est réglée par défaut sur « autoriser » . Vous pouvez définir le réglage par défaut sur la position désactivée ou requise à l’aide de l’outil de ligne de commande dsconfigad. Les options de chiffrement et de signature de paquets garantissent la protection de toutes les données vers et depuis le domaine Active Directory pour les recherches d’enregistrements.

Le connecteur Active Directory génère de façon dynamique un identifiant d’utilisateur unique et un identifiant de groupe principal en fonction de l’identifiant global unique (GUID) du compte utilisateur dans le domaine Active Directory. L’identifiant d’utilisateur et celui du groupe principal qui sont générés sont identiques pour chaque compte utilisateur, même si le compte est utilisé pour ouvrir une session sur des ordinateurs Mac OS X différents.

Par ailleurs, vous pouvez forcer le connecteur Active Directory à mapper l’identifiant d’utilisateur sur les attributs Active Directory que vous spécifiez.

Le connecteur Active Directory génère un identifiant de groupe basé sur le GUID du compte de groupe Active Directory. Vous pouvez également forcer le module à mapper l’identifiant de groupe des comptes de groupe avec les attributs Active Directory de votre choix.

Quand un utilisateur ouvre une session sur Mac OS X avec un compte utilisateur Active Directory, le connecteur Active Directory peut monter le dossier de départ réseau Windows spécifié dans le compte utilisateur Active Directory comme dossier de départ Mac OS X de l’utilisateur. Vous pouvez choisir d’utiliser le dossier de départ réseau spécifié par l’attribut du répertoire de départ standard d’Active Directory ou par l’attribut du répertoire de départ de Mac OS X (si le schéma Active Directory a été étendu à cet effet).

Vous pouvez sinon configurer le module afin de créer un dossier de départ local sur le volume de démarrage de l’ordinateur client Mac OS X. Dans ce cas, le module monte également le dossier de départ réseau Windows de l’utilisateur (spécifié dans le compte utilisateur Active Directory) en tant que volume réseau, comme un point de partage. À l’aide du Finder, l’utilisateur peut alors copier des fichiers entre le volume réseau du dossier de départ Windows et le dossier de départ Mac OS X local.

Le connecteur Active Directory peut aussi créer des comptes mobiles pour les utilisateurs. Un compte mobile possède un dossier de départ local sur le volume de démarrage de l’ordinateur client Mac OS X. (L’utilisateur possède également un dossier de départ réseau défini dans le compte Active Directory de l’utilisateur).

Un compte mobile met en cache les informations d’authentification Active Directory de l’utilisateur sur l’ordinateur client Mac OS X. Les informations d’authentification mises en cache permettent à l’utilisateur d’ouvrir une session avec le nom et le mot de passe Active Directory lorsque l’ordinateur client est déconnecté du serveur Active Directory.

Un compte mobile possède un dossier de départ local sur le volume de démarrage de l’ordinateur client Mac OS X. (L’utilisateur possède également un dossier de départ réseau défini dans le compte Active Directory de l’utilisateur).

Si le schéma Active Directory a été étendu pour inclure des types d’enregistrements (classes d’objets) et des attributs Mac OS X, le connecteur Active Directory les détecte et y accède.

Par exemple, il est possible de modifier le schéma Active Directory à l’aide d’outils d’administration Windows afin d’inclure des attributs de clients gérés Mac OS X. Cette modification du schéma permet au connecteur Active Directory de prendre en charge les réglages du client géré qui ont été réalisés via l’application Gestionnaire de groupe de travail de Mac OS X Server.

Les clients Mac OS X ont un accès complet aux attributs ajoutés à l’annuaire. Par conséquent, il peut s’avérer nécessaire de modifier l’ACL de ces attributs pour permettre aux groupes d’ordinateurs de lire ces attributs ajoutés.

Le connecteur Active Directory détecte tous les domaines d’une forêt Active Directory. Vous pouvez configurer le module pour autoriser les utilisateurs de n’importe quel domaine de la forêt à s’authentifier auprès d’un ordinateur Mac OS X. Vous pouvez également n’autoriser que certains domaines à être authentifiés sur le client.

Le connecteur Active Directory prend en charge intégralement la réplique et le basculement Active Directory. Il détecte différents contrôleurs de domaine et identifie le plus proche d’entre eux. Si un contrôle de domaine devient indisponible, le module bascule sur un autre contrôleur de domaine à proximité.

Le connecteur Active Directory utilise le protocole LDAP pour accéder aux comptes utilisateur Active Directory et Kerberos pour les authentifier. Le connecteur Active Directory n’utilise pas l’interface propriétaire ADSI (Active Directory Services Interface) de Microsoft pour obtenir des services d’annuaire ou d’authentification.
