Configuration de dossiers de départ pour des comptes utilisateurs Active Directory

Sur un ordinateur configuré pour utiliser le connecteur Active Directory d’Utilitaire d’annuaire, vous pouvez activer ou désactiver des dossiers de départ réseau ou locaux pour des comptes utilisateur Active Directory.

Dans le cas de dossiers de départ réseau, le dossier de départ réseau Windows d’un utilisateur est monté en tant que dossier de départ Mac OS X chaque fois que l’utilisateur ouvre une session.

Vous pouvez déterminer si l’emplacement du dossier de départ réseau est défini par l’attribut de dossier de départ standard d’Active Directory ou par l’attribut de dossier de départ de Mac OS X, si le schéma Active Directory a été étendu à cet effet.

Dans le cas de dossiers de départ locaux, chaque utilisateur Active Directory qui ouvre une session possède un dossier de départ sur le disque de démarrage de Mac OS X. En outre, le dossier de départ réseau de l’utilisateur est monté en tant que volume réseau, comme pour un point de partage. L’utilisateur peut copier des fichiers entre ce volume réseau et le dossier de départ local.

Pour configurer des dossiers de départ pour des comptes utilisateurs Active Directory :

  1. Ouvrez les Préférences Système, puis cliquez sur Comptes.

  2. Si l’icône de verrouillage est activée, déverrouillez-la en cliquant dessus puis en saisissant le nom et le mot de passe d’un administrateur.

  3. Cliquez sur Options, puis cliquez sur Se connecter ou sur Modifier.

  4. Cliquez sur Utilitaire Open Directory.

  5. Si l’icône de verrouillage est activée, déverrouillez-la en cliquant dessus puis en saisissant le nom et le mot de passe d’un administrateur.

  6. Cliquez sur Services.

  7. Dans la liste de services, sélectionnez Active Directory, puis cliquez sur le bouton de modification (/).

  8. Si la liste des options avancées est masquée, cliquez sur "Afficher les options avancées".

  9. Cliquez sur Expérience utilisateur.

  10. Pour que les comptes utilisateur Active Directory possèdent des dossiers de départ locaux dans le dossier /Utilisateurs de l’ordinateur, cliquez sur l’option « Forcer l’utilisation du répertoire de départ local sur le disque de démarrage ».

    Cette option n’est pas disponible si "Créer un compte mobile lors de l’ouverture de session" est activée.

  11. Pour pouvoir utiliser l’attribut standard Active Directory pour l’emplacement du dossier de départ, sélectionnez « Utiliser le chemin UNC depuis Active Directory pour déduire l’emplacement du répertoire de départ réseau », puis choisissez un protocole parmi ceux présentés ci-dessous pour accéder au dossier de départ :

    • Pour utiliser le protocole SMB standard Windows, choisissez smb dans le menu local « Protocole réseau à utiliser ».

    • Pour utiliser le protocole AFP standard Macintosh, choisissez afp dans le menu local « Protocole réseau à utiliser ».

  12. Pour utiliser l’attribut Mac OS X définissant l’emplacement du dossier de départ, désélectionnez « Utiliser le chemin UNC depuis Active Directory pour déduire l’emplacement du répertoire de départ réseau ».

    Pour pouvoir utiliser l’attribut Mac OS X, le schéma Active Directory doit être étendu à cet effet.

  13. Cliquez sur OK.

Si vous modifiez le nom d’un compte utilisateur dans le domaine Active Directory, le serveur créera un dossier de départ (et des sous-dossiers) pour ce compte la fois suivante où il sera employé pour une ouverture de session sur un ordinateur Mac OS X. L’utilisateur peut toujours accéder à l’ancien dossier de départ et afficher son contenu dans le Finder.

Vous pouvez empêcher la création d’un dossier de départ en renommant l’ancien dossier avant que l’utilisateur ouvre une nouvelle session.
