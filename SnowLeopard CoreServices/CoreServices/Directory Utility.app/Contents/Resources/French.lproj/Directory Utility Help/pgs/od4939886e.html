Définition de réglages de fichiers de configuration BSD

Traditionnellement, les ordinateurs UNIX stockent des données d’administration dans des fichiers de configuration tels que /etc/master.passwd, /etc/group et /etc/hosts. Mac OS X est conçu d’après sur une version BSD d’UNIX mais obtient généralement les données d’administration auprès de systèmes d’annuaires.

Mac OS X Server prend en charge un ensemble défini de fichiers de configuration BSD. Il n’est pas possible de choisir les fichiers de configuration à utiliser, ni de mapper leur contenu avec des attributs et types d’enregistrements Mac OS X.

Depuis la version 10.2 de Mac OS X et de Mac OS X Server, Open Directory peut récupérer des données d’administration enregistrées dans des fichiers de configuration BSD. Cette fonctionnalité permet aux organisations possédant des fichiers de configuration d’utiliser des copies des fichiers existants sur des ordinateurs Mac OS X. Les fichiers de configuration BSD sont utilisables seuls ou avec d’autres domaines de répertoires.

Pour utiliser des fichiers de configuration BSD :

  1. Assurez-vous que les fichiers de configuration contiennent les données requises par les services d’annuaire Mac OS X.

    Pour en savoir plus, reportez-vous à la rubrique Configuration de données des fichiers de configuration BSD.

  2. Ouvrez les Préférences Système, puis cliquez sur Comptes.

  3. Si l’icône de verrouillage est activée, déverrouillez-la en cliquant dessus puis en saisissant le nom et le mot de passe d’un administrateur.

  4. Cliquez sur Options, puis cliquez sur Se connecter ou sur Modifier.

  5. Cliquez sur Utilitaire Open Directory.

  6. Si l’icône de verrouillage est activée, déverrouillez-la en cliquant dessus puis en saisissant le nom et le mot de passe d’un administrateur.

  7. Cliquez sur Services.

  8. Dans la liste de services, sélectionnez « Fichier plat BSD et NIS » puis cliquez sur le bouton de modification (/).

  9. Sélectionnez l’option “Utiliser les fichiers locaux BSD (/etc) pour l’authentification”, puis cliquez sur OK.

    Le domaine des fichiers de configuration BSD est ajouté à la règle de recherche d’authentification de l’ordinateur sous la forme /BSD/local.
