Modification des réglages de connexion pour un annuaire LDAP

Utilitaire d’annuaire vous permet de modifier les réglages de connexion de la configuration du mode d’accès à un annuaire LDAPv3 ou LDAPv2 pour l’ordinateur.

Pour modifier les réglages de connexion en vue de l’accès à un annuaire LDAP :

  1. Ouvrez les Préférences Système, puis cliquez sur Comptes.

  2. Si l’icône de verrouillage est activée, déverrouillez-la en cliquant dessus puis en saisissant le nom et le mot de passe d’un administrateur.

  3. Cliquez sur Options, puis sur Modifier.

  4. Cliquez sur Utilitaire Open Directory.

  5. Si l’icône de verrouillage est activée, déverrouillez-la en cliquant dessus puis en saisissant le nom et le mot de passe d’un administrateur.

  6. Cliquez sur Services.

  7. Dans la liste des services, sélectionnez LDAPv3 et cliquez sur le bouton de modification (/).

  8. Si la liste des configurations serveur est masquée, cliquez sur Afficher les options.

  9. Dans la liste, sélectionnez une configuration serveur, puis cliquez sur Édition.

  10. Cliquez sur Connexion et modifiez les réglages suivants à votre guise :

    • Nom de la configuration  :   identifie cette configuration dans la liste des configurations d’annuaire LDAP. (Vous pouvez également changer ce nom dans la liste des configurations d’annuaire LDAP.)

    • Nom du serveur ou adresse IP  :   indique le nom DNS ou l’adresse IP du serveur. (Vous pouvez également le changer dans la liste des configurations d’annuaire LDAP.)

    • L’ouverture/fermeture expire après  :   indique la durée maximale avant l’annulation d’une tentative de connexion.

    • La requête expire après  :   indique la durée maximale avant l’annulation d’une requête.

    • Nouvelle tentative de liaison après  :   indique le délai d’attente, en secondes, avant une nouvelle tentative de connexion si le serveur LDAP ne répond pas. Afin d’éviter des tentatives de reconnexion continuelles, indiquez une valeur supérieure.

    • La connexion expire après  :   indique la durée du maintien, en minutes, d’une connexion inactive ou sans réponse.

    • Chiffrer via SSL  :   détermine si les communications avec l’annuaire LDAP doivent être chiffrées à l’aide d’une connexion SSL. (Vous pouvez également changer ce réglage dans la liste des configurations d’annuaire LDAP.) Avant de cocher la case SSL, demandez à votre administrateur Open Directory si cette option est nécessaire.

    • Utiliser le port personnalisé  :   indique un numéro de port différent du port standard pour les connexions LDAP (389 sans SSL, 636 avec SSL).

    • Ignorer les références du serveur  :   détermine si les références du serveur LDAP permettant de rechercher des informations sur d’autres serveurs ou répliques LDAP doivent être ignorées ou suivies. Les références de serveur peuvent être utiles pour trouver des informations, mais elles peuvent également retarder l’ouverture de session ou entraîner d’autres retards si l’ordinateur doit vérifier des références à d’autres serveurs LDAP.

    • Utiliser LDAPv2 (lecture seule) :  détermine si l’ancien protocole LDAPv2 doit être utilisé pour accéder en lecture seule à un annuaire LDAP.
