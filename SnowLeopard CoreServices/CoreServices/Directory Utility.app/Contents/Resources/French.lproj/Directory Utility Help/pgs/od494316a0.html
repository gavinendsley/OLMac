Configuration des recherches et mappages LDAP

Utilitaire d’annuaire vous permet de modifier les mappages, bases et étendues de recherche qui indiquent la méthode utilisée par Mac OS X pour rechercher des éléments de données spécifiques dans un annuaire LDAP. Vous pouvez modifier ces réglages séparément pour chaque configuration d’annuaire LDAP figurant dans la liste d’Utilitaire d’annuaire. Chaque configuration d’annuaire LDAP spécifie le mode d’accès aux données d’annuaire LDAPv3 ou LDAPv2 pour Mac OS X.

Vous pouvez modifier les éléments suivants :

  • Le mappage de chaque type d’enregistrement Mac OS X, avec des classes d’objets LDAP

  • Le mappage de types de données (ou attributs) Mac OS X, avec des attributs LDAP pour chaque type d’enregistrement

  • La base et l’étendue de recherche LDAP qui détermine l’endroit où Mac OS X recherche un type d’enregistrement Mac OS X dans un annuaire LDAP

Lors du mappage d’attributs utilisateur Mac OS X sur un domaine de répertoire LDAP en lecture et écriture (à savoir un domaine LDAP qui n’est pas en lecture seule), l’attribut LDAP mappé sur RealName ne doit pas être identique au premier attribut d’une liste d’attributs LDAP mappés sur RecordName.

Par exemple, l’attribut cn ne doit pas être le premier attribut mappé sur RecordName si cn est également mappé sur RealName.

Si l’attribut LDAP mappé sur RealName est identique au premier attribut mappé sur RecordName, cela occasionnera des problèmes lorsque vous essayerez de modifier le nom complet (long) ou le premier nom abrégé dans Gestionnaire de groupe de travail.

Pour en savoir plus sur les types d’enregistrement et attributs Mac OS X, consultez Administration Open Directory (disponible à l’adresse www.apple.com/fr/server/resources/).

Pour modifier les bases de recherche et les mappages d’un serveur LDAP :

  1. Ouvrez les Préférences Système, puis cliquez sur Comptes.

  2. Si l’icône de verrouillage est activée, déverrouillez-la en cliquant dessus puis en saisissant le nom et le mot de passe d’un administrateur.

  3. Cliquez sur Options, puis sur Modifier.

  4. Cliquez sur Utilitaire Open Directory.

  5. Si l’icône de verrouillage est activée, déverrouillez-la en cliquant dessus puis en saisissant le nom et le mot de passe d’un administrateur.

  6. Cliquez sur Services.

  7. Dans la liste des services, sélectionnez LDAPv3 et cliquez sur le bouton de modification (/).

  8. Si la liste des configurations serveur est masquée, cliquez sur Afficher les options.

  9. Dans la liste, sélectionnez une configuration serveur, puis cliquez sur Édition.

  10. Cliquez sur Recherche et mappages.

  11. Sélectionnez les mappages que vous souhaitez utiliser comme point de départ ; ou bien choisissez Personnalisé pour commencer sans aucun mappage prédéfini.

    Si vous choisissez un des modèles de mappage LDAP, une extension de base de recherche s’affiche : vous pouvez soit la modifier soit accepter cette extension de base de recherche par défaut en cliquant sur OK.

    Cliquez sur le menu local « Accéder à ce serveur LDAPv3 via » et sélectionnez un modèle pour utiliser ses mappages comme point de départ.

  12. Ajoutez des types d’enregistrement et modifiez leurs bases de recherche comme il convient :

    • Pour ajouter des types d’enregistrement, cliquez sur Ajouter (sous la liste « Types d’enregistrement et attributs »), puis sur la feuille qui apparaît. Sélectionnez Types d’enregistrement, sélectionnez-en un dans la liste et cliquez sur OK.

    • Pour modifier la base et l’étendue de recherche d’un type d’enregistrement, sélectionnez celui-ci dans la liste « Types d’enregistrement et attributs », puis modifiez le champ Base de recherche. Sélectionnez « tous les sous-arbres » pour définir l’étendue de recherche de manière à inclure la hiérarchie de l’annuaire LDAP en commençant par la base de recherche, ou bien sélectionnez « le premier niveau » pour inclure uniquement la base de recherche et un niveau en dessous dans la hiérarchie de l’annuaire LDAP.

    • Pour supprimer un type d’enregistrement, sélectionnez-le dans la liste "Types d’enregistrement et attributs" puis cliquez sur Supprimer.

    • Pour ajouter un mappage à un type d’enregistrement, sélectionnez celui-ci dans la liste « Types d’enregistrement et attributs », puis cliquez sur Ajouter (en dessous de « Mapper sur __ éléments listés ») et tapez le nom d’une classe d’objets de l’annuaire LDAP.

      Pour ajouter une autre catégorie d’objet LDAP, appuyez sur la touche Retour et tapez son nom, puis indiquez les catégories qui seront utilisées à l’aide du menu local situé au-dessus de la liste.

    • Pour changer le mappage d’un type d’enregistrement, sélectionnez celui-ci dans la liste « Types d’enregistrement et attributs », puis double-cliquez sur la catégorie d’objets LDAP à modifier dans « Mapper sur __ éléments listés ») et modifiez-la. Spécifiez les catégories d’objet LDAP à utiliser à l’aide du menu local situé au-dessus de la liste.

    • Pour supprimer le mappage d’un type d’enregistrement, sélectionnez celui-ci dans la liste « Types d’enregistrement et attributs », sélectionnez la catégorie d’objet LDAP à supprimer dans « Mapper sur __ éléments listés », puis cliquez sur Supprimer (en dessous de « Mapper sur __ éléments listés »).

  13. Ajoutez des attributs et modifiez leurs mappages comme il convient :

    • Pour ajouter des attributs à un type d’enregistrement, sélectionnez celui-ci dans la liste « Types d’enregistrement et attributs » et cliquez sur Ajouter (en dessous de la liste « Types d’enregistrement et attributs ») ; ensuite, sur la feuille qui apparaît, cliquez sur Types d’attributs, puis sélectionnez les types d’attributs et cliquez sur OK.

    • Pour ajouter un mappage à un type d’enregistrement, sélectionnez celui-ci dans la liste « Types d’enregistrement et attributs », puis cliquez sur Ajouter (en dessous de « Mapper sur __ éléments listés ») et saisissez le nom d’un attribut de l’annuaire LDAP. Pour ajouter un autre attribut LDAP, cliquez sur Retour puis saisissez son nom.

    • Pour modifier le mappage d’un attribut, sélectionnez celui-ci dans la liste « Types d’enregistrement et attributs », double-cliquez sur l’élément à modifier dans « Mapper sur __ éléments listés ») et modifiez ensuite le nom de l’élément.

    • Pour supprimer le mappage d’un attribut, sélectionnez celui-ci dans la liste « Types d’enregistrement et attributs », sélectionnez l’élément à supprimer dans « Mapper sur __ éléments listés », puis cliquez sur Supprimer (en dessous de « Mapper sur __ éléments listés »).

    • Pour changer l’ordre des attributs qui apparaissent sur la liste de droite, faites-les glisser vers le haut ou vers le bas.

  14. Pour enregistrer vos mappages en tant que modèle, cliquez sur Enregistrer le modèle.

    Les modèles enregistrés à l’emplacement par défaut seront énumérés dans des menus locaux de modèles de mappage LDAP à la prochaine ouverture d’Utilitaire d’annuaire. L’emplacement par défaut des modèles enregistrés correspond au chemin suivant de votre dossier de départ :

    ~/Bibliothèque/Application Support/Directory Access/LDAPv3/Templates

  15. Pour stocker les mappages dans l’annuaire LDAP afin qu’il puisse les fournir automatiquement à ses clients, cliquez sur "Écrire sur le serveur" puis saisissez une base de recherche en vue du stockage, un nom d’administrateur ou autre utilisateur doté des droits en écriture pour la base de recherche (par exemple, uid=diradmin,cn=utilisateurs,dc=ods,dc=exemple,dc=com) et enfin un mot de passe.

    Si vous écrivez des mappages sur un serveur LDAP Open Directory, la base de recherche correcte est « cn=config,extension (où extension représente l’extension de la base de recherche, comme « dc=ods,dc=example,dc=com »).

L’annuaire LDAP fournit ses mappages aux clients Mac OS X dont la règle de recherche personnalisée inclut une connexion configurée pour recevoir les mappages du serveur LDAP.

L’annuaire LDAP fournit également ses mappages à tous les clients Mac OS X dotés d’une règle de recherche automatique. Pour en savoir plus, consultez Configuration de l’accès à un annuaire LDAP et Utilisation des réglages avancés de règles de recherche.
