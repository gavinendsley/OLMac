Modification du règlement de sécurité pour une connexion LDAP

Utilitaire d’annuaire vous permet de configurer un règlement de sécurité plus strict pour une connexion LDAPv3 par rapport à celui de l’annuaire LDAP. Par exemple, si le règlement de sécurité de l’annuaire LDAP autorise les mots de passe en clair, vous pouvez définir une connexion LDAPv3 qui ne permette pas ce type de mot de passe.

La définition d’un règlement de sécurité plus strict permet de protéger votre ordinateur contre des pirates informatiques qui tenteraient d’utiliser un serveur LDAP malveillant pour prendre le contrôle de votre ordinateur.

L’ordinateur doit communiquer avec le serveur LDAP pour afficher l’état des options de sécurité. Par conséquent, lorsque vous modifiez les options de sécurité d’une connexion LDAPv3, la règle de recherche d’authentification de l’ordinateur doit inclure la connexion LDAPv3.

Les réglages acceptables des options de sécurité d’une connexion LDAPv3 sont soumis aux conditions requises et fonctionnalités de sécurité du serveur LDAP. Par exemple, si le serveur LDAP ne prend pas en charge l’authentification Kerberos, plusieurs options de sécurité propres à la connexion LDAPv3 sont désactivées.

Pour modifier les options de sécurité d’une connexion LDAPv3 :

  1. Ouvrez les Préférences Système, puis cliquez sur Comptes.

  2. Si l’icône de verrouillage est activée, déverrouillez-la en cliquant dessus puis en saisissant le nom et le mot de passe d’un administrateur.

  3. Cliquez sur Options, puis sur Modifier.

  4. Cliquez sur Utilitaire Open Directory.

  5. Si l’icône de verrouillage est activée, déverrouillez-la en cliquant dessus puis en saisissant le nom et le mot de passe d’un administrateur.

  6. Cliquez sur Règles de recherche.

  7. Cliquez sur Authentification et vérifiez que l’annuaire LDAPv3 de votre choix figure sur la liste des règles de recherche.

    Pour en savoir plus sur l’ajout d’annuaire LDAPv3 à la règle de recherche d’authentification, consultez la rubrique Définition de règles de recherche personnalisées.

  8. Cliquez sur Services.

  9. Dans la liste des services, sélectionnez LDAPv3 et cliquez sur le bouton de modification (/).

  10. Si la liste des configurations serveur est masquée, cliquez sur Afficher les options.

  11. Sélectionnez la configuration d’annuaire souhaitée, puis cliquez sur Modifier.

  12. Cliquez sur Sécurité et modifiez les réglages suivants à votre guise.

    Remarque  :   les réglages de sécurité affichés ici et sur le serveur LDAP correspondant sont déterminés lorsque la connexion LDAP est configurée. Ces réglages ne sont pas mis à jour lorsque les réglages du serveur sont modifiés.

    Si une ou plusieurs des quatre dernières options est sélectionnée mais désactivée, elle est requise par l’annuaire LDAP. Si ces options ne sont pas sélectionnées et sont également désactivées, cela signifie qu’elles ne sont pas prises en charge par le serveur LDAP.

    • Utiliser l’authentification lors de la connexion  :   détermine si la connexion LDAPv3 s’authentifie auprès de l’annuaire LDAP en fournissant le nom et le mot de passe spécifiés. Cette option n’est pas visible si la connexion LDAPv3 utilise la liaison fiable avec l’annuaire LDAP.

    • Relié à l’annuaire en tant que  :   spécifie les informations d’authentification utilisées par la connexion LDAPv3 pour la liaison sécurisée avec l’annuaire LDAP. Il n’est pas possible de modifier cette option et les références à cet endroit. Vous avez toutefois la possibilité de rompre la liaison et d’en créer une nouvelle avec d’autres références.

      Pour en savoir plus, consultez Configuration d’une liaison sécurisée pour un annuaire LDAP et Configuration d’une liaison sécurisée pour un annuaire LDAP.

      Cette option n’est visible que si la connexion LDAPv3 utilise la liaison fiable.

    • Désactiver les mots de passe en clair  :   détermine si le mot de passe doit être envoyé en clair au cas où il est impossible de le valider à l’aide d’une méthode d’authentification qui envoie un mot de passe chiffré.

    • Signer tous les paquets numériquement (requiert Kerberos)  :   certifie que les données de répertoire provenant du serveur LDAP n’ont pas été interceptées et modifiées par un autre ordinateur pendant le trajet vers votre ordinateur.

    • Chiffrer tous les paquets (requiert SSL ou Kerberos)  :   demande au serveur LDAP de chiffrer les données de répertoire au moyen de SSL ou Kerberos avant de les envoyer à votre ordinateur. Avant de cocher la case « Chiffrer tous les paquets (requiert SSL ou Kerberos) », demandez à votre administrateur Open Directory si SSL est nécessaire.

    • Bloquer les attaques Man-in-the-Middle (requiert Kerberos)  :   protège contre un serveur malveillant se faisant passer pour un serveur LDAP. À utiliser de préférence avec l’option « Signer tous les paquets numériquement ».
