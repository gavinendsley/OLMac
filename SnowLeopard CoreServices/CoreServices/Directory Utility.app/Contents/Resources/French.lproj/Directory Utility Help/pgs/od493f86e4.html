Configuration de l’accès à un domaine Active Directory

Le connecteur Active Directory répertorié par Utilitaire d’annuaire permet de configurer Mac OS X pour accéder à des informations élémentaires de compte utilisateur dans un domaine Active Directory sur un serveur Windows.

Le connecteur Active Directory génère tous les attributs requis pour l’authentification Mac OS X. Aucune modification du schéma Active Directory n’est requise.

Si le schéma Active Directory a été étendu à cet effet, le connecteur Active Directory détecte les types et attributs d’enregistrements Mac OS X standard (tels que les attributs requis pour la gestion des clients Mac OS X) et y accède.

ATTENTION :  grâce aux options avancées du connecteur Active Directory, vous pouvez mapper sur l’identifiant d’utilisateur unique (UID) Mac OS X, l’identifiant de groupe principal (GID), et l’attribut GID de groupe sur les attributs appropriés qui ont été ajoutés au schéma Active Directory. Si vous modifiez le réglage de ces options de mappage ultérieurement, il se peut que les utilisateurs perdent l’accès aux fichiers déjà créés.
Important :  si le nom de votre ordinateur contient un trait d’union, cela peut vous empêcher de joindre ou de créer un lien avec un domaine de répertoire tel que LDAP ou Active Directory. Pour en établir une, utilisez un nom d’ordinateur ne comportant pas de trait d’union.
Pour configurer l’accès à un domaine Active Directory :

  1. Ouvrez les Préférences Système, puis cliquez sur Comptes.

  2. Si l’icône de verrouillage est activée, déverrouillez-la en cliquant dessus puis en saisissant le nom et le mot de passe d’un administrateur.

  3. Cliquez sur Options, puis cliquez sur Se connecter ou sur Modifier.

  4. Cliquez sur Utilitaire Open Directory.

  5. Si l’icône de verrouillage est activée, déverrouillez-la en cliquant dessus puis en saisissant le nom et le mot de passe d’un administrateur.

  6. Cliquez sur Services.

  7. Dans la liste de services, sélectionnez Active Directory, puis cliquez sur le bouton de modification (/).

  8. Saisissez le nom DNS du domaine Active Directory que vous souhaiter associer à l’ordinateur en cours de configuration.

    Prenez contact avec l’administrateur du domaine Active Directory pour connaître le nom DNS à saisir.

  9. Si nécessaire, modifiez l’identifiant de l’ordinateur.

    L’identifiant de l’ordinateur est le nom sous lequel il est enregistré dans le domaine Active Directory ; par défaut, il s’agit du nom de l’ordinateur. Vous pouvez le modifier conformément aux règles d’attribution de noms d’ordinateurs en vigueur dans votre entreprise pour le domaine Active Directory. En cas de doute, prenez contact avec l’administrateur du domaine Active Directory.

  10. (Facultatif) définissez les options avancées.

    Si les options avancées sont masquées, cliquez sur "Afficher les options avancées" dans les sous-fenêtres Expérience utilisateur, Mappages et Administratif. Vous pouvez également modifier les réglages d’options avancées ultérieurement.

    Pour plus d’informations sur les options avancées, consultez les rubriques suivantes :

    • Configuration de comptes utilisateurs mobiles dans Active Directory

    • Configuration de dossiers de départ pour des comptes utilisateurs Active Directory

    • Configuration d’un shell UNIX pour des comptes utilisateur Active Directory

    • Mappage de l’UID avec un attribut Active Directory

    • Mappage de l’identifiant du groupe principal avec un attribut Active Directory

    • Mappage de l’identifiant de groupe dans les comptes de groupe avec un attribut Active Directory

    • Choix d’un serveur Active Directory préféré

    • Modification des groupes Active Directory autorisés à administrer l’ordinateur

    • Contrôle de l’authentification à partir de tous les domaines d’une forêt Active Directory

  11. Cliquez sur Liaison, utilisez ce qui suit pour vous authentifier en tant qu’utilisateur autorisé à établir une liaison avec le domaine Active Directory, sélectionnez les règles de recherche auxquelles vous souhaiter ajouter Active Directory (voir ci-après), puis cliquez sur OK :

    • Nom d’utilisateur et Mot de passe  :   vous devriez pouvoir vous authentifier en saisissant le nom et le mot de passe de votre compte utilisateur Active Directory. Sinon, demandez à l’administrateur du domaine Active Directory de vous fournir un nom et un mot de passe.

    • Clé OU ordinateur  :   saisissez l’unité organisationnelle (Organizational unit, OU) de l’ordinateur que vous configurez.

    • Utiliser pour l’authentification  :   cette option permet de définir si Active Directory doit être ajouté aux règles de recherche d’authentification de l’ordinateur.

    • Utiliser pour les contacts  :   cette option permet de définir si Active Directory doit être ajouté aux règles de recherche des contacts de l’ordinateur.

    Quand vous cliquez sur OK, Utilitaire d’annuaire configure une liaison fiable entre l’ordinateur que vous configurez et le serveur Active Directory. Les règles de recherche de l’ordinateur sont définies selon les options sélectionnées lors de l’authentification et Active Directory est activé dans la sous-fenêtre Services d’Utilitaire d’annuaire.

    Avec les réglages par défaut des options avancées d’Active Directory, la forêt Active Directory est ajoutée aux règles de recherche d’authentification et de contacts de l’ordinateur si vous avez sélectionné « Utiliser pour l’authentification » ou « Utiliser pour les contacts ».

    Cependant, si vous désélectionnez « Autoriser l’authentification depuis n’importe quel domaine de la forêt » dans la sous-fenêtre des options avancées Administratif avant de cliquer sur Liaison, c’est le domaine Active Directory le plus proche (et non pas la forêt) qui est ajouté.

    Vous pouvez modifier les règles de recherche ultérieurement en ajoutant ou en supprimant la forêt ou des domaines individuels Active Directory. Pour plus d’informations, reportez-vous à la rubrique Définition de règles de recherche personnalisées.

  12. (Facultatif) Connectez le serveur au domaine Kerberos Active Directory :

    • Sur le serveur ou un ordinateur administrateur pouvant se connecter au serveur, ouvrez Admin Serveur, puis sélectionnez Open Directory pour ce serveur.

    • Cliquez sur Réglages, puis sur Général.

    • Cliquez sur Joindre Kerberos, choisissez le domaine Kerberos Active Directory dans le menu local et saisissez les informations d’authentification d’un administrateur local de ce serveur.
