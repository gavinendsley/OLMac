Modification du mappage RFC 2307 pour permettre la création d’utilisateurs

Afin de pouvoir utiliser Gestionnaire de groupe de travail pour créer des utilisateurs sur un serveur d’annuaires LDAP non-Apple qui utilise des mappages RFC 2307 (UNIX), vous devez modifier le mappage du type d’enregistrement d’Utilisateurs. Pour ce faire, utilisez Utilitaire d’annuaire.

Pour activer la création d’enregistrements d’utilisateur dans un annuaire LDAP avec mappages RFC 2307 :

  1. Ouvrez les Préférences Système, puis cliquez sur Comptes.

  2. Si l’icône de verrouillage est activée, déverrouillez-la en cliquant dessus puis en saisissant le nom et le mot de passe d’un administrateur.

  3. Cliquez sur Options, puis sur Modifier.

  4. Cliquez sur Utilitaire Open Directory.

  5. Si l’icône de verrouillage est activée, déverrouillez-la en cliquant dessus puis en saisissant le nom et le mot de passe d’un administrateur.

  6. Cliquez sur Services.

  7. Dans la liste des services, sélectionnez LDAPv3 et cliquez sur le bouton de modification (/).

  8. Si la liste des configurations serveur est masquée, cliquez sur Afficher les options.

  9. Sélectionnez la configuration d’annuaire avec mappages RFC 2307, puis cliquez sur Édition.

  10. Cliquez sur Recherche et mappages.

  11. Dans la liste de gauche, sélectionnez Utilisateurs.

    Par défaut, l’option "Mapper sur __ éléments listés" est réglée sur "n’importe lequel des" et la liste de droite contient posixAccount, inetOrgPerson et shadowAccount.

  12. Remplacez l’option « Mapper sur __ éléments listés » par « tous les », puis modifiez la liste située à droite pour qu’elle comprenne l’ensemble des classes d’objets LDAP que vous souhaitez mapper avec le type d’enregistrements Utilisateurs.

Vous pouvez par exemple supprimer shadowAccount de la liste afin que les utilisateurs soient mappés avec posixAccount et inetOrgPerson uniquement. Vous pouvez aussi mapper Utilisateurs avec account, posixAccount et shadowAccount.

  • Pour changer un élément dans la liste, double-cliquez dessus.

  • Pour ajouter un élément dans la liste, cliquez sur Ajouter.

  • Pour supprimer un élément de la liste, cliquez sur Supprimer.

  • Pour modifier l’ordre des éléments, faites-les glisser dans la liste, vers le haut ou vers le bas.

Vous pouvez rechercher les classes d’objets d’enregistrements d’utilisateurs dans l’annuaire LDAP à l’aide de l’outil UNIX ldapsearch dans Terminal. Par exemple, le code suivant recherche les classes d’objet d’enregistrement d’utilisateur dont l’attribut cn est Léonard de Vinci :

$ ldapsearch -x -h ldapserver.exemple.com -b “dc=exemple, dc=com” ’cn=Léonard de Vinci’ objectClass

Le résultat affiché pour cet exemple serait le suivant :

# Léonard de Vinci, exemple.com
dn: cn=Léonard de Vinci, dc=exemple, dc=com
objectClass: inetOrgPerson
objectClass: posixAccount
