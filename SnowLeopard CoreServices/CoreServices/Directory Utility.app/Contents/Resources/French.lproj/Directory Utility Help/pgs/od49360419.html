Configuration manuelle de l’accès à un annuaire LDAP

Vous avez la possibilité de créer manuellement une configuration déterminant le mode d’accès à un annuaire LDAPv3 ou LDAPv2 pour Mac OS X . Pour cela, vous devez connaître le nom DNS ou l’adresse IP du serveur d’annuaire LDAP.

Si le répertoire n’est pas hébergé par Mac OS X Server, vous devez connaître la base de recherche et le modèle de mappage des données Mac OS X sur celles du répertoire. Voici les modèles de mappage pris en charge :

  • Du serveur, pour un répertoire qui fournit ses propres mappages et sa propre base de recherche, comme Mac OS X Server

  • Serveur Open Directory, pour un répertoire utilisant le schéma Mac OS X Server

  • Active Directory, pour un répertoire hébergé par un serveur Windows 2000, Windows 2003, ou ultérieur

  • RFC 2307, pour la plupart des répertoires hébergés par des serveurs UNIX

  • Personnalisé, pour les répertoires qui n’utilisent aucun des mappages ci-dessus

Le module LDAPv3 prend en charge intégralement la réplique et le basculement Open Directory. En cas d’indisponibilité de l’objet principal Open Directory, le module se replie sur une réplique voisine.

Important  :   si le nom de votre ordinateur contient un trait d’union, cela peut vous empêcher de vous connecter à un domaine de répertoire tel que LDAP ou Active Directory ou de créer un lien avec de tels domaines. Pour en établir une, utilisez un nom d’ordinateur ne comportant pas de trait d’union.
Pour configurer manuellement l’accès à un annuaire LDAP :

  1. Ouvrez les Préférences Système, puis cliquez sur Comptes.

  2. Si l’icône de verrouillage est activée, déverrouillez-la en cliquant dessus puis en saisissant le nom et le mot de passe d’un administrateur.

  3. Cliquez sur Options, puis cliquez sur Se connecter ou sur Modifier.

  4. Cliquez sur Utilitaire Open Directory.

  5. Si l’icône de verrouillage est activée, déverrouillez-la en cliquant dessus puis en saisissant le nom et le mot de passe d’un administrateur.

  6. Cliquez sur Services.

  7. Dans la liste des services, sélectionnez LDAPv3 et cliquez sur le bouton de modification (/).

    Vous pouvez sélectionner LDAPv3 dans la liste des services sans cocher la case Activer pour LDAPv3.

  8. Cliquez sur Nouveau, puis sur Manuel.

  9. Saisissez un nom pour la configuration.

  10. Appuyez sur la touche de tabulation et saisissez le nom DNS ou l’adresse IP du serveur qui héberge l’annuaire LDAP auquel vous désirez accéder.

  11. À côté du nom DNS ou de l’adresse IP, cliquez sur le menu local et choisissez un modèle ou une méthode de mappage :

    • Si vous avez choisi le modèle Du serveur, il n’est pas nécessaire d’indiquer une extension de base de recherche. Dans ce cas, Open Directory considère que l’extension de la base de recherche correspond au premier niveau de l’annuaire LDAP.

    • Si vous avez choisi un modèle, saisissez l’extension de la base de recherche pour l’annuaire LDAP, puis cliquez sur OK. Vous devez saisir une extension pour la base de recherche sinon l’ordinateur ne peut pas rechercher d’informations dans l’annuaire LDAP.

      En général, l’extension de la base de recherche provient du nom DNS du serveur. Par exemple, l’extension de la base de recherche pourrait être « dc=ods,dc=exemple,dc=com » pour un serveur dont le nom DNS est ods.exemple.com.

    • Si vous avez choisi le modèle Personnalisé, vous devez configurer des mappages entre les types et attributs d’enregistrements Mac OS X et les classes et attributs de l’annuaire LDAP auquel vous vous connectez. Pour en savoir plus, consultez la rubrique Configuration des recherches et mappages LDAP.

  12. Avant d’activer la case « Chiffrer via SSL », vérifiez avec votre administrateur Open Directory si le protocole SSL est nécessaire.

  13. Pour modifier les réglages suivants de cette configuration LDAP, cliquez sur Modifier afin d’afficher les options pour la configuration LDAP sélectionnée, effectuez vos modifications, puis cliquez sur OK lorsque vous avez terminé.

    • Cliquez sur Connexion pour définir les options de délai, spécifier un port personnalisé, ignorer les références du serveur ou forcer l’utilisation du protocole LDAPv2 (en lecture seule). Pour en savoir plus, consultez la rubrique Modification des réglages de connexion pour un annuaire LDAP.

    • Cliquez sur Recherche et mappages pour configurer des options de recherche et de mappage pour un serveur LDAP. Pour en savoir plus, consultez Configuration d’une liaison sécurisée pour un annuaire LDAP.

    • Cliquez sur Sécurité pour configurer une connexion authentifiée (au lieu d’une liaison fiable) et d’autres options de règlement de sécurité. Pour en savoir plus, consultez la rubrique Modification du règlement de sécurité pour une connexion LDAP.

    • Cliquez sur Liaison pour configurer des liaisons fiables (si l’annuaire LDAP les prend en charge). Pour en savoir plus, consultez Configuration d’une liaison sécurisée pour un annuaire LDAP.

  14. Cliquez sur OK pour terminer la création manuelle de la configuration d’accès à un annuaire LDAP.

  15. Si vous souhaitez que l’ordinateur accède à l’annuaire LDAP pour lequel vous avez créé une configuration, ajoutez le répertoire à une règle de recherche personnalisée dans les sous-fenêtres Authentification et Contacts de la règle de recherche dans Utilitaire d’annuaire, puis vérifiez que LDAPv3 est bien activé dans la sous-fenêtre Services.

Pour en savoir plus, consultez la rubrique Activation ou désactivation des services d’annuaire LDAP et Définition de règles de recherche personnalisées.

Remarque  :   pour pouvoir utiliser Gestionnaire de groupe de travail afin de créer des utilisateurs sur un serveur LDAP non Apple utilisant les mappages RFC 2307 (UNIX), vous devez modifier le mappage du type d’enregistrement Utilisateurs. Pour en savoir plus, consultez la rubrique Modification du mappage RFC 2307 pour permettre la création d’utilisateurs.
Important  :   si vous modifiez l’adresse IP et le nom de votre ordinateur à l’aide de la commande changeip alors que vous êtes connecté à un serveur d’annuaire, vous devez vous déconnecter et vous reconnecter à ce serveur d’annuaire pour mettre à jour le répertoire avec le nouveau nom et la nouvelle adresse IP de l’ordinateur. Si vous ne procédez pas ainsi, l’annuaire n’est pas mis à jour et il continue d’utiliser l’ancien nom et l’ancienne adresse IP de l’ordinateur.
