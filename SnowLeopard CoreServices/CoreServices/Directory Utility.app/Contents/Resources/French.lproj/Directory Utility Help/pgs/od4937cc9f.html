Configuration de l’accès à un annuaire LDAP

Utilitaire d’annuaire vous permet de spécifier le mode d’accès de Mac OS X à un annuaire LDAPv3 si vous connaissez le nom DNS ou l’adresse IP du serveur d’annuaire LDAP.

Si le répertoire n’est pas hébergé par un serveur assurant son propre mappage (par ex. Mac OS X Server), vous devez connaître la base de recherche et le modèle de mappage des données Mac OS X sur celles du répertoire.

Voici les modèles de mappage pris en charge :

  • Serveur Open Directory, pour un répertoire utilisant le schéma Mac OS X Server

  • Active Directory, pour un répertoire hébergé par un serveur Windows 2000, Windows 2003 ou ultérieur

  • RFC 2307, pour la plupart des répertoires hébergés par des serveurs UNIX

Le module LDAPv3 prend en charge intégralement la réplique et le basculement Open Directory. En cas d’indisponibilité de l’objet principal Open Directory, le module se replie sur une réplique voisine.

Pour spécifier des mappages personnalisés pour les données d’annuaire, suivez les instructions de la rubrique Configuration manuelle de l’accès à un annuaire LDAP à la place de celles indiquées ici.

Important  :   si le nom de votre ordinateur contient un trait d’union, cela peut vous empêcher de vous connecter à un domaine de répertoire tel que LDAP ou Active Directory ou de créer un lien avec de tels domaines. Pour en établir une, utilisez un nom d’ordinateur ne comportant pas de trait d’union.
Pour qu’Utilitaire d’annuaire vous aide à configurer l’accès à un annuaire LDAP :

  1. Ouvrez les Préférences Système, puis cliquez sur Comptes.

  2. Si l’icône de verrouillage est activée, déverrouillez-la en cliquant dessus puis en saisissant le nom et le mot de passe d’un administrateur.

  3. Cliquez sur Options, puis cliquez sur Se connecter ou sur Modifier.

  4. Cliquez sur Utilitaire Open Directory.

  5. Si l’icône de verrouillage est activée, déverrouillez-la en cliquant dessus puis en saisissant le nom et le mot de passe d’un administrateur.

  6. Cliquez sur Services.

  7. Dans la liste des services, sélectionnez LDAPv3 et cliquez sur le bouton de modification (/).

    Vous pouvez sélectionner LDAPv3 dans la liste des services sans cocher la case Activer pour LDAPv3.

  8. Cliquez sur Nouveau et saisissez le nom DNS ou l’adresse IP du serveur LDAP.

  9. Sélectionnez les options d’accès à l’annuaire :

    • Sélectionnez « Chiffrer via SSL » si vous désirez qu’Open Directory utilise le protocole SSL (Secure Sockets Layer) pour les connexions avec l’annuaire LDAP. Avant de sélectionner cette option, demandez à votre administrateur Open Directory de déterminer si SSL est nécessaire.

    • Sélectionnez « Utiliser pour l’authentification » si ce répertoire contient des comptes utilisateur qui serviront à ouvrir une session ou à s’authentifier auprès de services.

    • Sélectionnez l’option « Utiliser pour les contacts » si l’annuaire contient des adresses de messagerie et d’autres informations que vous souhaitez utiliser dans Carnet d’adresses.

    Si Utilitaire d’annuaire ne parvient pas à entrer en contact avec le serveur LDAP, un message apparaît et vous devez configurer l’accès manuellement ou annuler le processus de configuration. Pour en savoir plus sur les instructions de configuration manuelle, consultez la rubrique Configuration manuelle de l’accès à un annuaire LDAP.

    Si la zone de dialogue se développe en faisant apparaître des options de mappage, choisissez le modèle de mappage dans le menu local, saisissez l’extension de la base de recherche, puis cliquez sur Continuer.

    En général, l’extension de la base de recherche provient du nom DNS du serveur. Par exemple, l’extension de la base de recherche pourrait être « dc=ods,dc=exemple,dc=com » pour un serveur dont le nom DNS est ods.exemple.com.

    Si aucun des modèles de mappage disponibles ne s’applique à la connexion que vous êtes en train de configurer, cliquez sur Manuel. Pour en savoir plus, reportez-vous à la rubrique Configuration manuelle de l’accès à un annuaire LDAP.

  10. Pour qu’Utilitaire d’annuaire obtienne des informations du serveur LDAP, cliquez sur Continuer.

  11. Si la zone de dialogue se développe pour afficher des options de liaison sécurisée, entrez le nom de l’ordinateur, ainsi que le nom et le mot de passe d’un administrateur de répertoire. (La liaison peut être facultative.)

    La zone de dialogue vous indique si une liaison fiable est requise ou facultative pour l’annuaire LDAP. Une liaison sécurisée est mutuelle : chaque fois que l’ordinateur se connecte à l’annuaire LDAP, ils s’authentifient l’un l’autre. Si une liaison fiable est configurée ou si l’annuaire LDAP ne prend pas en charge la liaison fiable, le bouton de liaison n’apparaît pas. Vérifiez que vous avez bien indiqué le nom de l’ordinateur.

    Si un message d’avertissement vous indique qu’il existe un enregistrement de l’ordinateur, cliquez sur Annuler pour revenir en arrière et modifier le nom de l’ordinateur ou cliquez sur Écraser pour remplacer l’enregistrement existant.

    Il est possible que l’enregistrement de l’ordinateur ne soit plus d’usage ou qu’il appartienne à un autre ordinateur.

    Si vous remplacez un enregistrement d’ordinateur existant, informez-en l’administrateur d’annuaire LDAP au cas où ce remplacement désactiverait un autre ordinateur. Dans ce cas, l’administrateur d’annuaire LDAP doit donner un autre nom à l’ordinateur désactivé et l’ajouter de nouveau au groupe d’ordinateurs auquel il appartenait.

  12. Si la zone de dialogue se développe pour afficher des options de connexion, sélectionnez « Utiliser l’authentification lors de la sélection » et saisissez le nom et le mot de passe d’un compte utilisateur dans le répertoire.

    Les options de connexion authentifiée apparaissent si le serveur LDAP prend en charge ce type de connexion et non la liaison fiable. Une connexion authentifiée n’est pas réciproque : le serveur LDAP authentifie le client mais le client n’authentifie pas le serveur.

    L’option « Utiliser l’authentification lors de la sélection » est présélectionnée mais estompée si le serveur LDAP exige que vous saisissiez un nom et un mot de passe de compte utilisateur pour une connexion authentifiée.

    Ce nom peut correspondre à n’importe quel compte utilisateur autorisé à consulter des données dans le répertoire. Ainsi, un compte utilisateur dont le nom abrégé est dirauth sur un serveur LDAP et dont l’adresse est ods.exemple.com a le nom suivant : uid=dirauth,cn=users,dc=ods,dc=exemple,dc=com.

    Important  :   si le nom ou le mot de passe est incorrect, vous pouvez ouvrir une session sur l’ordinateur à l’aide de comptes utilisateur issus de l’annuaire LDAP.

  13. Cliquez sur OK pour finir de créer la connexion LDAP.

  14. Cliquez sur OK pour finir de configurer les options LDAPv3.

Si vous avez sélectionné « Utiliser pour l’authentification » ou « Utiliser pour les contacts » lors de l’étape 5, la configuration que vous avez créée pour l’annuaire LDAP vient s’ajouter à une règle de recherche personnalisée dans la sous-fenêtre Authentification (ou Contacts) d’Utilitaire d’annuaire.

Vérifiez que LDAPv3 est bien activé dans la sous-fenêtre Services afin que l’ordinateur utilise la configuration LDAP que vous avez créée. Pour en savoir plus, reportez-vous à la rubrique Activation ou désactivation des services d’annuaire LDAP.
