Configuration de l’accès LDAP à des domaines Active Directory

Utilitaire d’annuaire permet de définir une configuration LDAPv3 afin d’accéder à un domaine Active Directory sur un serveur Windows. Une configuration LDAPv3 permet un contrôle total sur le mappage des attributs et types d’enregistrements Mac OS X avec les classes d’objets, bases de recherche et attributs Active Directory.

Le mappage de certains attributs et types d’enregistrements Mac OS X importants, tels que l’UID (identifiant utilisateur unique), requiert l’extension du schéma Active Directory.

Une configuration LDAPv3 ne comprend pas les fonctions suivantes du connecteur Active Directory répertoriées dans Utilitaire d’annuaire :

  • Génération dynamique d’un identifiant utilisateur unique et d’un identifiant de groupe principal

  • Création d’un dossier de départ Mac OS X local

  • Montage automatique du dossier de départ Windows

  • Comptes utilisateur mobiles avec informations d’authentification mises en cache

  • Détection de tous les domaines d’une forêt Active Directory

  • Prise en charge de la réplication et du basculement Active Directory

Pour plus d’informations, reportez-vous à la rubrique À propos de l’accès Active Directory.

Pour créer une configuration serveur Active Directory :

  1. Ouvrez les Préférences Système, puis cliquez sur Comptes.

  2. Si l’icône de verrouillage est activée, déverrouillez-la en cliquant dessus puis en saisissant le nom et le mot de passe d’un administrateur.

  3. Cliquez sur Options, puis cliquez sur Se connecter ou sur Modifier.

  4. Cliquez sur Utilitaire Open Directory.

  5. Si l’icône de verrouillage est activée, déverrouillez-la en cliquant dessus puis en saisissant le nom et le mot de passe d’un administrateur.

  6. Cliquez sur Services.

  7. Dans la liste des services, sélectionnez LDAPv3 et cliquez sur le bouton de modification (/).

  8. Cliquez sur Nouveau, puis saisissez le nom DNS ou l’adresse IP du serveur Active Directory.

  9. Faites votre choix parmi les options d’accès à l’annuaire suivantes, puis cliquez sur Continuer pour qu’Utilitaire d’annuaire récupère des informations sur le serveur Active Directory.

    • Sélectionnez « Chiffrer via SSL » si vous souhaitez qu’Open Directory utilise le protocole SSL pour les connexions au serveur Active Directory. Avant de cocher la case SSL, demandez à votre administrateur Open Directory si cette option est nécessaire.

    • Sélectionnez l’option « Utiliser pour l’authentification » si cet annuaire contient des comptes utilisateur qui serviront à ouvrir des sessions ou à s’authentifier auprès de services.

    • Sélectionnez l’option « Utiliser pour les contacts » si l’annuaire contient des adresses de messagerie et d’autres informations que vous souhaitez utiliser dans Carnet d’adresses.

    Si Utilitaire d’annuaire ne parvient pas à se connecter au serveur Active Directory, un message vous en informe : vous devez alors configurer manuellement l’accès ou annuler le processus de configuration. Pour en savoir plus, reportez-vous à la rubrique Configuration manuelle de l’accès à un annuaire LDAP.

    Si vous avez sélectionné « Utiliser pour l’authentification » ou « Utiliser pour les contacts », la connexion LDAPv3 au domaine Active Directory est ajoutée à la règle de recherche personnalisée dans les sous-fenêtres Authentification ou Contacts d’Utilitaire d’annuaire.

    Assurez-vous que l’option LDAPv3 soit activée dans la sous-fenêtre Services, de sorte que l’ordinateur puisse utiliser la connexion que vous venez de configurer. Pour en savoir plus, reportez-vous à la rubrique Activation ou désactivation des services d’annuaire LDAP.

  10. Lorsque la zone de dialogue s’agrandit pour afficher les options de mappages, choisissez Active Directory dans le menu local, saisissez la base de recherche, puis cliquez sur Continuer.

    Le modèle de mappage Active Directory d’une configuration LDAPv3 établit un mappage entre certains attributs et types d’enregistrements Mac OS X et des classes d’objets et attributs qui n’appartiennent pas à un schéma Active Directory standard. Vous pouvez modifier les mappages définis par le modèle ou étendre le schéma Active Directory.

    Sinon, il se peut que vous puissiez accéder à votre domaine Active Directory par l’intermédiaire du connecteur Active Directory au lieu de LDAPv3. Pour en savoir plus, reportez-vous à la rubrique«  Configuration de l’accès à un domaine Active Directory ».

  11. Lorsque la zone de dialogue s’agrandit pour afficher les options de connexion, saisissez le nom et le mot de passe d’un compte utilisateur Active Directory.

  12. Cliquez sur OK pour finir de créer la connexion LDAP.

  13. Cliquez sur OK pour finir de configurer les options LDAPv3.
