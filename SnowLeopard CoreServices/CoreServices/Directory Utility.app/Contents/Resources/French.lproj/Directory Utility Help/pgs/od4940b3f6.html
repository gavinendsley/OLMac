Configuration d’un shell UNIX pour des comptes utilisateur Active Directory

Sur un ordinateur configuré pour l’utilisation du connecteur Active Directory d’Utilitaire d’annuaire, vous pouvez définir le shell de ligne de commande qu’emploieront par défaut les utilisateurs de comptes Active Directory lors de leurs interactions avec le système Mac OS X dans l’application Terminal.

Le shell par défaut est également utilisé pour les interactions à distance via Secure Shell (SSH) ou Telnet. Chaque utilisateur peut remplacer le shell par défaut en changeant une préférence de Terminal.

Pour définir un shell UNIX pour des comptes utilisateurs Active Directory :

  1. Ouvrez les Préférences Système, puis cliquez sur Comptes.

  2. Si l’icône de verrouillage est activée, déverrouillez-la en cliquant dessus puis en saisissant le nom et le mot de passe d’un administrateur.

  3. Cliquez sur Options, puis cliquez sur Se connecter ou sur Modifier.

  4. Cliquez sur Utilitaire Open Directory.

  5. Si l’icône de verrouillage est activée, déverrouillez-la en cliquant dessus puis en saisissant le nom et le mot de passe d’un administrateur.

  6. Cliquez sur Services.

  7. Dans la liste de services, sélectionnez Active Directory, puis cliquez sur le bouton de modification (/).

  8. Si la liste des options avancées est masquée, cliquez sur "Afficher les options avancées".

  9. Cliquez sur Expérience utilisateur.

  10. Sélectionnez Shell utilisateur par défaut, puis tapez le chemin d’accès à ce shell.

  11. Cliquez sur OK.
