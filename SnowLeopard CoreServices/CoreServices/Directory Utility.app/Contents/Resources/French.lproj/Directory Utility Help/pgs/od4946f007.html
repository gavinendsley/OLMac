Modification du mot de passe utilisé pour authentifier une connexion LDAP

Utilitaire d’annuaire permet de mettre à jour une connexion LDAP authentifiée afin d’utiliser un mot de passe ayant été modifié sur le serveur LDAP. (Tous les ordinateurs possédant une connexion authentifiée auprès d’un serveur LDAP doivent être mis à jour si le mot de passe utilisé pour authentifier la connexion LDAP est modifié sur le serveur.)

Pour modifier le mot de passe d’une connexion LDAP :

  1. Ouvrez les Préférences Système, puis cliquez sur Comptes.

  2. Si l’icône de verrouillage est activée, déverrouillez-la en cliquant dessus puis en saisissant le nom et le mot de passe d’un administrateur.

  3. Cliquez sur Options, puis sur Modifier.

  4. Cliquez sur Utilitaire Open Directory.

  5. Si l’icône de verrouillage est activée, déverrouillez-la en cliquant dessus puis en saisissant le nom et le mot de passe d’un administrateur.

  6. Cliquez sur Services.

  7. Dans la liste des services, sélectionnez LDAPv3 et cliquez sur le bouton de modification (/).

  8. Si la liste des configurations serveur est masquée, cliquez sur Afficher les options.

  9. Dans la liste, sélectionnez une configuration serveur, puis cliquez sur Édition.

  10. Cliquez sur Sécurité et modifiez le réglage Mot de passe :

    • Si le réglage Mot de passe est estompé car l’option « Utiliser l’authentification lors de la connexion » n’est pas cochée, consultez Authentification d’une connexion LDAP.

    • Si le réglage Mot de passe est estompé car l’option « Relié à l’annuaire en tant que » est sélectionnée (mais estompée), la connexion n’est pas authentifiée avec un mot de passe utilisateur. La connexion utilise alors un enregistrement de l’ordinateur authentifié pour la liaison fiable.
