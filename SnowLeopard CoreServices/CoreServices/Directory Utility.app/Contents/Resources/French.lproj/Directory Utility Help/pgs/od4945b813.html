Mappage des attributs de l’enregistrement de configuration des annuaires LDAP

Pour stocker des informations pour des utilisateurs Mac OS X gérés dans un annuaire LDAP autre qu’Apple, vous devez mapper les attributs RealName et DataStamp de l’enregistrement de configuration.

Si vous ne mappez pas ces attributs, le message d’erreur suivant apparaîtra lorsque vous utiliserez Gestionnaire de groupe de travail pour modifier un enregistrement d’utilisateur résidant dans l’annuaire LDAP :

L’attribut intitulé « dsRecTypeStandard:Config » n’est pas mappé.

Vous pouvez ignorer ce message si vous n’utilisez pas la gestion de clients Mac OS X qui s’appuie sur les attributs RealName et DataStamp de l’enregistrement Config pour un cache.
