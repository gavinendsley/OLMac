Configuration de comptes utilisateurs mobiles dans Active Directory

Vous pouvez activer ou désactiver des comptes utilisateurs Active Directory mobiles sur un ordinateur configuré pour utiliser le connecteur Active Directory d’Utilitaire d’annuaire. Les utilisateurs disposant de comptes mobiles peuvent ouvrir une session à l’aide de leurs informations d’authentification Active Directory quand l’ordinateur est déconnecté du serveur Active Directory.

Le connecteur Active Directory met en cache les informations d’authentification d’un compte d’utilisateur mobile lorsque l’utilisateur ouvre une session pendant que l’ordinateur est connecté au domaine Active Directory. Cette mise en cache des informations d’authentification ne nécessite pas de modification du schéma Active Directory.

Si le schéma Active Directory a été étendu pour inclure les attributs de clients gérés par Mac OS X, ces réglages de compte mobile sont utilisés à la place du réglage de compte mobile du connecteur Active Directory.

Vous pouvez activer la création automatique de comptes mobiles ou exiger que les utilisateurs Active Directory confirment la création d’un compte mobile.

Pour activer ou désactiver les comptes mobiles dans un domaine Active Directory :

  1. Ouvrez les Préférences Système, puis cliquez sur Comptes.

  2. Si l’icône de verrouillage est activée, déverrouillez-la en cliquant dessus puis en saisissant le nom et le mot de passe d’un administrateur.

  3. Cliquez sur Options, puis cliquez sur Se connecter ou sur Modifier.

  4. Cliquez sur Utilitaire Open Directory.

  5. Si l’icône de verrouillage est activée, déverrouillez-la en cliquant dessus puis en saisissant le nom et le mot de passe d’un administrateur.

  6. Cliquez sur Services.

  7. Dans la liste de services, sélectionnez Active Directory, puis cliquez sur le bouton de modification (/).

  8. Si la liste des options avancées est masquée, cliquez sur "Afficher les options avancées".

  9. Cliquez sur Expérience utilisateur, puis sur « Créer un compte mobile lors de l’ouverture de session » et, le cas échéant, sur « Exiger une confirmation avant de créer un compte mobile ».

    • Si les deux options sont sélectionnées, chaque utilisateur décide s’il souhaite ou non créer un compte mobile lors de l’ouverture de session. Lorsqu’un utilisateur ouvre une session dans Mac OS X à l’aide d’un compte utilisateur Active Directory, ou en tant qu’utilisateur réseau, une zone de dialogue apparaît à l’écran avec des commandes permettant de créer immédiatement un compte mobile.

    • Si la première option est sélectionnée et que la seconde ne l’est pas, des comptes mobiles sont créés lors de l’ouverture de session.

    • Si la première option n’est pas sélectionnée, la seconde est désactivée.

  10. Cliquez sur OK.
