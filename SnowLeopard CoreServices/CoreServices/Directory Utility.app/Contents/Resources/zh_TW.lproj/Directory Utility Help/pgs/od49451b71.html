保護電腦不受惡意 DHCP 伺服器的侵害

若您比較擔心安全性問題，Apple 建議您不要對 DHCP 提供的 LDAP 伺服器或 DHCP 提供的共享目錄網域使用自動認證搜尋規則。

不懷好意的駭客可以利用假的 DHCP 伺服器和 LDAP 目錄（或共享目錄網域），並透過使用 root 使用者帳號控制您的電腦。

若駭客要連接您的網路，駭客的假 DHCP 伺服器必須是您區域網路或子網路的部分。因此，若您的電腦是您區域網路上的唯一部分，且它們透過 Mac OS X Server NAT 服務或 NAT 路由器連接 Internet，此類型的安全性侵害是不可能的。然而，使用無線區域網路會減少安全性，因為駭客加入無線區域網路比加入有線區域網路更加輕鬆。

若要保護您的 Mac 不會受到來自虛假的 DHCP 伺服器的惡意攻擊，請停用 DHCP 提供的 LDAP 目錄，並停用本機目錄網域和 DHCP 綁定（或停用本機目錄網域）。

若您有連接到 LDAP 伺服器的可攜式電腦，且當電腦連接到網路時您將電腦的搜尋規則從自動改為自定（在“目錄工具程式”的“搜尋規則”中的“認證”面板），則當電腦沒有連接到網路時啟動會延遲。

若電腦無法連接到電腦的自定搜尋規則所列的某個指定目錄網域，就會發生延遲。當電腦為睡眠狀態時而中斷了與網路的連接，則在您喚醒它時不會感覺到延遲。
