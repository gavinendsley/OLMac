設定 LDAP 搜尋和對應

使用“目錄工具程式”，您可以編輯對應表、搜尋基準和搜尋範圍，以指定 Mac OS X 在 LDAP 目錄中搜尋特定資料項目的方式。您可以分別為“目錄工具程式”列表上的每個 LDAP 目錄配置來編輯這些設定。每個 LDAP 目錄配置會指定 Mac OS X 在 LDAPv3 或 LDAP v2 目錄中存取資料的方式。

您可以編輯以下項目：

  • 每種 Mac OS X 記錄類型與 LDAP 物件類別之間的對應

  • 每個記錄類型的 Mac OS X 資料類型或屬性，與 LDAP 屬性的對應

  • LDAP 搜尋基準以及搜尋範圍，以決定 Mac OS X 在 LDAP 目錄中搜尋特定 Mac OS X 記錄類型的位置

將 Mac OS X 使用者屬性對應到一個可讀寫的 LDAP 目錄網域（非唯讀的 LDAP 網域）時，對應到 RealName 的 LDAP 屬性不能與對應到 RecordName 之 LDAP 屬性列表中的第一個屬性相同。

舉例來說，當 cn 已經對應到 RealName 屬性時，對應到 RecordName 的第一個屬性就不能為 cn。

若對應到 RealName（真實名稱）的 LDAP 屬性和對應到 RecordName（記錄名稱）的第一個屬性相同，則當您嘗試以 Workgroup Manage（工作群組管理程式）編輯完整（長）名稱或第一個簡稱時，便會發生問題。

如需更多 Mac OS X 記錄類型和屬性的相關資訊，請參閱「Open Directory 管理」（您可在 www.apple.com/tw/server/resources/ 找到相關資訊）。

若要編輯 LDAP 伺服器的搜尋基準和對應：

  1. 打開“系統偏好設定”並按“帳號”一下。

  2. 若鎖頭圖像是鎖定的，請按圖像一下，然後輸入管理者的名稱和密碼來解除鎖定。

  3. 按“登入選項”一下，然後按“編輯”一下。

  4. 按“打開目錄工具程式⋯”一下。

  5. 若鎖頭圖像是鎖定的，請按圖像一下，然後輸入管理者的名稱和密碼來解除鎖定。

  6. 按“服務”一下。

  7. 在服務列表中，請選擇 LDAPv3 並按“編輯”按鈕（/）一下。

  8. 若伺服器配置列表為隱藏狀態，請按“顯示選項”一下。

  9. 在列表中選擇伺服器配置，並按“編輯⋯”一下。

  10. 按“搜尋和對應”一下。

  11. 選擇您想要用來作為開始點的對應；否則，請選擇“自定”來開始處理沒有預先定義的對應。

    當您選擇任一 LDAP 對應樣板時，系統會顯示您可更改的搜尋基準網域尾碼，或者您可按一下“好”來接受預設的搜尋基準網域尾碼。

    按一下“連接這個 LDAPv3 伺服器”彈出式選單，然後選擇某個對應樣板以使用其對應作為開始點。

  12. 增加記錄類型並依需要更改其搜尋基準：

    • 若要加入記錄類型，請按“加入”一下（位於“記錄類型和屬性”列表下方）；然後在出現的表單中，選擇“記錄類型”，從列表中選擇記錄類型，然後按“好”一下。

    • 若要更改記錄類型的搜尋基準和搜尋範圍，請在“記錄類型和屬性”列表中選擇記錄類型，然後編輯“搜尋基準”欄位。選擇“所有子目錄樹”來設定搜尋範圍，以包含從搜尋基準以下的 LDAP 目錄階層，或選擇“僅第一層”來設定搜尋範圍，以僅包含搜尋基準和 LDAP 目錄階層的第一層。

    • 若要移除記錄類型，請在“記錄類型和屬性”列表中選擇類型，並按“刪除”一下。

    • 若要加入記錄類型的對應，請在“記錄類型和屬性”列表中選擇記錄類型，然後按“加入”一下（位於“對應至列表中的__項目”下方），然後輸入來自 LDAP 目錄的物件類別名稱。

      若要加入其他 LDAP 物件類別，請按下 Return，然後輸入物件類別的名稱，並透過使用列表上方的彈出式選單，來指定是否使用列出的 LDAP 物件類別。

    • 若要更改記錄類型的對應，請在“記錄類型和屬性”列表中選擇記錄類型，然後在“對應至列表中的__項目”中按兩下您想要更改的 LDAP 物件類別，然後進行編輯。使用列表上方的彈出式選單來選擇是否要使用列出的 LDAP 物件類型。

    • 若要移除記錄類型對應，請在“記錄類型和屬性”列表選擇記錄類型，然後從“對應至列表中的__項目”選擇您想要移除的 LDAP 物件類別，然後按“刪除”一下（位於“對應至列表中的__項目”的下方）。

  13. 新增屬性並依需要更改其對應：

    • 若要加入屬性到記錄類型，請在“記錄類型和屬性”列表中選擇記錄類型，然後按“加入”一下（位於“記錄類型和屬性”列表下方）；然後在出現的表單中，選擇“屬性類型”，選擇屬性類型，然後按“好”一下。

    • 若要加入屬性的對應，請在“記錄類型和屬性”列表中選擇屬性，然後按“加入”一下（位於“對應至列表中的__項目”下方），然後輸入來自 LDAP 目錄的屬性名稱。若要加入其他 LDAP 屬性，請按下 Return 鍵並輸入屬性的名稱。

    • 若要更改屬性的對應，請在“記錄類型和屬性”列表選擇屬性，然後在“對應至列表中的__項目”中按兩下您想要更改的項目，然後編輯項目名稱。

    • 若要移除屬性的對應，請在“記錄類型和屬性”列表選擇屬性，然後從“對應至列表中的__項目”選擇您想要移除的項目，然後按“刪除”一下（位於“對應至列表中的__項目”下方）。

    • 若要更改顯示於右方欄位列表中的屬性排列順序，請在列表中向上或向下拖移屬性。

  14. 若要將您的對應儲存為樣板，請按“儲存樣板”一下。

    下次當您打開“目錄工具程式”時，LDAP 對應樣板的彈出式選單會列出已儲存在預設位置的樣板。儲存樣板的預設位置是位於您“個人專屬”檔案夾的以下路徑：

    ~/資源庫/Application Support/Directory Access/LDAPv3/Templates

  15. 若要儲存 LDAP 目錄的對應以便可自動將它們提供給用戶端，請按“寫入伺服器”一下，然後輸入搜尋基準（以儲存對應）、管理者或其他使用者的可辨識名稱（對搜尋基準有不同的寫入權限，例如 uid=diradmin、cn=users、dc=ods、dc=example、dc=com）和密碼。

    若您是在 Open Directory LDAP 伺服器上寫入對應表，正確的搜尋基準是 cn=config,suffix（這裡的 suffix 指的是伺服器的搜尋基準網域尾碼，例如 dc=ods,dc=example,dc=com）。

LDAP 目錄提供它的對應給 Mac OS X 用戶端，而用戶端的自定搜尋規則包含了連線（連線的設定是從 LDAP 伺服器取得對應）。

LDAP 目錄也提供對應給具有自動搜尋規則的所有 Mac OS X 用戶端。如需更多資訊，請參閱「設定存取 LDAP 目錄」和「使用進階搜尋規則設定」。
