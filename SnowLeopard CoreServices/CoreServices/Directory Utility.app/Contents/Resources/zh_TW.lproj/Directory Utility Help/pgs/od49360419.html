手動設定存取 LDAP 目錄

您可以手動建立配置，此配置會指定 Mac OS X 存取 LDAPv3 或 LDAPv2 目錄的方式。您必須知道 LDAP 目錄伺服器的 DNS 名稱或 IP 位址。

若目錄的主機並不是 Mac OS X Server，您須知道搜尋基準和 Mac OS X 資料與目錄資料的對應樣板。支援的對應樣板有：

  • 從伺服器，適用於有自己對應和搜尋基準的目錄，例如 Mac OS X Server

  • Open Directory 伺服器，適用於使用 Mac OS X Server 描述語言的目錄

  • Active Directory，適用於主機Windows 2000 或 Windows 2003 或更新版本之伺服器的伺服器目錄

  • RFC 2307，適用於主機為 UNIX 伺服器的大部分目錄

  • 自定，適用於沒有使用以上任何對應的目錄

LDAPv3 外掛模組完全支援 Open Directory 的複寫與錯誤修復功能。若無法使用 Open Directory 的主體，外掛模組會自動換用另一個鄰近的複本。

重要事項：若您的電腦名稱包含連字號，您可能無法加入或綁定到“目錄網域”（如 LDAP 或 Active Directory）。若要建立綁定，請使用不含連字號的電腦名稱。
若要手動設定存取 LDAP 目錄：

  1. 打開“系統偏好設定”並按“帳號”一下。

  2. 若鎖頭圖像是鎖定的，請按圖像一下，然後輸入管理者的名稱和密碼來解除鎖定。

  3. 按“登入選項”一下，然後按一下“加入”或“編輯”。

  4. 按“打開目錄工具程式⋯”一下。

  5. 若鎖頭圖像是鎖定的，請按圖像一下，然後輸入管理者的名稱和密碼來解除鎖定。

  6. 按“服務”一下。

  7. 在服務列表中，請選擇 LDAPv3 並按“編輯”按鈕（/）一下。

    您可以在服務列表裡選擇 LDAPv3，而無須選擇 LDAPv3 的“啟用”註記框。

  8. 按“新增⋯”一下，然後按“手動”一下。

  9. 為此配置輸入名稱。

  10. 按下 Tab 鍵，並為您要連線的伺服器輸入 DNS 名稱或 IP 位址。

  11. 按一下在 DNS 名稱或是 IP 位址旁的彈出式選單，選擇一份對應樣板或方法：

    • 若您選擇“從伺服器”，則不需要搜尋基準網域尾碼。使用此設定時，Open Directory 會以 LDAP 目錄的第一層資訊作為尋基準網域尾碼。

    • 若您選擇樣板，請輸入 LDAP 目錄的基準網域尾碼，然後按“好”一下。您必須填入一組搜尋基準網域尾碼，否則電腦無法找到 LDAP 目錄中的資訊。

      基本上，搜尋基準網域尾碼可以透過伺服器的 DNS 名稱來推論得出。例如，若伺服器的 DNS 名稱為 ods.example.com，則其搜尋基準網域尾碼可能為“dc=ods,dc=example, dc=com”。

    • 若您選擇“自定”，您必須設定 Mac OS X 記錄類型和屬性與您打算連接的 LDAP 目錄類別和屬性之間的對應樣板。如需更多資訊，請參閱「設定 LDAP 搜尋和對應」。

  12. 在您選擇“使用 SSL 加密”註記框前，請與您的 Open Directory 管理員確認，以決定是否需要使用 SSL。

  13. 若要更改此 LDAP 設定的下列設定，請按“編輯”一下來顯示所選 LDAP 設定的選項，進行更動，然後在完成編輯 LDAP 設定選項時，按“好”一下。

    • 按“連線”一下來設定逾時選項、指定自定埠、忽略伺服器指示或強制使用 LDAPv2（唯讀）通訊協定。如需更多資訊，請參閱「更改 LDAP 目錄的連線設定」。

    • 按“搜尋和對應”一下來設定 LDAP 伺服器的搜尋和對應。如需更多資訊，請參閱「對 LDAP 目錄設定信任綁定」。

    • 按“安全性”一下來設定已認證的連線（而非信任綁定）和其他安全性規則選項。如需更多資訊，請參閱「為 LDAP 連線更改安全性規則」。

    • 按“綁定”一下來設定信任綁定（若 LDAP 目錄支援它）。如需更多資訊，請參閱「對 LDAP 目錄設定信任綁定」。

  14. 按“好”一下來結束手動建立存取 LDAP 目錄的配置。

  15. 若您希望電腦存取您已為其建立配置的 LDAP 目錄，請在“目錄工具程式”的“認證”和“聯絡資訊”面板，將目錄加入為自定搜尋規則，然後確定“服務”面板的 LDAPv3 已啟用。

如需更多資訊，請參閱「啟用或停用 LDAP 目錄服務」和「定義自定搜尋規則」。

注意：在使用 RFC 2307（UNIX）對應表的非 Apple LDAP 伺服器上，若要使用 Workgroup Manager（工作群組管理程式）製作使用者，您必須先編輯“使用者”記錄類型的對應。如需更多資訊，請參閱「編輯 RFC 2307 對應表以啟用建立使用者的功能」。
重要事項：若您在連接到目錄伺服器時使用 changeip 更改 IP 位址和電腦名稱，您必須先中斷連接並重新連接到目錄伺服器，藉此使用新的電腦名稱和 IP 位址來更新目錄。若您未中斷連接並重新連接到目錄伺服器，目錄將不會進行更新，並會繼續使用舊有電腦名稱和 IP 位址。
