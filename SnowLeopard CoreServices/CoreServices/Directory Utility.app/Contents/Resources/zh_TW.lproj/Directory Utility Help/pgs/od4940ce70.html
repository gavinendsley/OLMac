設定 LDAP 存取 Active Directory 網域

使用“目錄工具程式”，您可以設定 LDAPv3 配置來存取 Windows 伺服器上的 Active Directory 網域。 LDAPv3 配置提供完整的控制能力，讓您可以將 Mac OS X 記錄類型與屬性對應到 Active Directory 的物件類別、搜尋基準及屬性。

若要對應某些重要的 Mac OS X 記錄類型與屬性，如使用者獨有 ID（UID）等，則需延伸 Active Directory 描述語言。

LDAPv3 配置並未包含 Active Directory 連接器（列於“目錄工具程式”中）所提供的下列功能：

  • 動態產生使用者獨有 ID 和主要群組 ID

  • 製作本機 Mac OS X 個人專屬檔案夾

  • 自動裝載 Windows 個人專屬檔案夾

  • 行動使用者帳號的憑證資料快取功能

  • 搜尋 Active Directory 樹系裡的所有網域

  • 支援 Active Directory 的複寫與錯誤修復

如需更多資訊，請參閱「關於存取 Active Directory」。

若要建立 Active Directory 伺服器配置：

  1. 打開“系統偏好設定”並按“帳號”一下。

  2. 若鎖頭圖像是鎖定的，請按圖像一下，然後輸入管理者的名稱和密碼來解除鎖定。

  3. 按“登入選項”一下，然後按一下“加入”或“編輯”。

  4. 按“打開目錄工具程式⋯”一下。

  5. 若鎖頭圖像是鎖定的，請按圖像一下，然後輸入管理者的名稱和密碼來解除鎖定。

  6. 按“服務”一下。

  7. 在服務列表中，請選擇 LDAPv3 並按“編輯”按鈕（/）一下。

  8. 按“新增⋯”一下，並輸入 Active Directory 伺服器的 DNS 名稱或 IP 位址。

  9. 從下列選項中選擇存取目錄所需的選項，然後按“繼續”一下，以便“目錄工具程式”從 Active Directory 伺服器上取得資訊。

    • 如果您要讓 Open Directory 與 Active Directory 伺服器連線時使用 SSL，請選擇“使用 SSL 加密”。在您勾選 SSL 註記框前，請詢問您的 Open Directory 管理者是否需要使用 SSL。

    • 如果此目錄包含他人用於登入或服務認證的使用者帳號，請選擇“用於認證”。

    • 如果此目錄包含您要用於“通訊錄”的電子郵件位址和其他資訊，請選擇“用於聯絡資訊”。

    若“目錄工具程式”無法聯絡 Active Directory 伺服器，便會顯示訊息，而您必須手動設定存取或取消設定過程。如需更多資訊，請參閱「手動設定存取 LDAP 目錄」。

    若您選擇“用於認證”或“用於聯絡資訊”，“目錄工具程式”的“認證”或“聯絡資訊”面板裡便會將 LDAPv3 至 Active Directory 網域的連線加入到自定搜尋規則中。

    確定“服務”面板中的 LDAPv3 已啟用，這樣電腦才會使用您設定的連線。如需更多資訊，請參閱「啟用或停用 LDAP 目錄服務」。

  10. 當該對話框延伸以顯示對應選項時，請從彈出式選單裡選擇 Active Directory，並輸入搜尋基準，然後按“繼續”一下。

    LDAPv3 配置的 Active Directory 的對應表範本，會將一些 Mac OS X 的記錄類型與屬性對應到不在標準 Active Directory 描述語言裡的物件類別與屬性上。您可以自行更改範本中定義的對照表，或是延伸 Active Directory 描述語言。

    或者您也可改用 Active Directory 連接器來連接 Active Directory 網域，而不使用 LDAPv3。如需更多資訊，請參閱「設定 Active Directory 網域存取」。

  11. 當此對話框延伸以顯示連線選項時，請輸入辨別 Active Directory 使用者帳號的名稱和密碼。

  12. 按“好”一下來完成建立 LDAP 連線。

  13. 按“好”一下來完成設定 LDAPv3 選項。
