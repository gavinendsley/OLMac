認證 LDAP 連線

有了“目錄工具程式”，您可以設定經過認證的 LDAP 目錄連線。這種認證僅是單向的。 電腦驗證身分來連接 LDAP 目錄，但 LDAP 目錄沒有為電腦驗證其可靠性。 如需相互認證，請參閱「對 LDAP 目錄設定信任綁定」。

注意：如果電腦與 LDAP 目錄之間已設定了信任綁定，您便無需再設定認證連線，同時您也無法建立此連線。
若要設定經認證的 LDAPv3 連線：

  1. 打開“系統偏好設定”並按“帳號”一下。

  2. 若鎖頭圖像是鎖定的，請按圖像一下，然後輸入管理者的名稱和密碼來解除鎖定。

  3. 按“登入選項”一下，然後按一下“加入”或“編輯”。

  4. 按“打開目錄工具程式⋯”一下。

  5. 若鎖頭圖像是鎖定的，請按圖像一下，然後輸入管理者的名稱和密碼來解除鎖定。

  6. 按“服務”一下。

  7. 在服務列表中，請選擇 LDAPv3 並按“編輯”按鈕（/）一下。

  8. 若伺服器配置列表為隱藏狀態，請按“顯示選項”一下。

  9. 在列表中選擇伺服器配置，並按“編輯⋯”一下。

  10. 按“安全性”一下。

  11. 請選擇“連接時使用認證”，然後輸入使用者的識別名稱和密碼。

    可識別的名稱可以指定任何擁有權限的使用者帳號查看目錄裡的資料。例如，LDAP 伺服器上簡稱為“authenticator”且位址為 ods.example.com 的使用者帳號，其識別名稱為 uid=authenticator,cn=users,dc=ods,dc=example,dc=com。

    重要事項：如果可識別的名稱或密碼不正確，則沒有人可以使用來自 LDAP 目錄的使用者帳號登入電腦。
