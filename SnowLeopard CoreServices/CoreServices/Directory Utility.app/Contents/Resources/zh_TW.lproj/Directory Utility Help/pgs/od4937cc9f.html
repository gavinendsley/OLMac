設定存取 LDAP 目錄

透過使用“目錄工具程式”，且若您知道 LDAP 目錄伺服器的 DNS 名稱或 IP 位址，您就可以指定 Mac OS X 如何存取 LDAPv3 目錄。

此外，若此目錄的所在伺服器並不會提供自有的對應（如 Mac OS X Server），您必須知道搜尋基準和對應 Mac OS X 資料與目錄資料的樣板。

支援的對應樣板有：

  • Open Directory 伺服器，適用於使用 Mac OS X Server 描述語言的目錄

  • Active Directory，適用於主機Windows 2000 或 Windows 2003 或更新版本之伺服器的伺服器目錄

  • RFC 2307，適用於主機為 UNIX 伺服器的大部分目錄

LDAPv3 外掛模組完全支援 Open Directory 的複寫與錯誤修復功能。若無法使用 Open Directory 的主體，外掛模組會自動換用另一個鄰近的複本。

若要指定自定目錄資料的對應，請依照「手動設定存取 LDAP 目錄」的說明指示，而非此文的說明指示。

重要事項：若您的電腦名稱包含連字號，您可能無法加入或綁定到“目錄網域”（如 LDAP 或 Active Directory）。若要建立綁定，請使用不含連字號的電腦名稱。
若要“目錄工具程式”幫助您設定存取 LDAP 目錄：

  1. 打開“系統偏好設定”並按“帳號”一下。

  2. 若鎖頭圖像是鎖定的，請按圖像一下，然後輸入管理者的名稱和密碼來解除鎖定。

  3. 按“登入選項”一下，然後按一下“加入”或“編輯”。

  4. 按“打開目錄工具程式⋯”一下。

  5. 若鎖頭圖像是鎖定的，請按圖像一下，然後輸入管理者的名稱和密碼來解除鎖定。

  6. 按“服務”一下。

  7. 在服務列表中，請選擇 LDAPv3 並按“編輯”按鈕（/）一下。

    您可以在服務列表裡選擇 LDAPv3，而無須選擇 LDAPv3 的“啟用”註記框。

  8. 按“新增”一下，然後輸入 LDAP 伺服器的 DNS 名稱或 IP 位址。

  9. 選擇存取目錄的選項：

    • 若您希望 Open Directory 使用“安全編碼傳輸技術”（SSL, Secure Sockets Layer）與 LDAP 目錄進行連線，請勾選“使用 SSL 加密”註記框。 選擇此選項之前，請詢問您的 Open Directory 管理者以決定是否需要使用 SSL。

    • 若此目錄包含他人用於登入或認證服務的使用者帳號，請選擇“用於認證”。

    • 如果此目錄包含您要用於“通訊錄”的電子郵件位址和其他資訊，請選擇“用於聯絡資訊”。

    若“目錄工具程式”無法連接 LDAP 伺服器，會顯示一個訊息，您得手動設定連線或取消設定程序。如需更多手動設定說明的相關資訊，請參閱「手動設定存取 LDAP 目錄」。

    若對話框展開以顯示對應選項，請從彈出式選單中選擇對應樣板，輸入搜尋基準網域尾碼，然後按“繼續”一下。

    基本上，搜尋基準網域尾碼可以透過伺服器的 DNS 名稱來推論得出。例如，若伺服器的 DNS 名稱為 ods.example.com，則其搜尋基準網域尾碼可能為“dc=ods,dc=example, dc=com”。

    若沒有可用的對應樣板可套用到您設定的連線，請按“手動”一下。如需更多資訊，請參閱「手動設定存取 LDAP 目錄」。

  10. 若要讓“目錄工具程式”從 LDAP 伺服器取得資訊，請按“繼續”一下。

  11. 若對話框展開以顯示信任綁定的選項，請輸入電腦名稱與目錄管理者的名稱和密碼。（綁定選項可留空。）

    對話框會告訴您 LDAP 目錄是否需要信任綁定或令其可留空。信任綁定具有相互關聯：每次電腦連接到 LDAP 目錄時，它們會相互認證。若已設定好信任綁定或 LDAP 目錄無法支援信任綁定，“綁定”按鈕便不會出現。請確定您提供正確的電腦名稱。

    如果您看見警告訊息告知已存在一份電腦記錄，請按“取消”並返回上一步來更改電腦名稱，或是按“覆寫”來取代已存在的電腦記錄。

    現有的電腦記錄可能已經被遺棄或它是屬於另外一台電腦。

    若您要取代現有的電腦記錄，請通知 LDAP 目錄管理者以防取代電腦記錄會停用其他電腦。若發生這種情況，LDAP 目錄管理者必須給已停用的電腦不同的名稱，然後將其加回到原屬的電腦列表。

  12. 若對話框展開以顯示連線選項，請選擇“當選取時使用認證”，然後在目錄裡輸入可辨別的使用者帳號的名稱和密碼。

    若 LDAP 伺服器支援已認證的連線但不支援信任綁定，已認證的連線選項便會出現。已認證的連線不具有相互關聯：LDAP 伺服器會認證用戶端，而用戶端則不會認證伺服器。

    “當選取時使用認證”是為您預先選好的，若 LDAP 伺服器要求您為某個已認證的連線輸入使用者帳號可辨別的名稱和密碼時，它會變暗。

    可識別的名稱可以指定任何擁有權限的使用者帳號查看目錄裡的資料。例如，某個使用者帳號（其在 LDAP 伺服器中的簡稱為“dirauth”）的位址是 ods.example.com，則其可辨別的名稱為 uid=dirauth,cn=users,dc=ods,dc=example,dc=com。

    重要事項：若可辨別的名稱或密碼不正確，您可以使用來自 LDAP 目錄的使用者帳號登入電腦。

  13. 按“好”一下來完成建立 LDAP 連線。

  14. 按“好”一下來完成設定 LDAPv3 選項。

若您在步驟 5 選擇了“用於認證”或“用於聯絡資訊”，您剛才製作的 LDAP 目錄設定，會加入到“目錄工具程式”的“認證”或“聯絡資訊”面板的自定搜尋規則。

請確定在“服務”面板裡啟用了 LDAPv3，這樣電腦才會使用您製作的 LDAP 設定。如需更多資訊，請參閱「啟用或停用 LDAP 目錄服」。
