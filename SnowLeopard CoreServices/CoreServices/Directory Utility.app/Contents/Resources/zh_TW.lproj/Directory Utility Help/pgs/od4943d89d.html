關於存取 Active Directory

在 Windows 2000 Server 或更新版本伺服器的 Active Directory 網域上，您可以設定 Mac OS X 來存取使用者帳號的基本資訊。有了“目錄工具程式”的 Active Directory 連接器，您便可以完成這項動作。Active Directory 連接器列在“目錄工具程式”的“服務”面板中。

您無須更改 Active Directory 網域的描述語言，就能取得使用者帳號的基本資訊。您可能要更改特定屬性的“連線權限控制列表”（ACL）預設值，來讓電腦帳號可以讀取使用者屬性。

Active Directory 連接器會以 Active Directory 使用者帳號中的標準屬性來產生 Mac OS X 認證所需的所有屬性。該連接器亦支援 Active Directory 認證規則，包含密碼更動、帳號有效期限、強制更動，以及安全性選項。

Mac OS X v10.6 支援所有 Windows Active Directory 網域的封包加密和封包簽名選項。依照預設，會啟用此功能，並設為“允許”。您可以使用 dsconfigad 指令行工具來將預設設定更改為停用或要求。封包加密與封包簽名選項可確保在 Active Directory 網域往來之所有資料的安全性，因為記錄查詢值是受到保護的。

Active Directory 連接器會根據 Active Directory 網域中使用者帳號的“全域唯一識別碼”（GUID），動態地產生出一個使用者獨有 ID 和一個主要群組 ID。對每個使用者帳號來說，即使在不同的 Mac OS X 電腦上登入，產生出來的使用者 ID 與主要群組 ID 仍會相同。

或者，您可以強制 Active Directory 連接器將使用者 ID 對應至您所指定的 Active Directory 屬性。

Active Directory 連接器會根據 Active Directory 群組帳號的 GUID 製作一個群組 ID。您還可以強制外掛模組將群組帳號的群組 ID 對應於您指定的 Active Directory 屬性。

當有人使用 Active Directory 使用者帳號登入 Mac OS X 時，Active Directory 連接器會裝載 Windows 網路的個人專屬檔案夾，此 檔案夾在 Active Directory 使用者帳號中被指定為使用者的 Mac OS X 個人專屬檔案夾。您就可以指定是否使用網路上由 Active Directory 的標準個人專屬“目錄”屬性指定的網路個人專屬，或由 Mac OS X 的 homeDirectory 屬性指定的網路個人專屬（如果 Active Directory 描述語言延伸到包含這些功能）。

或者，您可以設定外掛模組來在 Mac OS X 用戶端電腦的啟動卷宗上建立本機個人專屬檔案夾。 在這樣的情況下，外掛模組也會將使用者的 Windows 網路個人專屬檔案夾（Active Directory 使用者帳號裡指定的目錄）作為網路卷宗進行裝載，就像共享點一樣。有了此檔案夾，使用者可以在 Windows 個人專屬檔案夾的網路卷宗與本機 Mac OS X 個人專屬檔案夾之間，使用 Finder 進行檔案拷貝的工作。

Active Directory 連接器還可以為使用者製作行動帳號。行動帳號在 Mac OS X 用戶端電腦的啟動卷宗上有一個本機個人專屬檔案夾。（使用者在其 Active Directory 帳號裡也會有一個指定好的網路個人專屬檔案夾）。

行動帳號會為 Mac OS X 用戶端電腦上該使用者的 Active Directory 認證憑證進行快取。在用戶端電腦從 Active Directory 伺服器中斷連線時，已快取的憑證允許使用者使用 Active Directory 名稱和密碼來進行登入。

行動帳號在 Mac OS X 用戶端電腦的啟動卷宗上有一個本機個人專屬檔案夾。（使用者同樣有一個在使用者 Active Directory 帳號裡指定的網路個人專屬檔案夾）。

如果 Active Directory 的描述語言已延伸為包含 Mac OS X 的記錄類型（物件類別）和屬性，Active Directory 連接器便會偵測並取用這些設定。

例如，您可以使用 Windows 管理工具更改 Active Directory 的描述語言來包含 Mac OS X 所管理的用戶端屬性設定。描述語言的此種更動將會啟用 Active Directory 連接器來支援所管理的用戶端設定，該設定是由 Mac OS X Server 的 Workgroup Manager（工作群組管理程式）應用程式所製作的。

Mac OS X 用戶端會以完整的讀取權限來存取已經加入目錄的屬性。因此，若要允許電腦群組讀取這些新增的屬性，您可能需要更改這些屬性的 ACL。

Active Directory 連接器會找出 Active Directory 樹系中的所有網域。您可以設定外掛模組允許同一樹系中所有網域的使用者在 Mac OS X 電腦上進行認證。或者，您可以只允許持定網域在客戶端上進行認證。

Active Directory 連接器完全支援 Active Directory 的複寫和錯誤修復功能。外掛模組會尋找數個網域控制站並決定使用最接近的控制站。如果無法使用網域控制器，外掛模組會使用另一個鄰近的網域控制器。

Active Directory 連接器是使用 LDAP 取用 Active Directory 使用者帳號和 Kerberos 來進行認證。Active Directory 連接器並未使用 Microsoft 專用的“Active Directory 服務介面”（ADSI）來取得目錄服務或是認證服務。
