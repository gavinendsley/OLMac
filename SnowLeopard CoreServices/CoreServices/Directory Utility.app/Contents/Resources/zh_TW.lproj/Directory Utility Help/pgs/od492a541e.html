編寫密碼

當使用者為登入作業或其他服務進行認證時，必須輸入與使用者帳號相關的密碼。密碼區分大小寫（SMB-LAN Manager 的密碼除外），而且您在螢幕上輸入密碼時，系統會將其遮蔽。

無論您為使用者選擇何種密碼類型，以下指示皆適用於編寫 Mac OS X Server 使用者帳號的密碼：

  • 密碼應當包含字母、數字和符號的組合，這樣的話未經授權的使用者便無法輕易猜出。 密碼不應該由單字組成。安全性高的密碼包括數字和符號（如 # 或 $），或者是由某詞句中所有單字的第一個字母組成。同時使用大小寫字母。

  • 避免使用空格，以及 Option 與按鍵的組合。

  • 避免使用以下字元：無法在使用者要使用的電腦上輸入的字元，或可能需要知道特殊按鍵組合才能在不同鍵盤和平台上正確輸入的字元。

  • 某些網路通訊協定不支援包含前置空格、內嵌空格或結尾空格的密碼。

  • 不建議使用零長度的密碼。Open Directory 和某些系統（如 LDAP 綁定）就不支援零長度的密碼。

  • 若要與使用者可取用的電腦和服務取得最大相容性，請只使用 ASCII 字元作為密碼。
