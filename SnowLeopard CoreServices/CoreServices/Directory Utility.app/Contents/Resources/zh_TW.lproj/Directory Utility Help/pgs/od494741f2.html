更改 LDAP 目錄的存取配置

您可以使用“目錄工具程式”來更改 LDAP 目錄配置的設定。這個配置設定會指定 Open Directory 存取 LDAPv3 或 LDAPv2 目錄的方式。

如果 LDAP 配置是由 DHCP 所提供，則無法進行更改，因此此類型的配置在 LDAP 配置列表裡是變暗的。

若要編輯 LDAP 目錄的存取配置：

  1. 打開“系統偏好設定”並按“帳號”一下。

  2. 若鎖頭圖像是鎖定的，請按圖像一下，然後輸入管理者的名稱和密碼來解除鎖定。

  3. 按“登入選項”一下，然後按“編輯”一下。

  4. 按“打開目錄工具程式⋯”一下。

  5. 若鎖頭圖像是鎖定的，請按圖像一下，然後輸入管理者的名稱和密碼來解除鎖定。

  6. 按“服務”一下。

  7. 在服務列表中，請選擇 LDAPv3 並按“編輯”按鈕（/）一下。

  8. 若伺服器配置列表為隱藏狀態，請按“顯示選項”一下。

  9. 對下列設定進行必要的更動：

    • 啟用：按註記框一下以啟用或停用對 LDAP 目錄伺服器的存取。

    • 配置名稱：按兩下配置名稱，可以進行編輯。

    • 伺服器名稱或 IP 位址：按兩下伺服器名稱或 IP 位址，可以更改其內容。

    • LDAP 對應：從彈出式選單選擇一種樣板，然後輸入 LDAP 目錄的搜尋基準網域尾碼，並按“好”一下。

      若您選擇使用一種樣板，必須輸入一組搜尋基準網域尾碼，否則電腦無法找到 LDAP 目錄中的資訊。 基本上，搜尋基準網域尾碼可以透過伺服器的 DNS 名稱來推論得出。例如，若伺服器的 DNS 名稱為 ods.example.com，則其搜尋基準網域尾碼可能為“dc=ods,dc=example,dc=com”。

      若您選擇“從伺服器”，而非樣板，則不需要搜尋基準網域尾碼。使用此設定時，Open Directory 會以 LDAP 目錄的第一層資訊作為尋基準網域尾碼。

      若您選擇“自定”，您必須設定 Mac OS X 記錄類別和屬性與您打算連接的 LDAP 目錄類別和屬性的對應樣板。如需更多資訊，請參閱「設定 LDAP 搜尋和對應」。

    • SSL：按註記框一下來啟用或停用加密的通訊（使用 SSL 通訊協定）。在您勾選 SSL 註記框前，請詢問您的 Open Directory 管理者是否需要使用 SSL。

  10. 若要更改此 LDAP 配置的下列預設設定，請按“編輯”一下來顯示所選 LDAP 配置的選項，進行更動，然後在完成編輯 LDAP 配置選項時按“好”一下：

    • 按“連線”一下來設定逾時選項、指定自定埠、忽略伺服器指示或強制使用 LDAPv2（唯讀）通訊協定。如需更多資訊，請參閱「更改 LDAP 目錄的連線設定」。

    • 按“搜尋和對應”一下來設定 LDAP 伺服器的搜尋和對應。如需更多資訊，請參閱「對 LDAP 目錄設定信任綁定」。

    • 按“安全性”一下來設定已認證的連線（而非信任綁定）和其他安全性規則選項。如需更多資訊，請參閱「為 LDAP 連線更改安全性規則」。

    • 按“綁定”按鈕一下來設定信任綁定，或按“移除綁定”按鈕來停止信任綁定。（若 LDAP 目錄不允許信任綁定，您可能看不到這些按鈕。）如需更多資訊，請參閱「對 LDAP 目錄設定信任綁定」。

  11. 按“好”一下來結束更改存取 LDAP 目錄的配置。
