Konfigurowanie dostępu LDAP do domen Active Directory

Za pomocą programu Narzędzie katalogowe można utworzyć konfigurację LDAPv3 tak, aby mieć dostęp do domeny Active Directory na serwerze Windows. Konfiguracja LDAPv3 daje pełną kontrolę nad odwzorowaniami typów rekordów i atrybutów Mac OS X na klasy obiektów, podstawy wyszukiwania i atrybuty Active Directory.

Odwzorowanie pewnych istotnych typów rekordów i atrybutów Mac OS X, takich jak unikalny ID użytkownika (UID), wymaga rozszerzenia schematu Active Directory.

Konfiguracja LDAPv3 nie zawiera następujących cech wtyczki Active Directory z Narzędzia katalogowego:

  • Dynamiczne generowanie unikalnych ID użytkownika oraz ID grupy podstawowej

  • Tworzenie lokalnych katalogów domowych Mac OS X

  • Automatyczne montowanie katalogów domowych Windows

  • Przenośne konta użytkowników z przechowywanym uwierzytelnieniem

  • Wykrywanie wszystkich domen lasu Active Directory

  • Wsparcie dla replikacji i przejmowania Active Directory

Więcej informacji można zobaczyć w Dostęp do Active Directory.

Jak utworzyć konfigurację serwera Active Directory?

  1. Otwórz Preferencje systemowe i kliknij w Konta.

  2. Jeśli ikona kłódki jest zamknięta, otwórz ją klikając w nią i wpisując nazwę oraz hasło administratora.

  3. Kliknij w Opcje logowania, a następnie kliknij w Przyłącz lub Edycja.

  4. Kliknij w przycisk „Otwórz Narzędzie katalogowe”.

  5. Jeśli ikona kłódki jest zamknięta, otwórz ją klikając w nią i wpisując nazwę oraz hasło administratora.

  6. Kliknij w Usługi.

  7. Na liście usług, zaznacz LDAPv3 i kliknij w przycisk Edycja (/).

  8. Kliknij w Nowa i wpisz nazwę DNS lub adres IP serwera Active Directory.

  9. Wybierz z następujących opcji dostępu do katalogu, po czym kliknij w Dalej, aby Narzędzie katalogowe pobrało informację z serwera Active Directory.

    • Zaznacz „Szyfruj używając SSL“, jeśli Open Directory ma używać SSL dla połączeń z serwerem Active Directory. Przed zaznaczeniem pola SSL zapytaj administratora Open Directory, czy potrzebny jest SSL.

    • Zaznacz „Używaj do uwierzytelnienia“, jeśli katalog zawiera kont użytkowników, których ktoś może używać do logowania lub uwierzytelnienia usług.

    • Zaznacz „Używaj do kontaktów“, jeśli katalog zawiera adresy email oraz inne informacje, których chcemy używać w Książce adresowej.

    Jeśli Narzędzie katalogowe nie może połączyć się z serwerem Active Directory, pojawi się komunikat i trzeba będzie ręcznie skonfigurować dostęp lub poniechać konfiguracji. Więcej informacji można zobaczyć w Ręczne konfigurowanie dostępu do katalogu LDAP.

    Jeżeli zaznaczone jest „Używaj do uwierzytelnienia” lub „Używaj do kontaktów”, połączenie LDAPv3 z domeną Active Directory zostaje dodane do własnej zasady wyszukiwania w panelu Uwierzytelnienie lub Kontakty w Narzędziu katalogowym.

    Upewnij się, że LDAPv3 jest włączone w panelu Usługi tak, aby komputer używał tego połączenia. Więcej informacji można zobaczyć w Włączanie lub wyłączanie usług katalogowych LDAP.

  10. Gdy okno dialogowe powiększy się, żeby wyświetlić opcje odwzorowania, z rozwijanego menu wybierz Active Directory, wpisz podstawę wyszukiwania i kliknij w Dalej.

    Szablon odwzorowania Active Directory dla konfiguracji LDAPv3 odwzorowuje niektóre typy rekordów i atrybuty Mac OS X na klasy obiektów i atrybuty, które nie są częścią podstawowego schematu Active Directory. Można zmienić odwzorowania zdefiniowane w szablonie lub rozszerzyć schemat Active Directory.

    Zamiennie, można mieć dostęp do domeny Active Directory przez wtyczkę Active Directory zamiast przez LDAPv3. Więcej informacji można przeczytać w sekcji Konfigurowanie dostępu do domeny Active Directory.

  11. Gdy okno dialogowe powiększy się, żeby wyświetlić opcje połączenia, wpisz unikalną nazwę i hasło konta użytkownika Active Directory.

  12. Kliknij w OK, aby zakończyć tworzenie połączenia LDAP.

  13. Kliknij w OK, aby zakończyć konfigurowanie opcji LDAPv3.
