Użycie zaawansowanych ustawień zasad wyszukiwania

Narzędzie katalogowe definiuje następujące zasady wyszukiwania:

  • Uwierzytelnienie: Mac OS X używa zasad wyszukiwania danych uwierzytelnienia do zlokalizowania i pobrania informacji o uwierzytelnieniu użytkownika oraz innych danych administracyjnych z domen katalogowych.

  • Kontakty: Mac OS X używa zasad wyszukiwania do zlokalizowania i pobrania nazwiska, adresu oraz innych informacji o kontakcie z domen katalogowych. Informacji tych używa Książka adresowa Mac OS X. Inne programy również mogą zostać zaprogramowane do jej używania.

    Każda zasada wyszukiwania zawiera listę domen katalogowych. Kolejność domen katalogowych na liście określa zasady wyszukiwania. Zaczynając od góry listy, Mac OS X przeszukuje każdą podaną domenę katalogową, dopóki nie znajdzie potrzebnej informacji lub dojdzie do końca listy nie znajdując informacji.

    Zasady wyszukiwania uwierzytelnienia i kontaktów mogą zawierać jedno z poniższych ustawień:

  • Automatycznie: Zaczyna od lokalnej domeny katalogowej i może zawierać katalog LDAP dostarczony przez DHCP oraz domeny katalogowe, do których podłączony jest komputer. Jest to domyślne ustawienie dla Mac OS X v10.2 lub nowszych, oferujące najwięcej elastyczności komputerom przenośnym.

  • Katalog lokalny: Zawiera jedynie lokalną domenę katalogową.

  • Własna ścieżka: Rozpoczyna od lokalnej domeny katalogowej i zawiera dodane katalogi LDAP, domenę Active Directory, udostępniane domeny katalogowe, pliki konfiguracyjne BSD oraz domenę NIS.

    Katalog /BSD/local zawsze jest zawarty w ścieżce wyszukiwania i jest zawsze wyszarzony.

Ważne:Wybór automatycznej zasady wyszukiwania i serwera LDAP z DHCP zwiększa potencjalne ryzyko ataku i przejęcia kontroli nad komputerem. Ryzyko jest jeszcze większe, gdy komputer jest podłączony do sieci bezprzewodowej. Więcej informacji można zobaczyć w sekcji Zabezpieczanie komputerów przez złośliwym serwerem DHCP.
