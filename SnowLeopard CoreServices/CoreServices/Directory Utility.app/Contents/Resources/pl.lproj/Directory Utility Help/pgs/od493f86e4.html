Konfigurowanie dostępu do domeny Active Directory

Za pomocą wtyczki Active Directory dostępnej w programie „Narzędzie katalogowe” można w Mac OS X skonfigurować dostęp do podstawowych danych o koncie użytkownika w domenie Active Directory na serwerze Windows.

Wtyczka Active Directory generuje wszystkie atrybuty wymagane dla uwierzytelniania w Mac OS X. Nie trzeba dokonywać zmian schematu Active Directory.

Wtyczka Active Directory wykrywa i ma dostęp do standardowych typów rekordów i atrybutów Mac OS X (takich jak atrybuty wymagane przy zarządzaniu klientem Mac OS X), jeśli schemat Active Directory został rozszerzony, żeby je zawierać.

UWAGA:  Zaawansowane opcje wtyczki Active Directory mogą odwzorować unikalny ID użytkownika (UID), ID grupy podstawowej (GID) oraz atrybut GID grupy na poprawne atrybuty, które zostały dodane do schematu Active Directory. Jeśli później zmienimy ustawienia tych opcji, użytkownicy mogą utracić dostęp do poprzednio utworzonych plików.
Ważne:  Jeśli nazwa komputera zawiera myślnik, może wystąpić problem z łączeniem lub powiązaniem z domeną katalogową LDAP lub Active Directory. Powiązanie można ustanowić używając nazwy komputera, która nie zawiera myślnika.
Jak skonfigurować dostęp do domeny Active Directory?

  1. Otwórz Preferencje systemowe i kliknij w Konta.

  2. Jeśli ikona kłódki jest zamknięta, otwórz ją klikając w nią i wpisując nazwę oraz hasło administratora.

  3. Kliknij w Opcje logowania, a następnie kliknij w Przyłącz lub Edycja.

  4. Kliknij w przycisk „Otwórz Narzędzie katalogowe”.

  5. Jeśli ikona kłódki jest zamknięta, otwórz ją klikając w nią i wpisując nazwę oraz hasło administratora.

  6. Kliknij w Usługi.

  7. Na liście usług, zaznacz Active Directory i kliknij w przycisk Edycja (/).

  8. Wpisz nazwę DNS domeny Active Directory, z którą chcesz powiązać konfigurowany komputer.

    Administrator domeny Active Directory może podać nazwę DNS do wpisania.

  9. Jeśli trzeba, edytuj ID komputera.

    ID komputera jest nazwą, pod którą komputer występuje w domenie Active Directory i jest domyślnie ustawione na nazwę komputera. Można go zmienić tak, aby odpowiadał schematowi nazywania komputerów w domenie Active Directory ustalonemu w organizacji. Jeśli nie jesteś pewien, zapytaj administratora domeny Active Directory.

  10. (Opcjonalnie) Ustaw opcje zaawansowane.

    Jeśli opcje zaawansowane są ukryte, kliknij w Pokaż opcje zaawansowane i ustaw opcje na panelach Użytkownik, Odwzorowania oraz Administrowanie. Opcje zaawansowane można też zmienić później.

    Więcej informacji o zaawansowanych opcjach można zobaczyć w:

    • Konfigurowanie przenośnych kont użytkowników w Active Directory

    • Konfigurowanie katalogów domowych dla kont użytkowników Active Directory

    • Konfigurowanie powłoki UNIX dla kont użytkowników Active Directory

    • Odwzorowanie UID na atrybut Active Directory

    • Odwzorowanie ID grupy podstawowej na atrybut Active Directory

    • Odwzorowanie ID grupy w kontach grup na atrybut Active Directory

    • Określanie preferowanego serwera Active Directory

    • Zmiana grup, które mogą administrować komputerami w Active Directory

    • Kontrolowanie uwierzytelniania ze wszystkich domen z lasu Active Directory

  11. Kliknij w Powiąż, uwierzytelnij się jako użytkownik posiadający prawa do powiązania komputera z domeną Active Directory, zaznacz zasady wyszukiwania, do których chcesz dodać Active Directory (zobacz poniżej) i kliknij w OK:

    • Nazwa użytkownika i hasło: Można uwierzytelnić się podając nazwę i hasło konta użytkownika Active Directory, albo administrator domeny Active Directory musi podać nazwę i hasło.

    • OU komputera: Podaj jednostkę organizacyjną (OU) konfigurowanego komputera.

    • Używaj do uwierzytelnienia: Używaj do określenia, czy Active Directory jest dodawana do zasad wyszukiwania danych uwierzytelnienia komputera.

    • Używaj do kontaktów: Używaj do określenia, czy Active Directory jest dodawana do zasad wyszukiwania kontaktów komputera.

    Po kliknięciu w OK, Narzędzie katalogowe ustanawia zaufane powiązanie pomiędzy konfigurowanym komputerem a serwerem Active Directory. Zasady wyszukiwania komputera zostają ustawione zgodnie z opcjami zaznaczonymi przy uwierzytelnieniu, a Active Directory zostaje włączony w panelu Usługi w Narzędziu katalogowym.

    Dla domyślnych ustawień zaawansowanych opcji Active Directory, las Active Directory zostaje dodany do zasad wyszukiwania uwierzytelnienia i kontaktów komputera, jeśli zostało zaznaczone „Używaj do uwierzytelnienia” lub „Używaj do kontaktów”.

    Jednak, jeśli przed kliknięciem w Powiąż nie jest zaznaczone „Pozwól na uwierzytelnienie z dowolnej domeny lasu” w panelu zaawansowanych opcji administracyjnych, zamiast lasu zostaje dodana najbliższa domena Active Directory .

    Można później zmienić zasady wyszukiwania dodając lub usuwając las Active Directory lub poszczególne domeny. Więcej informacji można przeczytać w sekcji Definiowanie własnych zasad wyszukiwania.

  12. (Opcjonalnie) Połącz serwer z dziedziną Kerberos Active Directory:

    • Na serwerze lub komputerze administratora, który może połączyć się z serwerem, otwórz program Server Admin i zaznacz Open Directory dla serwera.

    • Kliknij w Settings, a potem w General.

    • Kliknij w „Join Kerberos”, z rozwijanego menu wybierz dziedzinę Kerberos Active Directory i wpisz dane uwierzytelnienia lokalnego administratora serwera.
