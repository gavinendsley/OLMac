Dostęp do Active Directory

Można skonfigurować Mac OS X tak, aby miał dostęp do podstawowej informacji o koncie użytkownika w domenie Active Directory z serwera Windows 2000 lub nowszego. Jest to możliwe dzięki wtyczce Active Directory programu Narzędzie katalogowe. Jest ona wyświetlona w panelu Usługi w programie Narzędzie katalogowe.

Nie potrzeba dokonywać zmian schematu domeny Active Directory, żeby otrzymać podstawową informację o koncie użytkownika. Można zmienić domyślną listę kontroli dostępu (ACL) określonych atrybutów, żeby konta na komputerze mogły odczytywać właściwości użytkownika.

Wtyczka Active Directory generuje wszystkie wymagane atrybuty dla uwierzytelniania w Mac OS X ze standardowych atrybutów kont użytkowników Active Directory. Wtyczka obsługuje również zasady uwierzytelnienia Active Directory, włączając zmiany haseł, ich wygasanie, wymuszanie zmian oraz opcje bezpieczeństwa.

Mac OS X v10.6 obsługuje zarówno opcje szyfrowania pakietów, jak i podpisywania pakietów dla wszystkich domen Windows Active Directory. Ta funkcjonalność jest domyślnie włączona jako „pozwalaj”. Można zmienić domyślne ustawienia na „wyłączone” lub „wymagane”, używając narzędzia linii poleceń dsconfigad. Opcje szyfrowania i podpisywania pakietów zapewniają zabezpieczenie wszystkich danych wyszukiwania rekordów do i z domeny Active Directory.

Wtyczka Active Directory dynamicznie generuje unikalny ID użytkownika oraz ID grupy podstawowej w oparciu o GUID konta użytkownika w domenie Active Directory. Wygenerowane ID użytkownika i ID grupy podstawowej są takie same dla każdego konta użytkownika, nawet gdy konto używane jest do logowania na innych komputerach Mac OS X.

Zamiennie, można zmusić wtyczkę Active Directory do odwzorowania ID użytkownika na określone atrybuty Active Directory.

Wtyczka Active Directory generuje ID grupy w oparciu o GUID konta grupowego Active Directory. Można też zmusić wtyczkę do odwzorowania ID grupy dla kont grupowych na określone atrybuty Active Directory.

Kiedy ktoś loguje się do Mac OS X za pomocą konta użytkownika Active Directory, wtyczka Active Directory może zamontować sieciowy katalog domowy Windows określony w koncie użytkownika Active Directory jako katalog domowy Mac OS X. Można określić, czy używać sieciowego katalogu domowego określonego przez standardowy atrybut home Directory Active Directory lub przez atrybut home Directory Mac OS X (jeżeli schemat Active Directory został rozszerzony tak, żeby go zawierać).

Zamiennie, można skonfigurować wtyczkę tak, żeby tworzyła lokalny katalog domowy na woluminie startowym komputera klienta Mac OS X. W takim wypadku, wtyczka montuje również sieciowy katalog domowy użytkownika Windows (określony w koncie użytkownika Active Directory) jako wolumin sieciowy, tak jak punkt udostępniania. Używając Finder, użytkownik może wtedy kopiować pliki pomiędzy woluminem sieciowym katalogu domowego Windows a lokalnym katalogiem domowym Mac OS X.

Wtyczka Active Directory może też tworzyć przenośne konta dla użytkowników. Konto przenośne posiada lokalny katalog domowy na woluminie startowym komputera klienta Mac OS X. (Użytkownik ma też sieciowym katalog domowy zdefiniowany na koncie Active Directory użytkownika).

Przenośne konto zapamiętuje dane uwierzytelnienia użytkownika Active Directory na komputerze kliencie Mac OS X. Zapamiętanie danych uwierzytelnienia pozwala użytkownikowi logować się za pomocą nazwy i hasła Active Directory, gdy komputer klienta odłączony jest od serwera Active Directory.

Konto przenośne posiada lokalny katalog domowy na woluminie startowym komputera klienta Mac OS X. (Użytkownik ma też sieciowy katalog domowy określony na koncie Active Directory użytkownika).

Jeżeli schemat Active Directory został rozszerzony i zwiera typy rekordów (klasy obiektów) i atrybuty Mac OS X, wtyczka Active Directory wykrywa i używa ich.

Na przykład, schemat Active Directory może zostać zmieniony za pomocą narzędzi administracyjnych Windows i zawierać atrybuty zarządzanego klienta Mac OS X. Taka zmiana schematu pozwala wtyczce Active Directory na obsługę zarządzalnych ustawień klienta wykonanych za pomocą programu Workgroup Manager z Mac OS X Server.

Klienci Mac OS X zakładają pełen dostęp do odczytu atrybutów dodanych do katalogu. Tak więc, może być konieczna zmiana ACL tych atrybutów, żeby pozwalały grupie komputerów na odczytanie tych dodanych atrybutów.

Wtyczka Active Directory wykrywa wszystkie domeny w lesie Active Directory. Można skonfigurować wtyczkę, żeby pozwalała użytkownikom z dowolnych domen w lesie na uwierzytelnianie na komputerze Mac OS X. Zamiennie, można pozwolić tylko określonym domenom na uwierzytelnienie na kliencie.

Wtyczka Active Directory w pełni wspiera replikację i awaryjne przejmowanie Active Directory. Wykrywa wiele kontrolerów domen i wybiera najbliższy. Jeżeli kontroler domeny staje się niedostępny, wtyczka przechodzi na inny niedaleki kontroler domeny.

Wtyczka Active Directory używa LDAP do dostępu do kont użytkowników Active Directory oraz Kerberos do ich uwierzytelnienia. Wtyczka Active Directory nie używa interfejsu Microsoft Active Directory Services Interface (ADSI) z usługami katalogowymi i uwierzytelniającymi.
