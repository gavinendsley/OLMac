Ręczne konfigurowanie dostępu do katalogu LDAP

Można ręcznie utworzyć konfigurację, która określa dostęp Mac OS X do katalogu LDAPv3 lub LDAPv2. Trzeba znać nazwę DNS lub adres IP serwera katalogowego LDAP.

Jeżeli katalog nie jest utrzymywany na Mac OS X Server, trzeba znać podstawę wyszukiwania oraz szablon dla odwzorowań danych Mac OS X na dane katalogu. Obsługiwane szablony odwzorowań to:

  • Z serwera, dla katalogu dostarczającego swoje własne odwzorowania i podstawę wyszukiwania, takich jak Mac OS X Server

  • Serwer Open Directory, dla katalogu używającego schematu Mac OS X Server

  • Active Directory, dla katalogów z Windows 2000, Windows 2003 lub nowszych serwerów

  • RFC 2307, dla większości katalogów z serwerów UNIX

  • Własne, dla katalogów, które nie używają żadnych z powyższych odwzorowań

Wtyczka LDAPv3 w pełni obsługuje powielanie i przejmowanie Open Directory. Gdy nadrzędny serwer Open Directory staje się niedostępny, wtyczka połączy się z jego repliką.

Ważne:  Jeśli nazwa komputera zawiera myślnik, może wystąpić problem z łączeniem lub powiązaniem z domeną katalogową LDAP lub Active Directory. Powiązanie można ustanowić używając nazwy komputera, która nie zawiera myślnika.
Jak ręcznie skonfigurować dostęp do katalogu LDAP?

  1. Otwórz Preferencje systemowe i kliknij w Konta.

  2. Jeśli ikona kłódki jest zamknięta, otwórz ją klikając w nią i wpisując nazwę oraz hasło administratora.

  3. Kliknij w Opcje logowania, a następnie kliknij w Przyłącz lub Edycja.

  4. Kliknij w przycisk „Otwórz Narzędzie katalogowe”.

  5. Jeśli ikona kłódki jest zamknięta, otwórz ją klikając w nią i wpisując nazwę oraz hasło administratora.

  6. Kliknij w Usługi.

  7. Na liście usług, zaznacz LDAPv3 i kliknij w przycisk Edycja (/).

    Można zaznaczyć LDAPv3 na liście usług bez zaznaczania pola wyboru Włącz dla LDAPv3.

  8. Kliknij w Nowa, a potem kliknij w Ręcznie.

  9. Wpisz nazwę dla konfiguracji.

  10. Wciśnij tabulator i wprowadź nazwę DNS lub adres IP serwera udostępniającego katalog LDAP, do którego chcesz mieć dostęp.

  11. Obok nazwy DNS lub adresu IP, kliknij w rozwijane menu i wybierz szablon lub metodę odwzorowania:

    • Jeśli wybierzesz Z serwera, sufiks podstawy wyszukiwania nie jest potrzebny. W takim wypadku, Open Directory zakłada, że sufiks podstawy wyszukiwania jest pierwszym poziomem katalogu LDAP.

    • Jeśli wybierzesz szablon, wpisz sufiks podstawy wyszukiwania dla katalogu LDAP i kliknij w OK. Musisz wpisać sufiks podstawy wyszukiwania, bo inaczej komputer nie będzie mógł znaleźć informacji w katalogu LDAP.

      Zwykle, sufiks podstawy wyszukiwania jest pobierany z nazwy DNS serwera. Na przykład, sufiksem podstawy wyszukiwania może być „dc=ods,dc=przyklad,dc=com” dla serwera, którego nazwą DNS jest ods.przyklad.com.

    • Jeśli wybierzesz Własne, musisz skonfigurować odwzorowania pomiędzy typami rekordów Mac OS X a atrybutami i klasami atrybutów katalogu LDAP, z którym się łączysz. Więcej informacji można przeczytać w sekcji Konfigurowanie szukania i odwzorowań LDAP.

  12. Przed zaznaczeniem pola „Szyfruj używając SSL“, zapytaj administratora Open Directory, czy potrzebny jest SSL.

  13. Dla tej konfiguracji LDAP można zmienić następujące ustawienie, klikając w Edycja w celu wyświetlenia opcje dla zaznaczonej konfiguracji LDAP, dokonując zmian i klikając w OK po zakończeniu edytowania opcji konfiguracyjnych LDAP.

    • Kliknij w Połączenie, aby ustawić opcje limitu czasu, określić własny port, ignorować odwołania serwera lub wymusić używanie protokołu LDAPv2 (tylko do odczytu). Więcej informacji można przeczytać w sekcji Zmiana ustawień połączenia z katalogiem LDAP.

    • Kliknij w Szukanie i odwzorowania, aby ustawić wyszukiwanie oraz odwzorowania dla serwera LDAP. Więcej informacji można przeczytać w sekcji Konfigurowanie zaufanego powiązania z katalogiem LDAP.

    • Kliknij w Zabezpieczenia, aby ustawić uwierzytelnione połączenie (zamiast zaufanego powiązania) oraz inne opcje zasad bezpieczeństwa. Więcej informacji można przeczytać w sekcji Zmiana zasad bezpieczeństwa połączenia LDAP.

    • Kliknij w Powiąż, aby ustawić zaufane powiązanie (o ile katalog LDAP takie obsługuje). Więcej informacji można przeczytać w sekcji Konfigurowanie zaufanego powiązania z katalogiem LDAP.

  14. Kliknij w OK, aby zakończyć ręczne tworzenie konfiguracji dostępu do katalogu LDAP.

  15. Jeżeli komputer ma mieć dostęp do katalogu LDAP, którego konfigurację utworzyliśmy, należy dodać katalog do własnych zasad wyszukiwania w panelu Uwierzytelnienie oraz panelu Kontakty w Zasadach wyszukiwania w Narzędziu katalogowym, po czym upewnić się, że włączony jest LDAPv3 w panelu Usługi.

Więcej informacji można przeczytać w sekcji Włączanie lub wyłączanie usług katalogowych LDAP oraz Definiowanie własnych zasad wyszukiwania.

Uwaga:  Przed użyciem programu Workgroup Manager do tworzenia użytkowników na serwerach LDAP innych niż Apple, używających odwzorowania RFC 2307 (UNIX), trzeba zmienić odwzorowanie rekordu typu Użytkownicy. Więcej informacji można przeczytać w sekcji Edycja odwzorowań RFC 2307 do tworzenia użytkowników.
Ważne:  Jeżeli zmieniony został adres IP i nazwa komputera za pomocą changeip, podczas przyłączenia do serwera katalogowego, trzeba odłączyć się i ponownie podłączyć do serwera katalogowego, aby uaktualnić katalog nową nazwą komputera i adresem IP. Jeżeli się nie rozłączysz i ponownie podłączysz do serwera katalogowego, katalog nie zostanie uaktualniony i będzie nadal używał starej nazwy komputera i adresu IP.
