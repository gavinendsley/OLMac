Określanie ustawień plików konfiguracyjnych BSD

Historycznie, komputery UNIX przechowywały dane administracyjne w plikach konfiguracyjnych, takich jak /etc/master.passwd, /etc/group, czy /etc/hosts. Mac OS X opiera się na wersji BSD systemu UNIX, ale zwykle pobiera dane administracyjne z systemów katalogowych.

Mac OS X Server obsługuje stały zestaw plików konfiguracyjnych BSD. Nie można określić, których plików konfiguracyjnych używać, ani nie można odwzorować ich zawartości na typy rekordów i atrybuty Mac OS X.

W Mac OS X v10.2 lub nowszych (w tym Mac OS X Server v10.2 lub nowsze), Open Directory może pobierać dane administracyjne z plików konfiguracyjnych BSD. Możliwość ta pozwala organizacjom posiadającym pliki konfiguracyjne BSD na używanie kopi istniejących plików na komputerach Mac OS X. Pliki konfiguracyjne BSD mogą być używane same lub z innymi domenami katalogowymi.

Jak używać plików konfiguracyjnych BSD?

  1. Upewnij się, że pliki konfiguracyjne BSD zawierają dane wymagane przez usługi katalogowe Mac OS X.

    Więcej informacji można zobaczyć w Konfigurowanie danych w plikach konfiguracyjnych BSD.

  2. Otwórz Preferencje systemowe i kliknij w Konta.

  3. Jeśli ikona kłódki jest zamknięta, otwórz ją klikając w nią i wpisując nazwę oraz hasło administratora.

  4. Kliknij w Opcje logowania, a następnie kliknij w Przyłącz lub Edycja.

  5. Kliknij w przycisk „Otwórz Narzędzie katalogowe”.

  6. Jeśli ikona kłódki jest zamknięta, otwórz ją klikając w nią i wpisując nazwę oraz hasło administratora.

  7. Kliknij w Usługi.

  8. Na liście usług, zaznacz „Plik jednorodny BSD oraz NIS“ i kliknij w przycisk Edycja (/).

  9. Zaznacz „Użyj rekordów Użytkowników i grup w lokalnym węźle BSD“ i kliknij w OK.

    Domena plików konfiguracyjnych BSD zostanie dodana do zasady wyszukiwania uwierzytelnienia komputera jako /BSD/local.
