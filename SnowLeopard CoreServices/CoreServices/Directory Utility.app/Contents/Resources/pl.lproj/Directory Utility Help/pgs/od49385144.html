Konfigurowanie katalogów domowych dla kont użytkowników Active Directory

Na komputerze skonfigurowanym w Narzędziu katalogowym do używania wtyczki Active Directory można włączyć lub wyłączyć sieciowe lub lokalne katalogi domowe na kontach użytkowników Active Directory.

Dla sieciowych katalogów domowych, kiedy użytkownik loguje się, sieciowy katalog domowy użytkownika Windows jest montowany jako katalog domowy Mac OS X.

Określamy, czy położenie sieciowego katalogu domowego jest pobierane ze standardowego atrybutu homeDirectory Active Directory, czy z atrybutu homeDirectory Mac OS X, jeśli schemat Active Directory został rozszerzony i go zawiera.

Dla lokalnych katalogów domowych, każdy użytkownik Active Directory, który się loguje, ma katalog domowy na dysku startowym Mac OS X. Dodatkowo, sieciowy katalog domowy użytkownika jest montowany jako wolumin sieciowy, tak jak punkt udostępniania. Użytkownik może kopiować pliki pomiędzy takim woluminem sieciowym a lokalnym katalogiem domowym.

Jak skonfigurować katalogi domowe dla kont użytkowników Active Directory?

  1. Otwórz Preferencje systemowe i kliknij w Konta.

  2. Jeśli ikona kłódki jest zamknięta, otwórz ją klikając w nią i wpisując nazwę oraz hasło administratora.

  3. Kliknij w Opcje logowania, a następnie kliknij w Przyłącz lub Edycja.

  4. Kliknij w przycisk „Otwórz Narzędzie katalogowe”.

  5. Jeśli ikona kłódki jest zamknięta, otwórz ją klikając w nią i wpisując nazwę oraz hasło administratora.

  6. Kliknij w Usługi.

  7. Na liście usług, zaznacz Active Directory i kliknij w przycisk Edycja (/).

  8. Jeśli ukryte są opcje zaawansowane, kliknij w Pokaż opcje zaawansowane.

  9. Kliknij w Użytkownik.

  10. Jeśli konta użytkowników Active Directory mają mieć lokalne katalogi domowe w katalogu /Użytkownicy komputera, kliknij w „Wymuś lokalny katalog domowy na dysku startowym“.

    Opcja ta jest niedostępna, jeśli zaznaczone jest „Utwórz konto przenośne podczas logowania”.

  11. Zaznacz „Użyj ścieżki UNC z Active Directory do utworzenia ścieżki katalogu domowego“, aby używać standardowego atrybutu Active Directory dla położenia katalogu domowego, po czym wybierz jeden z dostępnych protokołów dostępu do katalogu domowego:

    • Wybierz smb z rozwijanego menu „Używany protokół sieciowy“, żeby używać standardowego protokołu Windows: SMB.

    • Wybierz aft z rozwijanego menu „Używany protokół sieciowy“, żeby używać standardowego protokołu Macintosh: AFP.

  12. Zdejmij zaznaczenie z opcji „Użyj ścieżki UNC z Active Directory do utworzenia ścieżki katalogu domowego“, żeby używać atrybutu Mac OS X dla położenia katalogu domowego.

    Schemat Active Directory musi być rozszerzony, żeby używać atrybutu Mac OS X.

  13. Następnie kliknij w przycisk OK.

Jeśli zmieni się nazwę konta użytkownika w domenie Active Directory, serwer tworzy katalog domowy (i podkatalogi) dla konta użytkownika, gdy następnym razem jest ono używane do logowania na komputerze Mac OS X. Użytkownik może ciągle przechodzić do starego katalogu domowego i oglądać jego zawartość w Finder.

Można zapobiec tworzeniu nowych katalogów domowych zmieniając nazwę starego katalogu przed następnym logowaniem użytkownika.
