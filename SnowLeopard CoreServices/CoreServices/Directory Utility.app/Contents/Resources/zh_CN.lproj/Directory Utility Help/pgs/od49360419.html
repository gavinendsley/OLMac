手动配置 LDAP 目录访问

您可以手动创建一个配置来指定 Mac OS X 如何访问 LDAPv3 或 LDAPv2 目录。您必须知道 LDAP 目录服务器的 DNS 名称或 IP 地址。

如果该目录不是由 Mac OS X Server 托管，您必须知道将 Mac OS X 数据映射至目录数据的搜索基准和模板。支持的映射模板有：

  • “来自服务器”，用于提供自己的映射和搜索基准的目录，例如 Mac OS X Server

  • “Open Directory 服务器”，用于使用 Mac OS X Server 模式的目录

  • “Active Directory”，用于由 Windows 2000、Windows 2003 或更高版本的服务器托管的目录

  • “RFC 2307”，用于由 UNIX 服务器托管的大多数目录

  • “自定”，用于不使用以上任何映射的目录

LDAPv3 插件完全支持 Open Directory 复制和故障转移。如果 Open Directory 主服务器不可用，插件会退回到附近的复制服务器。

重要事项：  如果电脑名称包含连字符，您可能无法加入或捆绑到 LDAP 或 Active Directory 等目录域。要建立捆绑，请使用不包含连字符的电脑名称。
要手动配置 LDAP 目录访问：

  1. 打开“系统偏好设置”并点按“帐户”。

  2. 如果锁图标已锁定，点按该图标然后输入管理员的名称和密码，即可将其解锁。

  3. 点按“登陆选项”，然后点按“加入”或“编辑”。

  4. 点按“打开目录实用工具”。

  5. 如果锁图标已锁定，点按该图标然后输入管理员的名称和密码，即可将其解锁。

  6. 点按“服务”。

  7. 在服务列表中选择“LDAPv3”，然后点按编辑按钮 (/)。

    您可以在服务列表中选择“LDAPv3”，而不用选择 LDAPv3 的“启用”注记格。

  8. 点按“新建”，然后点按“手动”。

  9. 输入配置的名称。

  10. 按下 Tab 键并输入托管您要访问的 LDAP 目录的那个服务器的 DNS 名称或 IP 地址。

  11. 在 DNS 名称或 IP 地址旁边，点按弹出式菜单，选择一个映射模板或方法：

    • 如果选择“来自服务器”，则不需要搜索基准后缀。在此情况下，Open Directory 假设搜索基准后缀是 LDAP 目录的第一级。

    • 如果您选取一个模板，请输入 LDAP 目录的搜索基准后缀，然后点按“好”。您必须输入一个搜索基准后缀，否则电脑无法在 LDAP 目录中查找信息。

      通常，搜索基准后缀源自服务器的 DNS 名称。例如，如果服务器的 DNS 名称是 ods.example.com，搜索基准后缀就可能是“dc=ods,dc=example,dc=com”。

    • 如果您选取“自定”，则需要在 Mac OS X 记录类型和属性与您要连接到的 LDAP 目录的类和属性之间设置映射。有关更多信息，请参阅“配置 LDAP 搜索和映射”。

  12. 在选择“使用 SSL 加密”注记格之前，请与 Open Directory 管理员核实是否需要 SSL。

  13. 要更改此项 LDAP 配置的以下设置，点按“编辑”可显示出所选 LDAP 配置的选项，进行更改，在完成编辑 LDAP 配置选项后点按“好”。

    • 点按“连接”以设置超时选项、指定自定端口、忽略服务器推荐或者强制使用 LDAPv2（只读）协议。有关更多信息，请参阅“更改 LDAP 目录的连接设置”。

    • 点按“搜索与映射”设置 LDAP 服务器的搜索和映射。有关更多信息，请参阅“建立与 LDAP 目录的信任捆绑”。

    • 点按“安全”以设置鉴定连接（不是信任捆绑）和其他安全策略选项。有关更多信息，请参阅“更改 LDAP 连接的安全策略”。

    • 点按“捆绑”建立信任捆绑（如果 LDAP 目录支持）。有关更多信息，请参阅“建立与 LDAP 目录的信任捆绑”。

  14. 点按“好”完成手动创建访问 LDAP 目录的配置。

  15. 如果您想要电脑访问您为其创建了配置的 LDAP 目录，请将该目录添加到“目录实用工具”的“鉴定”面板和“联系”面板中的自定搜索策略，并确定在“服务”面板中启用了 LDAPv3。

有关更多信息，请参阅“启用或停用 LDAP 目录服务”和“定义自定搜索策略”。

注：您必须先编辑“用户”记录类型的映射，才可以使用 Workgroup Manager 在使用 RFC 2307 (UNIX) 映射的非 Apple LDAP 服务器上创建用户。有关更多信息，请参阅“编辑 RFC 2307 映射以启用创建用户”。
重要事项：在您连接到目录服务器时如果要使用 changeip 更改 IP 地址和电脑名称，则必须断开连接并重新连接目录服务器，以使用新的电脑名称和 IP 地址更新目录。如果您不断开连接并重新连接目录服务器，则目录将不更新，并继续使用旧的电脑名称和 IP 地址。
