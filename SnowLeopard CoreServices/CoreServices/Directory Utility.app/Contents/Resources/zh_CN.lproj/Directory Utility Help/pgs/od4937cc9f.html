配置 LDAP 目录访问

使用“目录实用工具”，您可以指定 Mac OS X 如何访问 LDAPv3，但是您必须知道 LDAP 目录服务器的 DNS 名称或 IP 地址。

如果该目录不是由提供它自己的映射的服务器（例如 Mac OS X Server）托管，则必须知道将 Mac OS X 数据映射至目录数据的搜索基准和模板。

支持的映射模板有：

  • “Open Directory 服务器”，用于使用 Mac OS X Server 模式的目录

  • “Active Directory”，用于由 Windows 2000、Windows 2003 或更高版本的服务器托管的目录

  • “RFC 2307”，用于由 UNIX 服务器托管的大多数目录

LDAPv3 插件完全支持 Open Directory 复制和故障转移。如果 Open Directory 主服务器不可用，插件会退回到附近的复制服务器。

要为目录数据指定自定映射，请按照手动配置 LDAP 目录访问中的说明操作，而不是按照此处的说明。

重要事项：  如果电脑名称包含连字符，您可能无法加入或捆绑到 LDAP 或 Active Directory 等目录域。要建立捆绑，请使用不包含连字符的电脑名称。
要使用“目录实用工具”帮助配置 LDAP 目录访问：

  1. 打开“系统偏好设置”并点按“帐户”。

  2. 如果锁图标已锁定，点按该图标然后输入管理员的名称和密码，即可将其解锁。

  3. 点按“登陆选项”，然后点按“加入”或“编辑”。

  4. 点按“打开目录实用工具”。

  5. 如果锁图标已锁定，点按该图标然后输入管理员的名称和密码，即可将其解锁。

  6. 点按“服务”。

  7. 在服务列表中选择“LDAPv3”，然后点按编辑按钮 (/)。

    您可以在服务列表中选择“LDAPv3”，而不用选择 LDAPv3 的“启用”注记格。

  8. 点按“新建”，然后输入 LDAP 服务器的 DNS 名称或 IP 地址。

  9. 选择用于访问目录的选项：

    • 如果您想要 Open Directory 为与 LDAP 目录的连接使用安全套接字层 (SSL)，请选择“使用 SSL 加密”。 在选择此选项之前，请询问 Open Directory 管理员是否需要 SSL。

    • 如果此目录包含某人登录或鉴定服务时要使用的用户帐户，请选择“用于鉴定”。

    • 如果此目录包含“地址簿”中要使用的邮件地址或其他信息，请选择“用于联系”。

    如果“目录实用工具”不能联系到 LDAP 服务器，会显示一则信息，您必须手动配置访问或取消设置过程。有关手动配置说明的更多信息，请参阅“手动配置 LDAP 目录访问”。

    如果展开对话框以显示映射选项，请从弹出式菜单中选取映射模板，输入搜索基准后缀，然后点按“继续”。

    通常，搜索基准后缀源自服务器的 DNS 名称。例如，如果服务器的 DNS 名称是 ods.example.com，搜索基准后缀就可能是“dc=ods,dc=example,dc=com”。

    如果没有可用的映射模板应用于您正在设置的连接，请点按“手动”。有关更多信息，请参阅“手动配置 LDAP 目录访问”。

  10. 要让“目录实用工具”从 LDAP 服务器获得信息，请点按“继续”。

  11. 如果对话框展开，显示信任捆绑的选项，请输入电脑的名称以及目录管理员的名称和密码。（捆绑可能是可选的。）

    该对话框说明 LDAP 目录是要求信任捆绑还是使其成为可选。信任捆绑是相互的：每次电脑连接到 LDAP 目录时，它们会相互鉴定。如果已设置信任捆绑或 LDAP 目录不支持信任捆绑，“捆绑”按钮不会显示。确定您提供了正确的电脑名称。

    如果您看到一则警告，提示某个电脑记录存在，请点按“取消”返回并更改电脑名称，或点按“覆盖”替换该电脑记录。

    现有的电脑记录可能被放弃，也可能属于另一电脑。

    如果您替换现有的电脑记录，请通知 LDAP 目录管理员，以防替换该记录会停用另一电脑。在此情况下，LDAP 目录管理员必须为被停用的电脑指定一个不同的名称，并且将其重新添加到所属的电脑组别。

  12. 如果对话框展开，显示连接选项，请选择“连接时使用鉴定”，并在目录中输入用户帐户的识别名称和密码。

    如果 LDAP 服务器支持鉴定连接，但不支持信任捆绑，将会出现鉴定连接的选项。鉴定连接不是相互的：LDAP 服务器鉴定客户端，但客户端不会鉴定服务器。

    对于已鉴定的连接，如果 LDAP 服务器要求您输入用户帐户识别名称和密码，“连接时使用鉴定”将会被预先选定，但是显示为暗淡。

    识别名称可以指明有权限查看目录数据的任何用户帐户。例如，LDAP 服务器上短名称是“dirauth”、地址是 ods.example.com 的用户帐户，它的识别名称将为“uid=dirauth,cn=users,dc=ods,dc=example,dc=com”。

    重要事项：如果识别名称或密码不正确，您可以使用 LDAP 目录中的用户帐户登录电脑。

  13. 点按“好”完成创建 LDAP 连接。

  14. 点按“好”完成配置 LDAPv3 选项。

如果您在步骤 5 中选择了“用于鉴定”或者“用于联系”，您刚才创建的 LDAP 目录配置将会添加到“目录实用工具”的“鉴定”或“联系”面板的自定搜索策略中。

确定在“服务”面板中已启用 LDAPv3，这样电脑将使用您刚才创建的 LDAP 配置。有关更多信息，请参阅“启用或停用 LDAP 目录服务”。
