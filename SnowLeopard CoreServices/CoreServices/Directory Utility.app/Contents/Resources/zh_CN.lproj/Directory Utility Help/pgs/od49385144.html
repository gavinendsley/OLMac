设置 Active Directory 用户帐户的个人文件夹

您可以在配置为使用“目录实用工具”的 Active Directory 连接器的电脑上启用或停用 Active Directory 用户帐户的网络个人文件夹或本地个人文件夹。

有了网络个人文件夹，用户就可以在登录时像装载 Mac OS X 个人文件夹一样装载其 Windows 网络个人文件夹。

如果 Active Directory 模式已经扩展为包含网络个人文件夹，您可以确定网络个人文件夹位置是来自 Active Directory 的标准 HomeDirectory 属性还是来自 Mac OS X 的 HomeDirectory 属性。

有了本地个人文件夹，每个登录的 Active Directory 用户都将在 Mac OS X 启动盘上拥有一个个人文件夹。此外，用户的网络个人文件夹将装载为网络宗卷（如共享点）。用户可以在此网络宗卷和本地个人文件夹之间拷贝文件。

要设置 Active Directory 用户帐户的个人文件夹：

  1. 打开“系统偏好设置”并点按“帐户”。

  2. 如果锁图标已锁定，点按该图标然后输入管理员的名称和密码，即可将其解锁。

  3. 点按“登陆选项”，然后点按“加入”或“编辑”。

  4. 点按“打开目录实用工具”。

  5. 如果锁图标已锁定，点按该图标然后输入管理员的名称和密码，即可将其解锁。

  6. 点按“服务”。

  7. 在服务列表中选择“Active Directory”，然后点按编辑按钮 (/)。

  8. 如果高级选项处于隐藏状态，请点按“显示高级选项”。

  9. 点按“用户体验”。

  10. 如果您想要 Active Directory 用户帐户的本地个人文件夹位于电脑的 /Users 文件夹中，请点按“强制本地个人文件夹位于启动盘”。

    如果选择了“登录时创建移动帐户”，此选项将不可用。

  11. 要使用 Active Directory 的标准属性确定个人文件夹位置，请选择“使用来自 Active Directory 的 UNC 路径获得网络个人位置”，然后从下面选取用于访问此个人文件夹的协议：

    • 要使用标准 Windows 协议 SMB，请从“要使用的网络协议”弹出式菜单中选取“smb”。

    • 要使用标准 Macintosh 协议 AFP，请从“要使用的网络协议”弹出式菜单中选取“afp”。

  12. 要使用 Mac OS X 的属性确定个人文件夹位置，请取消选择“使用来自 Active Directory 的 UNC 路径获得网络个人位置”。

    要使用 Mac OS X 的属性，必须将 Active Directory 模式扩展为包含 Mac OS X 的属性。

  13. 点按“好”。

如果更改 Active Directory 域中一个用户帐户的名称，则下次使用该用户帐户登录到 Mac OS X 电脑时，服务器将为该用户帐户创建个人文件夹（以及子文件夹）。用户仍可以导航到旧个人文件夹，在 Finder 中查看其内容。

您可以在用户下次登录前重新命名旧文件夹，阻止创建新个人文件夹。
