保护电脑免受来自恶意 DHCP 服务器的攻击

Apple 建议，在安全是一个重要考虑因素的情况下，请不要使用自动鉴定搜索策略（包括 DHCP 提供的 LDAP 服务器和/或 DHCP 提供的共享目录域）。

具有您的网络访问权的恶意电脑黑客可以利用 root 用户帐户，使用伪 DHCP 服务器和伪 LDAP 目录（或共享的目录域）来控制您的电脑。

黑客要访问您的网络，他的伪 DHCP 服务器必须是您的本地网络或子网的组成部分。因此，如果在您的本地网络上只包括您的电脑，而且这些电脑通过 Mac OS X Server NAT 服务或 NAT 路由器访问 Internet，则不会存在这种安全漏洞。不过，无线本地网络会使安全性降低，因为和有线本地网络相比，黑客更容易进入无线本地网络。

您可以保护您的 Mac 免受来自伪 DHCP 服务器的恶意攻击，方法是停用 DHCP 提供的 LDAP 目录并停用广播和 DHCP 对本地目录域的捆绑（或者停用本地目录域）。

如果您有一台移动电脑，在将电脑连接到网络时，您将其连接到 LDAP 服务器，并且您将电脑的搜索策略从自动更改为自定（在“目录实用工具”中“搜索策略”的“鉴定”面板中），则电脑在没有连接到网络时会发生启动延迟。

发生延迟的原因是电脑不能连接到在电脑的自定搜索策略中列出的特定目录域。将与网络断开连接的电脑从睡眠状态唤醒时，不会注意到延迟。
