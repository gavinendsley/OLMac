更改 LDAP 连接的安全策略

使用“目录实用工具”，您可以配置比 LDAP 目录的安全策略更严格的 LDAPv3 连接安全策略。例如，如果 LDAP 目录的安全策略允许明文密码，您可以将 LDAPv3 连接设置为不允许明文密码。

设置更严格的安全策略将防止恶意黑客使用伪 LDAP 服务器控制您的电脑。

电脑必须与 LDAP 服务器通信，才能准确地显示安全选项的状态。因此在更改 LDAPv3 连接的安全选项时，电脑的鉴定搜索策略应该包括 LDAPv3 连接。

LDAPv3 连接的安全选项中可允许的设置是由 LDAP 服务器的安全功能和要求决定的。例如，如果 LDAP 服务器不支持 Kerberos 鉴定，则 LDAPv3 安全连接选项中有多个选项被停用。

要更改 LDAPv3 连接的安全选项：

  1. 打开“系统偏好设置”并点按“帐户”。

  2. 如果锁图标已锁定，点按该图标然后输入管理员的名称和密码，即可将其解锁。

  3. 点按“登陆选项”，然后点按“编辑”。

  4. 点按“打开目录实用工具”。

  5. 如果锁图标已锁定，点按该图标然后输入管理员的名称和密码，即可将其解锁。

  6. 点按“搜索策略”。

  7. 点按“鉴定”，确定在搜索策略中列出了您需要的 LDAPv3 目录。

    有关将 LDAPv3 目录添加到鉴定搜索策略的更多信息，请参阅“定义自定搜索策略”。

  8. 点按“服务”。

  9. 在服务列表中选择“LDAPv3”，然后点按编辑按钮 (/)。

  10. 如果服务器配置列表处于隐藏状态，请点按“显示选项”。

  11. 选择您需要的目录的配置，然后点按“编辑”。

  12. 点按“安全”，然后更改以下任何设置。

    注：当 LDAP 连接建立时，即确定了此处以及对应 LDAP 服务器上的安全设置。当服务器设置更改时，这些设置不会更新。

    如果最后四个选项中的任何一个已选定但已停用，则说明 LDAP 目录要求使用它们。如果上述选项中的任何选项没有选定并且已停用，则表示 LDAP 服务器不支持它们。

    • 连接时使用鉴定：确定 LDAPv3 连接是否提供指定的识别名称和密码以在 LDAP 目录进行自我鉴定。如果 LDAPv3 使用与 LDAP 目录的信任捆绑，则该选项不可见。

    • 捆绑到目录，身份为：指定 LDAPv3 连接用于与 LDAP 目录进行信任捆绑的凭证。此处不能更改该选项和凭证。您可以先解开捆绑，然后以不同的凭证再次捆绑。

      有关更多信息，请参阅“停止 LDAP 目录的信任捆绑”和“建立与 LDAP 目录的信任捆绑”。

      如果 LDAPv3 没有使用信任捆绑，该选项则不可见。

    • 停用明码文本密码：确定如果使用发送加密密码的鉴定方法无法验证密码时是否以明码文本发送该密码。

    • 通过数码方式为所有数据包签名（要求 Kerberos）：确保来自 LDAP 服务器的目录数据在传输到您的电脑的途中没有被另一电脑截取或修改。

    • 加密所有数据包（要求 SSL 或 Kerberos）：要求 LDAP 服务器在将目录数据发送到您的电脑之前使用 SSL 或 Kerberos 加密这些数据。在选择“加密所有数据包（要求 SSL 或 Kerberos）”注记格之前，请询问 Open Directory 管理员是否需要 SSL。

    • 阻止中间人攻击（要求 Kerberos）：防止伪服务器冒充 LDAP 服务器。与“通过数码签名方式为所有数据包签名”选项一起使用将获得最佳效果。
