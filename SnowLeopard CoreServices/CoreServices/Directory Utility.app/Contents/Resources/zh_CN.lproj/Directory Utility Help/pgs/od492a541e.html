编写密码

当用户对登录或其他服务进行鉴定时，需要输入与用户帐户关联的密码。密码是区分大小写的（除了 SMB-LAN Manager 密码），并且输入时在屏幕上被遮盖起来。

不管您为用户选取了哪种密码类型，都应该遵守以下 Mac OS X Server 用户帐户密码编写指导原则：

  • 密码应该包含字母、数字和符号的组合，而且不会被未授权用户轻易猜出。 密码不应由词语组成。好的密码包括数字和符号（如 # 或 $），或者由短语中所有词语的第一个字母组成。同时使用大写和小写字母。

  • 避免使用空格和 Option 按键组合。

  • 避免使用在用户电脑上无法输入的字符或可能需要了解特殊的按键组合才能在另一种键盘和平台上正确输入的字符。

  • 有些网络协议不支持含有前置空格、内嵌空格或拖尾空格的密码。

  • 也不建议使用长度为零的密码。Open Directory 和有些系统（如 LDAP 捆绑）不支持零长度的密码。

  • 为了与您的用户可能需要访问的电脑和服务保持最大的兼容性，请只在密码中使用 ASCII 字符。
