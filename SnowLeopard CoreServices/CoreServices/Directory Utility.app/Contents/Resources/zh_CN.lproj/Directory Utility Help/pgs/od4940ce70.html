设置对 Active Directory 域的 LDAP 访问

使用“目录实用工具”，您可以设置 LDAPv3 配置，以访问 Windows 服务器上的 Active Directory 域。 LDAPv3 配置让您可以充分控制如何将 Mac OS X 记录类型和属性映射至 Active Directory 对象类、搜索基准和属性。

映射唯一用户 ID (UID) 等某些重要的 Mac OS X 记录类型和属性时需要扩展 Active Directory 模式。

LDAPv3 配置不包括“目录实用工具”中列出的 Active Directory 连接器的以下功能：

  • 动态生成唯一用户 ID 和首选组别 ID

  • 创建本地 Mac OS X 个人文件夹

  • 自动装载 Windows 个人文件夹

  • 使用高速缓存的鉴定凭证的移动用户帐户

  • 发现 Active Directory 网域树系中的所有域

  • 支持 Active Directory 复制和故障转移

有关更多信息，请参阅“关于 Active Directory 访问”。

要创建 Active Directory 服务器配置：

  1. 打开“系统偏好设置”并点按“帐户”。

  2. 如果锁图标已锁定，点按该图标然后输入管理员的名称和密码，即可将其解锁。

  3. 点按“登陆选项”，然后点按“加入”或“编辑”。

  4. 点按“打开目录实用工具”。

  5. 如果锁图标已锁定，点按该图标然后输入管理员的名称和密码，即可将其解锁。

  6. 点按“服务”。

  7. 在服务列表中选择“LDAPv3”，然后点按编辑按钮 (/)。

  8. 点按“新建”，然后输入 Active Directory 服务器的 DNS 名称或 IP 地址。

  9. 选择以下用于访问该目录的选项，然后点按“继续”让“目录实用工具”从 Active Directory 服务器获得信息。

    • 如果您想要 Open Directory 对于与 Active Directory 的连接使用 SSL，请选择“使用 SSL 加密”。在选择 SSL 注记格之前，请询问 Open Directory 管理员是否需要 SSL。

    • 如果此目录包含某人登录时或鉴定服务时要使用的用户帐户，请选择“用于鉴定”。

    • 如果此目录包含“地址簿”中要使用的邮件地址或其他信息，请选择“用于联系”。

    如果“目录实用工具”不能联系到 Active Directory 服务器，会显示一则信息，您必须手动配置访问或取消设置过程。有关更多信息，请参阅“手动配置 LDAP 目录访问”。

    如果您选择了“用于鉴定”或“用于联系”，则与 Active Directory 域的 LDAPv3 连接将会添加到“目录实用工具”的“鉴定”或“联系”面板的自定搜索策略中。

    确定在“服务”面板中启用了 LDAPv3，这样电脑才可以使用您设置的连接。有关更多信息，请参阅“启用或停用 LDAP 目录服务”。

  10. 当对话框展开显示出映射选项时，请从弹出式菜单中选取“Active Directory”，输入搜索基准，然后点按“继续”。

    LDAPv3 配置的 Active Directory 映射模板将某些 Mac OS X 记录类型和属性映射至不属于标准 Active Directory 模式的对象类和属性。您可以更改该模板定义的映射，也可以扩展 Active Directory 模式。

    另外，您也可以通过 Active Directory 连接器而不是 LDAPv3 来访问您的 Active Directory 域。有关更多信息，请参阅“配置 Active Directory 域访问”。

  11. 当对话框展开显示出连接选项时，输入 Active Directory 用户帐户的识别名和密码。

  12. 点按“好”完成创建 LDAP 连接。

  13. 点按“好”完成配置 LDAPv3 选项。
