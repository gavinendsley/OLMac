关于 Active Directory 访问

您可以配置 Mac OS X 以便访问 Windows 2000 或更高版本服务器的 Active Directory 域中的基本用户帐户信息。之所以可能这样做，是因为有了“目录实用工具”的 Active Directory 连接器。Active Directory 连接器在“目录实用工具”的“服务”面板中列出。

您无需对 Active Directory 域进行任何模式更改，便可以获得基本用户帐户信息。您可以更改特定属性的默认访问控制列表 (ACL)，使电脑帐户能够读取用户属性。

Active Directory 连接器根据 Active Directory 用户帐户的标准属性生成所有 Mac OS X 鉴定所需的属性。该连接器还支持 Active Directory 鉴定策略，包括密码更改、过期、强制更改和安全选项。

Mac OS X v10.6 支持所有 Windows Active Directory 域的数据包加密和数据包签名选项。这项功能为打开状态，默认为“允许”。您可以使用命令行工具 dsconfigad 将默认设置更改为“停用”或“需要”。数据包加密和数据包签名选项，保证所有进出 Active Directory 域用于记录查找的数据都能受到保护。

Active Directory 连接器根据 Active Directory 域中用户帐户的全局唯一标识符 (GUID) 动态生成唯一的用户 ID 和首选组别 ID。为每个用户帐户生成的用户 ID 和首选组别 ID 是相同的，即使是用该帐户登录不同的 Mac OS X 电脑。

另外，您可以强制 Active Directory 连接器将用户 ID 映射至指定的 Active Directory 属性。

Active Directory 连接器会根据 Active Directory 组别帐户的 GUID 生成一个组别 ID。您还可以强制 Active Directory 插件将组别帐户的组别 ID 映射至指定的 Active Directory 属性。

有人使用 Active Directory 用户帐户登录 Mac OS X 时，Active Directory 连接器可以将 Active Directory 用户帐户中指定的 Windows 网络个人文件夹装载为用户的 Mac OS X 个人文件夹。如果 Active Directory 模式已经扩展为包括网络个人目录，您可以指定是使用由 Active Directory 的标准 homeDirectory 属性还是 Mac OS X 的 HomeDirectory 属性指定的网络个人目录。

另外，您可以配置插件，以便在 Mac OS X 客户端电脑的启动宗卷中创建本地个人文件夹。 在这种情况下，该插件还将用户的 Windows 网络个人目录（在 Active Directory 用户帐户中指定）装载为网络宗卷（如共享点）。使用 Finder，用户可以在 Windows 个人文件夹网络宗卷和本地 Mac OS X 个人文件夹之间拷贝文件。

Active Directory 连接器还可以为用户创建移动帐户。Mac OS X 客户端电脑的启动宗卷中有移动帐户的本地个人文件夹。（用户还有一个网络个人文件夹，该文件夹在用户的 Active Directory 帐户中指定。）

移动帐户将用户的 Active Directory 鉴定凭证缓存在 Mac OS X 客户端电脑中。客户端电脑与 Active Directory 服务器的连接断开时，缓存的凭证允许用户使用 Active Directory 名称和密码登录。

Mac OS X 客户端电脑的启动宗卷中有移动帐户的本地个人文件夹。（用户还有一个网络个人文件夹，该文件夹在用户的 Active Directory 帐户中指定。）

如果 Active Directory 模式已经扩展为包括 Mac OS X 记录类型（对象类）和属性，Active Directory 连接器将对其进行探测和访问。

例如，可以使用 Windows 管理工具将 Active Directory 模式更改为包括 Mac OS X 管理的客户端属性。该模式更改将启用 Active Directory 连接器，以支持使用 Mac OS X Server 的 Workgroup Manager 应用程序对被管理客户端进行的设置。

Mac OS X 客户端假定对添加到目录的属性具有完全的读取权限。因此，可能必须更改这些属性的 ACL，以允许电脑组读取这些添加的属性。

Active Directory 连接器会寻找 Active Directory 网域树系中的所有域。您可以配置该插件，允许来自该网域树系中任何域的用户在 Mac OS X 电脑上鉴定。或者，您可以只允许在客户端上鉴定特定的域。

Active Directory 连接器完全支持 Active Directory 复制和故障转移。它会寻找多个域控制器，然后确定最近的一个。如果一个域控制器变为不可用，该插件将自动后退到附近的另一个域控制器。

Active Directory 连接器使用 LDAP 访问 Active Directory 用户帐户，使用 Kerberos 对其进行鉴定。Active Directory 连接器不使用 Microsoft 专用的 Active Directory Services Interface（活动目录服务接口，ADSI）获得目录或鉴定服务。
