建立与 LDAP 目录的信任捆绑

您可以使用“目录实用工具”来建立电脑与支持信任捆绑的 LDAP 目录之间的信任捆绑。 通过在建立信任捆绑时在目录中创建的已鉴定电脑记录来相互鉴定该捆绑。

不能将电脑配置为使用信任的 LDAP 捆绑和 DHCP 提供的 LDAP 目录。信任的 LDAP 捆绑本质上是静态的捆绑，而 DHCP 提供的 LDAP 则是动态的捆绑。

要建立与 LDAP 目录的信任捆绑：

  1. 打开“系统偏好设置”并点按“帐户”。

  2. 如果锁图标已锁定，点按该图标然后输入管理员的名称和密码，即可将其解锁。

  3. 点按“登陆选项”，然后点按“加入”或“编辑”。

  4. 点按“打开目录实用工具”。

  5. 如果锁图标已锁定，点按该图标然后输入管理员的名称和密码，即可将其解锁。

  6. 点按“服务”。

  7. 在服务列表中选择“LDAPv3”，然后点按编辑按钮 (/)。

  8. 如果服务器配置列表处于隐藏状态，请点按“显示选项”。

  9. 选择您想要编辑的服务器配置，然后点按“编辑”。

  10. 点按“捆绑”，输入下面的凭证，然后点按“好”。

    输入电脑名称和 LDAP 目录域管理员的名称和密码。该电脑名称不能由另一电脑用于信任捆绑或其他网络服务。

    如果没有显示“捆绑”按钮，则表示 LDAP 目录不支持信任捆绑。

  11. 确定您提供了正确的电脑名称。

    如果您看到一则警告，提示某个电脑记录已存在，请点按“取消”返回并更改电脑名称或点按“覆盖”替换现有的电脑记录。

    现有的电脑记录可能被放弃，也可能属于另一台电脑。如果您替换现有的电脑记录，请通知 LDAP 目录管理员，以防替换该记录会停用另一电脑。

    在此情况下，LDAP 目录管理员必须为停用的电脑指定另一个名称，使用不同的名称将该电脑添加到它所属的电脑组别。

  12. 要完成建立信任捆绑，请点按“好”。
