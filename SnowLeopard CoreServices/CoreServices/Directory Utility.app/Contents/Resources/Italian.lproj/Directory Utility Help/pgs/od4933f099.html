Duplicare una configurazione per l'accesso a una directory LDAP

Puoi usare Utility Directory per duplicare una configurazione in grado di specificare come deve accedere Mac OS X a una directory LDAPv3 o LDAPv2. Dopo aver duplicato la configurazione di una directory LDAP, ne potrai cambiare le impostazioni per differenziarla dalla configurazione originaria.

Per duplicare una configurazione per l'accesso a una directory LDAP:

  1. Apri “Preferenze di Sistema” e fare clic su Account.

  2. Se l'icona del lucchetto è bloccata, fai clic su di essa e inserisci il nome e la password di un amministratore per sbloccarla.

  3. Fai clic su “Opzioni login” e quindi su Modifica.

  4. Fai clic su “Apri Utility Directory”.

  5. Se l'icona del lucchetto è bloccata, fai clic su di essa e inserisci il nome e la password di un amministratore per sbloccarla.

  6. Fai clic su Servizi.

  7. Nell'elenco dei servizi, seleziona LDAPv3, quindi fai clic sul pulsante Modifica (/).

  8. Se l'elenco delle configurazioni dei server è nascosto, fai clic su Mostra Opzioni.

  9. Seleziona la configurazione di un server dall'elenco e fai clic su Duplica.

  10. Modifica le impostazioni della configurazione duplicata:

    • Abilita: fai clic su un riquadro per abilitare o disabilitare l'accesso a un server di directory LDAP.

    • Nome configurazione: fai doppio clic sul nome di una configurazione per modificarlo.

    • Nome server o Indirizzo IP: fai doppio clic sul nome di un server o su un indirizzo IP per modificarlo.

    • Mappature LDAP: seleziona un modello dal menu a comparsa, quindi inserisci il suffisso della base di ricerca per la directory LDAP e fai clic su OK.

      Se scegli un modello, devi inserire un suffisso per la base di ricerca o il computer non sarà in grado di trovare informazioni nella directory LDAP. Generalmente, il suffisso per la base di ricerca viene ricavato dal nome DNS del server. Ad esempio, per un server il cui nome DNS sia ods.example.com, il suffisso per la base di ricerca è “dc=ods,dc=example,dc=com.”

      Se scegli un server invece di un modello, non è necessario un suffisso per la base di ricerca. In tal caso, Open Directory considererà come suffisso per la base di ricerca il primo livello della directory LDAP.

      Se scegli Personalizzato, devi configurare la mappatura tra i tipi di record e gli attributi di Mac OS X e le classi e gli attributi della directory LDAP alla quale ti stai collegando. Per ulteriori informazioni, consulta Configurare le ricerche e le mappature LDAP.

    • SSL: fai clic sul riquadro per abilitare o disabilitare le comunicazioni criptate utilizzando il protocollo SSL. Prima di selezionare il riquadro SSL, verifica con l'amministratore Open Directory se è necessario utilizzare SSL.

  11. Per modificare le impostazioni di default di seguito riportate per la configurazione LDAP duplicata, fai clic su Modifica per visualizzare le opzioni, effettua le modifiche e fai clic su OK una volta terminata la modifica.

    • Fai clic su Connessione per impostare il trusted binding (se la directory LDAP lo supporta), specificare una porta personalizzata, ignorare i rinvii del server o forzare l'utilizzo del protocollo LDAPv2 (solo lettura). Per ulteriori informazioni, consulta Modificare le impostazioni di connessione di una directory LDAP.

    • Fai clic su Ricerca & Assegnazione per configurare le ricerche e le mappature per un server LDAP. Per ulteriori informazioni, consulta Impostare il trusted binding per una directory LDAP”.

    • Fai clic su Sicurezza per impostare una connessione autenticata (invece di un trusted binding) e altre opzioni sulla politica di sicurezza. Per ulteriori informazioni, consulta Modificare la politica di sicurezza per una connessione LDAP.

    • Fai clic su Vincola per configurare il trusted binding oppure fai clic su Non vincolare per arrestare il trusted binding (questi pulsanti non appaiono se la directory LDAP non consente il trusted binding). Per ulteriori informazioni, consulta Impostare il trusted binding per una directory LDAP”.

  12. Fai clic su OK per terminare la modifica della configurazione duplicata.

  13. Se vuoi che il computer acceda alla directory LDAP specificata dalla configurazione duplicata che hai creato, aggiungi la directory a una ricerca personalizzata nel pannello Autenticazione e nel pannello Contatti dei criteri di ricerca in Utility Directory, quindi assicurati che LDAPv3 sia abilitato nel pannello Servizi.

Per ulteriori informazioni, consulta Abilitare o disabilitare i servizi di directory LDAP e Definire i criteri di ricerca personalizzata.
