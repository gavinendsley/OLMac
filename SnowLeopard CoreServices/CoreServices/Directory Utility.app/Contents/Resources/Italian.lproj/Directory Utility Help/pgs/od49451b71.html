Proteggere i computer dall'intrusione di un server DHCP

Apple raccomanda di non utilizzare un criterio di ricerca automatico per l'autenticazione con un server LDAP o un dominio di directory condivisa con DHCP in ambienti in cui l'obiettivo principale sia quello della sicurezza.

Un hacker malintenzionato potrebbe accedere al network e utilizzare un server DHCP, una directory LDAP (o un dominio di directory condivisa) falsi per controllare il tuo computer utilizzando l'account utente root.

Per un hacker che accede al tuo network, il server DHCP falso deve essere parte del tuo network locale o della sottorete. Pertanto, se i tuoi computer sono i soli sul tuo network locale e hanno accesso a Internet tramite il servizio Mac OS X Server NAT o un router NAT, questo tipo di attentato alla sicurezza non è possibile. Tuttavia, un network locale wireless diminuisce la sicurezza perché un hacker può accedervi più facilmente che a un network locale cablato.

Puoi proteggere il tuo Mac dall'intrusione di server DHCP falsi disabilitando l'utilizzo di una directory LDAP con DHCP e disabilitando la connessione con i protocolli DHCP e broadcast per il dominio di directory locale (o disabilitando il dominio di directory locale).

Se ti colleghi con un computer portatile a un server LDAP mentre il computer è collegato a un network e modifichi la politica di ricerca del computer da automatica a personalizzata (nel pannello Autenticazione della Politica di ricerca in Utility Directory), si verifica un ritardo all’avvio del computer quando questo non sarà collegato al network.

Il ritardo si verifica perché il computer non riesce a collegarsi al dominio di una directory specifico elencato nel criterio di ricerca personalizzato del computer. Non si verificherà alcun ritardo quando si riattiva un computer la cui connessione al network sia stata interrotta durante lo stato di stop.
