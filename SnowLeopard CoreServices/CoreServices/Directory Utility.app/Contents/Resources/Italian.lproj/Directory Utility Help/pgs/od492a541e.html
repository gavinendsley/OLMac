Comporre una password

La password associata a un account utente deve essere inserita dall'utente durante l'autenticazione per il login o altri servizi. La password distingue tra maiuscole e minuscole (tranne le password per SMB-LAN Manager) e non viene visualizzata quando viene inserita.

Indipendentemente dal tipo di password scelta da un utente, di seguito vengono riportate le linee guida per comporre una password per gli account utente di Mac OS X Server:

  • Una password deve contenere lettere, numeri e simboli in combinazioni che non possano essere facilmente scoperte da utenti non autorizzati. Le password non devono essere composte da parole. Le password migliori contengono cifre e simboli (come # o $) o sono composte dalla prima lettera di tutte le parole di una frase. Utilizza sia lettere maiuscole che lettere minuscole.

  • Evita spazi e combinazioni di tasti Opzione.

  • Evita caratteri che non possono essere inseriti sui computer che l'utente utilizzerà o che possono richiedere la conoscenza di una speciale sequenza di tasti su diverse tastiere e piattaforme.

  • Alcuni protocolli di network non supportano password che contengono spazi iniziali, spazi interni o spazi finali.

  • Una password di lunghezza zero non è consigliabile. Open Directory e alcuni sistemi (come un vincolo LDAP) non supportano password di lunghezza zero.

  • Per la massima compatibilità con computer e servizi ai quali l'utente può accedere, componi le password utilizzando solo caratteri ASCII.
