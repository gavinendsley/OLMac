Configurare l'accesso a un dominio Active Directory

Se utilizzi il connettore Active Directory elencato in Utility Directory, puoi configurare Mac OS X in modo che acceda alle informazioni di base dell'account utente in un dominio Active Directory su un server Windows.

Il connettore Active Directory genera tutti gli attributi richiesti per l'autenticazione in Mac OS X. Non sono richieste modifiche nello schema Active Directory.

Il connettore Active Directory rileva e accede ai tipi di registrazioni e di attributi standard di Mac OS X, ad esempio agli attributi richiesti dalla gestione client di Mac OS X, se lo schema Active Directory è stato ampliato per includerli.

ATTENZIONE:   le opzioni avanzate del connettore Active Directory permettono di assegnare l'attributo UID (unique user ID), GID (primary group ID) e GID di gruppo di Mac OS X agli attributi appropriati che sono stati aggiunti allo schema Active Directory. Se in seguito modifichi l'impostazione di queste opzioni di assegnazione, gli utenti potrebbero perdere l'accesso ai documenti creati precedentemente.
Importante:   se il nome del tuo computer contiene un trattino, potresti non essere in grado di collegarti a un dominio di directory quale LDAP o Active Directory. Per stabilire il vincolo, utilizza un nome di computer che non contiene simboli.
Per configurare l'accesso a un dominio Active Directory:

  1. Apri “Preferenze di Sistema” e fare clic su Account.

  2. Se l'icona del lucchetto è bloccata, fai clic su di essa e inserisci il nome e la password di un amministratore per sbloccarla.

  3. Fai clic su “Opzioni login” e quindi su Accedi o Modifica.

  4. Fai clic su “Apri Utility Directory”.

  5. Se l'icona del lucchetto è bloccata, fai clic su di essa e inserisci il nome e la password di un amministratore per sbloccarla.

  6. Fai clic su Servizi.

  7. Nell'elenco dei servizi, seleziona Active Directory, quindi fai clic sul pulsante Modifica (/).

  8. Inserisci il nome DNS del dominio Active Directory che desideri vincolare al computer che stai configurando.

    L'amministratore del dominio Active Directory può informarti a proposito del nome DNS da inserire.

  9. Se necessario, modifica l'ID del computer.

    L’ID del computer è il nome mediante il quale il computer è noto nel dominio Active Directory ed è preimpostato sul nome del computer. Potresti modificarlo per conformarti allo schema stabilito dalla tua azienda per la denominazione dei computer nel dominio Active Directory. Se non sei sicuro, chiedi all’amministratore del dominio Active Directory.

  10. (Facoltativo) Imposta le opzioni avanzate.

    Se le opzioni avanzate sono nascoste, fai clic su Mostra opzioni avanzate e imposta le opzioni nei pannelli Esperienza utente, Mappature e Amministratore. Puoi anche modificare le impostazioni delle opzioni avanzate in seguito.

    Per ulteriori informazioni sulle opzioni avanzate, consulta:

    • Impostare account utente mobili in Active Directory

    • Impostare le cartelle Inizio per gli account utente di Active Directory

    • Impostare una shell UNIX per gli account utente di Active Directory

    • Assegnare la UID a un attributo Active Directory

    • Assegnare l'ID gruppo primario a un attributo Active Directory

    • Assegnare l'ID gruppo negli account di gruppo a un attributo Active Directory

    • Specificare un server Active Directory preferito

    • Modificare i gruppi Active Directory che possono amministrare il computer

    • Controllare l'autenticazione da tutti i domini in Forest Active Directory

  11. Fai clic su Vincola, autenticati come utente con diritti per vincolare un computer al dominio Active Directory, seleziona le politiche di ricerca a cui aggiungere Active Directory (vedi di seguito) e fai clic su OK:

    • Nome utente e Password: puoi autenticarti inserendo il nome e la password del tuo account utente di Active Directory, altrimenti l'amministratore del dominio Active Directory dovrà fornire un nome e una password.

    • OU computer: inserisci l'unità organizzativa (OU, organizational unit) per il computer che stai configurando.

    • Usa per l'autenticazione: utilizzato per determinare se Active Directory viene aggiunto alle politiche di ricerca di autenticazione del computer.

    • Usa per i contatti: utilizzato per determinare se Active Directory viene aggiunto alle politiche di ricerca di autenticazione del computer.

    Quando fai clic su OK, Utility Directory imposta il trusted binding tra il computer che stai configurando e il server Active Directory. I criteri di ricerca del computer sono impostati in accordo con le opzioni che hai selezionato quando hai effettuato l’autenticazione e Active Directory è abilitato nel pannello Servizi di Utility Directory.

    Con le impostazioni di default per le opzioni avanzate di Active Directory, la forest di Active Directory è aggiunta ai criteri di ricerca autenticazioni e contatti se hai selezionato "Usa per l'autenticazione" o "Usa per i contatti".

    Tuttavia, se deselezioni "Consenti autenticazione da tutti i domini nella forest" nel pannello Amministratore delle opzioni avanzate prima di fare clic su Vincola, il dominio Active Directory più vicino viene aggiunto al posto della forest.

    Puoi modificare i criteri di ricerca in seguito, aggiungendo o rimuovendo la forest di Active Directory o i domini individuali. Per ulteriori informazioni, consulta Definire i criteri di ricerca personalizzatae”.

  12. (Facoltativo) Collega il server alla zona Kerberos di Active Directory:

    • Sul server o un computer amministratore che può collegarsi al server, apri Server Admin e seleziona Open Directory per il server.

    • Fai clic su Impostazioni, quindi fai clic su Generale.

    • Fai clic su "Accedi a Kerberos", quindi scegli la zona Kerberos in Active Directory dal menu a comparsa e inserisci le credenziali per un amministratore locale su questo server.
