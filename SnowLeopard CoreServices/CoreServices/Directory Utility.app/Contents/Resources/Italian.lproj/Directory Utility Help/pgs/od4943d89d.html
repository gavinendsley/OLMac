Informazioni sull'accesso ad Active Directory

Puoi impostare Mac OS X in modo che acceda alle informazioni dell'account di base dell'utente nel dominio Active Directory del server Windows 2000 o versione successiva. Questo è possibile grazie a un connettore Active Directory per Utility Directory. Il connettore Active Directory è elencato nel pannello Servizi di Utility Directory.

Non devi apportare nessuna modifica allo schema del dominio Active Directory per ottenere informazioni di base sull'account utente. Potresti modificare l'ACL (Access Control List) di default di attributi specifici in modo che gli account del computer possano leggere le proprietà dell'utente.

Il connettore Active Directory genera tutti gli attributi richiesti per l'autenticazione di Mac OS X dagli attributi standard negli account utente di Active Directory. Il connettore supporta anche le politiche di autenticazione Active Directory, compresi le modifiche, le scadenze, i cambiamenti forzati e le opzioni di sicurezza della password.

Mac OS X v10.6 supporta le opzioni di crittografia e di firma dei pacchetti per tutti i domini Active Directory di Windows. Questa funzionalità è attiva per default con l'opzione "Consenti" abilitata. Puoi modificare l'impostazione di default in disabilitato o richiesto mediante lo strumento della linea di comando dsconfigad. Le opzioni di crittografia e di firma dei pacchetti consentono di proteggere tutti i dati da e verso il dominio Active Directory per le ricerche dei record.

Il connettore Active Directory genera dinamicamente un ID utente univoco e un ID del gruppo principale, basato sulle GUID (Globally Unique ID) nel dominio Active Directory. L'ID generata dell'utente e l'ID primaria del gruppo sono le stesse per ogni account utente anche se l'account è utilizzato per eseguire il login su diversi computer Mac OS X.

In alternativa, puoi forzare il connettore Active Directory per assegnare all'ID dell'utente gli attributi Active Directory che specifichi.

Il connettore Active Directory genera un ID del gruppo basato sull’account gruppo GUID di Active Directory. Puoi anche forzare il plugin per assegnare all'ID del gruppo gli attributi Active Directory che specifichi.

Quando qualcuno accede a Mac OS X con un account utente Active Directory, il connettore Active Directory può attivare la directory Inizio del network di Windows specificata nell'account utente di Active Directory come cartella Inizio dell'utente Mac OS X. Puoi specificare se utilizzare la cartella Inizio del network specificata dall’attributo standard home Directory di Active Directory oppure dall’attributo home Directory di Mac OS X, se lo schema Active Directory è stato ampliato in modo da includerlo.

In alternativa, puoi configurare il plugin per creare una cartella Inizio locale sul volume di avvio del computer client Mac OS X. In questo caso, il plugin attiva anche la cartella Inizio del network Windows (specificata nell’account utente di Active Directory) come un volume network, allo stesso modo di un punto di condivisione. Utilizzando il Finder, l'utente può quindi copiare documenti fra la cartella Inizio di Windows sul volume network e la cartella Inizio di Mac OS X locale.

Il connettore Active Directory può anche creare account mobili per gli utenti. Un account mobile ha una cartella Inizio locale sul volume di avvio del computer client Mac OS X (l’utente ha anche una cartella Inizio network come specificato nell'account utente di Active Directory).

Un account mobile memorizza le credenziali di autenticazione dell’utente Active Directory sul computer client Mac OS X. Le credenziali memorizzate consentono all'utente di eseguire il login utilizzando il nome e la password Active Directory quando il computer client è disconnesso dal server Active Directory.

Un account mobile ha una cartella Inizio locale sul volume di avvio del computer client Mac OS X (l’utente ha anche una cartella Inizio network come specificato nell'account utente di Active Directory).

Se lo schema di Active Directory è stato esteso in modo da includere i tipi di record (classi di oggetti) e gli attributi di Mac OS X, il connettore Active Directory li rileva e vi accede.

Ad esempio, lo schema Active Directory può essere modificato utilizzando gli strumenti di amministrazione di Windows, per includere gli attributi dei client gestiti da Mac OS X. Questa modifica dello schema attiva il connettore Active Directory in modo che supporti le impostazioni del client gestito definite utilizzando l'applicazione Workgroup Manager di Mac OS X Server.

I client Mac OS X ammettono il pieno accesso di lettura agli attributi che vengono aggiunti alla directory. Pertanto, potrebbe essere necessario modificare l'ACL di quegli attributi che consentono ai gruppi del computer di leggere questi attributi aggiunti.

Il connettore Active Directory ricerca tutti i domini in una forest di Active Directory. Puoi configurare i plugin di modo che consentano a tutti gli utenti provenienti da qualsiasi dominio nella foresta di autenticarsi su un computer Mac OS X. In alternativa, puoi consentire l'autenticazione sul client esclusivamente a domini specifici.

Il connettore Active Directory supporta in maniera completa la replica e il failover di Active Directory. Trova i controllori dei domini multipli e determina qual è il più vicino. Se il controllore non è disponibile, il plugin retrocede verso un altro controllore di dominio che si trovi nelle vicinanze.

Il connettore Active Directory utilizza LDAP per accedere agli account utente di Active Directory e Kerberos per autenticarli. Il connettore Active Directory non utilizza l'ADSI (Active Directory Services Interface) proprietaria di Microsoft per ottenere servizi di directory o di autenticazione.
