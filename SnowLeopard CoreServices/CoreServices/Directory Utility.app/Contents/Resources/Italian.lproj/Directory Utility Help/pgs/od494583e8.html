Impostare il trusted binding per una directory LDAP

Puoi utilizzare Utility Directory per impostare il trusted binding tra il computer e una directory LDAP che supporta il trusted binding. Il binding ("vincolo") è autenticato reciprocamente dai parametri di un "record computer" autenticato che viene creato nella directory quando imposti il trusted binding.

Il computer non può essere configurato per utilizzare sia il trusted binding LDAP, sia la directory LDAP fornita da DHCP. Il trusted binding LDAP è inerente a un vincolo statico, ma quello LDAP fornito da DHCP è un vincolo dinamico.

Per impostare il trusted binding con una directory LDAP:

  1. Apri “Preferenze di Sistema” e fare clic su Account.

  2. Se l'icona del lucchetto è bloccata, fai clic su di essa e inserisci il nome e la password di un amministratore per sbloccarla.

  3. Fai clic su “Opzioni login” e quindi su Accedi o Modifica.

  4. Fai clic su “Apri Utility Directory”.

  5. Se l'icona del lucchetto è bloccata, fai clic su di essa e inserisci il nome e la password di un amministratore per sbloccarla.

  6. Fai clic su Servizi.

  7. Nell'elenco dei servizi, seleziona LDAPv3, quindi fai clic sul pulsante Modifica (/).

  8. Se l'elenco delle configurazioni dei server è nascosto, fai clic su Mostra Opzioni.

  9. Seleziona la configurazione di un server che desideri e fai clic su Modifica.

  10. Fai clic su Vincola, inserisci le seguenti credenziali e fai clic su OK.

    Inserisci il nome del computer e il nome e la password di un amministratore per il dominio directory. Il nome del computer non può essere in uso da un altro computer per il trusted binding o per altri servizi di network.

    Se non viene visualizzato il pulsante Vincola, la directory LDAP non supporta il trusted binding.

  11. Verifica di avere inserito correttamente il nome del computer.

    Se viene visualizzato un messaggio che ti avvisa dell'esistenza di un “record computer”, fai clic su Annulla per tornare indietro e modificare il nome del computer, oppure su Riscrivere per sostituire il “record computer” esistente.

    Il “record computer” esistente potrebbe essere stato abbandonato o poterebbe appartenere a un altro computer. Se sostituisci un "record computer" esistente, dovresti avvertire l'amministratore della directory LDAP nel caso in cui la sostituzione disabiliti un altro computer.

    In una tale situazione, l'amministratore della directory LDAP deve assegnare al computer disabilitato un altro nome e aggiungerlo al gruppo di computer a cui appartiene, utilizzando un nome diverso per quel computer.

  12. Fai clic su OK per completare l'impostazione del trusted binding.
