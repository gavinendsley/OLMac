Configurare manualmente l'accesso a una directory LDAP

Puoi creare manualmente una configurazione che definisce la modalità di accesso di Mac OS X a una directory LDAPv3 o LDAPv2. Devi conoscere il nome DNS o l'indirizzo IP del server delle directory LDAP.

Se la directory non è su un Mac OS X Server, devi conoscere la base di ricerca e il modello per la mappatura dei dati di Mac OS a quelli della directory. I modelli di mappatura supportati sono:

  • Dal server, per una directory che fornisce le proprie mappature e la propria base di ricerca, come Mac OS X Server.

  • Server Open Directory, per una directory che utilizza lo schema Mac OS X Server.

  • Active Directory, per una directory che risiede su un server Windows 2000, Windows 2003 o versione successiva.

  • RFC 2307, per la maggior parte delle directory che risiedono su server UNIX.

  • Personalizzato, per le directory che non utilizzano le mappature sopra elencate.

Il plugin LDAPv3 supporta in maniera completa la replica e il failover di Open Directory. Se il master di Open Directory non è disponibile, il plugin retrocede verso una replica vicina.

Importante:   se il nome del computer contiene un trattino, potresti non essere in grado di collegarti a un Directory Domain quale LDAP o Active Directory. Per stabilire il vincolo, utilizza un nome di computer che non contiene simboli.
Per configurare manualmente l'accesso a una directory LDAP:

  1. Apri “Preferenze di Sistema” e fai clic su Account.

  2. Se l'icona del lucchetto è bloccata, fai clic su di essa e inserisci il nome e la password di un amministratore per sbloccarla.

  3. Fai clic su “Opzioni login”, quindi fai clic su Accedi o Modifica.

  4. Fai clic su “Apri Utility Directory”.

  5. Se l'icona del lucchetto è bloccata, fai clic su di essa e inserisci il nome e la password di un amministratore per sbloccarla.

  6. Fai clic su Servizi.

  7. Nell'elenco dei servizi, seleziona LDAPv3, quindi fai clic sul pulsante Modifica (/).

    Puoi selezionare LDAPv3 nell'elenco dei servizi senza selezionare il riquadro Attiva per LDAPv3.

  8. Fai clic su Nuovo, quindi su Manuale.

  9. Inserisci un nome per la configurazione.

  10. Premi il tasto di tabulazione e scrivi il nome DNS o l'indirizzo IP del server dove si trova la directory LDAP a cui desideri accedere.

  11. Fai clic sul menu a comparsa accanto al nome DNS o all'indirizzo IP e seleziona un modello o un metodo per la mappatura:

    • Se scegli “Dal server”, non è necessario un suffisso per la base di ricerca. In tal caso, Open Directory considererà come suffisso per la base di ricerca il primo livello della directory LDAP.

    • Se scegli un modello, inserisci il suffisso per la base di ricerca per la directory LDAP e fai clic su OK. Devi inserire un suffisso per la base di ricerca oppure il computer non riesce a cercare le informazioni nella directory LDAP.

      Generalmente, il suffisso per la base di ricerca viene ricavato dal nome DNS del server. Ad esempio, il suffisso per la base di ricerca potrebbe essere "dc=ods, dc=example, dc=com" per un server il cui nome DNS sia ods.example.com.

    • Se scegli Personalizzato, devi configurare le mappature fra i tipi di registrazioni e gli attributi di Mac OS X e le classi e gli attributi della directory LDAP a cui ti stai collegando. Per ulteriori informazioni, consulta Configurare le ricerche e le mappature LDAP.

  12. Prima di selezionare il riquadro “Cripta usando SSL”, verifica con l'amministratore Open Directory se è necessario utilizzare SSL.

  13. Per modificare le impostazioni riportate di seguito per questa configurazione LDAP, fai clic su Modifica per visualizzare le opzioni per la configurazione LDAP selezionata, effettua le modifiche e fai clic su OK una volta terminata la modifica delle opzioni di configurazione LDAP.

    • Fai clic su Connessione per impostare le opzioni di timeout, specificare una porta personalizzata, ignorare i rinvii del server o forzare l'utilizzo del protocollo LDAPv2 (solo lettura). Per ulteriori informazioni, consulta Modificare le impostazioni di connessione di una directory LDAP.

    • Fai clic su Ricerca & Assegnazione per configurare le ricerche e le mappature per un server LDAP. Per ulteriori informazioni, consulta Impostare il trusted binding per una directory LDAP”.

    • Fai clic su Sicurezza per impostare una connessione autenticata (invece di un trusted binding) e altre opzioni sulla politica di sicurezza. Per ulteriori informazioni, consulta Modificare la politica di sicurezza per una connessione LDAP.

    • Fai clic su Vincola per configurare il trusted binding (se la directory LDAP lo supporta). Per ulteriori informazioni, consulta Impostare il trusted binding per una directory LDAP”.

  14. Fai clic su OK per terminare la creazione manuale della configurazione per l'accesso a una directory LDAP.

  15. Se vuoi che il computer acceda alla directory LDAP per cui hai creato una configurazione, aggiungi la directory a una ricerca personalizzata nel pannello Autenticazione e nel pannello Contatti dei criteri di ricerca in Utility Directory, quindi assicurati che LDAPv3 sia abilitato nel pannello Servizi.

Per ulteriori informazioni, consulta Abilitare o disabilitare i servizi di directory LDAP e Definire i criteri di ricerca personalizzata.

Nota:   prima di poter utilizzare Workgroup Manager per creare utenti su un server LDAP non Apple che utilizza mappature RFC 2307 (UNIX), dovrai modificare la mappatura del tipo di registrazione Users. Per ulteriori informazioni, consulta Modificare la mappatura RFC 2307 per abilitare la creazione di utenti.
Importante:   se modifichi l'indirizzo IP e il nome del computer utilizzando changeip mentre sei connesso a un server di directory, devi disconnetterti e riconnetterti al server di directory per aggiornare la directory con il nome del computer e l'indirizzo IP nuovi. Se non ti disconnetti e riconnetti al server di directory, la directory non verrà aggiornata e continuerà a utilizzare il nome del computer e l'indirizzo IP precedenti.
