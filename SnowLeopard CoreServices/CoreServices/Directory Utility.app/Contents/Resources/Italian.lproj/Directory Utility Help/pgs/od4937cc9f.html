Configurare l'accesso a una directory LDAP

Utilizzando Utility Directory, puoi specificare come Mac OS X deve accedere a una directory LDAPv3 se conosci il nome DNS o l'indirizzo IP del server di directory LDAP.

Se la directory non è su un server che fornisce la sua mappatura, come ad esempio Mac OS X Server, devi conoscere la base di ricerca e il modello per la mappatura dei dati di Mac OS X con quelli della directory.

I modelli di mappatura supportati sono:

  • Server Open Directory, per una directory che utilizza lo schema Mac OS X Server.

  • Active Directory, per una directory che risiede su un server Windows 2000, Windows 2003 o versione successiva.

  • RFC 2307, per la maggior parte delle directory che risiedono su server UNIX.

Il plugin LDAPv3 supporta in maniera completa la replica e il failover di Open Directory. Se il master di Open Directory non è disponibile, il plugin retrocede verso una replica vicina.

Per specificare una mappatura personalizzata per i dati della directory, segui le istruzioni in Configurare manualmente l'accesso a una directory LDAP.

Importante:   se il nome del computer contiene un trattino, potresti non essere in grado di collegarti a un dominio di directory quale LDAP o Active Directory. Per stabilire il vincolo, utilizza un nome di computer che non contiene simboli.
Per fare in modo che Utility Directory ti aiuti a configurare l'accesso a una directory LDAP:

  1. Apri “Preferenze di Sistema” e fare clic su Account.

  2. Se l'icona del lucchetto è bloccata, fai clic su di essa e inserisci il nome e la password di un amministratore per sbloccarla.

  3. Fai clic su “Opzioni login” e quindi su Accedi o Modifica.

  4. Fai clic su “Apri Utility Directory”.

  5. Se l'icona del lucchetto è bloccata, fai clic su di essa e inserisci il nome e la password di un amministratore per sbloccarla.

  6. Fai clic su Servizi.

  7. Nell'elenco dei servizi, seleziona LDAPv3, quindi fai clic sul pulsante Modifica (/).

    Puoi selezionare LDAPv3 nell'elenco dei servizi senza selezionare il riquadro Attiva per LDAPv3.

  8. Fai clic su Nuovo e inserisci il nome DNS o l’indirizzo IP del server LDAP.

  9. Seleziona le opzioni per accedere alla directory:

    • Seleziona “Cripta usando SSL” se desideri che Open Directory utilizzi SSL (Secure Sockets Layer) per le connessioni con la directory LDAP. Prima della selezione, verifica con l'amministratore di Open Directory se è necessario utilizzare SSL.

    • Seleziona “Usa per l'autenticazione” se questa directory contiene account utente utilizzati per il login o per l'autenticazione dei servizi.

    • Seleziona “Usa per i contatti” se questa directory contiene indirizzi e-mail e altre informazioni che desideri utilizzare in Rubrica Indirizzi.

    Se Utility Directory non riesce a contattare il server LDAP, viene visualizzato un messaggio e devi configurare manualmente l’accesso o annullare il processo di configurazione. Per ulteriori informazioni sulla configurazione manuale, consulta Configurare manualmente l'accesso a una directory LDAP.

    Se la finestra di dialogo si espande per visualizzare le opzioni di mappatura, scegli il modello di mappatura dal menu a comparsa, inserisci il suffisso per la base di ricerca, quindi fai clic su Continua.

    Generalmente, il suffisso per la base di ricerca viene ricavato dal nome DNS del server. Ad esempio, il suffisso per la base di ricerca potrebbe essere "dc=ods, dc=example, dc=com" per un server il cui nome DNS sia ods.example.com.

    Se nessuno modello di mappatura disponibile è applicabile alla connessione che stai configurando, fai clic su Manuale. Per ulteriori informazioni, consulta Configurare manualmente l'accesso a una directory LDAP.

  10. Per fare in modo che Utility Directory ottenga le informazioni dal server LDAP, fai clic su Continua.

  11. Se la finestra di dialogo si espande per visualizzare le opzioni per il trusted binding, inserisci il nome del computer e il nome e la password dell'amministratore della directory (il binding potrebbe essere facoltativo).

    La finestra di dialogo ti informa se la directory LDAP richiede il trusted binding o lo rende facoltativo. Il trusted binding è reciproco: ogni volta che il computer si collega alla directory LDAP, viene eseguita un'autenticazione reciproca. Se il trusted binding è stato configurato o se la directory LDAP non supporta il trusted binding, il pulsante Vincola non viene visualizzato. Verifica di aver inserito correttamente il nome del computer.

    Se viene visualizzato un messaggio che ti avvisa che esiste un "record computer", fai clic su Annulla per tornare indietro e modificare il nome del computer, oppure fai clic su Riscrivere per sostituire il "record computer" esistente.

    Il "record computer" esistente potrebbe essere stato abbandonato o poterebbe appartenere a un altro computer.

    Se sostituisci un "record computer" esistente, dovresti avvertire l'amministratore della directory LDAP nel caso in cui la sostituzione disabiliti un altro computer. In questo caso, l'amministratore della directory LDAP deve assegnare al computer disabilitato un nome diverso e aggiungerlo di nuovo all'elenco di computer al quale apparteneva.

  12. Se la finestra di dialogo si espande per visualizzare le opzioni di connessione, seleziona “Usa autenticazione quando ti colleghi” e inserisci il nome distintivo e la password di un account utente nella directory.

    Le opzioni per una connessione autenticata appaiono se il server LDAP supporta una connessione autenticata ma non i trusted binding. Una connessione autenticata non è reciproca: il server LDAP autentica il client ma non viceversa.

    L’opzione "Usa autenticazione quando ti colleghi" è selezionata ma appare in grigio se il server LDAP richiede l'inserimento del nome distintivo e della password di un account utente per una connessione autenticata.

    Il nome distintivo può specificare qualsiasi account utente che possiede i privilegi per vedere i dati nella directory. Ad esempio, un account utente con nome breve "dirauth" su un server LDAP e indirizzo ods.esempio.com avrebbe il nome distintivo uid=dirauth,cn=users,dc=ods,dc=esempio,dc=com.

    Importante:   se il nome distintivo o la password non sono corretti, puoi eseguire il login al computer utilizzando gli account utente della directory LDAP.

  13. Fai clic su OK per terminare la creazione della connessione LDAP.

  14. Fai clic su OK per terminare la configurazione delle opzioni LDAPv3.

Se selezioni l'opzione "Usa per l'autenticazione" o l'opzione "Usa per i contatti" nel passaggio 5, la configurazione della directory LDAP appena creata viene aggiunta al criterio di ricerca personalizzato all'interno dei pannelli Autenticazione o Contatti di Utility Directory.

Assicurati che LDAPv3 sia abilitato nel pannello Servizi in modo che il computer utilizzi la configurazione LDAP che hai creato. Per ulteriori informazioni, consulta Abilitare o disabilitare i servizi di directory LDAP”.
