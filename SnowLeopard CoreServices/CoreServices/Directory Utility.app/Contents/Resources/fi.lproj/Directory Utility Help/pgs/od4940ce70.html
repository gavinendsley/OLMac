LDAP-pääsyn käyttöönotto Active Directory -domaineilla

Hakemistotyökalulla voit asettaa LDAPv3-määrityksen käyttämään Windows-palvelimella olevaa Active Directory -domainia. LDAPv3-määritys antaa täyden kontrollin Mac OS X:n tietuetyyppien ja attribuuttien kuvaamiseen Active Directoryn objektiluokkiin, hakukantoihin ja attribuutteihin.

Joidenkin tärkeiden Mac OS X -tietuetyyppien ja attribuuttien, kuten yksilöllisen käyttäjätunnuksen (UID), kuvaaminen edellyttää Active Directory -mallin laajentamista.

LDAPv3-määrittely ei sisällä seuraavia Hakemistotyökalussa lueteltuja Active Directory -liittimen ominaisuuksia:

  • Yksilöllisen käyttäjätunnuksen ja ensisijaisen ryhmätunnuksen dynaaminen luominen

  • Paikallisen Mac OS X -kotikansion luominen

  • Windows-kotikansion automaattinen näkyviintuominen

  • Liikkuvien käyttäjätunnusten välimuistiin tallennettuja valtakirjoja

  • Kaikkien Active Directory -metsän domainien löytäminen

  • Tuki Active Directory -kahdentamiselle ja vikaantumisesta toipumiselle

Lisätietoja löytyy kohdasta Tietoja Active Directory -käytöstä.

Active Directory -palvelinmäärittelyn luominen:

  1. Avaa Järjestelmäasetukset ja osoita Tilit.

  2. Jos lukkosymboli on lukittu, avaa se osoittamalla sitä ja kirjoittamalla ylläpitäjän käyttäjätunnus ja salasana.

  3. Osoita Kirjautumisvalinnat ja osoita sitten Liity tai Muokkaa.

  4. Osoita Avaa Hakemistotyökalu.

  5. Jos lukkosymboli on lukittu, avaa se osoittamalla sitä ja kirjoittamalla ylläpitäjän käyttäjätunnus ja salasana.

  6. Osoita Palvelut.

  7. Valitse palveluluettelosta LDAPv3 ja osoita muokkauspainiketta (/).

  8. Osoita Uusi ja syötä Active Directory -palvelimen DNS-nimi tai IP-osoite.

  9. Valitse seuraavia valintoja hakemiston käyttöön ja osoita Jatka, jolloin Hakemistotyökalu hakee tietoja Active Directory -palvelimelta.

    • Valitse ”Salaa käyttäen SSL:ää”, jos haluat Open Directoryn käyttävän SSL-yhteyksiä Active Directory -palvelimen kanssa. Ennen kuin valitset SSL-valintaneliön, kysy Open Directory -ylläpitäjältä tarvitaanko SSL:ää.

    • Valitse ”Käytä todentamiseen”, jos hakemisto sisältää käyttäjätilejä, joita joku aikoo käyttää sisäänkirjautumiseen tai todentamiseen palveluita varten.

    • Valitse ”Käytä yhteystietoihin”, jos hakemisto sisältää sähköpostiosoitteita ja muita tietoja, joita haluat käyttää Osoitekirjassa.

    Jos Hakemistotyökalu ei saa yhteyttä Active Directory -palvelimeen, näkyviin tulee viesti ja pääsy on määriteltävä käsin tai määrittelyprosessi on kumottava. Lisätietoja löytyy kohdasta Pääsyn määritteleminen LDAP-hakemistoon käsin.

    Jos valitsit ”Käytä todentamiseen” tai ”Käytä yhteystietoihin”, LDAPv3-yhteys Active Directory -domainiin lisätään muokattuun hakukäytäntöön Hakemistotyökalun Todentaminen- tai Yhteystiedot-paneeleissa.

    Varmista, että LDAPv3 on otettu käyttöön Palvelut-osiossa, jotta tietokone käyttää käyttöönottamaasi yhteyttä. Lisätietoja löytyy kohdasta LDAP-hakemistopalveluiden salliminen tai estäminen.

  10. Jos valintataulu laajenee ja siinä näkyvät kuvausasetukset, valitse ponnahdusvalikosta Active Directory, syötä etsintäkanta ja osoita Jatka.

    LDAPv3-määrityksen Active Directory -kuvauspohja kuvaa tietyt Mac OS X:n tietuetyypit ja attribuutit objektiluokkiin ja attribuutteihin, jotka eivät ole osa normaalia Active Directory -mallia. Voit muuttaa työpohjassa määriteltyjä kuvauksia tai voit laajentaa Active Directory -mallia.

    Vaihtoehtoisesti voit ehkä käyttää Active Directory -domainia Active Directory -liittimen kautta LDAPv3:n sijaan. Lisätietoja löytyy kohdasta Pääsyn määritteleminen Active Directory -domainiin.

  11. Kun valintataulu laajenee näyttämään yhteysasetukset, syötä Active Directoryn käyttäjätunnuksen yksilöllinen nimi ja salasana.

  12. Viimeistele LDAP-yhteyden luominen osoittamalla OK.

  13. Viimeistele LDAPv3-valintojen määrittely osoittamalla OK.
