Salasanan muodostaminen

Käyttäjän on syötettävä tiliinsä liittyvä salasana, kun hän suorittaa todentamisen sisäänkirjautumista tai muita palveluita varten. Salasanassa isot ja pienet kirjaimet tulkitaan eri merkeiksi (paitsi SMB-LAN Manager -salasanoissa) ja merkit piilotetaan näytöllä, kun sana syötetään.

Tässä on käyttäjälle valitun salasanan tyypistä riippumattomia ohjeita Mac OS X Server -käyttäjätilien salasanojen muodostamiseen:

  • Salasanassa tulee olla kirjaimia, numeroita ja symboleita yhdistelmänä, jota luvattomien käyttäjien ei ole helppo arvata. Salasanan ei tule koostua sanoista. Hyvässä salasanassa on numeroita ja symboleja (kuten # tai $) tai se voi koostua lauseen kaikkien sanojen alkukirjaimista. Käytä sekä isoja että pieniä kirjaimia.

  • Vältä välilyöntejä ja Optio-näppäinyhdistelmiä.

  • Vältä merkkejä, joita ei voi syöttää tietokoneilla, joita käyttäjä ehkä käyttää, tai jotka edellyttävät erityisten näppäinyhdistelmien tuntemusta, jotta ne voidaan syöttää erilaisilla näppäimistöillä ja eri alustoilla.

  • Jotkut verkkoprotokollat eivät tue salasanoja, joiden alussa, keskellä tai lopussa on välilyöntejä.

  • Nollapituista salasanaa ei suositella. Open Directory ja jotkin muut järjestelmät (kuten LDAP-sidonta) eivät tue nollapituista salasanaa.

  • Jotta yhteensopivuus käyttäjien mahdollisesti käyttämien tietokoneiden ja palveluiden kanssa olisi mahdollisimman hyvä, kannattaa salasanoissa käyttää vain ASCII-merkkejä.
