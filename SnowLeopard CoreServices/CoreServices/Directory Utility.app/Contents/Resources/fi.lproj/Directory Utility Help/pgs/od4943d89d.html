Tietoja Active Directoryn käytöstä

Voit määritellä Mac OS X:n käyttämään Windows 2000- tai uudempien palvelimien Active Directory -domainien käyttäjätunnustietoja. Tämän tekee mahdolliseksi Hakemistotyökalun Active Directory -liitin. Tämä Active Directory -liitin on luettelossa Hakemistotyökalun Palvelut-osiossa.

Active Directory -domainin malliin ei tarvitse tehdä muutoksia käyttätunnusten perustietojen hakemista varten. Voit muuttaa tiettyjen attribuuttien oletusarvoista pääsynhallintaluetteloa (ACL), jotta tietokoneen tilit voivat lukea käyttäjien ominaisuuksia.

Active Directory -liitin luo kaikki Mac OS X:n todentamiseen tarvitsemat attribuutit Active Directory -käyttäjätunnusten normaaleista attribuuteista. Liitin tukee myös Active Directoryn todentamismenetelmiä, mukaan lukien salasanan vaihtaminen, vanheneminen, pakotettu vaihtaminen ja suojausasetukset.

Mac OS X 10.6 tukee pakettien salausta ja allekirjoitusta kaikille Windows Active Directory -domaineille. Tämän toiminnon oletusasetus on ”salli”. Voit muuttaa oletusasetuksen arvoon ”ei käytössä” tai ”vaadi” komentorivityökalulla dsconfigad. Pakettien salaus ja allekirjoitus takaavat, että kaikki Active Directory -domainille tietuehakuja varten lähetetty ja siltä vastaanotettu data on suojattua.

Active Directory -liitin luo dynaamisesti yksilöllisen käyttäjätunnuksen ja ensisijaisen ryhmätunnuksen Active Directory -domainissa olevan käyttäjätunnuksen GUID (Globally Unique ID) perusteella. Luotu käyttäjätunnus ja ensisijainen ryhmätunnus ovat samoja jokaiselle käyttäjätunnukselle, vaikka tunnusta käytettäisiin eri Mac OS X -tietokoneille kirjautumiseen.

Vaihtoehtoisesti voit pakottaa Active Directory -liittimen kuvaamaan käyttäjätunnuksen määrittelemillesi Active Directory -attribuuteille.

Active Directory -liitin luo ryhmätunnuksen, joka perustuu Active Directoryn ryhmätunnuksen GUID:iin. Voit myös pakottaa liitännäisen kuvaamaan ryhmätunnusten ryhmänumeron määrittelemillesi Active Directory -attribuuteille.

Kun käyttäjä kirjautuu Mac OS X:ään Active Directory -käyttäjätunnuksella, Active Directory -liitin voi tuoda näkyviin käyttäjän Mac OS X -kotikansiona Windows-verkon kotikansion, joka on määritelty Active Directory -käyttäjätilillä. Voit määritellä, käytetäänkö Active Directoryn normaalin home Directory -attribuutin vai Mac OS X:n HomeDirectory-attribuutin määrittelemää verkkokotikansiota (jos Active Directory -malli on laajennettu sisältämään sen).

Vaihtoehtoisesti voit määritellä liitännäisen luomaan paikallisen kotikansion Mac OS X:n työasematietokoneen käynnistystaltiolle. Tässä tapauksessa liitännäinen tuo näkyviin myös käyttäjän Windows-verkkokotikansion (määritelty Active Directory -käyttäjätunnuksessa) verkkotaltiona, ikään kuin jakopisteenä. Finderin avulla käyttäjä voi sitten kopioida tiedostoja Windows-kotikansion verkkotaltion ja paikallisen Mac OS X:n kotikansion välillä.

Active Directory -liitin voi luoda käyttäjille myös siirrettäviä tilejä. Siirrettävällä tunnuksella on paikallinen kotikansio Mac OS X -työasematietokoneen käynnistystaltiolla. (Käyttäjällä on myös verkkokotikansio, kuten on määritelty käyttäjän Active Directory -tunnuksessa.)

Siirrettävä tunnus tallentaa käyttäjän Active Directory -todentamisen valtakirjat välimuistiin Mac OS X -työasematietokoneelle. Välimuistiin tallennetut valtakirjat sallivat käyttäjän kirjautua sisään Active Directory -käyttäjätunnusta ja salasanaa käyttäen, kun asiakastietokone ei ole yhteydessä Active Directory -palvelimeen.

Siirrettävällä tunnuksella on paikallinen kotikansio Mac OS X -työasematietokoneen käynnistystaltiolla. (Käyttäjällä on myös verkkokotikansio kuten on määritelty käyttäjän Active Directory -tunnuksessa.)

Jos Active Directory -malli on laajennettu sisältämään Mac OS X:n tietuetyyppejä (objektiluokkia) ja attribuutteja, Active Directory -liitin havaitsee ne ja käyttää niitä.

Esimerkiksi, Active Directory -mallia voidaan muuttaa Windowsin ylläpitotyökaluilla siten, että se sisältää Mac OS X:n hallitsemat asiakasattribuutit. Tämä mallimuutos sallii Active Directory -liittimen tukea Mac OS X Serverin Workgroup Managerilla tehtyjä hallittuja asiakasasetuksia.

Mac OS X -asiakkaat olettavat täysiä lukuoikeuksia hakemistoon lisättyihin attribuutteihin. Siksi saattaa olla tarpeen muuttaa niiden attribuuttien pääsynhallintaluetteloa, jotta tietokoneryhmien sallitaan lukea näitä lisättyjä attribuutteja.

Active Directory -liitin löytää kaikki domainit Active Directory -metsästä. Voit määritellä liitännäisen sallimaan metsän kaikkien domainien käyttäjien todentautua Mac OS X -tietokoneella. Vaihtoehtoisesti voit sallia vain tiettyjen domainien todentautumisen asiakkaalla.

Active Directory -liitin tukee täysin Active Directoryn kahdentamis- ja vikaantumisestatoipumisominaisuuksia. Se etsii useita domainohjaimia ja käyttää niistä lähintä. Jos domainohjain lakkaa olemasta käytettävissä, liitännäinen palaa toiseen lähellä olevaan domainohjaimeen.

Active Directory -liitin käyttää LDAP:ta Active Directory -käyttäjätunnusten hakemiseen ja Kerberosta niiden todentamiseen. Active Directory -liitin ei käytä Microsoftin suljettua ADSI (Active Directory Services Interface) -liittymää hakemisto- ja todentamispalveluiden hakemiseen.
