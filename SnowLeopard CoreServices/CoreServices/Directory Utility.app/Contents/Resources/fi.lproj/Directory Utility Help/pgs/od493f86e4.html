Pääsyn määritteleminen Active Directory -domainille

Hakemistotyökalun Active Directory -liitintä käyttäen voit määritellä Mac OS X:n pääsemään Windows-palvelimen Active Directory -domainissa oleviin käyttäjätilien perustietoihin.

Active Directory -liitin luo kaikki Mac OS X:n todentamiseen tarvitsemat attribuutit. Active Directory -malliin ei tarvitse tehdä muutoksia.

Active Directory -liitin tunnistaa Mac OS X:n normaalit tietuetyypit ja attribuutit (kuten Mac OS X:n asiakashallintaan tarvittavat attribuutit) ja käyttää niitä, jos Active Directory -malli on laajennettu sisältämään ne.

VAROITUS:Active Directory -liittimen lisävalinnoilla voidaan kuvata Mac OS X:n yksilöllinen käyttäjätunnus (UID), ensisijainen ryhmätunnus (GID) ja ryhmän GID-attribuutti oikeisiin attribuutteihin, jotka on lisätty Active Directory -malliin. Jos muutat näiden kuvausvalintojen asetuksia myöhemmin, käyttäjät saattavat menettää pääsyn aiemmin luotuihin tiedostoihin.
Tärkeää:  Jos tietokoneen nimi sisältää väliviivan, et ehkä voi liittyä tai muodostaa sidosta esimerkiksi LDAP- tai Active Directory -hakemistodomainiin. Jos haluat muodostaa sidoksen, käytä tietokoneen nimeä, joka ei sisällä väliviivaa.
Pääsyn määrittäminen Active Directory -hakemistoon:

  1. Avaa Järjestelmäasetukset ja osoita Tilit.

  2. Jos lukkosymboli on lukittu, avaa se osoittamalla sitä ja kirjoittamalla ylläpitäjän käyttäjätunnus ja salasana.

  3. Osoita Kirjautumisvalinnat ja osoita sitten Liity tai Muokkaa.

  4. Osoita Avaa Hakemistotyökalu.

  5. Jos lukkosymboli on lukittu, avaa se osoittamalla sitä ja kirjoittamalla ylläpitäjän käyttäjätunnus ja salasana.

  6. Osoita Palvelut.

  7. Valitse palveluluettelosta Active Directory ja osoita muokkauspainiketta (/).

  8. Syötä sen Active Directory -domainin DNS-nimi, johon haluat sitoa määriteltävän tietokoneen.

    Active Directory -domainin ylläpitäjä voi kertoa syötettävän DNS-nimen.

  9. Muokkaa tarvittaessa tietokoneen ID:tä.

    Tietokoneen tunnus on nimi, jolla tietokone tunnetaan Active Directory -domainissa, ja sen esiasetuksena on tietokoneen nimi. Tämä voidaan muuttaa vastaamaan organisaation mallia tietokoneiden nimeämisestä Active Directory -domainissa. Jos et ole varma, kysy Active Directory -domainin ylläpitäjältä.

  10. (Valinnainen) Tee lisävalinnat.

    Jos lisäasetukset on kätketty, osoita Näytä lisävalinnat ja tee asetukset Käyttökokemus-, Kuvaukset- ja Ylläpidollinen-osissa. Voit muuttaa lisäasetuksia myöhemminkin.

    Lisätietoja lisävalinnoista löytyy kohdista:

    • Siirrettävien käyttäjätunnusten asettaminen Active Directoryyn

    • Kotikansioiden asettaminen Active Directory -käyttäjätunnuksille

    • UNIX-komentotulkin asettaminen Active Directory -käyttäjätunnuksille

    • UID:n kuvaaminen Active Directory -attribuuttiin

    • Ensisijaisen ryhmänumeron kuvaaminen Active Directory -attribuuttiin

    • Ryhmätunnuksien ryhmänumeron kuvaaminen Active Directory -attribuuttiin

    • Suosittavan Active Directory -palvelimen määritteleminen

    • Tietokonetta ylläpitävien Active Directory -ryhmien muuttaminen

    • Kaikilta Active Directory -metsän domaineilta tehtävän todentamisen hallitseminen

  11. Osoita Sido, käytä seuraavaa todentaaksesi itsesi käyttäjänä, jolla on oikeus sitoa tietokone Active Directory -domainiin, valitse hakukäytännöt, joihin haluat Active Directoryn lisättävän (katso jäljempänä) ja osoita OK.

    • Käyttäjätunnus ja salasana: voit ehkä suorittaa todentamisen syöttämällä Active Directory -käyttäjätunnuksen ja salasanan, tai sitten Active Directory -domainin ylläpitäjän on annettava käyttäjätunnus ja salasana.

    • Tietokoneen OU: syötä määriteltävän tietokoneen organisaatioyksikkö (organizational unit, OU).

    • Käytä todentamiseen: määrittelee lisätäänkö Active Directory tietokoneen todentamisen hakukäytäntöön.

    • Käytä yhteystietoihin: määrittelee lisätäänkö Active Directory tietokoneen yhteystietojen hakukäytäntöön.

    Kun osoitat OK, Hakemistotyökalu asettaa luotetun sidonnan määriteltävän tietokoneen ja Active Directory -palvelimen välille. Tietokoneen hakukäytännöt asetetaan todentauduttaessa valittujen asetusten mukaan ja Active Directory sallitaan Hakemistotyökalun Palvelut-paneelissa.

    Active Directoryn oletusasetuksilla Active Directory -metsä lisätään tietokoneen todentamisen ja yhteystietojen hakukäytäntöön, jos valitsit Käytä todentamiseen- ja Käytä yhteystietoihin -valintaneliöt.

    Kuitenkin, jos poistat Ylläpidollinen-osan lisävalinnoista Salli todentaminen miltä tahansa metsän domainilta -valintaneliön valinnan ennen Sido-painikkeen osoittamista, lähin Active Directory -domain lisätään metsän sijaan.

    Voit muuttaa hakukäytäntöjä lisäämällä tai poistamalla Active Directory -metsän tai yksittäisiä domaineja. Lisätietoja löytyy kohdasta Muokattujen hakukäytäntöjen määritteleminen.

  12. (Valinnainen) Liitä palvelin Active Directoryn Kerberos-realmiin.

    • Avaa Server Admin palvelimella tai ylläpitotietokoneella, joka voi muodostaa yhteyden palvelimeen, ja valitse palvelimen Open Directory.

    • Osoita Settings ja General.

    • Osoita Join Kerberos, valitse ponnahdusvalikosta Active Directory Kerberos realm ja syötä palvelimen paikallisen ylläpitäjän valtakirjat.
