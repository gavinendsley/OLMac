Ensisijaisen ryhmänumeron kuvaaminen Active Directory -attribuuttiin

Jos tietokone on määritelty käyttämään Hakemistotyökalun Active Directory -liitintä, voit määritellä Active Directory -attribuutin, jonka haluat kuvata Mac OS X:n käyttäjätilien ensisijaiseen ryhmätunnusattribuuttiin (GID).

Yleensä Active Directory -malli täytyy laajentaa sisältämään ensisijaisen GID:n kuvaamiseen sopivan attribuutin:

  • Jos Active Directory -ylläpitäjä laajentaa Active Directory -mallia asentamalla Microsoftin Services for UNIX -ohjelmiston, voit kuvata ensisijaisen GID:n msSFU-30-Gid-Number-attribuuttiin.

  • Kun Active Directoryn ylläpitäjä laajentaa Active Directory -mallin käsin sisältämään RFC 2307 -attribuutteja, voit kuvata ensisijaisen GID:n gidNumberiin.

  • Kun Active Directoryn ylläpitäjä laajentaa Active Directory -mallin käsin sisältämään Mac OS X PrimaryGroupID -attribuutin, voit kuvata ensisijaisen GID:n siihen.

Jos ensisijaisen GID:n kuvaus on estetty, Active Directory -liitin luo ensisijaisen GID:n Active Directoryn normaalin GUID-attribuutin perusteella.

VAROITUS: Jos muutat ensisijaisen GID:n kuvausta myöhemmin, käyttäjät saattavat menettää pääsyn aiemmin luotuihin tiedostoihin.
Ensisijaisen GID:n kuvaaminen laajennetun Active Directory -mallin attribuuttiin:

  1. Avaa Järjestelmäasetukset ja osoita Tilit.

  2. Jos lukkosymboli on lukittu, avaa se osoittamalla sitä ja kirjoittamalla ylläpitäjän käyttäjätunnus ja salasana.

  3. Osoita Kirjautumisvalinnat ja osoita sitten Muokkaa.

  4. Osoita Avaa Hakemistotyökalu.

  5. Jos lukkosymboli on lukittu, avaa se osoittamalla sitä ja kirjoittamalla ylläpitäjän käyttäjätunnus ja salasana.

  6. Osoita Palvelut.

  7. Valitse palveluluettelosta Active Directory ja osoita muokkauspainiketta (/).

  8. Jos lisäasetukset on kätketty, osoita Näytä lisäasetukset.

  9. Osoita Kuvaukset.

  10. Valitse ”Kuvaa käyttäjän GID attribuuttiin” ja syötä sen Active Directory -attribuutin nimi, johon haluat kuvata käyttäjätilien ensisijaisen ryhmätunnuksen.

  11. Osoita OK.
