Pääsyn määrittäminen LDAP-hakemistoon käsin

Voit luoda käsin määrittelyn, jonka mukaan Mac OS X käyttää LDAPv3- tai LDAPv2-hakemistoa. Sinun on tiedettävä LDAP-hakemistopalvelimen DNS-nimi tai IP-osoite.

Jos hakemisto ei sijaitse Mac OS X Serverillä, sinun on tiedettävä hakukanta ja kuvauspohja Mac OS X -tietojen kuvaamiseen hakemiston tietoihin. Tuetut kuvauspohjat ovat:

  • Palvelimelta hakemistolle, joka tarjoaa omat kuvauksensa ja hakukantansa, kuten Mac OS X Server

  • Open Directory -palvelin hakemistolle, joka käyttää Mac OS X Server -mallia

  • Active Directory hakemistolle, joka sijaitsee Windows 2000-, Windows 2003- tai uudemmalla Windows-palvelimella

  • RFC 2307 useimmille hakemistoille, jotka sijaitsevat UNIX-palvelimilla

  • Muokattu hakemistoille, jotka eivät käytä mitään yllä olevista kuvauksista

LDAPv3-liitännäinen tukee täysin Open Directoryn kahdennusominaisuuksia. Jos Open Directory master lakkaa olemasta käytettävissä, liitännäinen palaa lähellä olevaan kopioon.

Tärkeää:  Jos tietokoneen nimi sisältää väliviivan, et ehkä voi liittyä tai muodostaa sidosta esimerkiksi LDAP- tai Active Directory -hakemistodomainiin. Jos haluat muodostaa sidoksen, käytä tietokoneen nimeä, joka ei sisällä väliviivaa.
Pääsyn määrittäminen LDAP-hakemistoon manuaalisesti:

  1. Avaa Järjestelmäasetukset ja osoita Tilit.

  2. Jos lukkosymboli on lukittu, avaa se osoittamalla sitä ja kirjoittamalla ylläpitäjän käyttäjätunnus ja salasana.

  3. Osoita Kirjautumisvalinnat ja osoita sitten Liity tai Muokkaa.

  4. Osoita Avaa Hakemistotyökalu.

  5. Jos lukkosymboli on lukittu, avaa se osoittamalla sitä ja kirjoittamalla ylläpitäjän käyttäjätunnus ja salasana.

  6. Osoita Palvelut.

  7. Valitse palveluluettelosta LDAPv3 ja osoita muokkauspainiketta (/).

    Voit valita palveluluettelosta LDAPv3:n ilman, että sinun tarvitsee valita sallimisvalintaneliötä LDAPv3:lle.

  8. Osoita Uusi ja osoita sitten Käsin.

  9. Syötä määrittelylle nimi.

  10. Paina sarkainnäppäintä ja syötä sen palvelimen DNS-nimi tai IP-osoite, jossa haluamasi LDAP-hakemisto sijaitsee.

  11. Osoita DNS-nimen tai IP-osoitteen vieressä olevaa ponnahdusvalikkoa ja valitse kuvauspohja tai menetelmä:

    • Jos valitset Palvelimelta, hakukannan tarkennetta ei tarvita. Tässä tapauksessa Open Directory olettaa etsinnän peruspäätteen olevan LDAP-hakemiston ensimmäinen taso.

    • Jos valitset mallin, syötä LDAP-hakemiston hakukannan tarkenne ja osoita OK. Syötä hakukannan tarkenne tai muuten tietokone ei löydä tietoja LDAP-hakemistosta.

      Hakukannan tarkenne saadaan yleensä palvelimen DNS-nimestä. Hakukannan tarkenne voisi esimerkiksi olla ”dc=ods,dc=example, dc=com” palvelimelle, jonka DNS-nimi on ods.example.com.

    • Jos valitset Muokattu, sinun on asetettava Mac OS X -tietuetyyppien ja attribuuttien ja yhdistettävän LDAP-hakemiston luokkien ja attribuuttien väliset kuvaukset. Lisätietoja löytyy kohdasta LDAP-hakujen ja -kuvausten määritteleminen.

  12. Ennen kuin valitset Salaa käyttäen SSL:ää -valintaneliön, tarkista Open Directory -ylläpitäjältä tarvitaanko SSL:ää.

  13. Jos haluat muuttaa LDAP-määrittelyn asetuksia, tuo valitun LDAP-määrittelyn asetukset näkyviin osoittamalla Muokkaa, tee muutokset ja lopeta muokkaaminen osoittamalla OK.

    • Osoittamalla Yhteys voit määrittää aikakatkaisun asetukset, määritellä muokatun portin, jättää huomiotta viittaukset palvelimeen tai pakottaa käyttämään LDAPv2 (kirjoitussuojattu) -protokollaa. Lisätietoja löytyy kohdasta LDAP-hakemiston yhteysasetusten muuttaminen.

    • Aseta hakuja ja kuvauksia LDAP-palvelimelle osoittamalla Haut ja kuvaukset. Lisätietoja löytyy kohdasta Luotetun sidonnan käyttöönotto LDAP-hakemistolle.

    • Osoittamalla Turvallisuus voit asettaa todennetun yhteyden (luotetun sidonnan sijaan) ja muita turvallisuuteen liittyviä asetuksia. Lisätietoja löytyy kohdasta LDAP-yhteyden turvallisuuskäytännön muuttaminen.

    • Jos haluat asettaa luotetun sidonnan, osoita Sido (jos LDAP-hakemisto tukee sitä). Lisätietoja löytyy kohdasta Luotetun sidonnan käyttöönotto LDAP-hakemistolle.

  14. Osoittamalla OK voit lopettaa määrittelyiden luomisen LDAP-hakemistoon pääsyä varten.

  15. Jos haluat tietokoneen käyttävän LDAP-hakemistoa, jolle loit määrittelyn, lisää hakemisto Hakemistotyökalun Hakukäytännön Todentaminen- ja Yhteystieto-paneeleissa olevaan muokattuun hakukäytäntöön ja varmista, että LDAPv3 on sallittuna Palvelut-paneelissa.

Lisätietoja löytyy kohdista LDAP-hakemistopalveluiden salliminen ja estäminen ja Muokattujen hakukäytäntöjen määritteleminen.

Huomaa:  Ennen kun voit käyttää Workgroup Manageria käyttäjien luomiseen muun valmistajan kuin Applen RFC 2307 (UNIX) -kuvauksia käyttävällä LDAP-palvelimella, sinun täytyy muokata Käyttäjät-tietuetyypin kuvausta. Lisätietoja löytyy kohdasta RFC 2307 -kuvauksen muokkaaminen käyttäjien luomisen sallimiseksi.
Tärkeää:  Jos muutat IP-osoitetta ja tietokoneen nimeä changeip:llä, kun olet yhteydessä hakemistopalvelimeen, yhteys palvelimeen on katkaistava ja muodostettava uudelleen, jotta hakemisto päivitetään tietokoneen uudella nimellä ja IP-osoitteella. Jos yhteyttä hakemistopalvelimeen ei katkaista ja muodosteta uudelleen, hakemistoa ei päivitetä ja se jatkaa tietokoneen vanhan nimen ja IP-osoitteen käyttöä.
