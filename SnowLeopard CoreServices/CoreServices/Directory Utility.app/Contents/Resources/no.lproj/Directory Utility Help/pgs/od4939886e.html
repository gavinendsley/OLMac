Angi BSD-konfigurasjonsfilinnstillinger

Tidligere arkiverte UNIX-maskiner administrative data i konfigurasjonsfiler som for eksempel /etc/master.passwd, /etc/group og /etc/hosts. Mac OS X er basert på en BSD-versjon av UNIX, men får vanligvis administrative data fra katalogsystemene.

Mac OS X Server støtter et fast sett med BSD-konfigurasjonsfiler. Det er ikke mulig å spesifisere hvilke konfigurasjonsfiler som skal brukes, og du kan heller ikke tilordne innholdet i dem til Mac OS X-oppføringstyper og -attributter.

I Mac OS X versjon 10.2 eller nyere (inkludert Mac OS X Server versjon 10.2 eller nyere) kan Open Directory hente administrative data fra BSD-konfigurasjonsfiler. Denne egenskapen gjør det mulig for organisasjoner som allerede har BSD-konfigurasjonsfiler, å bruke kopier av eksisterende filer på Mac OS X-maskiner. BSD-konfigurasjonsfiler kan brukes alene eller sammen med andre katalogdomener.

Slik bruker du BSD-konfigurasjonsfiler:

  1. Kontroller at BSD-konfigurasjonsfilen inneholder dataene som kreves av katalogtjenestene i Mac OS X.

    Hvis du vil vite mer, kan du lese Konfigurere data i BSD-konfigurasjonsfiler.

  2. Åpne Systemvalg, og klikk på Kontoer.

  3. Hvis låssymbolet er låst, klikker du på det for å åpne det og skriver deretter inn brukernavnet og passordet for en administratorbruker.

  4. Klikk på Påloggingsvalg, og klikk på Koble til eller Rediger.

  5. Klikk på Åpne Katalogverktøy.

  6. Hvis låssymbolet er låst, klikker du på det for å åpne det og skriver deretter inn brukernavnet og passordet for en administratorbruker.

  7. Klikk på Tjenester.

  8. I listen med tjenester velger du «BSD Flat File og NIS» og klikker på Rediger-knappen (/).

  9. Marker «Bruk Bruker og gruppe-oppføringer i BSD-lokalnode», og klikk deretter på OK.

    BSD-konfigurasjonsfilerdomenet legges til i Godkjenning-søkekriteriene på maskinen som /BSD/local.
