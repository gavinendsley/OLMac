Angi UNIX-shell for Active Directory-brukerkontoer

På en maskin som er konfigurert til å bruke Katalogverktøys Active Directory-tilkobling, kan du angi hvilket kommandolinje-shell som brukere med Active Directory-kontoer skal bruke som standard når de arbeider mot Mac OS X i Terminal-programmet.

Standard-shell brukes også til ekstern kommunikasjon via SSH (Secure Shell) eller Telnet. Brukere kan overstyre standardshell ved å endre en innstilling i Terminal.

Slik angir du UNIX-shell for Active Directory-brukerkontoer:

  1. Åpne Systemvalg, og klikk på Kontoer.

  2. Hvis låssymbolet er låst, klikker du på det for å åpne det og skriver deretter inn brukernavnet og passordet for en administratorbruker.

  3. Klikk på Påloggingsvalg, og klikk på Koble til eller Rediger.

  4. Klikk på Åpne Katalogverktøy.

  5. Hvis låssymbolet er låst, klikker du på det for å åpne det og skriver deretter inn brukernavnet og passordet for en administratorbruker.

  6. Klikk på Tjenester.

  7. I listen med tjenester velger du Active Directory og klikker på Rediger-knappen (/).

  8. Hvis de avanserte innstillingene er skjult, klikker du på Vis avanserte innstillinger.

  9. Klikk på Brukeropplevelse.

  10. Marker Standardshell for bruker, og angi banen for brukerens standardshell.

  11. Klikk på OK.
