Lage et passord

Passordet som er tilknyttet en brukers konto, må oppgis av brukeren når han eller hun godkjennes for pålogging eller andre tjenester. Passordet skiller mellom små og store bokstaver (med unntak av SMB-LAN Manager-passord), og skjules på skjermen når det skrives inn.

Uavhengig av passordtypen du velger for en bruker, finnes det noen retningslinjer for å lage passord for Mac OS X Server-brukerkontoer:

  • Et passord bør inneholde bokstaver, tall og symboler i kombinasjoner som er vanskelige å gjette for uautoriserte brukere. Passord bør ikke bestå av ord. Gode passord inneholder tall og symboler (for eksempel # eller $), eller de består av den første bokstaven i alle ordene i et uttrykk. Bruk både store og små bokstaver.

  • Unngå mellomrom og kombinasjoner som bruker Tilvalg-tasten.

  • Unngå tegn som ikke kan skrives på maskiner som brukeren skal bruke, eller som krever kunnskap om en spesiell tastaturkombinasjon for å kunne skrives inn riktig på forskjellige tastaturer og plattformer.

  • Enkelte nettverksprotokoller støtter ikke passord som inneholder innledende mellomrom, mellomrom i passordet eller avsluttende mellomrom.

  • Nullengdepassord anbefales ikke. Open Directory og enkelte systemer (for eksempel LDAP bind) støtter ikke nullengdepassord.

  • For maksimal kompatibilitet med maskiner og tjenere brukerne din kan ha tilgang til, bør du kun bruke ASCII-tegn for passord.
